### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for three-column Kanban layout
- Props: columns (array), tasks (array), loading (boolean), error (string)
- State: draggedTask, columnCounts, isLoading, errorMessage
- Events: taskMoved, taskCreated, taskUpdated, columnUpdated
- API Binding: GET /youth-accounts/{id}/dashboard for task data

**KanbanColumnComponent:**
- Purpose: Individual column container (To Do, In Progress, Done)
- Props: columnId (string), title (string), tasks (array), count (number), indicator (string)
- State: isDragOver, isMenuOpen
- Events: taskDropped, addTaskClicked, menuToggled
- API Binding: None (receives data from parent)

**TaskCardComponent:**
- Purpose: Individual task display with priority, assignee, due date
- Props: task (object), draggable (boolean), priority (string)
- State: isDragging, isSelected
- Events: taskClicked, dragStarted, dragEnded
- API Binding: None (receives data from parent)

**HeaderComponent:**
- Purpose: Top navigation with search and user actions
- Props: user (object), notifications (array)
- State: searchQuery, isUserMenuOpen, notificationCount
- Events: searchSubmitted, userMenuToggled, notificationClicked
- API Binding: GET /user/profile, GET /notifications

**SidebarNavigationComponent:**
- Purpose: Left navigation menu with sections
- Props: activeRoute (string), user (object)
- State: activeMenuItem, isCollapsed
- Events: navigationClicked, menuToggled
- API Binding: None (static navigation)

**TaskDetailModalComponent:**
- Purpose: Modal for creating/editing tasks
- Props: task (object), isOpen (boolean), mode (string)
- State: formData, isSubmitting, validationErrors
- Events: taskSaved, modalClosed, formSubmitted
- API Binding: POST /tasks, PUT /tasks/{id}

**ButtonComponent (from library):**
- Purpose: Reusable button with multiple variants
- Props: label, variant, size, disabled, processing, iconName
- State: isPressed, isFocused
- Events: userClick
- API Binding: None

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **View Dashboard**
   - User clicks "Dashboard" in sidebar
   - в†’ Navigate to dashboard page
   - в†’ Call GET /youth-accounts/{id}/dashboard
   - в†’ Display account balance, metrics, recent activity

2. **Create New Task**
   - User clicks "Add Task" button
   - в†’ Open TaskDetailModal
   - в†’ User fills form and submits
   - в†’ Call POST /tasks
   - в†’ Refresh kanban board data
   - в†’ Show success notification

3. **Move Task Between Columns**
   - User drags task card to different column
   - в†’ Update UI optimistically
   - в†’ Call PATCH /tasks/{id}/status
   - в†’ Update column counts
   - в†’ Announce change to screen readers

4. **Search Tasks**
   - User types in search input
   - в†’ Filter tasks in real-time
   - в†’ Call GET /tasks?search={query}
   - в†’ Update kanban board display

5. **View Task Details**
   - User clicks on task card
   - в†’ Open TaskDetailModal with task data
   - в†’ Allow editing if user has permissions
   - в†’ Save changes with PUT /tasks/{id}
