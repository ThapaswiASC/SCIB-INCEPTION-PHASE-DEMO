# UI COMPONENT ARCHITECTURE

## 1. JIRA REQUIREMENT SUMMARY

**Story Description:**
Create Angular component for Kanban board container with three-column layout. Implement component class with @Component decorator and selector 'app-kanban-board'. Define template with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr)). Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header. Implement error handling with *ngIf directive to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

**Acceptance Criteria:**
- KanbanBoardComponent created with proper Angular structure and decorators
- Template renders three distinct column containers with proper semantic HTML
- CSS Grid layout implemented with responsive breakpoints for desktop, tablet, and mobile
- ARIA attributes added for accessibility compliance
- Error state template implemented with conditional rendering
- Component compiles without errors and passes linting checks

**UI Tasks:**
- Create Angular component at src/app/features/kanban/components/kanban-board/kanban-board.component.ts
- Implement three-column layout with CSS Grid
- Add responsive breakpoints and ARIA attributes
- Initialize column data structure with OnInit lifecycle hook

## 6. COMPONENT DEFINITIONS

### KanbanBoardComponent:
**Purpose:** Main container component implementing three-column Kanban layout with drag/drop functionality
**Props:**
- columns: KanbanColumn[] = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- loading: boolean = false
- error: string | null = null
- dragEnabled: boolean = true
**State:**
- draggedTask: TaskCard | null
- columnData: KanbanColumn[]
- isLoading: boolean
- errorMessage: string | null
**Events:**
- onTaskMove: (taskId: string, fromColumn: string, toColumn: string) => void
- onTaskCreate: (task: TaskCard) => void
- onTaskUpdate: (task: TaskCard) => void
- onTaskDelete: (taskId: string) => void
**API Binding:**
- GET /youth-accounts/{youthAccountId}/dashboard - Fetch dashboard data
- PUT /tasks/:id/status - Update task status on drag/drop
- POST /tasks - Create new task

### HeaderComponent:
**Purpose:** Top navigation bar with search functionality and user actions
**Props:**
- user: User
- notificationCount: number
- searchPlaceholder: string = "Search tasks, projects, or team members..."
**State:**
- searchQuery: string
- showNotifications: boolean
- showUserMenu: boolean
**Events:**
- onSearch: (query: string) => void
- onNotificationToggle: () => void
- onUserMenuToggle: () => void
- onSettingsClick: () => void
**API Binding:**
- GET /notifications - Fetch notification count
- GET /user/profile - Fetch user profile data

### SidebarComponent:
**Purpose:** Left navigation panel with hierarchical menu structure
**Props:**
- activeRoute: string
- navigationItems: NavigationSection[]
- isCollapsed: boolean = false
**State:**
- expandedSections: string[]
- focusedItem: string | null
**Events:**
- onNavigate: (route: string) => void
- onSectionToggle: (sectionId: string) => void
**API Binding:** None (static navigation structure)

### KanbanColumnComponent:
**Purpose:** Individual column in Kanban board with drag/drop zone
**Props:**
- column: KanbanColumn
- tasks: TaskCard[]
- allowDrop: boolean = true
- maxTasks: number | null = null
**State:**
- isDragOver: boolean
- taskCount: number
- isExpanded: boolean = true
**Events:**
- onTaskDrop: (task: TaskCard) => void
- onAddTask: (columnId: string) => void
- onColumnMenuToggle: () => void
**API Binding:**
- GET /tasks?status={columnId} - Fetch tasks by column status

### TaskCardComponent:
**Purpose:** Individual task card with priority indicators and metadata
**Props:**
- task: TaskCard
- draggable: boolean = true
- showPriority: boolean = true
- showAssignee: boolean = true
- showDueDate: boolean = true
- showTags: boolean = true
**State:**
- isDragging: boolean
- isSelected: boolean
**Events:**
- onTaskClick: (taskId: string) => void
- onDragStart: (task: TaskCard) => void
- onDragEnd: () => void
**API Binding:**
- GET /tasks/:id - Fetch detailed task information

### ButtonComponent (from library):
**Purpose:** Reusable button with multiple variants and interactive states
**Props:**
- label: string
- variant: 'default' | 'primary' | 'secondary' | 'success' | 'warning' | 'danger' | 'info'
- size: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
- disabled: boolean = false
- processing: boolean = false
- pill: boolean = false
- outlined: boolean = false
- clear: boolean = false
- iconName: string | null = null
- iconPosition: 'left' | 'right' | 'only' = 'left'
- hoverEffect: 'torch' | 'glow' | 'tint' | 'scale' | 'none' = 'none'
- pressedEffect: 'ripple' | 'inset' | 'solid' | 'none' = 'none'
**Events:**
- userClick: () => void
**API Binding:** None (UI component)

### ModalComponent:
**Purpose:** Overlay modal for task details, forms, and dialogs
**Props:**
- isOpen: boolean = false
- title: string
- size: 'sm' | 'md' | 'lg' | 'xl' = 'md'
- closable: boolean = true
- backdrop: boolean = true
**State:**
- isVisible: boolean
- isAnimating: boolean
**Events:**
- onClose: () => void
- onSubmit: (data: any) => void
- onCancel: () => void
**API Binding:** Context-dependent based on modal content

### AnalyticsPanelComponent:
**Purpose:** Dashboard analytics with performance metrics
**Props:**
- metrics: PerformanceMetric[]
- timeRange: 'week' | 'month' | 'quarter' = 'week'
- showTrends: boolean = true
**State:**
- isLoading: boolean
- selectedMetric: string | null
**Events:**
- onMetricClick: (metricId: string) => void
- onTimeRangeChange: (range: string) => void
**API Binding:**
- GET /youth-accounts/{youthAccountId}/dashboard - Fetch dashboard metrics
- GET /analytics/performance - Fetch performance data

## 8. API SUMMARY

**Available APIs:**
- GET /youth-accounts/{youthAccountId}/dashboard - Retrieve youth account dashboard information
- GET /health - API health check endpoint

**Expected Additional Endpoints:**
- GET /tasks - Fetch all tasks with filtering
- GET /tasks/:id - Fetch task details
- POST /tasks - Create new task
- PUT /tasks/:id - Update task
- PUT /tasks/:id/status - Update task status
- DELETE /tasks/:id - Delete task
- GET /users - Fetch users for assignment
- GET /notifications - Fetch user notifications
- GET /analytics/performance - Fetch team performance metrics
- GET /user/profile - Fetch user profile

**Authentication:**
- OAuth 2.0 with JWT Bearer tokens
- Required scopes: youth-account:read, youth-account:write, fund-transfer:execute, spending-limit:manage

**Rate Limiting:**
- General API: 1000 requests per hour per user
- Fund Transfer: 10 requests per hour per user
- Dashboard: 100 requests per hour per user
- Transaction History: 50 requests per hour per user

   в†’ TaskCardComponent.onDragStart()

   в†’ KanbanColumnComponent.onDrop()
   в†’ TaskService.updateTaskStatus(taskId, 'inprogress')
   в†’ PUT /tasks/:id/status
   в†’ Update UI state optimistically
   в†’ Show success/error feedback

   в†’ AddTaskButtonComponent.onClick()
   в†’ ModalService.openTaskModal()
   в†’ TaskModalComponent displays form
   в†’ User fills form and submits
   в†’ TaskService.createTask(taskData)
   в†’ POST /tasks
   в†’ Add task to appropriate column
   в†’ Close modal and show success message

   - User clicks on task card
   в†’ TaskCardComponent.onClick()
   в†’ TaskService.getTaskDetails(taskId)
   в†’ GET /tasks/:id
   в†’ ModalService.openTaskDetailModal()

   в†’ GET /tasks?search={query}
