## UI COMPONENT ARCHITECTURE

### 1. JIRA REQUIREMENT SUMMARY

**Story Description:**
Create Angular component for Kanban board container with three-column layout

**Acceptance Criteria:**
- KanbanBoardComponent created with proper Angular structure and decorators
- Template renders three distinct column containers with proper semantic HTML
- CSS Grid layout implemented with responsive breakpoints for desktop, tablet, and mobile
- ARIA attributes added for accessibility compliance
- Error state template implemented with conditional rendering
- Component compiles without errors and passes linting checks

**UI Tasks:**
- Implement component class with @Component decorator and selector 'app-kanban-board'
- Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr))
- Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px)
- Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header
- Implement error handling with *ngIf directive to display error message when column configuration fails

### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for three-column kanban layout
- Props: columns: KanbanColumn[], loading: boolean, error: string
- State: draggedTask: Task, activeColumn: string (mobile)
- Events: taskMoved, taskCreated, taskUpdated, taskDeleted
- API Binding: GET /api/youth-accounts/{id}/dashboard for initial data

**KanbanColumnComponent:**
- Purpose: Individual column container with header and task list
- Props: column: KanbanColumn, tasks: Task[], isActive: boolean
- State: isDropTarget: boolean, taskCount: number
- Events: taskDropped, addTaskClicked, columnActionClicked
- API Binding: None (receives data from parent)

**TaskCardComponent:**
- Purpose: Individual task display with drag functionality
- Props: task: Task, draggable: boolean, focused: boolean
- State: isDragging: boolean, isSelected: boolean
- Events: taskClicked, taskDragStart, taskDragEnd, taskKeydown
- API Binding: None (receives data from parent)

**HeaderComponent:**
- Purpose: Top navigation and search functionality
- Props: user: User, breadcrumb: BreadcrumbItem[], searchEnabled: boolean
- State: searchQuery: string, isSearchFocused: boolean
- Events: searchSubmitted, themeToggled, userMenuClicked
- API Binding: Search API (if implemented)

**SidebarComponent:**
- Purpose: Main navigation and project menu
- Props: isOpen: boolean, currentPage: string, projects: Project[]
- State: activeSection: string, isCollapsed: boolean
- Events: navigationClicked, sidebarToggled, projectSelected
- API Binding: None (static navigation)

**ModalComponent:**
- Purpose: Task creation and editing modal
- Props: isOpen: boolean, mode: 'create'|'edit', task: Task
- State: formData: TaskFormData, isSubmitting: boolean, validationErrors: ValidationError[]
- Events: modalClosed, taskSubmitted, formChanged
- API Binding: POST /api/youth-accounts/{id}/fund-transfer for task creation

### 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data including account balance and recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (can be adapted for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits (can be adapted for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history (can be adapted for task history)

### 9. API TO UI MAPPING

**Component → API:**
- KanbanBoardComponent → GET /api/youth-accounts/{id}/dashboard (initial load)
- TaskCardComponent → PUT /api/youth-accounts/{id}/spending-limit (task updates)
- ModalComponent → POST /api/youth-accounts/{id}/fund-transfer (task creation)
- TaskHistoryComponent → GET /api/youth-accounts/{id}/transactions (task history)

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **Load Kanban Board:**
   - User navigates to kanban page
   → KanbanBoardComponent.ngOnInit()
   → Call GET /api/youth-accounts/{id}/dashboard
   → Display columns and tasks

2. **Create New Task:**
   - User clicks "Add Task" button
   → Open ModalComponent
   → User fills form and submits
   → Call POST /api/youth-accounts/{id}/fund-transfer (adapted)
   → Update task list and close modal

3. **Drag and Drop Task:**
   - User drags task to different column
   → TaskCardComponent.onDragStart()
   → KanbanColumnComponent.onDrop()
   → Call PUT /api/youth-accounts/{id}/spending-limit (adapted)
   → Update task status and column counts

4. **Mobile Column Switch:**
   - User taps mobile tab
   → MobileTabsComponent.onTabClick()
   → KanbanBoardComponent.setActiveColumn()
   → Show selected column

5. **Search Tasks:**
   - User types in search input
   → HeaderComponent.onSearchInput()
   → Filter displayed tasks
   → Update kanban board view
