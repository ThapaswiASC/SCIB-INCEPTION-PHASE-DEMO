## UI COMPONENT ARCHITECTURE

1. JIRA REQUIREMENT SUMMARY

Story Description:
Create Angular component for Kanban board container with three-column layout. Implementation details include creating Angular component at src/app/features/kanban/components/kanban-board/kanban-board.component.ts with @Component decorator and selector 'app-kanban-board'. Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr)). Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header. Implement error handling with *ngIf directive to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

Acceptance Criteria:
- Kanban Board Component created with proper Angular structure and decorators
- Template renders three distinct column containers with proper semantic HTML
- CSS Grid layout implemented with responsive breakpoints for desktop, tablet, and mobile
- ARIA attributes added for accessibility compliance
- Error state template implemented with conditional rendering
- Component compiles without errors and passes linting checks

UI Tasks:

6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout as per Jira requirements
Props:
- columns: KanbanColumn[] (todo, inprogress, done)
- loading: boolean
- error: string | null
State:
- draggedTask: Task | null
- columnData: KanbanColumn[]
Events:
- taskMoved: EventEmitter<TaskMoveEvent>
- taskCreated: EventEmitter<Task>
API Binding:
- GET /api/youth-accounts/{id}/dashboard for initial data

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn
- tasks: Task[]
- columnId: string
State:
- isDropTarget: boolean
Events:
- taskDropped: EventEmitter<DropEvent>
- addTaskClicked: EventEmitter<string>
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task display with drag/drop functionality
Props:
- task: Task
- draggable: boolean
State:
- isDragging: boolean
Events:
- taskClicked: EventEmitter<Task>
- taskDragStart: EventEmitter<Task>
API Binding:
- None (receives data from parent)

TaskModalComponent:
Purpose: Modal for creating and editing tasks
Props:
- isOpen: boolean
- task: Task | null
- defaultColumn: string
State:
- formData: TaskFormData
- isSubmitting: boolean
Events:
- taskSaved: EventEmitter<Task>
- modalClosed: EventEmitter<void>
API Binding:
- POST /api/youth-accounts/{id}/fund-transfer for task creation

HeaderComponent:
Purpose: Top navigation bar with search, notifications, and user menu

- user: User

State:
- searchQuery: string
Events:
- searchSubmitted: EventEmitter<string>

API Binding:
- None (receives data from parent)

SidebarComponent:

Events:

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data with account balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits (can be repurposed for task limits)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history (can be repurposed for task history)

Authentication: OAuth 2.0 with JWT tokens
Rate Limiting: 100 requests per minute
Supported Currencies: USD, EUR, GBP

9. API TO UI MAPPING

Component → API:
- KanbanBoardComponent → GET /api/youth-accounts/{id}/dashboard (fetch initial kanban data)
- TaskModalComponent → POST /api/youth-accounts/{id}/fund-transfer (create new tasks)
- KanbanBoardComponent → PUT /api/youth-accounts/{id}/spending-limit (update column limits)
- TaskCardComponent → GET /api/youth-accounts/{id}/transactions (fetch task history)

13. INTERACTION FLOWS

User Action → UI → API:

1. Load Kanban Board:
   → KanbanBoardComponent.ngOnInit()
   → KanbanService.getDashboard()
   → GET /api/youth-accounts/{id}/dashboard
   → Update component state with columns and tasks

2. Create New Task:
   → Click "Add Task" button
   → Open TaskModalComponent
   → Fill form and submit
   → POST /api/youth-accounts/{id}/fund-transfer
   → Add task to appropriate column
   → Update task counts

3. Drag and Drop Task:
   → TaskCardComponent.onDragStart()
   → KanbanColumnComponent.onDrop()
   → Update local state
   → PUT /api/youth-accounts/{id}/spending-limit (update column)
   → Announce change to screen readers

4. Search Tasks:
   → HeaderComponent.onSearchSubmit()
   → Filter tasks locally or
   → GET /api/youth-accounts/{id}/transactions (with filters)
   → Update displayed tasks

5. Toggle Theme:
   → HeaderComponent.onThemeToggle()
   → Update CSS custom properties
   → Store preference in localStorage
   → Announce change to screen readers
