# UI COMPONENT ARCHITECTURE

## 1. JIRA REQUIREMENT SUMMARY

**Story Description:**
AS a parent or guardian I WANT to allocate and manage funds for my child's youth account SO THAT I can teach financial responsibility while maintaining visibility and control over spending

**Acceptance Criteria:**
- Scenario 1 – Access youth account dashboard: Given the parent is logged into the banking web application When the parent navigates to the youth account section Then the system should display the youth account dashboard with current balance and recent activity
- Scenario 2 – Allocate funds to youth account: Given the parent is on the youth account dashboard When the parent selects "Add Funds" Then the system should allow the parent to transfer funds from their primary account to the youth account
- Scenario 3 – Set spending limit: Given the parent is viewing the youth account management section When the parent configures a weekly spending limit Then the system should save and display the limit on the dashboard
- Scenario 4 – View youth spending activity: Given the parent is viewing the youth account dashboard When the parent selects the activity section Then the system should display recent transactions made from the youth account
- Scenario 5 – Handle insufficient balance during fund allocation: Given the parent attempts to transfer funds When the selected parent account does not have sufficient balance Then the system should display an error message indicating insufficient funds

**UI Tasks:**
- Design youth account dashboard layout
- Design fund transfer interaction for youth account
- Design spending limit configuration interface
- Design youth account activity view

## 6. COMPONENT DEFINITIONS

### YouthAccountDashboard
**Purpose:** Main container component for youth account management
**Props:**
- youthAccountId: string
- parentAccountId: string
**State:**
- dashboardData: DashboardData
- loading: boolean
- error: string
**Events:**
- onRefresh: void
- onError: (error: string) => void
**API Binding:**
- GET /api/youth-accounts/{youthAccountId}/dashboard

### AccountOverviewCard
**Purpose:** Display current balance and account summary
**Props:**
- balance: number
- currency: string
- spendingLimit: number
- accountId: string
**State:**
- refreshing: boolean
**Events:**
- onRefresh: void
**API Binding:**
- GET /api/youth-accounts/{youthAccountId}/dashboard

### FundTransferModal
**Purpose:** Handle fund transfer from parent to youth account
**Props:**
- isOpen: boolean
- youthAccountId: string
- parentAccounts: Account[]
**State:**
- transferAmount: number
- sourceAccountId: string
- description: string
- processing: boolean
- validationErrors: ValidationError[]
**Events:**
- onTransfer: (transferData: FundTransferRequest) => void
- onClose: void
- onSuccess: (result: FundTransferResponse) => void
**API Binding:**
- POST /api/youth-accounts/{youthAccountId}/fund-transfer

### SpendingLimitConfigModal
**Purpose:** Configure weekly spending limits
**Props:**
- isOpen: boolean
- youthAccountId: string
- currentLimit: number
**State:**
- weeklyLimit: number
- effectiveDate: string
- processing: boolean
**Events:**
- onSave: (limitData: SpendingLimitRequest) => void
- onClose: void
**API Binding:**
- PUT /api/youth-accounts/{youthAccountId}/spending-limit

### ActivityTimeline
**Purpose:** Display transaction history and activity
**Props:**
- youthAccountId: string
- pageSize: number
**State:**
- transactions: Transaction[]
- pagination: PaginationInfo
- loading: boolean
- filters: TransactionFilters
**Events:**
- onLoadMore: void
- onFilter: (filters: TransactionFilters) => void
**API Binding:**
- GET /api/youth-accounts/{youthAccountId}/transactions

### TransactionCard
**Purpose:** Individual transaction display component
**Props:**
- transaction: TransactionDetail
- showCategory: boolean
**State:**
- expanded: boolean
**Events:**
- onExpand: void

## 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data with balance, limits, and recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds from parent to youth account
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Configure weekly spending limits
- GET /api/youth-accounts/{youthAccountId}/transactions → Retrieve paginated transaction history with filtering

## 9. API TO UI MAPPING

## 13. INTERACTION FLOWS

**User Action → UI → API**

1. **View Dashboard:**
   - User navigates to youth account section
   → YouthAccountDashboard loads
   → GET /api/youth-accounts/{id}/dashboard
   → Display balance, limits, recent activity

2. **Add Funds:**
   - User clicks "Add Funds" button
   → FundTransferModal opens
   → User fills transfer form
   → User clicks "Transfer"
   → POST /api/youth-accounts/{id}/fund-transfer
   → Success notification via Snackbar
   → Dashboard refreshes

3. **Set Spending Limit:**
   - User clicks "Configure Limit" button
   → SpendingLimitConfigModal opens
   → User sets weekly limit
   → User clicks "Save"
   → PUT /api/youth-accounts/{id}/spending-limit
   → Success notification
   → Dashboard updates limit display

4. **View Activity:**
   - User clicks on activity section
   → ActivityTimeline loads
   → GET /api/youth-accounts/{id}/transactions
   → Display paginated transaction list
   → User can filter/load more

5. **Handle Insufficient Funds:**
   - User attempts transfer with insufficient balance
   → API returns 422 error
   → Error Snackbar displays "Insufficient funds" message
   → Modal remains open for correction
