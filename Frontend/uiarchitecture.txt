### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for three-column Kanban layout
- Props: columns (array), boardId (string)
- State: draggedTask, columnCounts, errorState
- Events: taskMoved, taskCreated, columnUpdated
- API Binding: GET /api/boards/{boardId}/tasks

**HeaderComponent:**
- Purpose: Top navigation and search functionality
- Props: user (object), notifications (array)
- State: searchQuery, notificationOpen, userMenuOpen
- Events: search, notificationClick, userMenuToggle
- API Binding: Search API, Notifications API

**SidebarComponent:**
- Purpose: Main navigation and branding
- Props: activeRoute (string), navigationItems (array)
- State: collapsed, activeItem
- Events: navigationChange, toggleSidebar
- API Binding: None (static navigation)

**KanbanColumnComponent:**
- Purpose: Individual column container (To Do, In Progress, Done)
- Props: columnId (string), title (string), tasks (array), allowDrop (boolean)
- State: taskCount, dragOver
- Events: taskDropped, addTaskClicked, columnMenuToggle
- API Binding: Task movement API

**TaskCardComponent:**
- Purpose: Individual task display and interaction
- Props: task (object), draggable (boolean), priority (string)
- State: selected, dragging
- Events: taskClick, taskDrag, taskEdit
- API Binding: Task details API

**TaskModalComponent:**
- Purpose: Task creation and editing form
- Props: task (object), mode (create/edit), visible (boolean)
- State: formData, validation, saving
- Events: taskSave, modalClose, formChange
- API Binding: POST /api/boards/{boardId}/tasks, PUT /api/tasks/{taskId}

### 7. DATA FLOW / PROPS

**Parent → Child Mapping:**
- App → HeaderComponent (user, notifications)
- App → SidebarComponent (activeRoute, navigationItems)
- KanbanBoardComponent → KanbanColumnComponent (columnData, tasks)
- KanbanColumnComponent → TaskCardComponent (taskData, draggable)
- App → TaskModalComponent (modalState, taskData)
- TaskCardComponent → AavaAvatarsComponent (assigneeData)
- HeaderComponent → AavaSearchBarComponent (searchConfig)
- TaskModalComponent → AavaButtonComponent (buttonConfig)

### 8. API SUMMARY

**Available APIs:**
- POST /api/boards/{boardId}/tasks → Create single task
- POST /api/boards/{boardId}/tasks/batch → Create multiple tasks
- POST /api/boards/{boardId}/tasks/validate → Validate task data
- GET /health → Service health check
- WebSocket: wss://api.scib.com/ws/boards/{boardId} → Real-time events

### 9. API TO UI MAPPING

**Component → API:**
- KanbanBoardComponent → GET /api/boards/{boardId}/tasks (load tasks)
- TaskModalComponent → POST /api/boards/{boardId}/tasks (create task)
- TaskModalComponent → POST /api/boards/{boardId}/tasks/validate (validate form)
- TaskCardComponent → WebSocket events (real-time updates)
- KanbanColumnComponent → Task movement API (drag & drop)
- HeaderComponent → AavaSearchBarComponent → Search API

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **Create Task:**
   - Click AavaButtonComponent "Add Task"
   → Open TaskModalComponent
   → Fill form and submit
   → Call POST /api/boards/{boardId}/tasks
   → WebSocket broadcast to all users
   → Update local task list

2. **Move Task:**
   - Drag TaskCardComponent
   → Update UI optimistically
   → Call task movement API
   → WebSocket broadcast position change
   → Revert if API fails

3. **Search Tasks:**
   - Type in AavaSearchBarComponent
   → Trigger search event
   → Call search API
   → Filter displayed tasks
   → Update column counts

4. **Real-time Updates:**
   - WebSocket receives TASK_CREATED event
   → Update KanbanColumnComponent
   → Animate new task appearance
   → Update column counts
