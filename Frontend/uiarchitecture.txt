### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for the three-column kanban layout
- Props: columns (array), loading (boolean), error (string)
- State: draggedTask, columnData, isLoading, errorMessage
- Events: taskMoved, taskCreated, taskUpdated, taskDeleted
- API Binding: GET /api/youth-accounts/{id}/dashboard for initial data

**KanbanColumnComponent:**
- Purpose: Individual column container with header and task list
- Props: columnId (string), title (string), tasks (array), count (number)
- State: isDropTarget, isCollapsed
- Events: taskDropped, addTaskClicked, columnOptionsClicked
- API Binding: None (receives data from parent)

**TaskCardComponent:**
- Purpose: Individual task display with all task information
- Props: task (object), draggable (boolean), selectable (boolean)
- State: isDragging, isSelected, isExpanded
- Events: taskClicked, taskDragStart, taskDragEnd, taskSelected
- API Binding: None (receives data from parent)

**HeaderComponent:**
- Purpose: Top navigation bar with search, notifications, and user menu
- Props: user (object), notifications (array), breadcrumb (array)
- State: searchQuery, notificationCount, userMenuOpen
- Events: searchSubmitted, notificationClicked, userMenuToggled, themeToggled
- API Binding: GET /api/notifications, GET /api/user/profile

**SidebarComponent:**
- Purpose: Left navigation panel with menu items and project links
- Props: navigationItems (array), currentPage (string), isOpen (boolean)
- State: activeSection, collapsedSections
- Events: navigationClicked, sidebarToggled, sectionToggled
- API Binding: GET /api/navigation, GET /api/projects

**TaskModalComponent:**
- Purpose: Modal for creating and editing tasks
- Props: isOpen (boolean), task (object), mode ('create'|'edit')
- State: formData, isSubmitting, validationErrors
- Events: taskSaved, modalClosed, formSubmitted
- API Binding: POST /api/tasks (create), PUT /api/tasks/{id} (update)

**ButtonComponent (aava-button):**
- Purpose: Reusable button with various styles and states
- Props: label, variant, size, iconName, iconPosition, disabled, processing
- State: isPressed, isHovered
- Events: userClick
- API Binding: None

**PopupComponent (aava-popup):**
- Purpose: Reusable popup/tooltip/dropdown container
- Props: isOpen, trigger, placement, closeOnClickOutside
- State: position, isVisible
- Events: opened, closed, positioned
- API Binding: None

### 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Add funds
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update limit
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history

**Authentication:**
- OAuth 2.0 with JWT tokens
- Required scopes: youth-account:read, youth-account:write, fund-transfer:execute
- Rate limiting: 100 requests per minute

**Response Formats:**
- Dashboard: account balance, spending limit, recent transactions
- Fund Transfer: transfer ID, status, updated balance, confirmation
- Spending Limit: updated limit, effective date, previous limit
- Transactions: paginated list with filtering and summary

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **Load Dashboard:**
   - User navigates to kanban page
   → KanbanBoardComponent.ngOnInit()
   → GET /api/youth-accounts/{id}/dashboard
   → Display columns and tasks

2. **Create Task:**
   - User clicks "Add Task" button
   → Open TaskModalComponent
   → User fills form and submits
   → POST /api/tasks (if available) or local state update
   → Refresh kanban board

3. **Move Task (Drag & Drop):**
   - User drags task to different column
   → TaskCardComponent.onDragStart()
   → KanbanColumnComponent.onDrop()
   → PUT /api/tasks/{id}/status (if available)
   → Update local state and UI

4. **Transfer Funds:**
   - User clicks "Add Funds" button
   → Open FundTransferModal
   → User enters amount and submits
   → POST /api/youth-accounts/{id}/fund-transfer
   → Update balance display

5. **Update Spending Limit:**
   - User clicks "Set Limit" button
   → Open SpendingLimitModal
   → User sets new limit
   → PUT /api/youth-accounts/{id}/spending-limit
   → Update limit indicators

6. **View Transaction History:**
   - User clicks "View History" button
   → Navigate to transactions page
   → GET /api/youth-accounts/{id}/transactions
   → Display paginated transaction list
