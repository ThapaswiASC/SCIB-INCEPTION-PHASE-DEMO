6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout
Props:
- columns: KanbanColumn[]
- loading: boolean
- error: string | null
State:
- draggedTask: Task | null
- columnData: KanbanColumn[]
Events:
- taskMoved: EventEmitter<TaskMoveEvent>
- taskCreated: EventEmitter<Task>
API Binding:
- GET /api/youth-accounts/{id}/dashboard for initial data

KanbanColumnComponent:
Purpose: Individual column container with tasks
Props:
- column: KanbanColumn
- tasks: Task[]
- columnId: string
State:
- isDragOver: boolean
- taskCount: number
Events:
- taskDropped: EventEmitter<TaskDropEvent>
- addTaskClicked: EventEmitter<string>
API Binding:
- None (receives data from parent)

API Binding:
- None (receives data from parent)

- POST /api/youth-accounts/{youthAccountId}/fund-transfer в†’ Transfer funds to youth account (can be adapted for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit в†’ Configure spending limits (can be adapted for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions в†’ Get transaction history with pagination and filtering
