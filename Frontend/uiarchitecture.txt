### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for three-column kanban layout
- Props:
  - columns: KanbanColumn[]
  - loading: boolean
  - error: string
- State:
  - draggedTask: Task | null
  - columnData: KanbanColumn[]
- Events:
  - taskMoved: (task: Task, fromColumn: string, toColumn: string)
  - taskCreated: (task: Task)
- API Binding:
  - GET /api/youth-accounts/{id}/dashboard for initial data

**KanbanColumnComponent:**
- Purpose: Individual column container with tasks
- Props:
  - column: KanbanColumn
  - tasks: Task[]
  - allowDrop: boolean
- State:
  - isDropTarget: boolean
  - taskCount: number
- Events:
  - taskDropped: (task: Task)
  - addTaskClicked: (columnId: string)
- API Binding: None (data flows from parent)

**TaskCardComponent:**
- Purpose: Individual task display with drag functionality
- Props:
  - task: Task
  - draggable: boolean
- State:
  - isDragging: boolean
- Events:
  - taskClicked: (task: Task)
  - dragStarted: (task: Task)
- API Binding: None (data flows from parent)

**HeaderComponent:**
- Purpose: Top navigation and search
- Props:
  - user: User
  - breadcrumb: BreadcrumbItem[]
- State:
  - searchQuery: string
  - notificationCount: number
- Events:
  - searchSubmitted: (query: string)
  - themeToggled: ()
- API Binding: None

**ModalComponent:**
- Purpose: Task creation and editing modal
- Props:
  - isOpen: boolean
  - mode: 'create' | 'edit'
  - task?: Task
- State:
  - formData: TaskFormData
  - isSubmitting: boolean
- Events:
  - taskSubmitted: (task: Task)
  - modalClosed: ()
- API Binding:
  - POST /api/youth-accounts/{id}/fund-transfer for task creation

### 7. DATA FLOW / PROPS

**Parent → Child Mapping:**
- AppComponent → HeaderComponent (user, breadcrumb)
- AppComponent → SidebarComponent (navigationItems, activeRoute)
- AppComponent → KanbanBoardComponent (columns, tasks, loading, error)
- KanbanBoardComponent → KanbanColumnComponent (column, tasks, allowDrop)

### 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data including account balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits (can be repurposed for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history (can be repurposed for task history)

### 9. API TO UI MAPPING

**Component → API:**
- KanbanBoardComponent → GET /api/youth-accounts/{id}/dashboard (initial load)
- TaskCardComponent → GET /api/youth-accounts/{id}/transactions (task details)
- ModalComponent → POST /api/youth-accounts/{id}/fund-transfer (create task)
- ModalComponent → PUT /api/youth-accounts/{id}/spending-limit (update task)

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **Load Kanban Board**
   → KanbanBoardComponent.ngOnInit()
   → Call GET /api/youth-accounts/{id}/dashboard
   → Transform response to kanban columns
   → Render three columns with tasks

2. **Create New Task**
   → Click "Add Task" button
   → Open ModalComponent
   → Fill task form
   → Submit form
   → Call POST /api/youth-accounts/{id}/fund-transfer
   → Update local state
   → Close modal

3. **Drag and Drop Task**
   → Start drag on TaskCardComponent
   → Drop on target KanbanColumnComponent
   → Update local state
   → Call PUT /api/youth-accounts/{id}/spending-limit
   → Persist changes

4. **Search Tasks**
   → Type in SearchComponent
   → Filter local task data
   → Update kanban board display
