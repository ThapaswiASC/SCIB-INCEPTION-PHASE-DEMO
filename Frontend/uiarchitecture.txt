6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for the three-column kanban layout
Props:
- columns: KanbanColumn[] = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- loading: boolean = false
- error: string | null = null
State:
- draggedTask: Task | null = null
- activeColumn: string = 'todo' (for mobile)
Events:
- onTaskMove: (taskId: string, fromColumn: string, toColumn: string) => void
- onTaskCreate: (task: Task, column: string) => void
API Binding:
- GET /api/youth-accounts/{youthAccountId}/dashboard for initial data
- GET /api/youth-accounts/{youthAccountId}/transactions for task history

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn
- tasks: Task[]
- isActive: boolean = false (for mobile)
State:
- isDragOver: boolean = false
Events:
- onTaskDrop: (task: Task) => void
- onAddTask: () => void
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task card with drag-and-drop functionality
Props:
- task: Task
- draggable: boolean = true
State:
- isDragging: boolean = false
Events:
- onDragStart: (task: Task) => void
- onTaskClick: (task: Task) => void
API Binding:
- None (receives data from parent)

TaskModalComponent:
Purpose: Modal for creating and editing tasks
Props:
- isOpen: boolean = false
- task: Task | null = null
- defaultColumn: string = 'todo'
State:
- formData: TaskFormData
- isSubmitting: boolean = false
- validationErrors: ValidationError[]
Events:
- onSubmit: (task: Task) => void
- onClose: () => void
API Binding:
- POST /api/youth-accounts/{youthAccountId}/fund-transfer for task creation

HeaderComponent:
Purpose: Top navigation bar with search, notifications, and user menu
Props:
- user: User
- notifications: Notification[]
State:
- searchQuery: string = ''
- theme: 'light' | 'dark' = 'light'
Events:
- onSearch: (query: string) => void
- onThemeToggle: () => void
- onNotificationClick: () => void
API Binding:
- GET /api/youth-accounts/{youthAccountId}/dashboard for user data

SidebarComponent:
Purpose: Left navigation sidebar with menu items
Props:
- isOpen: boolean = false (for mobile)
- activeRoute: string
State:
- None
Events:
- onNavigate: (route: string) => void
- onToggle: () => void
API Binding:
- None (static navigation)

ButtonComponent (from library):
Purpose: Reusable button with multiple variants and accessibility
Props:
- label: string
- variant: 'default' | 'primary' | 'secondary' | 'success' | 'warning' | 'danger' | 'info'
- size: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
- disabled: boolean = false
- processing: boolean = false
- iconName: string = ''
- iconPosition: 'left' | 'right' | 'only' = 'left'
State:
- None
Events:
- userClick: () => void
API Binding:
- None (pure UI component)

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data with account balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits (can be repurposed for task limits)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history (can be repurposed for task history)

9. API TO UI MAPPING

Component → API:
- KanbanBoardComponent → GET /api/youth-accounts/{youthAccountId}/dashboard (initial load)
- KanbanBoardComponent → GET /api/youth-accounts/{youthAccountId}/transactions (task history)
- TaskModalComponent → POST /api/youth-accounts/{youthAccountId}/fund-transfer (create task)
- TaskModalComponent → PUT /api/youth-accounts/{youthAccountId}/spending-limit (update task limits)
- HeaderComponent → GET /api/youth-accounts/{youthAccountId}/dashboard (user info, notifications)

13. INTERACTION FLOWS

User Action → UI → API:

1. Load Kanban Board:
   → KanbanBoardComponent.ngOnInit()
   → GET /api/youth-accounts/{youthAccountId}/dashboard
   → Update columns with task data

2. Create New Task:
   → Click "Add Task" button
   → Open TaskModalComponent
   → Fill form and submit
   → POST /api/youth-accounts/{youthAccountId}/fund-transfer
   → Add task to appropriate column
   → Update column counts

3. Move Task (Drag & Drop):
   → Drag TaskCardComponent
   → Drop on KanbanColumnComponent
   → Update local state
   → PUT /api/youth-accounts/{youthAccountId}/spending-limit (task status update)
   → Refresh dashboard data

4. Search Tasks:
   → Type in HeaderComponent search input
   → Filter tasks locally
   → GET /api/youth-accounts/{youthAccountId}/transactions (if needed)
   → Update filtered results

5. Toggle Theme:
   → Click theme button in HeaderComponent
   → Update theme service
   → Apply CSS variables
   → Persist preference

6. Mobile Column Switch:
   → Click mobile tab
   → Show/hide columns
   → Update active column state
   → Maintain accessibility focus
