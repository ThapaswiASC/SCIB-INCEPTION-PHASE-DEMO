### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for three-column Kanban layout with CSS Grid
- Props: columns (KanbanColumn[]), loading (boolean), error (string), viewMode (string)
- State: selectedTask (Task), draggedTask (Task), columnData (KanbanColumn[]), errorState (boolean)
- Events: taskMoved (TaskMoveEvent), taskSelected (Task), taskCreated (Task), columnUpdated (KanbanColumn)
- API Binding: GET /forms/sessions/{session_id} for task data, PUT /forms/sessions/{session_id} for updates

**KanbanColumnComponent:**
- Purpose: Individual column container with header, task list, and actions
- Props: column (KanbanColumn), tasks (Task[]), columnId (string), isDropTarget (boolean)
- State: taskCount (number), isDropTarget (boolean), isExpanded (boolean)
- Events: taskDropped (TaskDropEvent), addTaskClicked (string), columnActionClicked (string)
- API Binding: None (receives data from parent)

**TaskCardComponent:**
- Purpose: Individual task display with metadata, priority, and interactions
- Props: task (Task), priority (Priority), assignee (User), isDragging (boolean)
- State: isSelected (boolean), isHovered (boolean), isEditing (boolean)
- Events: taskClicked (Task), taskDragStart (Task), taskDragEnd (Task), taskEdited (Task)
- API Binding: None (receives data from parent)

**HeaderComponent:**
- Purpose: Top navigation with search, notifications, and user profile
- Props: user (User), searchQuery (string), notifications (Notification[])
- State: isSearchFocused (boolean), notificationCount (number), isProfileOpen (boolean)
- Events: searchChanged (string), profileClicked (User), notificationClicked (Notification)
- API Binding: GET /users/profile for user data

**SidebarComponent:**
- Purpose: Left navigation with workflow scenarios and quick actions
- Props: activeItem (string), menuItems (MenuItem[]), isCollapsed (boolean)
- State: selectedMenuItem (string), isCollapsed (boolean), quickActionsVisible (boolean)
- Events: menuItemClicked (MenuItem), sidebarToggled (boolean), quickActionClicked (string)
- API Binding: None (static navigation)

**TaskModalComponent:**
- Purpose: Task creation and editing dialog with form validation
- Props: isOpen (boolean), task (Task), mode (ModalMode), formConfig (FormConfig)
- State: formData (TaskFormData), validationErrors (ValidationError[]), isSubmitting (boolean)
- Events: taskSaved (Task), modalClosed (), formValidated (ValidationResult)
- API Binding: POST /forms/sessions (create task), PUT /forms/sessions/{session_id} (update task), POST /documents/upload (attachments)

**SearchContainer:**
- Purpose: Search input with icon and autocomplete functionality
- Props: placeholder (string), value (string), suggestions (SearchSuggestion[])
- State: isFocused (boolean), showSuggestions (boolean), selectedSuggestion (number)
- Events: searchChanged (string), suggestionSelected (SearchSuggestion), searchSubmitted (string)
- API Binding: None (local search functionality)

### 7. DATA FLOW / PROPS

**Parent → Child Mapping:**
- AppComponent → SidebarComponent (menuItems, activeItem, isCollapsed)
- AppComponent → HeaderComponent (user, notifications, searchQuery)
- AppComponent → MainContentComponent (currentView, pageData)
- MainContentComponent → KanbanBoardComponent (columns, tasks, loading, error)
- KanbanBoardComponent → KanbanColumnComponent (column, tasks, isDropTarget)
- KanbanColumnComponent → TaskCardComponent (task, priority, assignee)
- AppComponent → TaskModalComponent (isOpen, selectedTask, mode)
- HeaderComponent → SearchContainer (placeholder, value, suggestions)
- HeaderComponent → UserProfile (user, notifications, isOpen)

**State Management Flow:**
- Global State: User authentication, current view, task data
- Component State: UI interactions, form data, loading states
- Event Flow: User actions → Component events → State updates → API calls → UI updates

### 8. API SUMMARY

**Available APIs:**
- POST /auth/login → User authentication with JWT tokens
- GET /users/profile → User profile data and preferences
- POST /forms/sessions → Create new form session for task management
- GET /forms/sessions/{session_id} → Get session data with tasks and columns
- PUT /forms/sessions/{session_id} → Update session with task changes
- POST /documents/upload → Upload task attachments with quality validation
- GET /documents/{document_id} → Get document status and metadata
- POST /documents/validate → Real-time document quality validation
- GET /components/metadata → Component library information
- GET /health → System health check and service status

### 9. API TO UI MAPPING

**Component → API:**
- AppComponent → POST /auth/login (authentication)
- HeaderComponent → GET /users/profile (user data and notifications)
- KanbanBoardComponent → GET /forms/sessions/{session_id} (load tasks and columns)
- KanbanBoardComponent → PUT /forms/sessions/{session_id} (save task updates and moves)
- TaskModalComponent → POST /forms/sessions (create new tasks)
- TaskModalComponent → PUT /forms/sessions/{session_id} (update existing tasks)
- TaskModalComponent → POST /documents/upload (upload task attachments)
- TaskModalComponent → POST /documents/validate (validate attachments before upload)
- AppComponent → GET /health (system status monitoring)

**Data Transformation:**
- Form sessions → Kanban columns and tasks structure
- User profile → Header user display and preferences
- Document uploads → Task attachment metadata
- API errors → User-friendly error messages

### 13. INTERACTION FLOWS

**User Action → UI → API:**

**Load Kanban Board:**
1. User navigates to Kanban view
2. KanbanBoardComponent.ngOnInit() executes
3. Call GET /forms/sessions/{session_id}
4. Transform API response to column/task structure
5. Display columns and tasks with loading states
6. Handle errors with user-friendly messages

**Create New Task:**
1. User clicks "Create Task" button
2. Open TaskModalComponent with empty form
3. User fills task details (title, description, assignee, priority)
4. Form validation occurs in real-time
5. User submits form
6. Call POST /forms/sessions with task data
7. Update local state and close modal
8. Refresh board display with new task

**Move Task Between Columns:**
1. User drags task card from source column
2. Visual feedback shows drag state
3. User drops task in target column
4. Update local state immediately (optimistic update)
5. Call PUT /forms/sessions/{session_id} with updated task
6. Handle API response and revert if error
7. Show success/error notification

**Upload Task Attachment:**
1. User clicks attachment button in task modal
2. File picker opens for document selection
3. Call POST /documents/validate for quality check
4. Show validation feedback to user
5. If valid, call POST /documents/upload
6. Associate document with task
7. Update task display with attachment indicator

**Search Tasks:**
1. User types in search input
2. Debounced search triggers local filtering
3. Filter tasks by title, description, assignee
4. Update board display with filtered results
5. Show search result count and clear option

**User Authentication:**
1. User enters credentials in login form
2. Call POST /auth/login with credentials
3. Store JWT tokens securely
4. Redirect to Kanban board
5. Set up token refresh mechanism
6. Handle authentication errors gracefully
