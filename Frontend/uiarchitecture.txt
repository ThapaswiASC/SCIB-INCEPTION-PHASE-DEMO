6. COMPONENT DEFINITIONS

TaskCreationComponent:
Purpose: Create new tasks with comprehensive validation and API integration
Props:
- isOpen: boolean (modal visibility)
- onClose: () => void (close handler)
- onTaskCreated: (task: Task) => void (success callback)
State:
- taskForm: FormGroup (reactive form)
- isLoading: boolean (submission state)
- validationErrors: ValidationError[] (form errors)
- characterCounts: object (real-time counters)
Events:
- onSubmit: Form submission with validation
- onCancel: Modal close with confirmation
- onFieldChange: Real-time validation and character counting
API Binding:
- TaskService.createTask(taskData) → POST /api/tasks

KanbanBoardComponent:
Purpose: Display and manage task workflow columns
Props:
- tasks: Task[] (task data)
- columns: Column[] (board configuration)
- onTaskMove: (taskId, newStatus) => void
State:
- draggedTask: Task | null
- columnTasks: Map<string, Task[]>
Events:
- onDragStart, onDragOver, onDrop (drag and drop)
- onTaskClick: (taskId) => void (task detail navigation)
API Binding:
- TaskService.getTasks() → GET /api/tasks
- TaskService.updateTaskStatus() → PUT /api/tasks/{id}

TaskCardComponent:
Purpose: Display individual task information in Kanban columns
Props:
- task: Task (task data)
- onClick: (taskId) => void
- draggable: boolean
State:
- isHovered: boolean
Events:
- onClick: Task selection
- onDragStart: Drag initiation
API Binding: None (display only)

HeaderComponent:
Purpose: Application header with search and user actions
Props:
- user: User (current user data)
- onSearch: (query) => void
- onNotificationClick: () => void
- onSettingsClick: () => void
State:
- searchQuery: string
- notificationCount: number
Events:
- onSearchInput: Real-time search
- onUserMenuToggle: User menu display
API Binding:
- SearchService.searchTasks() → GET /api/tasks?search=query

SidebarComponent:
Purpose: Navigation menu with route management
Props:
- activeRoute: string
- onNavigate: (route) => void
State:
- expandedSections: string[]
Events:
- onMenuItemClick: Route navigation
- onSectionToggle: Menu expansion
API Binding: None (navigation only)

ModalComponent:
Purpose: Reusable modal wrapper with backdrop and accessibility
Props:
- isOpen: boolean
- title: string
- onClose: () => void
- size: 'sm' | 'md' | 'lg'
State:
- isAnimating: boolean
Events:
- onBackdropClick: Close on outside click
- onEscapeKey: Close on escape key
API Binding: None (UI only)

FormFieldComponent:
Purpose: Reusable form input with validation display
Props:
- label: string
- type: 'text' | 'email' | 'select' | 'textarea'
- value: any
- error: string | null
- required: boolean
- maxLength: number
- onChange: (value) => void
State:
- isFocused: boolean
- characterCount: number
Events:
- onFocus, onBlur: Focus state management
- onChange: Value updates with validation
API Binding: None (form control only)

8. API SUMMARY

Available APIs:
- POST /api/tasks → Create new task
- GET /api/tasks → List tasks with filtering and pagination
- GET /api/tasks/{id} → Get specific task details
- GET /health/live → Application liveness check
- GET /health/ready → Application readiness check

Authentication: JWT Bearer token required
Rate Limiting: 100 requests/minute for POST, 200 for GET list, 300 for GET by ID

9. API TO UI MAPPING

Component → API:
- TaskCreationComponent → POST /api/tasks (create task)
- KanbanBoardComponent → GET /api/tasks (load tasks)
- TaskDetailComponent → GET /api/tasks/{id} (task details)
- SearchComponent → GET /api/tasks?search=query (search tasks)
- AnalyticsComponent → GET /api/tasks (aggregate data)

10. API COMPATIBILITY

Compatible:
- Task creation form maps perfectly to CreateTaskRequest schema
- Task display components align with Task and TaskSummary schemas
- Pagination support matches API pagination structure
- Error handling covers all API error responses (400, 401, 403, 409, 422, 429, 500)

Missing:
- Task update/edit API (PUT /api/tasks/{id})
- Task deletion API (DELETE /api/tasks/{id})
- User management APIs for assignee selection
- Real-time collaboration APIs for live updates

Transformations:

- API TaskPriority enum (LOW, MEDIUM, HIGH, CRITICAL) maps to badge variants
- ISO date strings require formatting for UI display
- UUID fields need user-friendly display names

13. INTERACTION FLOWS

User Action → UI → API:

1. Create New Task Flow:
   - Click "Add Task" button
   → Open TaskCreationComponent modal
   → Fill form fields with real-time validation
   → Character counters update on input
   → Click "Create Task" button
   → Show loading spinner
   → Call TaskService.createTask()
   → POST /api/tasks with form data
   → Handle success: show toast, close modal, refresh board
   → Handle error: display validation errors, hide spinner

2. View Task Details Flow:
   - Click task card in Kanban board
   → Navigate to TaskDetailComponent
   → Call TaskService.getTaskById()
   → GET /api/tasks/{id}
   → Display task information and comments
   → Enable task editing and status updates

3. Search Tasks Flow:
   - Type in header search input
   → Real-time search with debouncing
   → Call SearchService.searchTasks()
   → GET /api/tasks?search=query
   → Update Kanban board with filtered results
   → Highlight matching terms

4. Drag and Drop Task Flow:
   - Drag task card to different column
   → Update UI optimistically
   → Call TaskService.updateTaskStatus()
   → PUT /api/tasks/{id} with new status
   → Handle success: confirm UI update
   → Handle error: revert UI change, show error
