### 8. API SUMMARY

**Available APIs:**
- **GET /api/youth-accounts/{youthAccountId}/dashboard** → Fetch dashboard data including balance, spending limit, recent transactions
- **POST /api/youth-accounts/{youthAccountId}/fund-transfer** → Transfer funds to youth account
- **PUT /api/youth-accounts/{youthAccountId}/spending-limit** → Configure weekly spending limits
- **GET /api/youth-accounts/{youthAccountId}/transactions** → Get paginated transaction history with filtering

**Authentication:**
- OAuth 2.0 with JWT tokens
- Required scopes: youth-account:read, youth-account:write, fund-transfer:execute
- Rate limiting: 100 requests per minute per token

**Response Models:**
- DashboardResponse: account balance, spending limit, recent transactions
- FundTransferResponse: transfer status, updated balance, confirmation
- SpendingLimitResponse: updated limits and effective dates
- TransactionHistoryResponse: paginated transaction list with summaries

### 13. INTERACTION FLOWS

**User Action → UI → API Flow:**

1. **View Dashboard:**
   - User clicks "Dashboard" → showPage('dashboard')
   - DashboardComponent loads → GET /api/youth-accounts/{id}/dashboard
   - Display balance, recent transactions, project progress

2. **Create Task:**
   - User clicks "Add Task" → openTaskModal()
   - TaskModalComponent opens → Form validation
   - User submits → POST /api/tasks (missing API)
   - Task added to Kanban board → updateColumnCounts()

3. **Move Task (Drag & Drop):**
   - User drags task → handleDragStart()
   - Drop on column → handleDrop()
   - Update UI immediately → PUT /api/tasks/{id} (missing API)
   - Announce to screen reader → announceToScreenReader()

4. **Transfer Funds (Adapted):**
   - User clicks "Add Funds" → openFundModal()
   - Submit transfer → POST /api/youth-accounts/{id}/fund-transfer
   - Update balance display → refresh dashboard

5. **Configure Limits (Adapted):**
   - User opens settings → showPage('settings')
   - Update spending limit → PUT /api/youth-accounts/{id}/spending-limit
   - Apply to project budget constraints

6. **Search Tasks:**
   - User types in search → handleSearchInput()
   - Submit search → GET /api/search (missing API)
   - Filter and display results
