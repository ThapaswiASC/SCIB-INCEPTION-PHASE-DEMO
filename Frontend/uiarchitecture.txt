## 6. COMPONENT DEFINITIONS

### **KanbanBoardComponent**
**Purpose:** Main container for three-column kanban layout
**Props:**
- columns: KanbanColumn[] - Array of column configurations
- loading: boolean - Loading state indicator
- error: string | null - Error message display

**State:**
- columns: KanbanColumn[] - Column data with tasks
- isLoading: boolean - Loading state
- errorMessage: string | null - Error handling

**Events:**
- taskMoved: EventEmitter<TaskMoveEvent> - Task drag/drop events
- taskSelected: EventEmitter<Task> - Task selection events
- columnAction: EventEmitter<ColumnActionEvent> - Column-level actions

**API Binding:**
- GET /api/youth-accounts/{id}/dashboard - Load initial data
- WebSocket connection for real-time updates

### **KanbanColumnComponent**
**Purpose:** Individual column with header and task list
**Props:**
- column: KanbanColumn - Column configuration and data
- allowDrop: boolean - Enable drag/drop functionality
- readonly: boolean - Read-only mode

**State:**
- tasks: Task[] - Tasks in this column
- isDropTarget: boolean - Drag/drop state

**Events:**
- taskDropped: EventEmitter<TaskDropEvent> - Task drop events
- addTask: EventEmitter<string> - Add task to column
- columnOptionsClicked: EventEmitter<string> - Column menu actions

**API Binding:**
- No direct API calls (managed by parent)

### **TaskCardComponent**
**Purpose:** Individual task display with all task information
**Props:**
- task: Task - Task data object
- draggable: boolean - Enable drag functionality
- selectable: boolean - Enable selection

**State:**
- isDragging: boolean - Drag state
- isSelected: boolean - Selection state

**Events:**
- taskClicked: EventEmitter<Task> - Task selection
- taskDragStart: EventEmitter<Task> - Drag start
- taskDragEnd: EventEmitter<Task> - Drag end

**API Binding:**
- No direct API calls (data passed from parent)

### **TaskModalComponent**
**Purpose:** Task creation and editing form
**Props:**
- task: Task | null - Task to edit (null for new task)
- isOpen: boolean - Modal visibility
- mode: 'create' | 'edit' - Modal mode

**State:**
- formData: TaskFormData - Form state
- isSubmitting: boolean - Submission state
- validationErrors: ValidationError[] - Form validation

**Events:**
- taskSaved: EventEmitter<Task> - Task save event
- modalClosed: EventEmitter<void> - Modal close event

**API Binding:**
- POST /api/tasks - Create new task
- PUT /api/tasks/{id} - Update existing task

### **HeaderComponent**
**Purpose:** Top navigation with search and user actions
**Props:**
- user: User - Current user information
- searchEnabled: boolean - Enable search functionality

**State:**
- searchQuery: string - Search input value
- notifications: Notification[] - User notifications

**Events:**
- searchChanged: EventEmitter<string> - Search query changes
- userMenuClicked: EventEmitter<void> - User menu actions
- notificationClicked: EventEmitter<Notification> - Notification actions

**API Binding:**
- GET /api/search - Search functionality
- GET /api/notifications - User notifications

### **SidebarComponent**
**Purpose:** Left navigation with workflow scenarios and quick actions
**Props:**
- currentView: string - Active view identifier
- collapsed: boolean - Sidebar collapse state

**State:**
- activeMenuItem: string - Currently selected menu item
- isCollapsed: boolean - Collapse state

**Events:**
- viewChanged: EventEmitter<string> - View navigation
- quickActionClicked: EventEmitter<string> - Quick action events
- sidebarToggled: EventEmitter<boolean> - Collapse toggle

**API Binding:**
- No direct API calls (navigation only)

## 8. API SUMMARY

**Available APIs:**
- **GET /api/youth-accounts/{youthAccountId}/dashboard** → Fetch dashboard data with account balance, spending limit, and recent transactions
- **POST /api/youth-accounts/{youthAccountId}/fund-transfer** → Transfer funds from parent to youth account
- **PUT /api/youth-accounts/{youthAccountId}/spending-limit** → Configure weekly spending limits
- **GET /api/youth-accounts/{youthAccountId}/transactions** → Get paginated transaction history with filtering

**Authentication:**
- OAuth 2.0 with JWT tokens
- Required scopes: youth-account:read, youth-account:write, fund-transfer:execute
- Rate limiting: 100 requests per minute

## 9. API TO UI MAPPING

**Component → API:**
- **KanbanBoardComponent** → GET /api/youth-accounts/{id}/dashboard (load initial data)
- **TaskModalComponent** → POST /api/youth-accounts/{id}/fund-transfer (create fund transfer task)
- **TaskModalComponent** → PUT /api/youth-accounts/{id}/spending-limit (update spending limit task)
- **TaskCardComponent** → GET /api/youth-accounts/{id}/transactions (view transaction details)
- **HeaderComponent** → GET /api/youth-accounts/{id}/dashboard (search/filter functionality)

**Data Transformation:**
- Dashboard API response → KanbanColumn[] format
- Transaction history → Task[] objects for display
- Fund transfer requests → Task creation workflow
- Spending limit updates → Task update workflow

## 13. INTERACTION FLOWS

**User Action → UI → API**

1. **Load Dashboard:**
   - User navigates to kanban board
   → KanbanBoardComponent.ngOnInit()
   → GET /api/youth-accounts/{id}/dashboard
   → Display columns with transaction-based tasks

2. **Create Fund Transfer Task:**
   - User clicks "Create Task" button
   → Open TaskModalComponent
   → User fills transfer form
   → POST /api/youth-accounts/{id}/fund-transfer
   → Update kanban board with new task

3. **Update Spending Limit:**
   - User clicks on spending limit task
   → Open TaskModalComponent in edit mode
   → User updates limit value
   → PUT /api/youth-accounts/{id}/spending-limit
   → Update task status and display

4. **View Transaction History:**
   - User clicks on transaction task card
   → TaskCardComponent.taskClicked()
   → GET /api/youth-accounts/{id}/transactions
   → Display detailed transaction information

5. **Search/Filter Tasks:**
   - User types in search box
   → HeaderComponent.searchChanged()
   → Filter local task data or API call
   → Update kanban board display

6. **Drag and Drop Task:**
   - User drags task between columns
   → TaskCardComponent.taskDragStart()
   → KanbanColumnComponent.taskDropped()
   → Update task status locally
   → Sync with backend if needed
