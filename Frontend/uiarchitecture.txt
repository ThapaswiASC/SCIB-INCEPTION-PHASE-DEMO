1. JIRA REQUIREMENT SUMMARY

Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr)). 6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout
Props:
- columns: KanbanColumn[]
- isLoading: boolean
- error: string | null
State:
- selectedTask: Task | null
- draggedTask: Task | null
Events:
- taskSelected: EventEmitter<Task>
- taskMoved: EventEmitter<{task: Task, fromColumn: string, toColumn: string}>
API Binding:
- GET /api/youth-accounts/{id}/dashboard for initial data

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn
- tasks: Task[]
- columnId: string
State:
- isDropTarget: boolean
Events:
- taskDropped: EventEmitter<{task: Task, columnId: string}>
- addTaskClicked: EventEmitter<string>
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task display with all task information
Props:
- task: Task
- isSelected: boolean
State:
- isHovered: boolean
Events:
- taskClicked: EventEmitter<Task>
- taskDragStart: EventEmitter<Task>
API Binding:
- None (receives data from parent)

TaskModalComponent:
Purpose: Modal for creating and editing tasks
Props:
- isOpen: boolean
- task: Task | null (for editing)
- availableAssignees: User[]
State:
- formData: TaskFormData
- isSubmitting: boolean
Events:
- taskCreated: EventEmitter<Task>
- taskUpdated: EventEmitter<Task>
- modalClosed: EventEmitter<void>
API Binding:
- POST /api/youth-accounts/{id}/fund-transfer for task creation
- PUT /api/youth-accounts/{id}/spending-limit for task updates

HeaderComponent:
Purpose: Top navigation with search and user actions
Props:
- user: User
- notificationCount: number
State:
- searchQuery: string
Events:
- searchChanged: EventEmitter<string>
- notificationClicked: EventEmitter<void>
- settingsClicked: EventEmitter<void>
- userProfileClicked: EventEmitter<void>
API Binding:
- None (receives data from parent)

SidebarComponent (from library):
Purpose: Navigation sidebar with menu items and user profile
Props:
- width: string
- collapsedWidth: string
- isCollapsed: boolean
- menuItems: MenuItem[]
State:
- activeMenuItem: string
Events:
- collapseToggle: EventEmitter<boolean>
- menuItemSelected: EventEmitter<string>
API Binding:
- None (static navigation)

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data with account balance, spending limit, and recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (adapted for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits (adapted for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history (adapted for task history)

9. API TO UI MAPPING

Component → API:
- KanbanBoardComponent → GET /api/youth-accounts/{id}/dashboard (initial load)
- TaskModalComponent → POST /api/youth-accounts/{id}/fund-transfer (create task)
- TaskModalComponent → PUT /api/youth-accounts/{id}/spending-limit (update task)
- TaskCardComponent → GET /api/youth-accounts/{id}/transactions (task history)

13. INTERACTION FLOWS

User Action → UI → API:

1. Load Dashboard:
   → AppComponent.ngOnInit()
   → Call GET /api/youth-accounts/{id}/dashboard
   → Transform response to kanban data
   → Update KanbanBoardComponent with columns and tasks

2. Create Task:
   → Click "Create Task" button
   → Open TaskModalComponent
   → Fill form and submit
   → Call POST /api/youth-accounts/{id}/fund-transfer
   → Transform request data
   → Update local state and refresh board

3. Move Task:
   → Drag task from one column to another
   → Update local state immediately
   → Call PUT /api/youth-accounts/{id}/spending-limit
   → Handle success/error response

4. Search Tasks:
   → Type in search input
   → Filter local task data
   → Update displayed tasks in real-time

5. View Task Details:
   → Click on task card
   → Show task details in modal or side panel
   → Load additional data if needed
