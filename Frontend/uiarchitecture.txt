6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container component implementing the three-column Kanban layout
Props:
- columns: KanbanColumn[] - Array of column configurations
- tasks: Task[] - Array of all tasks
- loading: boolean - Loading state
- error: string | null - Error message
State:
- selectedTask: Task | null - Currently selected task
- draggedTask: Task | null - Task being dragged
Events:
- taskMoved: EventEmitter<TaskMoveEvent> - Task moved between columns
- taskSelected: EventEmitter<Task> - Task selected
- taskCreated: EventEmitter<CreateTaskRequest> - New task created
API Binding:
- GET /api/tasks - Fetch tasks
- POST /api/tasks - Create new task

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn - Column configuration
- tasks: Task[] - Tasks in this column
- allowDrop: boolean - Whether column accepts drops
State:
- isDropTarget: boolean - Column is drop target
Events:
- taskDropped: EventEmitter<TaskDropEvent> - Task dropped in column
- addTaskClicked: EventEmitter<string> - Add task button clicked
API Binding:
- None (delegates to parent)

TaskCardComponent:
Purpose: Individual task display with all task information
Props:
- task: Task - Task data
- draggable: boolean - Whether task can be dragged
- selectable: boolean - Whether task can be selected
State:
- isSelected: boolean - Task is selected
- isDragging: boolean - Task is being dragged
Events:
- taskClicked: EventEmitter<Task> - Task clicked
- taskDragStart: EventEmitter<Task> - Drag started
- taskDragEnd: EventEmitter<Task> - Drag ended
API Binding:
- None (displays data only)

HeaderComponent:
Purpose: Application header with title and actions
Props:
- title: string - Page title
- subtitle: string - Page subtitle
- showActions: boolean - Show action buttons
State:
- None
Events:
- settingsClicked: EventEmitter<void> - Settings button clicked
- addTaskClicked: EventEmitter<void> - Add task button clicked
API Binding:
- None

SidebarComponent:
Purpose: Navigation sidebar with brand and menu
Props:
- navigationItems: NavigationItem[] - Menu items
- activeItem: string - Currently active menu item
State:
- collapsed: boolean - Sidebar collapsed state
Events:
- navigationChanged: EventEmitter<string> - Navigation item selected
API Binding:
- None

PriorityIndicatorComponent:
Purpose: Visual priority indicator dot

- priority: TaskPriority - Task priority level

Events:
- None

- None

7. DATA FLOW / PROPS

Parent в†’ Child Mapping:
KanbanBoardComponent в†’ HeaderComponent (title, subtitle, showActions)
KanbanBoardComponent в†’ SidebarComponent (navigationItems, activeItem)
KanbanBoardComponent в†’ KanbanColumnComponent (column, tasks, allowDrop)
KanbanColumnComponent в†’ TaskCardComponent (task, draggable, selectable)
TaskCardComponent в†’ PriorityIndicatorComponent (priority, size)
TaskCardComponent в†’ StatusIndicatorComponent (status, type)
TaskCardComponent в†’ AssigneeAvatarComponent (user, size)
TaskCardComponent в†’ TaskTagComponent (tag, color)

Data Flow:
1. KanbanBoardComponent fetches tasks from API
2. Tasks are filtered and passed to respective columns
3. Columns render individual task cards
4. User interactions bubble up through event emitters
5. Parent component handles API calls and state updates

8. API SUMMARY

Available APIs:
- POST /api/tasks в†’ Create new task
- GET /health в†’ Basic health check
- GET /health/detailed в†’ Detailed system health

API Models:
- CreateTaskRequest: {title, description?, status?, priority?, dueDate?, assignedTo?}
- TaskResponse: {id, title, description?, status, priority, dueDate?, createdAt, updatedAt, createdBy, assignedTo?}
- TaskStatus: 'TODO' | 'IN_PROGRESS' | 'DONE'
- TaskPriority: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL'

Authentication:
- OAuth 2.0 Bearer token required
- Scopes: tasks:create, tasks:read, tasks:update, tasks:delete

9. API TO UI MAPPING

Component в†’ API:
- KanbanBoardComponent в†’ GET /api/tasks (fetch all tasks)
- KanbanBoardComponent в†’ POST /api/tasks (create new task)
- HeaderComponent в†’ POST /api/tasks (add task button)
- KanbanColumnComponent в†’ POST /api/tasks (add task in column)

API Response Mapping:
- TaskResponse.status в†’ Column assignment (TODO в†’ To Do, IN_PROGRESS в†’ In Progress, DONE в†’ Done)
- TaskResponse.priority в†’ PriorityIndicatorComponent color
- TaskResponse.assignedTo в†’ AssigneeAvatarComponent user data
- TaskResponse.createdAt/updatedAt в†’ Task metadata display
