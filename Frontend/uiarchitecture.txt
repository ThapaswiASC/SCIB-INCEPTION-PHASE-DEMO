Implement error handling with *ngIf directive to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose:** Main container for three-column Kanban layout
- **Props:**
  - columns: KanbanColumn[]
  - loading: boolean
  - error: string | null
- **State:**
  - currentView: 'desktop' | 'mobile'
  - activeColumn: string (for mobile)
- **Events:**
  - columnChange: EventEmitter<string>
  - taskMove: EventEmitter<TaskMoveEvent>
- **API Binding:**
  - GET /api/youth-accounts/{id}/dashboard for initial data

**KanbanColumnComponent:**
- **Purpose:** Individual column container (To Do, In Progress, Done)
- **Props:**
  - column: KanbanColumn
  - tasks: Task[]
  - isActive: boolean
- **State:**
  - dragOver: boolean
  - taskCount: number
- **Events:**
  - taskDrop: EventEmitter<TaskDropEvent>
  - addTask: EventEmitter<string>
- **API Binding:** None (receives data from parent)

**TaskCardComponent:**
- **Purpose:** Individual task display and interaction
- **Props:**
  - task: Task
  - draggable: boolean
- **State:**
  - isDragging: boolean
  - isSelected: boolean
- **Events:**
  - taskClick: EventEmitter<Task>
  - taskDragStart: EventEmitter<Task>
- **API Binding:** None (receives data from parent)

**HeaderComponent:**
- **Purpose:** Top navigation and search functionality
- **Props:**
  - user: User
  - notifications: Notification[]
- **State:**
  - sidebarOpen: boolean
  - theme: 'light' | 'dark'
- **Events:**
  - sidebarToggle: EventEmitter<boolean>
  - themeChange: EventEmitter<string>
- **API Binding:** None (receives data from parent)

**TaskModalComponent:**
- **Purpose:** Task creation and editing modal
- **Props:**
  - isOpen: boolean
  - task: Task | null
  - defaultStatus: string
- **State:**
  - formData: TaskFormData
  - isSubmitting: boolean
- **Events:**
  - taskCreate: EventEmitter<Task>
  - taskUpdate: EventEmitter<Task>
  - modalClose: EventEmitter<void>
- **API Binding:**
  - POST /api/youth-accounts/{id}/fund-transfer for task creation

### 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data including balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits (can be repurposed for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history (can be repurposed for task history)

### 9. API TO UI MAPPING

**Component → API:**
- DashboardComponent → GET /api/youth-accounts/{id}/dashboard
- KanbanBoardComponent → GET /api/youth-accounts/{id}/dashboard (for initial task data)
- TaskModalComponent → POST /api/youth-accounts/{id}/fund-transfer (for task creation)
- TaskCardComponent → PUT /api/youth-accounts/{id}/spending-limit (for task updates)
- AllTasksComponent → GET /api/youth-accounts/{id}/transactions (for task history)

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **Load Kanban Board:**
   - User navigates to Kanban page
   → KanbanBoardComponent loads
   → Call GET /api/youth-accounts/{id}/dashboard
   → Display columns and tasks

2. **Create New Task:**
   - User clicks "Add Task" button (aava-button)
   → Open TaskModalComponent
   → User fills form and submits
   → Call POST /api/youth-accounts/{id}/fund-transfer (adapted)
   → Update UI with new task

3. **Move Task Between Columns:**
   - User drags task card
   → TaskCardComponent emits dragStart
   → KanbanColumnComponent handles drop
   → Call PUT /api/youth-accounts/{id}/spending-limit (adapted)
   → Update task status in UI

4. **Search Tasks:**
   - User types in search bar (aava-search-bar)
   → SearchBarComponent emits searchChange
   → Filter tasks in KanbanBoardComponent
   → Update displayed tasks

5. **Toggle Mobile View:**
   - User clicks mobile tab
   → MobileTabsComponent emits tabChange
   → KanbanBoardComponent updates activeColumn
   → Show/hide columns based on selection
