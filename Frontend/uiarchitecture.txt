### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose:** Main container for the three-column kanban layout
- **Props:** 
  - columns: KanbanColumn[] - Array of column configurations
  - loading: boolean - Loading state
  - error: string | null - Error message
- **State:**
  - draggedTask: Task | null - Currently dragged task
  - columnData: KanbanColumn[] - Column data with tasks
- **Events:**
  - taskMoved: (task: Task, fromColumn: string, toColumn: string) => void
  - taskCreated: (task: Task, column: string) => void
- **API Binding:** None (receives data from parent)

**KanbanColumnComponent:**
- **Purpose:** Individual column container with header and task list
- **Props:**
  - column: KanbanColumn - Column configuration and data
  - tasks: Task[] - Array of tasks in this column
  - allowDrop: boolean - Whether column accepts drops
- **State:**
  - isDragOver: boolean - Drag over state
  - taskCount: number - Number of tasks
- **Events:**
  - taskDropped: (task: Task) => void
  - addTaskClicked: () => void
- **API Binding:** None (receives data from parent)

**TaskCardComponent:**
- **Purpose:** Individual task card with drag-and-drop functionality
- **Props:**
  - task: Task - Task data object
  - draggable: boolean - Whether card is draggable
  - focused: boolean - Focus state for keyboard navigation
- **State:**
  - isDragging: boolean - Drag state
- **Events:**
  - taskClicked: (task: Task) => void
  - dragStarted: (task: Task) => void
  - dragEnded: () => void
- **API Binding:** None (receives data from parent)

**TaskModalComponent:**
- **Purpose:** Modal for creating and editing tasks
- **Props:**
  - isOpen: boolean - Modal visibility
  - task: Task | null - Task to edit (null for new task)
  - defaultColumn: string - Default column for new tasks
- **State:**
  - formData: TaskFormData - Form input data
  - isSubmitting: boolean - Submission state
  - validationErrors: ValidationError[] - Form validation errors
- **Events:**
  - taskSaved: (task: Task) => void
  - modalClosed: () => void
- **API Binding:** Task creation/update APIs

**HeaderComponent:**
- **Purpose:** Top navigation bar with search and user controls
- **Props:**
  - user: User - Current user data
  - notifications: Notification[] - User notifications
- **State:**
  - searchQuery: string - Search input value
  - notificationsOpen: boolean - Notifications dropdown state
- **Events:**
  - searchSubmitted: (query: string) => void
  - themeToggled: () => void
  - sidebarToggled: () => void
- **API Binding:** Search API, Notifications API

**SidebarComponent:**
- **Purpose:** Left navigation panel with menu items
- **Props:**
  - isOpen: boolean - Sidebar visibility (mobile)
  - currentPage: string - Active page identifier
- **State:**
  - expandedSections: string[] - Expanded menu sections
- **Events:**
  - navigationClicked: (page: string) => void
  - sidebarClosed: () => void
- **API Binding:** None (static navigation)

### 7. DATA FLOW / PROPS

**Parent → Child Mapping:**
- **AppComponent** → HeaderComponent (user, notifications)
- **AppComponent** → SidebarComponent (isOpen, currentPage)
- **AppComponent** → KanbanBoardComponent (columns, loading, error)
- **KanbanBoardComponent** → KanbanColumnComponent (column, tasks, allowDrop)
- **KanbanColumnComponent** → TaskCardComponent (task, draggable, focused)
- **AppComponent** → TaskModalComponent (isOpen, task, defaultColumn)

**Data Flow Pattern:**
1. AppComponent manages global state
2. API data flows down through props
3. User interactions bubble up through events
4. State updates trigger re-renders

### 8. API SUMMARY

**Available APIs:**
- **GET /api/youth-accounts/{youthAccountId}/dashboard** → Fetch dashboard data
- **POST /api/youth-accounts/{youthAccountId}/fund-transfer** → Transfer funds
- **PUT /api/youth-accounts/{youthAccountId}/spending-limit** → Update spending limit
- **GET /api/youth-accounts/{youthAccountId}/transactions** → Get transaction history

**API Characteristics:**
- OAuth 2.0 authentication required
- Rate limiting: 100 requests per minute
- JSON request/response format
- Comprehensive error handling
- Pagination support for transactions

### 9. API TO UI MAPPING

**Component → API:**
- **KanbanBoardComponent** → GET /dashboard (for initial data load)
- **TaskCardComponent** → GET /transactions (for task-related transactions)
- **TaskModalComponent** → POST /fund-transfer (for fund-related tasks)
- **HeaderComponent** → GET /dashboard (for notifications and user data)

**Data Transformation Required:**
- Dashboard API returns financial data, needs mapping to task structure
- Transaction history needs conversion to task activity timeline
- Fund transfer operations need mapping to task creation/updates

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **View Dashboard**
   - User navigates to kanban board
   - KanbanBoardComponent loads
   - Calls GET /dashboard API
   - Displays account data as project overview

2. **Create Task**
   - User clicks "Add Task" button
   - TaskModalComponent opens
   - User fills form and submits
   - Calls POST /fund-transfer API (simulating task creation)
   - Updates local state and closes modal

3. **Move Task (Drag & Drop)**
   - User drags task card
   - TaskCardComponent handles drag events
   - Drop triggers column update
   - Calls PUT /spending-limit API (simulating status change)
   - Updates UI with new task position

4. **Search Tasks**
   - User types in search input
   - SearchComponent filters local data
   - Calls GET /transactions API for detailed search
   - Updates task list with filtered results

5. **View Task Details**
   - User clicks task card
   - TaskModalComponent opens in edit mode
   - Displays task information
   - Calls GET /transactions API for task history
