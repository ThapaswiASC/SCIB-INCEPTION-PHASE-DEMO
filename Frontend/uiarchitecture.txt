### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose:** Main container for three-column Kanban layout
- **Props:**
  - `columns: KanbanColumn[]` - Array of column configurations
  - `tasks: Task[]` - Array of all tasks
  - `loading: boolean` - Loading state
  - `error: string | null` - Error message
- **State:**
  - `draggedTask: Task | null` - Currently dragged task
  - `columnData: KanbanColumn[]` - Column data with tasks
- **Events:**
  - `taskMoved: EventEmitter<TaskMoveEvent>` - Task moved between columns
  - `taskCreated: EventEmitter<Task>` - New task created
  - `taskUpdated: EventEmitter<Task>` - Task updated
- **API Binding:**
  - GET /api/youth-accounts/{id}/dashboard - Fetch dashboard data
  - POST /api/youth-accounts/{id}/fund-transfer - Create fund transfer tasks

**KanbanColumnComponent:**
- **Purpose:** Individual column container with header and task list
- **Props:**
  - `column: KanbanColumn` - Column configuration
  - `tasks: Task[]` - Tasks for this column
  - `allowDrop: boolean` - Whether column accepts drops
- **State:**
  - `isDropTarget: boolean` - Drop target state
- **Events:**
  - `taskDropped: EventEmitter<TaskDropEvent>` - Task dropped in column
  - `addTaskClicked: EventEmitter<string>` - Add task button clicked

**TaskCardComponent:**
- **Purpose:** Individual task display with drag/drop functionality
- **Props:**
  - `task: Task` - Task data
  - `draggable: boolean` - Whether task is draggable
- **State:**
  - `isDragging: boolean` - Drag state
- **Events:**
  - `taskClicked: EventEmitter<Task>` - Task clicked for editing
  - `dragStarted: EventEmitter<Task>` - Drag operation started

**HeaderComponent:**
- **Purpose:** Top navigation bar with breadcrumbs and user menu
- **Props:**
  - `currentPage: string` - Current page identifier
  - `user: User` - Current user data
- **State:**
  - `searchQuery: string` - Search input value
  - `userMenuOpen: boolean` - User menu state
- **Events:**
  - `searchSubmitted: EventEmitter<string>` - Search performed
  - `themeToggled: EventEmitter<void>` - Theme toggle clicked

**ModalComponent:**
- **Purpose:** Task creation/editing modal dialog
- **Props:**
  - `isOpen: boolean` - Modal visibility
  - `task: Task | null` - Task being edited (null for new)
  - `defaultColumn: string` - Default column for new tasks
- **State:**
  - `formData: TaskFormData` - Form input data
  - `isSubmitting: boolean` - Submission state
- **Events:**
  - `taskSaved: EventEmitter<Task>` - Task saved
  - `modalClosed: EventEmitter<void>` - Modal closed
- **API Binding:**
  - POST /api/youth-accounts/{id}/fund-transfer - Create fund transfer
  - PUT /api/youth-accounts/{id}/spending-limit - Update spending limits

### 7. DATA FLOW / PROPS

**Parent в†’ Child Mapping:**
- AppComponent в†’ HeaderComponent (user, currentPage)
- AppComponent в†’ SidebarComponent (navigationItems, currentPage)
- AppComponent в†’ KanbanBoardComponent (columns, tasks, loading, error)
- KanbanBoardComponent в†’ KanbanColumnComponent (column, tasks, allowDrop)
- KanbanColumnComponent в†’ TaskCardComponent (task, draggable)
- AppComponent в†’ ModalComponent (isOpen, task, defaultColumn)

**Data Flow:**
1. AppComponent fetches dashboard data from API
2. Data flows down to KanbanBoardComponent
3. KanbanBoardComponent distributes tasks to columns
4. User interactions bubble up through events
5. AppComponent handles API calls and state updates

### 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard в†’ Fetch dashboard data with balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer в†’ Transfer funds (create fund transfer tasks)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit в†’ Configure spending limits (update limit tasks)
- GET /api/youth-accounts/{youthAccountId}/transactions в†’ Get transaction history with pagination

**Authentication:** OAuth 2.0 with JWT tokens
**Rate Limiting:** 100 requests per minute per token
**Supported Currencies:** USD, EUR, GBP

### 9. API TO UI MAPPING

**Component в†’ API:**
- KanbanBoardComponent в†’ GET /api/youth-accounts/{id}/dashboard (load initial data)
- TaskCardComponent (Fund Transfer) в†’ POST /api/youth-accounts/{id}/fund-transfer (create transfer)
- TaskCardComponent (Spending Limit) в†’ PUT /api/youth-accounts/{id}/spending-limit (update limit)
- DashboardComponent в†’ GET /api/youth-accounts/{id}/dashboard (dashboard overview)
- AllTasksComponent в†’ GET /api/youth-accounts/{id}/transactions (transaction history)

**Data Mapping:**
- Dashboard API response maps to Kanban columns and task cards
- Fund transfer requests create "Fund Transfer" tasks in appropriate columns
- Spending limit updates create "Limit Configuration" tasks
- Transaction history populates "All Tasks" view and recent activity

### 10. API COMPATIBILITY

**Compatible:**
- Dashboard API fully supports UI dashboard and Kanban board data requirements
- Fund transfer API supports task creation workflow
- Spending limit API supports configuration tasks
- Transaction history API supports task listing and filtering

**Missing:**
- No dedicated task management endpoints (tasks are represented as financial operations)
- No task status update endpoints (status changes are implicit in financial operations)
- No task assignment or collaboration features

**Transformations:**
- Financial operations (transfers, limits) are presented as Kanban tasks
- Transaction history is transformed into task-like items for "All Tasks" view
- API responses need transformation to match Kanban task structure
- Task priorities and categories are derived from transaction types and amounts

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **Load Kanban Board**
   - User navigates to Kanban page
   - KanbanBoardComponent calls GET /api/youth-accounts/{id}/dashboard
   - Transform API response to Kanban structure
   - Render three columns with tasks

2. **Create Fund Transfer Task**
   - User clicks "Add Task" in To Do column
   - Modal opens with fund transfer form
   - User submits form
   - Call POST /api/youth-accounts/{id}/fund-transfer
   - Create task card in "In Progress" column
   - Update dashboard data

3. **Update Spending Limit**
   - User creates spending limit task
   - Modal opens with limit configuration
   - User sets new limit
   - Call PUT /api/youth-accounts/{id}/spending-limit
   - Move task to "Done" column
   - Update dashboard display

4. **Drag and Drop Task**
   - User drags task between columns
   - Update local state immediately
   - Trigger appropriate API call based on task type
   - Handle API response and update UI accordingly

5. **View Transaction History**
   - User clicks "All Tasks" navigation
   - Call GET /api/youth-accounts/{id}/transactions
   - Transform transactions to task-like display
   - Render in table format with pagination
