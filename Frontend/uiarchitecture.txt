## UI COMPONENT ARCHITECTURE

1. JIRA REQUIREMENT SUMMARY

Story Description:
Create Angular component for Kanban board container with three-column layout at src/app/features/kanban/components/kanban-board/kanban-board.component.ts. Implement component class with @Component decorator and selector 'app-kanban-board'. Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr)). Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header. Implement error handling with *ngIf directive to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column Kanban board layout
Props:
- columns: KanbanColumn[]
- loading: boolean
- error: string | null
State:
- draggedTask: Task | null
- columnData: KanbanColumn[]
Events:
- onTaskMove: (taskId: string, fromColumn: string, toColumn: string) => void
- onTaskCreate: (columnId: string) => void
- onTaskEdit: (taskId: string) => void
API Binding:
- GET /kanban/boards/{boardId}/columns
- PUT /kanban/tasks/{taskId}/move

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean
State:
- isDropTarget: boolean
- taskCount: number
Events:
- onTaskDrop: (task: Task) => void
- onAddTask: () => void
API Binding:
- GET /kanban/columns/{columnId}/tasks

TaskCardComponent:
Purpose: Individual task card with drag/drop functionality
Props:
- task: Task
- draggable: boolean
- selected: boolean
State:
- isDragging: boolean
Events:
- onDragStart: (task: Task) => void
- onDragEnd: () => void
- onClick: (task: Task) => void
API Binding:
- GET /tasks/{taskId}
- PUT /tasks/{taskId}

HeaderComponent:
Purpose: Top navigation bar with search and user actions
Props:
- user: User
- notifications: Notification[]
State:
- searchQuery: string
Events:
- onSearch: (query: string) => void
- onNotificationClick: () => void
API Binding:
- GET /users/profile
- GET /notifications

SidebarComponent:
Purpose: Left navigation menu with brand and navigation items
Props:
- navigationItems: NavigationItem[]
- activeItem: string
State:
- collapsed: boolean
Events:
- onNavigate: (itemId: string) => void
- onToggle: () => void
API Binding:
- GET /navigation/menu

ModalComponent:
Purpose: Reusable modal overlay for forms and dialogs
Props:
- isOpen: boolean
- title: string
- size: 'small' | 'medium' | 'large'
State:
- focusedElement: HTMLElement | null
Events:
- onClose: () => void
- onSubmit: (data: any) => void
API Binding: None (presentation only)

- POST /auth/login → User authentication

- GET /users/profile → User profile data
- PUT /users/profile → Update user profile
- POST /forms/sessions → Create form session
- GET /forms/sessions/{id} → Get form session
- PUT /forms/sessions/{id} → Update form session
- POST /documents/upload → Upload documents
- GET /documents/{id} → Get document status

- GET /components/metadata → UI component metadata

13. INTERACTION FLOWS

User Action → UI → API:

1. Load Kanban Board
   → KanbanBoardComponent.ngOnInit()
   → GET /forms/sessions/{sessionId}
   → Update columns and tasks state

2. Drag Task Between Columns
   → TaskCardComponent.onDragStart()
   → KanbanColumnComponent.onDrop()
   → PUT /forms/sessions/{sessionId}
   → Update task position

3. Create New Task
   → Click "Add Task" button
   → Open TaskCreationModal
   → Submit form
   → POST /forms/sessions (create new session)
   → Add task to column

4. Upload Task Attachment
   → Click attachment button in TaskCard
   → Open file picker
   → POST /documents/validate (quality check)
   → POST /documents/upload (if valid)
   → Update task with document reference

5. User Authentication
   → Enter credentials
   → POST /auth/login
   → Store JWT token
   → Redirect to Kanban board

6. Search Tasks
   → Type in search field
   → Filter local task data
   → Highlight matching tasks

7. Update User Profile
   → Click user avatar
   → Open profile modal
   → PUT /users/profile
   → Update header display
