## UI COMPONENT ARCHITECTURE

1. JIRA REQUIREMENT SUMMARY

Story Description:
Create Angular component for Kanban board container with three-column layout. Implementation details include creating Angular component at src/app/features/kanban/components/kanban-board/kanban-board.component.ts with @Component decorator and selector 'app-kanban-board'. Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr)). Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header. Implement error handling with *ngIf directive to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for kanban board with three-column layout
Props:
- columns: KanbanColumn[]
- loading: boolean
- error: string | null
State:
- draggedTask: Task | null
- columnData: KanbanColumn[]
Events:
- taskMoved: EventEmitter<TaskMoveEvent>
- taskCreated: EventEmitter<Task>
- taskUpdated: EventEmitter<Task>
API Binding:
- GET /api/youth-accounts/{id}/dashboard for initial data

KanbanColumnComponent:
Purpose: Individual column container with tasks
Props:
- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean
State:
- isDropTarget: boolean
Events:
- taskDropped: EventEmitter<TaskDropEvent>
- addTaskClicked: EventEmitter<string>
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task display with drag functionality
Props:
- task: Task
- draggable: boolean
- priority: 'high' | 'medium' | 'low'
State:
- isDragging: boolean
Events:
- taskClicked: EventEmitter<Task>
- taskDragStart: EventEmitter<Task>
API Binding:
- None (receives data from parent)

HeaderComponent:
Purpose: Top navigation with search and user menu
Props:
- user: User
- notifications: Notification[]
State:
- searchQuery: string
- isUserMenuOpen: boolean
Events:
- searchSubmitted: EventEmitter<string>
- themeToggled: EventEmitter<void>
- sidebarToggled: EventEmitter<void>
API Binding:
- None (receives data from parent)

SidebarComponent:
Purpose: Navigation sidebar with menu items
Props:
- navigationItems: NavigationItem[]
- isOpen: boolean
State:
- activeRoute: string
Events:
- navigationClicked: EventEmitter<string>
- sidebarClosed: EventEmitter<void>
API Binding:
- None (static navigation data)

ModalComponent:
Purpose: Reusable modal container
Props:
- isOpen: boolean
- title: string
- size: 'sm' | 'md' | 'lg'
State:
- None
Events:
- modalClosed: EventEmitter<void>
API Binding:
- None (content provided via projection)

TaskFormComponent:
Purpose: Form for creating/editing tasks
Props:
- task: Task | null
- mode: 'create' | 'edit'
State:
- formData: TaskFormData
- isSubmitting: boolean
Events:
- taskSubmitted: EventEmitter<Task>
- formCancelled: EventEmitter<void>
API Binding:
- POST /api/youth-accounts/{id}/fund-transfer for task creation

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data including balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits (can be repurposed for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history (can be repurposed for task history)

9. API TO UI MAPPING

Component → API:
- KanbanBoardComponent → GET /api/youth-accounts/{id}/dashboard (initial load)
- TaskFormComponent → POST /api/youth-accounts/{id}/fund-transfer (create task)
- TaskFormComponent → PUT /api/youth-accounts/{id}/spending-limit (update task)
- DashboardPageComponent → GET /api/youth-accounts/{id}/dashboard (dashboard data)
- TaskHistoryComponent → GET /api/youth-accounts/{id}/transactions (task history)

13. INTERACTION FLOWS

User Action → UI → API:

1. Load Kanban Board
   → KanbanBoardComponent.ngOnInit()
   → GET /api/youth-accounts/{id}/dashboard
   → Display columns and tasks

2. Create New Task
   → Click "Add Task" button
   → Open TaskFormComponent in Modal
   → Fill form and submit
   → POST /api/youth-accounts/{id}/fund-transfer
   → Update kanban board state

3. Move Task Between Columns
   → Drag task card
   → Drop in new column
   → PUT /api/youth-accounts/{id}/spending-limit
   → Update task status

4. View Task Details
   → Click task card
   → Open task detail modal
   → Display task information

5. Search Tasks
   → Type in search bar
   → Filter displayed tasks
   → No API call (client-side filtering)

6. Toggle Theme
   → Click theme button
   → Update CSS custom properties
   → Store preference in localStorage
