### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for the three-column Kanban layout
- Props:
  - columns: KanbanColumn[] - Array of column configurations
  - tasks: Task[] - Array of all tasks
  - loading: boolean - Loading state
  - error: string | null - Error message
- State:
  - draggedTask: Task | null - Currently dragged task
  - columnData: ColumnData[] - Column-specific data
- Events:
  - taskMoved: EventEmitter<TaskMoveEvent>
  - taskCreated: EventEmitter<Task>
  - taskUpdated: EventEmitter<Task>
- API Binding:
  - GET /api/youth-accounts/{id}/dashboard - Fetch initial data

**KanbanColumnComponent:**
- Purpose: Individual column container with tasks
- Props:
  - columnId: string - Column identifier
  - title: string - Column title
  - tasks: Task[] - Tasks in this column
  - count: number - Task count
- State:
  - isDragOver: boolean - Drag over state
- Events:
  - taskDropped: EventEmitter<TaskDropEvent>
  - addTaskClicked: EventEmitter<string>
- API Binding: None (receives data from parent)

**TaskCardComponent:**
- Purpose: Individual task display and interaction
- Props:
  - task: Task - Task data
  - draggable: boolean - Whether task is draggable
- State:
  - isDragging: boolean - Drag state
- Events:
  - taskClicked: EventEmitter<Task>
  - taskDragStart: EventEmitter<Task>
- API Binding: None (receives data from parent)

**HeaderComponent:**
- Purpose: Top navigation and user controls
- Props:
  - user: User - Current user data
  - breadcrumbs: Breadcrumb[] - Navigation breadcrumbs
- State:
  - searchQuery: string - Search input value
  - notificationCount: number - Unread notifications
- Events:
  - searchSubmitted: EventEmitter<string>
  - themeToggled: EventEmitter<void>
  - menuToggled: EventEmitter<void>
- API Binding: None (receives data from parent)

**ModalComponent:**
- Purpose: Task creation and editing modal
- Props:
  - isOpen: boolean - Modal visibility
  - task: Task | null - Task to edit (null for new)
- State:
  - formData: TaskFormData - Form state
  - isSubmitting: boolean - Submission state
- Events:
  - taskSubmitted: EventEmitter<Task>
  - modalClosed: EventEmitter<void>
- API Binding:
  - POST /api/youth-accounts/{id}/fund-transfer - Create task (mapped)

### 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard в†’ Fetch dashboard data
- POST /api/youth-accounts/{youthAccountId}/fund-transfer в†’ Add funds (mapped to task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit в†’ Update limit (mapped to task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions в†’ Get transaction history (mapped to task history)

### 9. API TO UI MAPPING

**Component в†’ API:**
- KanbanBoardComponent в†’ GET /api/youth-accounts/{id}/dashboard (fetch initial data)

- TaskCardComponent (Update Task) в†’ PUT /api/youth-accounts/{id}/spending-limit (update task)
- TaskHistoryComponent в†’ GET /api/youth-accounts/{id}/transactions (fetch task history)

**Data Transformation:**

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **Load Kanban Board:**
   - User navigates to Kanban page
   - KanbanBoardComponent calls GET /api/youth-accounts/{id}/dashboard
   - Transform API response to task/column data
   - Render three columns with tasks

2. **Create New Task:**
   - User clicks "Add Task" button
   - Open ModalComponent with empty form
   - User fills task details and submits
   - Call POST /api/youth-accounts/{id}/fund-transfer (adapted)
   - Update local state and refresh board

3. **Move Task Between Columns:**
   - User drags task to different column
   - Update local state immediately
   - Call PUT /api/youth-accounts/{id}/spending-limit (adapted)
   - Handle success/error responses

4. **Search Tasks:**
   - User types in search input
   - Filter local task data
   - Optionally call API for server-side search

5. **Mobile Column Navigation:**
   - User taps mobile tab
   - Switch active column view
   - Maintain keyboard accessibility
