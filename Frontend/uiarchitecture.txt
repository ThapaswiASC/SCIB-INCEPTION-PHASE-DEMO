## UI COMPONENT ARCHITECTURE

### 1. JIRA REQUIREMENT SUMMARY

**Story Description:**
Create Angular component for Kanban board container with three-column layout

**Acceptance Criteria:**
- KanbanBoardComponent created with proper Angular structure and decorators
- Template renders three distinct column containers with proper semantic HTML
- CSS Grid layout implemented with responsive breakpoints for desktop, tablet, and mobile
- ARIA attributes added for accessibility compliance
- Error state template implemented with conditional rendering
- Component compiles without errors and passes linting checks

**UI Tasks:**
- Implement component class with @Component decorator and selector 'app-kanban-board'
- Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr))
- Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px)
- Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header
- Implement error handling with *ngIf directive to display error message when column configuration fails
- Use Angular Material or custom CSS for styling
- Add responsive meta tags and viewport configuration

### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main kanban board container with three-column layout
- Props:
  - columns: KanbanColumn[] - Array of column configurations
  - tasks: Task[] - Array of all tasks
  - loading: boolean - Loading state
- State:
  - draggedTask: Task | null - Currently dragged task
  - error: string | null - Error message
- Events:
  - taskMoved: EventEmitter<TaskMoveEvent> - Task moved between columns
  - taskCreated: EventEmitter<Task> - New task created
- API Binding:
  - GET /youth-accounts/{id}/dashboard - Fetch dashboard data

**KanbanColumnComponent:**
- Purpose: Individual column container with tasks
- Props:
  - column: KanbanColumn - Column configuration
  - tasks: Task[] - Tasks in this column
  - allowDrop: boolean - Enable drop functionality
- State:
  - isDragOver: boolean - Drag over state
- Events:
  - taskDropped: EventEmitter<TaskDropEvent> - Task dropped in column
  - addTaskClicked: EventEmitter<string> - Add task button clicked
- API Binding: None (receives data from parent)

**TaskCardComponent:**
- Purpose: Individual task card with drag functionality
- Props:
  - task: Task - Task data
  - draggable: boolean - Enable drag functionality
- State:
  - isDragging: boolean - Dragging state
- Events:
  - taskClicked: EventEmitter<Task> - Task clicked
  - dragStarted: EventEmitter<Task> - Drag started
- API Binding: None (receives data from parent)

**HeaderComponent:**
- Purpose: Top navigation header with search and user menu
- Props:
  - user: User - Current user data
  - notifications: Notification[] - User notifications
- State:
  - searchQuery: string - Search input value
- Events:
  - searchSubmitted: EventEmitter<string> - Search performed
  - themeToggled: EventEmitter<void> - Theme toggle clicked
- API Binding: None (receives data from parent)

**SidebarComponent:**
- Purpose: Left navigation sidebar
- Props:
  - isOpen: boolean - Sidebar open state
  - currentPage: string - Active page
- State:
  - None
- Events:
  - navigationClicked: EventEmitter<string> - Navigation item clicked
- API Binding: None

**ModalComponent:**
- Purpose: Task creation/editing modal
- Props:
  - isOpen: boolean - Modal visibility
  - task: Task | null - Task to edit (null for new)
- State:
  - formData: TaskFormData - Form state
- Events:
  - taskSaved: EventEmitter<Task> - Task saved
  - modalClosed: EventEmitter<void> - Modal closed
- API Binding:
  - POST /youth-accounts/{id}/fund-transfer - Create task (simulated)

### 7. DATA FLOW / PROPS

**Parent → Child Mapping:**
- AppComponent → HeaderComponent (user, notifications)
- AppComponent → SidebarComponent (isOpen, currentPage)
- AppComponent → KanbanBoardComponent (columns, tasks, loading)
- KanbanBoardComponent → KanbanColumnComponent (column, tasks, allowDrop)
- KanbanColumnComponent → TaskCardComponent (task, draggable)

### 8. API SUMMARY

**Available APIs:**
- GET /youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data with account balance, spending limits, recent transactions
- POST /youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (can be adapted for task creation)
- PUT /youth-accounts/{youthAccountId}/spending-limit → Update spending limits (can be adapted for task updates)
- GET /youth-accounts/{youthAccountId}/transactions → Fetch transaction history (can be adapted for task history)

### 9. API TO UI MAPPING

**Component → API:**
- KanbanBoardComponent → GET /youth-accounts/{id}/dashboard (fetch initial data)
- KanbanBoardComponent → GET /youth-accounts/{id}/transactions (fetch task history)
- ModalComponent → POST /youth-accounts/{id}/fund-transfer (create new task - adapted)
- ModalComponent → PUT /youth-accounts/{id}/spending-limit (update task - adapted)

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **Load Kanban Board:**
   - User navigates to kanban page
   - KanbanBoardComponent calls GET /youth-accounts/{id}/dashboard
   - Transform dashboard data to kanban structure
   - Render columns and tasks

2. **Create New Task:**
   - User clicks "Add Task" button
   - Open ModalComponent
   - User fills form and submits
   - Call POST /youth-accounts/{id}/fund-transfer (adapted)
   - Add new task to appropriate column
   - Update column counts

3. **Move Task Between Columns:**
   - User drags task card
   - Update local state immediately
   - Call PUT /youth-accounts/{id}/spending-limit (adapted)
   - Handle success/error states

4. **Search Tasks:**
   - User types in search input
   - Filter tasks locally or call API
   - Update displayed tasks

5. **Toggle Theme:**
   - User clicks theme toggle
   - Update CSS custom properties
   - Save preference to localStorage
