6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout
Props:
- columns: KanbanColumn[] (array of column configurations)
- tasks: Task[] (array of all tasks)
- boardId: string (unique board identifier)
State:
- loading: boolean (loading state)
- error: string | null (error message)
- draggedTask: Task | null (currently dragged task)
Events:
- taskMoved: EventEmitter<TaskMoveEvent>
- taskCreated: EventEmitter<Task>
- taskUpdated: EventEmitter<Task>
API Binding:
- GET /api/boards/{boardId}/tasks (fetch tasks)
- POST /api/boards/{boardId}/tasks (create task)
- PUT /api/boards/{boardId}/tasks/{taskId} (update task)

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn (column configuration)
- tasks: Task[] (tasks for this column)
- allowDrop: boolean (enable drop functionality)
State:
- taskCount: number (number of tasks in column)
- isDropTarget: boolean (drag over state)
Events:
- taskDropped: EventEmitter<TaskDropEvent>
- addTaskClicked: EventEmitter<string>
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task display with drag/drop capability
Props:
- task: Task (task data)
- draggable: boolean (enable drag functionality)
- clickable: boolean (enable click to edit)
State:
- isDragging: boolean (drag state)
- isSelected: boolean (selection state)
Events:
- taskClicked: EventEmitter<Task>
- taskDragStart: EventEmitter<Task>
- taskDragEnd: EventEmitter<void>
API Binding:
- None (receives data from parent)

TaskModalComponent:
Purpose: Task creation/editing modal form
Props:
- task: Task | null (task to edit, null for new)
- visible: boolean (modal visibility)
- boardId: string (board context)
State:
- formData: TaskFormData (form state)
- isSubmitting: boolean (submission state)
- validationErrors: ValidationError[] (form errors)
Events:
- taskSaved: EventEmitter<Task>
- modalClosed: EventEmitter<void>
API Binding:
- POST /api/boards/{boardId}/tasks (create task)
- PUT /api/boards/{boardId}/tasks/{taskId} (update task)
- POST /api/boards/{boardId}/tasks/validate (validate task)

SidebarNavigationComponent:
Purpose: Main navigation sidebar with menu items
Props:
- currentPage: string (active page identifier)
- user: User (current user info)
State:
- isCollapsed: boolean (sidebar collapse state)
- activeMenuItem: string (active menu item)
Events:
- pageChanged: EventEmitter<string>
- userMenuClicked: EventEmitter<void>
API Binding:
- None (navigation only)

HeaderComponent:
Purpose: Top header with search and user actions
Props:
- user: User (current user info)
- notifications: Notification[] (user notifications)
State:
- searchQuery: string (search input value)
- showNotifications: boolean (notification panel state)
- showUserMenu: boolean (user menu state)
Events:
- searchChanged: EventEmitter<string>
- notificationClicked: EventEmitter<Notification>
- userActionClicked: EventEmitter<string>
API Binding:
- GET /api/search (search functionality)
- GET /api/notifications (fetch notifications)

AnalyticsPanelComponent:
Purpose: Dashboard analytics display with metrics
Props:
- boardId: string (board context)
- timeRange: string (analytics time range)
State:
- metrics: AnalyticsMetrics (performance data)
- loading: boolean (loading state)
- error: string | null (error state)
Events:
- refreshClicked: EventEmitter<void>
- timeRangeChanged: EventEmitter<string>
API Binding:
- GET /api/boards/{boardId}/analytics (fetch metrics)

8. API SUMMARY

Available APIs:
- GET /api/boards/{boardId}/tasks → Fetch all tasks for board
- POST /api/boards/{boardId}/tasks → Create new task
- PUT /api/boards/{boardId}/tasks/{taskId} → Update existing task
- DELETE /api/boards/{boardId}/tasks/{taskId} → Delete task
- POST /api/boards/{boardId}/tasks/validate → Validate task data
- POST /api/boards/{boardId}/tasks/batch → Create multiple tasks
- GET /api/boards/{boardId}/analytics → Fetch board analytics
- GET /api/search → Search tasks and users
- GET /api/notifications → Fetch user notifications
- GET /health → Service health check
- WebSocket: wss://api.scib.com/ws/boards/{boardId} → Real-time updates

13. INTERACTION FLOWS

User Action → UI → API:

1. Create New Task:
   - Click "Add Task" button
   → Open TaskModalComponent
   → Fill form and submit
   → Call POST /api/boards/{boardId}/tasks
   → WebSocket broadcast TASK_CREATED event
   → Update all connected clients

2. Move Task Between Columns:
   - Drag task card
   → Update TaskCardComponent state (isDragging)
   → Drop on KanbanColumnComponent
   → Call PUT /api/boards/{boardId}/tasks/{taskId}
   → WebSocket broadcast TASK_UPDATED event
   → Sync across all clients

3. Edit Existing Task:
   - Click task card
   → Open TaskModalComponent with task data
   → Modify form and submit
   → Call PUT /api/boards/{boardId}/tasks/{taskId}
   → WebSocket broadcast TASK_UPDATED event
   → Update UI optimistically

4. Search Tasks:
   - Type in search input
   → HeaderComponent captures input
   → Call GET /api/search with query
   → Filter and highlight results
   → Update KanbanBoardComponent display

5. View Analytics:
   - Navigate to Analytics page
   → AnalyticsPanelComponent loads
   → Call GET /api/boards/{boardId}/analytics
   → Display metrics and charts
   → Auto-refresh every 30 seconds

6. Real-time Synchronization:
   - WebSocket connection established
   → Listen for TASK_CREATED, TASK_UPDATED, TASK_DELETED events
   → Update local state without API calls
   → Animate changes for user feedback
   → Handle connection errors gracefully
