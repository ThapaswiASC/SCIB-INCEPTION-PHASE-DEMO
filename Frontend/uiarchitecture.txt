6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout with drag-drop functionality
Props:
- columns: KanbanColumn[] = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- loading: boolean = false
- error: string | null = null
State:
- draggedTask: TaskCard | null = null
- columnCounts: {[key: string]: number} = {}
Events:
- taskMoved: EventEmitter<{taskId: string, fromColumn: string, toColumn: string}>
- taskSelected: EventEmitter<string>
- addTaskClicked: EventEmitter<string>
API Binding:
- GET /youth-accounts/{youthAccountId}/dashboard for initial data

KanbanColumnComponent:
Purpose: Individual column container with header, task list, and add button
Props:
- column: KanbanColumn
- tasks: TaskCard[]
- allowDrop: boolean = true
State:
- isDragOver: boolean = false
- taskCount: number = 0
Events:
- taskDropped: EventEmitter<{task: TaskCard, columnId: string}>
- addTaskClicked: EventEmitter<string>
- menuToggled: EventEmitter<string>
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task display with drag functionality and task details
Props:
- task: TaskCard
- draggable: boolean = true
- clickable: boolean = true
State:
- isDragging: boolean = false
Events:
- taskClicked: EventEmitter<string>
- dragStarted: EventEmitter<TaskCard>
- dragEnded: EventEmitter<void>
API Binding:
- None (receives data from parent)

HeaderComponent:
Purpose: Top navigation bar with search and user actions
Props:
- user: User
- notifications: Notification[]
State:
- searchQuery: string = ''
- notificationsOpen: boolean = false
- userMenuOpen: boolean = false
Events:
- searchSubmitted: EventEmitter<string>
- notificationClicked: EventEmitter<void>
- userMenuToggled: EventEmitter<void>
API Binding:
- GET /notifications for notification data

SidebarNavigationComponent:
Purpose: Left navigation panel with menu items and brand
Props:
- currentPage: string
- navigationItems: NavigationItem[]
State:
- activeItem: string = 'dashboard'
Events:
- navigationChanged: EventEmitter<string>
API Binding:
- None (static navigation)

TaskModalComponent:
Purpose: Modal for creating and editing tasks
Props:
- task: TaskCard | null = null
- visible: boolean = false
- mode: 'create' | 'edit' = 'create'
State:
- formData: TaskFormData
- isSubmitting: boolean = false
- validationErrors: {[key: string]: string} = {}
Events:
- taskSaved: EventEmitter<TaskCard>
- modalClosed: EventEmitter<void>
API Binding:
- POST /tasks for creating tasks
- PUT /tasks/{id} for updating tasks

ButtonComponent (from library):
Purpose: Reusable button with multiple variants and states
Props:
- label: string = ''
- variant: 'default' | 'primary' | 'secondary' | 'success' | 'warning' | 'danger' | 'info' = 'default'
- size: 'xs' | 'sm' | 'md' | 'lg' | 'xl' = 'md'
- disabled: boolean = false
- processing: boolean = false
- iconName: string = ''
- iconPosition: 'left' | 'right' | 'only' = 'left'
State:
- isPressed: boolean = false
Events:
- userClick: EventEmitter<void>
API Binding:
- None (pure UI component)

8. API SUMMARY

Available APIs:
- GET /youth-accounts/{youthAccountId}/dashboard → Fetch youth account dashboard data
- GET /notifications → Fetch user notifications
- POST /tasks → Create new task
- PUT /tasks/{id} → Update existing task
- DELETE /tasks/{id} → Delete task
- GET /tasks → Fetch tasks with filtering
- POST /auth/login → User authentication
- GET /users/profile → Get user profile

9. API TO UI MAPPING

- KanbanBoardComponent → GET /youth-accounts/{youthAccountId}/dashboard

- TaskModalComponent → POST /tasks, PUT /tasks/{id}
- DashboardPageComponent → GET /youth-accounts/{youthAccountId}/dashboard
- AnalyticsPageComponent → GET /analytics/performance, GET /analytics/metrics

13. INTERACTION FLOWS

User Action → UI → API:

1. Load Dashboard:
   → Navigate to dashboard
   → KanbanBoardComponent.ngOnInit()
   → Call GET /youth-accounts/{youthAccountId}/dashboard
   → Update component state with dashboard data

2. Create Task:
   → Click "Add Task" button
   → Open TaskModalComponent
   → Fill task form
   → Submit form
   → Call POST /tasks
   → Update kanban board with new task
   → Close modal

3. Move Task (Drag & Drop):
   → Drag task card
   → Drop in new column
   → Update UI immediately
   → Call PUT /tasks/{id} with new status
   → Handle API response

4. Edit Task:
   → Click task card
   → Open TaskModalComponent in edit mode
   → Modify task details
   → Submit form
   → Call PUT /tasks/{id}
   → Update task display
   → Close modal

5. Search Tasks:
   → Type in search input
   → Debounce input
   → Call GET /tasks with search filter
   → Update task display

6. View Analytics:
   → Navigate to analytics page
   → Call GET /analytics/performance
   → Display metrics and charts
