## 6. COMPONENT DEFINITIONS

### KanbanBoardComponent
**Purpose:** Main kanban board container with three-column layout
**Props:**
- columns: KanbanColumn[] - Array of column configurations
- tasks: Task[] - Array of all tasks
- loading: boolean - Loading state indicator
**State:**
- draggedTask: Task | null - Currently dragged task
- error: string | null - Error message for display
**Events:**
- taskMoved: (taskId: string, newColumnId: string) => void
- taskCreated: (task: Task) => void
- taskUpdated: (task: Task) => void
**API Binding:**
- GET /api/youth-accounts/{id}/dashboard for initial data

### KanbanColumnComponent
**Purpose:** Individual column within kanban board
**Props:**
- column: KanbanColumn - Column configuration
- tasks: Task[] - Tasks for this column
- allowDrop: boolean - Whether column accepts drops
**State:**
- isDropTarget: boolean - Visual drop target state
**Events:**
- taskDropped: (taskId: string) => void
- addTaskClicked: () => void
**API Binding:** None (receives data from parent)

### TaskCardComponent
**Purpose:** Individual task card with drag-and-drop functionality
**Props:**
- task: Task - Task data object
- draggable: boolean - Whether card is draggable
**State:**
- isDragging: boolean - Dragging state
**Events:**
- taskClicked: (taskId: string) => void
- taskDragStart: (taskId: string) => void
**API Binding:** None (receives data from parent)

### HeaderComponent
**Purpose:** Application header with navigation and user controls
**Props:**
- user: User - Current user information
- notifications: Notification[] - User notifications
**State:**
- searchQuery: string - Current search input
- showUserMenu: boolean - User menu visibility
**Events:**
- searchSubmitted: (query: string) => void
- themeToggled: () => void
- userMenuToggled: () => void
**API Binding:** None (receives data from parent)

### ModalComponent
**Purpose:** Reusable modal for task creation/editing
**Props:**
- isOpen: boolean - Modal visibility state
- title: string - Modal title
- task?: Task - Task data for editing
**State:**
- formData: TaskFormData - Form input values
- isSubmitting: boolean - Form submission state
**Events:**
- modalClosed: () => void
- taskSubmitted: (task: Task) => void
**API Binding:**
- POST /api/youth-accounts/{id}/fund-transfer for task creation

## 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard в†’ Fetch dashboard data with account balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer в†’ Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit в†’ Update spending limits (can be repurposed for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions в†’ Get transaction history with pagination (can be repurposed for task history)

**Component в†’ API:**
- KanbanBoardComponent в†’ GET /api/youth-accounts/{id}/dashboard (fetch initial kanban data)

## 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **Create New Task**
   - Click "Add Task" button в†’ Open Modal в†’ Fill form в†’ Submit
   - Call POST /api/youth-accounts/{id}/fund-transfer (adapted for task creation)
   - Update local state в†’ Refresh kanban board

2. **Move Task Between Columns**
   - Drag task card в†’ Drop in new column в†’ Visual feedback
   - Call PUT /api/youth-accounts/{id}/spending-limit (adapted for task updates)
   - Update task status в†’ Refresh column counts

3. **View Task Details**
   - Click task card в†’ Open task detail modal
   - Display task information в†’ Allow editing
   - Save changes via API в†’ Update UI state

4. **Search Tasks**
   - Type in search bar в†’ Filter tasks locally
   - Advanced search в†’ Call GET /api/youth-accounts/{id}/transactions (adapted)
   - Display filtered results в†’ Maintain search state

5. **Mobile Column Navigation**
   - Tap mobile tab в†’ Switch active column
   - Update tab state в†’ Show/hide columns
   - Maintain responsive behavior
