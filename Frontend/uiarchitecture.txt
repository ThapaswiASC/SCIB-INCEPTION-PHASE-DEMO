## UI COMPONENT ARCHITECTURE

### 1. JIRA REQUIREMENT SUMMARY

**Story Description:**
Create Angular component for Kanban board container with three-column layout. Implementation details include creating Angular component at src/app/features/kanban/components/kanban-board/kanban-board.component.ts with @Component decorator and selector 'app-kanban-board'. Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr)). Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header. Implement error handling with *ngIf directive to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

**Acceptance Criteria:**
- KanbanBoardComponent created with proper Angular structure and decorators
- Template renders three distinct column containers with proper semantic HTML
- CSS Grid layout implemented with responsive breakpoints for desktop, tablet, and mobile
- ARIA attributes added for accessibility compliance
- Error state template implemented with conditional rendering
- Component compiles without errors and passes linting checks

### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose:** Main container for three-column kanban layout
- **Props:** 
  - columns: KanbanColumn[]
  - loading: boolean
  - error: string | null
- **State:**
  - draggedTask: Task | null
  - columnData: KanbanColumn[]
- **Events:**
  - taskMoved: (taskId: string, fromColumn: string, toColumn: string)
  - taskSelected: (taskId: string)
- **API Binding:**
  - GET /youth-accounts/{id}/dashboard for task data

**KanbanColumnComponent:**
- **Purpose:** Individual column container with header and task list
- **Props:**
  - column: KanbanColumn
  - tasks: Task[]
  - allowDrop: boolean
- **State:**
  - isDragOver: boolean
  - taskCount: number
- **Events:**
  - taskDropped: (task: Task)
  - addTaskClicked: (columnId: string)
- **API Binding:** None (receives data from parent)

**TaskCardComponent:**
- **Purpose:** Individual task display with drag/drop functionality
- **Props:**
  - task: Task
  - draggable: boolean
  - clickable: boolean
- **State:**
  - isDragging: boolean
- **Events:**
  - taskClicked: (taskId: string)
  - dragStarted: (task: Task)
- **API Binding:** None (receives data from parent)

**HeaderComponent:**
- **Purpose:** Top navigation bar with search and user actions
- **Props:**
  - user: User
  - notificationCount: number
- **State:**
  - searchQuery: string
  - showNotifications: boolean
- **Events:**
  - searchSubmitted: (query: string)
  - notificationClicked: ()
- **API Binding:** GET /user/profile, GET /notifications

**SidebarComponent:**
- **Purpose:** Left navigation panel with menu items
- **Props:**
  - activeRoute: string
  - collapsed: boolean
- **State:**
  - selectedMenuItem: string
- **Events:**
  - menuItemSelected: (route: string)
- **API Binding:** None (navigation only)

**ButtonComponent (from library):**
- **Purpose:** Reusable button with multiple variants
- **Props:**
  - label: string
  - variant: 'default' | 'primary' | 'secondary' | 'success' | 'warning' | 'danger'
  - size: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
  - disabled: boolean
  - processing: boolean
- **Events:**
  - userClick: ()

**AvatarComponent (from library):**
- **Purpose:** User profile display with status indicators
- **Props:**
  - imageUrl: string
  - altText: string
  - size: 'xxs' | 'xs' | 'sm' | 'md' | 'lg' | 'xl' | 'xxl'
  - badgeCount: number
- **Events:** None

**SearchBarComponent (from library):**
- **Purpose:** Search input with integrated functionality

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **Load Dashboard**
   - User navigates to dashboard
   - DashboardPageComponent loads
   - Call GET /youth-accounts/{id}/dashboard
   - Display account balance, spending limits, recent activity

2. **View Kanban Board**
   - User clicks "View Kanban Board" button
   - Navigate to KanbanBoardComponent
   - Load task data (simulated from dashboard API)
   - Display three-column layout with tasks

3. **Drag and Drop Task**
   - User drags task card
   - TaskCardComponent emits dragStarted event
   - KanbanColumnComponent handles drop
   - Update task status locally
   - (Future: Call PUT /tasks/{id} API)

4. **Create New Task**
   - User clicks "Add Task" button
   - Open task modal (ModalComponent)
   - User fills form and submits
   - (Future: Call POST /tasks API)
   - Update local task list

5. **Search Tasks**
   - User types in SearchBarComponent
   - Emit searchChange event
   - Filter displayed tasks locally
   - (Future: Call GET /tasks?search={query})

6. **User Authentication**
   - User login triggers OAuth flow
   - Store JWT token
   - Load user profile data
   - Update HeaderComponent with user info
