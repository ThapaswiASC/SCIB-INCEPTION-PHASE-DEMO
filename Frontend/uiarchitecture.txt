## 6. COMPONENT DEFINITIONS

### KanbanBoardComponent
**Purpose:** Main container for the three-column Kanban board
**Props:**
- columns: KanbanColumn[] - Array of column configurations
- loading: boolean - Loading state indicator
- error: string | null - Error message display

**State:**
- columns: KanbanColumn[] - Current column data
- selectedTask: Task | null - Currently selected task
- isLoading: boolean - Loading state
- errorMessage: string | null - Error state

**Events:**
- taskCreated: EventEmitter<Task> - Emitted when new task is created
- taskUpdated: EventEmitter<Task> - Emitted when task is updated
- taskMoved: EventEmitter<{task: Task, fromColumn: string, toColumn: string}> - Emitted when task is moved

**API Binding:**
- GET /api/youth-accounts/{youthAccountId}/dashboard - Fetch dashboard data
- GET /api/youth-accounts/{youthAccountId}/transactions - Fetch transaction history

### KanbanColumnComponent
**Purpose:** Individual column container with header and task list
**Props:**
- column: KanbanColumn - Column configuration and data
- tasks: Task[] - Array of tasks in this column
- allowDrop: boolean - Whether tasks can be dropped in this column

**State:**
- isExpanded: boolean - Column expansion state
- dragOver: boolean - Drag over state

**Events:**
- taskDropped: EventEmitter<{task: Task, column: string}> - Emitted when task is dropped
- addTaskClicked: EventEmitter<string> - Emitted when add task button is clicked
- columnOptionsClicked: EventEmitter<string> - Emitted when column options are clicked

**API Binding:** None (receives data from parent)

### TaskCardComponent
**Purpose:** Individual task card display with all task information
**Props:**
- task: Task - Task data object
- draggable: boolean - Whether card can be dragged
- selectable: boolean - Whether card can be selected

**State:**
- isDragging: boolean - Drag state
- isSelected: boolean - Selection state
- isHovered: boolean - Hover state

**Events:**
- taskClicked: EventEmitter<Task> - Emitted when task is clicked
- taskDragStart: EventEmitter<Task> - Emitted when drag starts
- taskDragEnd: EventEmitter<Task> - Emitted when drag ends

**API Binding:** None (receives data from parent)

### HeaderComponent
**Purpose:** Top navigation header with search and user actions
**Props:**
- user: User - Current user information
- showSearch: boolean - Whether to show search bar

**State:**
- searchQuery: string - Current search query
- notificationCount: number - Number of unread notifications

**Events:**
- searchChanged: EventEmitter<string> - Emitted when search query changes
- notificationClicked: EventEmitter<void> - Emitted when notifications are clicked
- settingsClicked: EventEmitter<void> - Emitted when settings are clicked
- profileClicked: EventEmitter<void> - Emitted when profile is clicked

**API Binding:** None (receives data from parent)

### SidebarNavigationComponent
**Purpose:** Left sidebar navigation with menu items and quick actions
**Props:**
- currentPage: string - Currently active page
- navigationItems: NavigationItem[] - Array of navigation items

**State:**
- isCollapsed: boolean - Sidebar collapse state
- activeItem: string - Currently active navigation item

**Events:**
- navigationChanged: EventEmitter<string> - Emitted when navigation item is selected
- quickActionClicked: EventEmitter<string> - Emitted when quick action is clicked

**API Binding:** None (static navigation)

### TaskModalComponent (Using Dialog Library)
**Purpose:** Modal dialog for creating and editing tasks
**Props:**
- task: Task | null - Task to edit (null for new task)
- assignees: User[] - Available assignees
- columns: string[] - Available columns

**State:**
- formData: TaskFormData - Form data object
- isSubmitting: boolean - Submission state
- validationErrors: ValidationError[] - Form validation errors

**Events:**
- taskSaved: EventEmitter<Task> - Emitted when task is saved
- modalClosed: EventEmitter<void> - Emitted when modal is closed

**API Binding:**
- POST /api/youth-accounts/{youthAccountId}/fund-transfer - Create new task (mapped to fund transfer)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit - Update task (mapped to spending limit)

## 7. DATA FLOW / PROPS

**Parent → Child Mapping:**
- AppComponent → SidebarNavigationComponent (navigationItems, currentPage)
- AppComponent → MainContentComponent (user, currentPage)
- MainContentComponent → HeaderComponent (user, showSearch)
- MainContentComponent → ContentAreaComponent (currentPage)
- ContentAreaComponent → KanbanBoardComponent (columns, tasks, loading, error)
- KanbanBoardComponent → KanbanColumnComponent (column, tasks, allowDrop)
- KanbanColumnComponent → TaskCardComponent (task, draggable, selectable)
- KanbanBoardComponent → TaskModalComponent (task, assignees, columns)

## 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data with account balance and recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (mapped to task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Configure spending limit (mapped to task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history (mapped to task history)

## 9. API TO UI MAPPING

**Component → API:**
- KanbanBoardComponent → GET /api/youth-accounts/{youthAccountId}/dashboard (fetch initial data)
- KanbanBoardComponent → GET /api/youth-accounts/{youthAccountId}/transactions (fetch task history)
- TaskModalComponent → POST /api/youth-accounts/{youthAccountId}/fund-transfer (create new task)
- TaskModalComponent → PUT /api/youth-accounts/{youthAccountId}/spending-limit (update task)

**Data Transformation:**
- Dashboard API response → Kanban columns and tasks
- Transaction history → Task activity timeline
- Fund transfer request → Task creation data
- Spending limit update → Task status/priority update

## 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **Load Kanban Board**
   → KanbanBoardComponent.ngOnInit()
   → Call GET /api/youth-accounts/{id}/dashboard
   → Transform response to column data
   → Render three columns with tasks

2. **Create New Task**
   → Click "Create Task" button
   → Open TaskModalComponent (Dialog)
   → Fill form and submit
   → Call POST /api/youth-accounts/{id}/fund-transfer
   → Add new task to appropriate column
   → Close modal and refresh board

3. **Move Task Between Columns**
   → Drag TaskCardComponent
   → Drop in different KanbanColumnComponent
   → Call PUT /api/youth-accounts/{id}/spending-limit
   → Update task status
   → Refresh column counts

4. **Search Tasks**
   → Type in SearchBarComponent
   → Filter displayed tasks locally
   → Optionally call transaction API for server-side search

5. **View Task Details**
   → Click on TaskCardComponent
   → Open TaskModalComponent in view/edit mode
   → Display task information
   → Allow editing and saving
