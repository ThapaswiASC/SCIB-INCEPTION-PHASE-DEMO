## UI COMPONENT ARCHITECTURE

1. JIRA REQUIREMENT SUMMARY

Story Description:
Create Angular component for Kanban board container with three-column layout at src/app/features/kanban/components/kanban-board/kanban-board.component.ts. Implement component class with @Component decorator and selector 'app-kanban-board'. Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr)). Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header. Implement error handling with *ngIf directive to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

Acceptance Criteria:
- KanbanBoardComponent created with proper Angular structure and decorators
- Template renders three distinct column containers with proper semantic HTML
- CSS Grid layout implemented with responsive breakpoints for desktop, tablet, and mobile
- ARIA attributes added for accessibility compliance
- Error state template implemented with conditional rendering
- Component compiles without errors and passes linting checks

6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout with error handling
Props:
- columns: KanbanColumn[]
- loading: boolean
- error: string | null
State:
- columns: [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- draggedTask: TaskCard | null
- errorMessage: string | null
Events:
- taskMoved: EventEmitter<{taskId: string, fromColumn: string, toColumn: string}>
- taskSelected: EventEmitter<string>
- errorOccurred: EventEmitter<string>
API Binding:
- GET /youth-accounts/{id}/dashboard for initial data

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn
- tasks: TaskCard[]
- columnId: string
State:
- isDragOver: boolean
- taskCount: number
Events:
- taskDropped: EventEmitter<DropEvent>
- addTaskClicked: EventEmitter<string>
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task display with drag/drop functionality
Props:
- task: TaskCard
- draggable: boolean
State:
- isDragging: boolean
Events:
- taskClicked: EventEmitter<string>
- dragStarted: EventEmitter<TaskCard>
API Binding:
- None (receives data from parent)

HeaderComponent:
Purpose: Top navigation bar with search and actions
Props:
- user: User
- notifications: Notification[]
State:
- searchQuery: string
- showNotifications: boolean
Events:
- searchChanged: EventEmitter<string>
- notificationClicked: EventEmitter<void>
API Binding:
- GET /notifications for notification data

SidebarComponent:
Purpose: Left navigation menu
Props:
- activeRoute: string
- navigationItems: NavItem[]
State:
- isCollapsed: boolean
Events:
- routeChanged: EventEmitter<string>
API Binding:
- None (static navigation)

TaskModalComponent:
Purpose: Task creation/editing modal using AavaModalComponent
Props:
- task: TaskCard | null
- isVisible: boolean
State:
- formData: TaskFormData
- isSubmitting: boolean
Events:
- taskSaved: EventEmitter<TaskCard>
- modalClosed: EventEmitter<void>
API Binding:
- POST /tasks for task creation
- PUT /tasks/{id} for task updates

AavaButtonComponent (from library):
Purpose: Reusable button with multiple variants and states
Props:
- label: string
- variant: 'default' | 'primary' | 'secondary' | 'success' | 'warning' | 'danger' | 'info'
- size: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
- disabled: boolean
- processing: boolean
- iconName: string
- iconPosition: 'left' | 'right' | 'only'
Events:
- userClick: EventEmitter

AavaModalComponent (from library):
Purpose: Modal dialog with service-based management
Props:
- width: string
- maxWidth: string
Events:
- modalClosed: EventEmitter<void>
API Binding:
- Managed via AavaDialogService

AavaCheckboxComponent (from library):
Purpose: Checkbox input with variants
Props:
- label: string
- isChecked: boolean
- variant: 'default' | 'with-bg' | 'animated'
- size: 'sm' | 'md' | 'lg'
- disabled: boolean
- indeterminate: boolean
Events:
- isCheckedChange: EventEmitter<boolean>
