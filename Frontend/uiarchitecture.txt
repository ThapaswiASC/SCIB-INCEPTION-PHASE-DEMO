### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for the Kanban board application
- Props: 
  - columns: KanbanColumn[]
  - tasks: Task[]
  - loading: boolean
  - error: string | null
- State:
  - selectedTask: Task | null
  - draggedTask: Task | null
  - filterOptions: FilterOptions
- Events:
  - onTaskMove(taskId: string, targetColumn: string)
  - onTaskCreate(task: Task)
  - onTaskUpdate(task: Task)
  - onTaskDelete(taskId: string)
- API Binding:
  - GET /api/youth-accounts/{id}/dashboard
  - GET /api/youth-accounts/{id}/transactions

**KanbanColumnComponent:**
- Purpose: Individual column container (Todo, In Progress, Done)
- Props:
  - column: KanbanColumn
  - tasks: Task[]
  - allowDrop: boolean
- State:
  - isDragOver: boolean
  - isLoading: boolean
- Events:
  - onDrop(event: DragEvent)
  - onDragOver(event: DragEvent)
  - onAddTask(columnId: string)
- API Binding: None (delegates to parent)

**TaskCardComponent:**
- Purpose: Individual task card with drag/drop functionality
- Props:
  - task: Task
  - draggable: boolean
  - selected: boolean
- State:
  - isDragging: boolean
- Events:
  - onDragStart(event: DragEvent)
  - onDragEnd(event: DragEvent)
  - onClick(task: Task)
- API Binding: None

**TaskModalComponent:**
- Purpose: Modal for creating/editing tasks
- Props:
  - isOpen: boolean
  - task: Task | null
  - mode: 'create' | 'edit'
- State:
  - formData: TaskFormData
  - isSubmitting: boolean
  - validationErrors: ValidationError[]
- Events:
  - onSubmit(taskData: TaskFormData)
  - onCancel()
  - onClose()
- API Binding:
  - POST /api/youth-accounts/{id}/fund-transfer (for task creation)

**HeaderComponent:**
- Purpose: Application header with search and user actions
- Props:
  - user: User
  - notifications: Notification[]
- State:
  - searchQuery: string
  - showNotifications: boolean
- Events:
  - onSearch(query: string)
  - onNotificationClick()
  - onUserMenuClick()
- API Binding: None

**SidebarComponent:**
- Purpose: Navigation sidebar with menu items
- Props:
  - menuItems: MenuItem[]
  - activeItem: string
  - collapsed: boolean
- State:
  - expandedSections: string[]
- Events:
  - onMenuItemClick(itemId: string)
  - onToggleCollapse()
- API Binding: None

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **Load Dashboard**
   - User navigates to Kanban board
   - KanbanBoardComponent calls GET /api/youth-accounts/{id}/dashboard
   - Display account balance, spending limits, and recent transactions as task cards

2. **Create Fund Transfer Task**
   - User clicks "Add Task" button
   - TaskModalComponent opens with form
   - User fills transfer details
   - Submit calls POST /api/youth-accounts/{id}/fund-transfer
   - New task card appears in appropriate column

3. **Update Spending Limit**
   - User creates "Set Limit" task
   - TaskModalComponent opens with limit form
   - Submit calls PUT /api/youth-accounts/{id}/spending-limit
   - Task moves to "Done" column on success

4. **View Transaction History**
   - User clicks on transaction-related task
   - Component calls GET /api/youth-accounts/{id}/transactions
   - Display detailed transaction history in modal

5. **Drag and Drop Tasks**
   - User drags task card between columns
   - Update local state immediately
   - Sync with backend APIs based on task type
