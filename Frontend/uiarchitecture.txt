### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container component managing the entire Kanban board
- Props: columns (array), loading (boolean), error (string)
- State: draggedTask, columnData, isLoading, errorMessage
- Events: onTaskMove, onTaskCreate, onTaskUpdate, onTaskDelete
- API Binding: GET /forms/sessions/{session_id}, PUT /forms/sessions/{session_id}

**HeaderComponent:**
- Purpose: Top navigation with search and user actions
- Props: user (object), notifications (array)
- State: searchQuery, isSearchFocused
- Events: onSearch, onNotificationClick, onUserMenuToggle
- API Binding: GET /users/profile

**SidebarComponent:**
- Purpose: Navigation sidebar with menu items
- Props: activeItem (string), menuItems (array)
- State: isCollapsed, selectedItem
- Events: onMenuItemClick, onToggleCollapse
- API Binding: None (static navigation)

**KanbanColumnComponent:**
- Purpose: Individual column container for tasks
- Props: columnId (string), title (string), tasks (array), taskCount (number)
- State: isDragOver, isAddingTask
- Events: onTaskDrop, onAddTask, onTaskClick
- API Binding: Inherited from parent

**TaskCardComponent:**
- Purpose: Individual task card display
- Props: task (object), isDragging (boolean)
- State: isHovered, isSelected
- Events: onDragStart, onDragEnd, onClick, onEdit
- API Binding: GET /documents/{document_id}, DELETE /documents/{document_id}

**TaskModalComponent:**
- Purpose: Task creation and editing modal
- Props: isOpen (boolean), task (object), mode (string)
- State: formData, validationErrors, isSubmitting
- Events: onSubmit, onCancel, onClose
- API Binding: POST /documents/upload, PUT /forms/sessions/{session_id}

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **Load Kanban Board:**
   - User navigates to Kanban page
   в†’ KanbanBoardComponent.ngOnInit()
   в†’ GET /forms/sessions/{session_id}
   в†’ Display columns and tasks

2. **Create New Task:**
   - User clicks "Add Task" button
   в†’ Open TaskModalComponent
   в†’ User fills form and submits
   в†’ POST /documents/upload (if attachments)
   в†’ PUT /forms/sessions/{session_id}
   в†’ Update UI with new task

3. **Move Task Between Columns:**
   - User drags task card
   в†’ TaskCardComponent.onDragStart()
   в†’ KanbanColumnComponent.onTaskDrop()
   в†’ PUT /forms/sessions/{session_id}
   в†’ Update column counts and UI

4. **Edit Task Details:**
   - User clicks task card
   в†’ Open TaskDetailModalComponent
   в†’ User modifies and saves
   в†’ PUT /forms/sessions/{session_id}
   в†’ Refresh task display
