### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose:** Main container for the three-column kanban layout as specified in Jira requirements
- **Props:** 
  - columns: KanbanColumn[] - Array of column configurations with id, title, and cards
  - loading: boolean - Loading state for data fetching
  - error: string | null - Error message for display
- **State:**
  - draggedTask: Task | null - Currently dragged task for drag-and-drop
  - columnData: KanbanColumn[] - Column data structure as per Jira spec
- **Events:**
  - taskMoved: (task: Task, fromColumn: string, toColumn: string) => void
  - taskCreated: (task: Task, column: string) => void
- **API Binding:** Receives data from parent component, no direct API calls

**KanbanColumnComponent:**
- **Purpose:** Individual column container with header, task list, and add button
- **Props:**
  - column: KanbanColumn - Column configuration and data
  - tasks: Task[] - Array of tasks in this column
  - allowDrop: boolean - Whether column accepts drag-and-drop
- **State:**
  - isDragOver: boolean - Visual feedback for drag operations
  - taskCount: number - Number of tasks for display
- **Events:**
  - taskDropped: (task: Task) => void
  - addTaskClicked: () => void
- **API Binding:** None (receives data from parent)

**TaskCardComponent:**
- **Purpose:** Individual task card with drag-and-drop and keyboard navigation
- **Props:**
  - task: Task - Task data object with title, description, priority, assignees
  - draggable: boolean - Whether card supports drag-and-drop
  - focused: boolean - Focus state for keyboard navigation
- **State:**
  - isDragging: boolean - Visual state during drag operations
- **Events:**
  - taskClicked: (task: Task) => void
  - dragStarted: (task: Task) => void
  - dragEnded: () => void
- **API Binding:** None (receives data from parent)

**TaskModalComponent:**
- **Purpose:** Modal for creating and editing tasks with comprehensive form
- **Props:**
  - isOpen: boolean - Modal visibility state
  - task: Task | null - Task to edit (null for new task)
  - defaultColumn: string - Default column for new tasks
- **State:**
  - formData: TaskFormData - Form input data
  - isSubmitting: boolean - Submission state
  - validationErrors: ValidationError[] - Form validation errors
- **Events:**
  - taskSaved: (task: Task) => void
  - modalClosed: () => void
- **API Binding:** Task creation/update APIs

**HeaderComponent:**
- **Purpose:** Top navigation bar with search, notifications, and user controls
- **Props:**
  - user: User - Current user data
  - notifications: Notification[] - User notifications
- **State:**
  - searchQuery: string - Search input value
  - notificationsOpen: boolean - Notifications dropdown state
- **Events:**
  - searchSubmitted: (query: string) => void
  - themeToggled: () => void
  - sidebarToggled: () => void
- **API Binding:** Search API, Notifications API

**SidebarComponent:**
- **Purpose:** Left navigation panel with hierarchical menu structure
- **Props:**
  - isOpen: boolean - Sidebar visibility (mobile)
  - currentPage: string - Active page identifier
- **State:**
  - expandedSections: string[] - Expanded menu sections
- **Events:**
  - navigationClicked: (page: string) => void
  - sidebarClosed: () => void
- **API Binding:** None (static navigation)

### 7. DATA FLOW / PROPS

**Parent в†’ Child Mapping:**
- **AppComponent** в†’ HeaderComponent (user, notifications)
- **AppComponent** в†’ SidebarComponent (isOpen, currentPage)
- **AppComponent** в†’ KanbanBoardComponent (columns, loading, error)
- **KanbanBoardComponent** в†’ KanbanColumnComponent (column, tasks, allowDrop)
- **KanbanColumnComponent** в†’ TaskCardComponent (task, draggable, focused)
- **AppComponent** в†’ TaskModalComponent (isOpen, task, defaultColumn)

**Data Flow Pattern:**
1. AppComponent manages global application state
2. API data flows down through component props
3. User interactions bubble up through event emissions
4. State updates trigger Angular change detection and re-renders
5. Drag-and-drop operations update local state before API calls

### 8. API SUMMARY

**Available APIs:**
- **GET /api/youth-accounts/{youthAccountId}/dashboard** в†’ Fetch dashboard data with account balance and recent transactions
- **POST /api/youth-accounts/{youthAccountId}/fund-transfer** в†’ Transfer funds between accounts
- **PUT /api/youth-accounts/{youthAccountId}/spending-limit** в†’ Update spending limits
- **GET /api/youth-accounts/{youthAccountId}/transactions** в†’ Get transaction history with pagination

**API Characteristics:**
- OAuth 2.0 authentication with JWT tokens
- Rate limiting: 100 requests per minute
- JSON request/response format
- Comprehensive error handling with structured error responses
- Pagination support for transaction history
- Idempotency key support for fund transfers

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **View Kanban Board**
   - User navigates to kanban board page
   - KanbanBoardComponent initializes with OnInit lifecycle hook
   - Calls GET /dashboard API to fetch account data
   - Transforms financial data to kanban structure
   - Displays three columns with task cards

2. **Create Task**
   - User clicks "Add Task" aava-button (variant="primary")
   - TaskModalComponent opens with form validation
   - User fills form and clicks "Create" aava-button
   - Calls POST /fund-transfer API (simulating task creation)
   - Updates local kanban state and closes modal
   - Emits taskCreated event to parent component

3. **Move Task (Drag & Drop)**
   - User drags TaskCardComponent between columns
   - TaskCardComponent handles dragStarted event
   - KanbanColumnComponent receives taskDropped event
   - Calls PUT /spending-limit API (simulating status change)
   - Updates UI with new task position
   - Provides keyboard alternative for accessibility

4. **Search Tasks**
   - User types in SearchComponent input field
   - Component filters local task data in real-time
   - Calls GET /transactions API for detailed search
   - Updates kanban board with filtered results
   - Maintains search state across navigation

5. **Toggle Theme**
   - User clicks theme aava-button in HeaderComponent
   - ThemeService updates application theme
   - CSS custom properties change globally
   - User preference saved to localStorage
