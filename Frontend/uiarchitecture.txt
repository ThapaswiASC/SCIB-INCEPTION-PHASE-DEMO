### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose:** Main container for three-column Kanban layout
- **Props:** 
  - columns: KanbanColumn[]
  - loading: boolean
  - error: string | null
- **State:**
  - draggedTask: Task | null
  - columnData: KanbanColumn[]
- **Events:**
  - taskMoved: (task: Task, fromColumn: string, toColumn: string)
  - taskCreated: (task: Task)
  - taskUpdated: (task: Task)
- **API Binding:**
  - GET /api/youth-accounts/{id}/dashboard for initial data
  - WebSocket connection for real-time updates

**KanbanColumnComponent:**
- **Purpose:** Individual column container (To Do, In Progress, Done)
- **Props:**
  - column: KanbanColumn
  - tasks: Task[]
  - allowDrop: boolean
- **State:**
  - dragOver: boolean
  - taskCount: number
- **Events:**
  - taskDropped: (task: Task)
  - addTaskClicked: (columnId: string)
- **API Binding:** None (receives data from parent)

**TaskCardComponent:**
- **Purpose:** Individual task display and interaction
- **Props:**
  - task: Task
  - draggable: boolean
  - compact: boolean
- **State:**
  - dragging: boolean
  - selected: boolean
- **Events:**
  - taskClicked: (task: Task)
  - taskDragStart: (task: Task)
  - taskDragEnd: (task: Task)
- **API Binding:** None (receives data from parent)

**TaskModalComponent:**
- **Purpose:** Task creation and editing modal
- **Props:**
  - isOpen: boolean
  - task: Task | null
  - mode: 'create' | 'edit'
- **State:**
  - formData: TaskFormData
  - loading: boolean
  - errors: ValidationErrors
- **Events:**
  - taskSaved: (task: Task)
  - modalClosed: ()
- **API Binding:**
  - POST /api/tasks for task creation
  - PUT /api/tasks/{id} for task updates

**HeaderComponent:**
- **Purpose:** Top navigation and user actions
- **Props:**
  - user: User
  - notifications: Notification[]
- **State:**
  - searchQuery: string
  - userMenuOpen: boolean
- **Events:**
  - searchSubmitted: (query: string)
  - themeToggled: ()
  - userMenuToggled: ()
- **API Binding:**
  - GET /api/notifications for notification count

**ButtonComponent (from library):**

### 8. API SUMMARY

**Available APIs:**
- **GET /api/youth-accounts/{youthAccountId}/dashboard** в†’ Fetch dashboard data with account balance, spending limits, recent transactions
- **POST /api/youth-accounts/{youthAccountId}/fund-transfer** в†’ Transfer funds from parent to youth account
- **PUT /api/youth-accounts/{youthAccountId}/spending-limit** в†’ Configure weekly spending limits
- **GET /api/youth-accounts/{youthAccountId}/transactions** в†’ Get paginated transaction history with filtering

**Authentication:** OAuth 2.0 with JWT tokens
**Rate Limiting:** 100 requests per minute per token
**Security:** PCI-DSS Level 1, GDPR compliant

### 9. API TO UI MAPPING

**Component в†’ API:**
- **DashboardComponent** в†’ GET /api/youth-accounts/{id}/dashboard
- **FundTransferModal** в†’ POST /api/youth-accounts/{id}/fund-transfer
- **SpendingLimitModal** в†’ PUT /api/youth-accounts/{id}/spending-limit
- **TransactionHistoryComponent** в†’ GET /api/youth-accounts/{id}/transactions

**Data Transformation Required:**
- Dashboard API returns financial data, but Kanban needs task data

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **View Kanban Board:**
   - User clicks "Kanban Board" в†’ Navigate to kanban page в†’ Load KanbanBoardComponent в†’ Call GET /api/tasks в†’ Display three columns with tasks

2. **Create New Task:**
   - User clicks "Add Task" в†’ Open TaskModalComponent в†’ Fill form в†’ Submit в†’ Call POST /api/tasks в†’ Update kanban board в†’ Close modal

3. **Move Task Between Columns:**
   - User drags task card в†’ Update UI optimistically в†’ Call PUT /api/tasks/{id}/status в†’ Confirm move or revert on error

4. **Fund Transfer (Dashboard):**
   - User clicks "Add Funds" в†’ Open FundTransferModal в†’ Enter amount в†’ Submit в†’ Call POST /api/youth-accounts/{id}/fund-transfer в†’ Update balance display

5. **Update Spending Limit:**
   - User navigates to settings в†’ Modify limit в†’ Submit в†’ Call PUT /api/youth-accounts/{id}/spending-limit в†’ Show confirmation

6. **View Transaction History:**
   - User clicks "All Tasks" в†’ Load TaskListComponent в†’ Call GET /api/youth-accounts/{id}/transactions в†’ Display paginated list

**Keyboard Navigation Flow:**
- Tab navigation between focusable elements
- Arrow keys for task navigation within columns
- Enter/Space for task selection and actions
- Escape for modal/dialog dismissal
