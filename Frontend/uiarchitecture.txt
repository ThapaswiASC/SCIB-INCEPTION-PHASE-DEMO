### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose:** Main container for the three-column Kanban layout
- **Props:** 
  - `columns: KanbanColumn[]` - Array of column configurations
  - `loading: boolean` - Loading state indicator
  - `error: string | null` - Error message display
- **State:**
  - `draggedTask: Task | null` - Currently dragged task
  - `columnData: KanbanColumn[]` - Column data with tasks
- **Events:**
  - `taskMoved: EventEmitter<TaskMoveEvent>` - Task movement between columns
  - `taskCreated: EventEmitter<Task>` - New task creation
  - `taskUpdated: EventEmitter<Task>` - Task updates
- **API Binding:**
  - GET /youth-accounts/{id}/dashboard - Fetch dashboard data

**KanbanColumnComponent:**
- **Purpose:** Individual column container with header and task list
- **Props:**
  - `column: KanbanColumn` - Column configuration and data
  - `tasks: Task[]` - Array of tasks in column
  - `allowDrop: boolean` - Enable drop functionality
- **State:**
  - `isDropTarget: boolean` - Drop zone active state
  - `taskCount: number` - Number of tasks in column
- **Events:**
  - `taskDropped: EventEmitter<TaskDropEvent>` - Task dropped in column
  - `addTaskClicked: EventEmitter<string>` - Add task button clicked
- **API Binding:** None (receives data from parent)

**TaskCardComponent:**
- **Purpose:** Individual task display with drag/drop functionality
- **Props:**
  - `task: Task` - Task data object
  - `draggable: boolean` - Enable drag functionality
  - `compact: boolean` - Compact display mode
- **State:**
  - `isDragging: boolean` - Drag state indicator
  - `isSelected: boolean` - Selection state
- **Events:**
  - `taskClicked: EventEmitter<Task>` - Task clicked for details
  - `dragStarted: EventEmitter<Task>` - Drag operation started
  - `dragEnded: EventEmitter<void>` - Drag operation ended
- **API Binding:** None (receives data from parent)

**HeaderComponent:**
- **Purpose:** Top navigation bar with search and user actions
- **Props:**
  - `user: User` - Current user information
  - `notifications: Notification[]` - User notifications
- **State:**
  - `searchQuery: string` - Current search term
  - `showNotifications: boolean` - Notifications panel visibility
- **Events:**
  - `searchSubmitted: EventEmitter<string>` - Search query submitted
  - `userMenuClicked: EventEmitter<void>` - User menu interaction
- **API Binding:** None (receives data from parent)

**SidebarComponent:**
- **Purpose:** Left navigation panel with menu items
- **Props:**
  - `currentPage: string` - Active page identifier
  - `navigationItems: NavItem[]` - Navigation menu structure
- **State:**
  - `isCollapsed: boolean` - Sidebar collapse state
  - `activeSection: string` - Currently active nav section
- **Events:**
  - `pageChanged: EventEmitter<string>` - Page navigation event
  - `sidebarToggled: EventEmitter<boolean>` - Sidebar toggle event
- **API Binding:** None (static navigation)

### 8. API SUMMARY

**Available APIs:**
- **GET /youth-accounts/{youthAccountId}/dashboard** → Fetch youth account dashboard data
  - Returns: account balance, spending limit, current week spent, remaining limit, account status
  - Headers: X-Request-ID, X-RateLimit-Remaining, X-RateLimit-Reset
  - Authentication: OAuth2 + JWT Bearer Token

**API Features:**
- OAuth 2.0 + JWT Bearer Token authentication
- Rate limiting and throttling
- Request correlation with X-Request-ID
- Comprehensive error responses (400, 401, 403, 404, 429, 500)
- PCI-DSS Level 1 compliant
- GDPR data protection compliant

### 9. API TO UI MAPPING

**Component → API:**
- **KanbanBoardComponent** → GET /youth-accounts/{id}/dashboard
  - Maps API response to dashboard metrics
  - Transforms account balance to display format
  - Calculates spending progress for visual indicators
- **AnalyticsPanelComponent** → GET /youth-accounts/{id}/dashboard
  - Maps spending data to performance metrics
  - Calculates trends and percentages
  - Formats currency and date displays

**Data Transformation:**
- API `accountBalance` → UI `currentBalance` (formatted currency)
- API `spendingLimit` → UI `weeklyLimit` (formatted currency)
- API `currentWeekSpent` → UI `spentThisWeek` (formatted currency)
- API `remainingLimit` → UI `remainingBudget` (formatted currency)

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **Load Dashboard**
   - User navigates to dashboard
   - → KanbanBoardComponent.ngOnInit()
   - → Call GET /youth-accounts/{id}/dashboard
   - → Update component state with API response
   - → Render dashboard metrics and Kanban board

2. **Create New Task**
   - User clicks "Add Task" button
   - → Open TaskModalComponent
   - → User fills form and submits
   - → Validate form data
   - → Add task to local state (API not available)
   - → Update column task count
   - → Close modal

3. **Move Task Between Columns**
   - User drags task card
   - → TaskCardComponent.onDragStart()
   - → KanbanColumnComponent.onDrop()
   - → Update task status in local state
   - → Emit taskMoved event
   - → Update column counts
   - → Announce change to screen readers

4. **Search Tasks**
   - User types in search bar
   - → SearchBarComponent.onSearchChange()
   - → Filter tasks in local state
   - → Update visible tasks in columns
   - → Highlight matching results

5. **View Task Details**
   - User clicks task card
   - → TaskCardComponent.onClick()
   - → Open TaskModalComponent with task data
   - → Display task details in modal
   - → Allow editing if permissions allow
