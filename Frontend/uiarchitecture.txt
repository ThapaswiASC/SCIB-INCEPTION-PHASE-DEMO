### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose:** Main container for three-column Kanban layout
- **Props:**
  - `columns`: KanbanColumn[] - Array of column configurations
  - `tasks`: Task[] - Array of all tasks
  - `loading`: boolean - Loading state indicator
- **State:**
  - `selectedTask`: Task | null - Currently selected task
  - `draggedTask`: Task | null - Task being dragged
  - `error`: string | null - Error message
- **Events:**
  - `taskCreated`: Emitted when new task is created
  - `taskUpdated`: Emitted when task is modified
  - `taskMoved`: Emitted when task is moved between columns
- **API Binding:**
  - GET /forms/sessions - Retrieve form session data
  - POST /forms/sessions - Create new form session

**KanbanColumnComponent:**
- **Purpose:** Individual column container with header and task list
- **Props:**
  - `column`: KanbanColumn - Column configuration
  - `tasks`: Task[] - Tasks in this column
  - `allowDrop`: boolean - Whether column accepts drops
- **State:**
  - `isDropTarget`: boolean - Whether column is drop target
  - `taskCount`: number - Number of tasks in column
- **Events:**
  - `taskDropped`: Emitted when task is dropped in column
  - `addTaskClicked`: Emitted when add task button is clicked
- **API Binding:**
  - PUT /forms/sessions/{session_id} - Update form session

**TaskCardComponent:**
- **Purpose:** Individual task card with metadata and interactions
- **Props:**
  - `task`: Task - Task data object
  - `draggable`: boolean - Whether card is draggable
  - `selectable`: boolean - Whether card is selectable
- **State:**
  - `isDragging`: boolean - Whether card is being dragged
  - `isSelected`: boolean - Whether card is selected
- **Events:**
  - `taskClicked`: Emitted when task is clicked
  - `taskDragStart`: Emitted when drag starts
  - `taskDragEnd`: Emitted when drag ends
- **API Binding:**
  - GET /documents/{document_id} - Get document status if task has attachments

**TaskModalComponent:**
- **Purpose:** Modal dialog for creating and editing tasks
- **Props:**
  - `isOpen`: boolean - Whether modal is visible
  - `task`: Task | null - Task being edited (null for new task)
  - `mode`: 'create' | 'edit' - Modal mode
- **State:**
  - `formData`: TaskFormData - Form input data
  - `validationErrors`: ValidationError[] - Form validation errors
  - `isSubmitting`: boolean - Whether form is being submitted
- **Events:**
  - `taskSaved`: Emitted when task is saved
  - `modalClosed`: Emitted when modal is closed
- **API Binding:**
  - POST /documents/upload - Upload task attachments
  - POST /documents/validate - Validate document quality

**HeaderComponent:**
- **Purpose:** Application header with search and user actions
- **Props:**
  - `user`: User - Current user data
  - `searchPlaceholder`: string - Search input placeholder
- **State:**
  - `searchQuery`: string - Current search query
  - `isSearchFocused`: boolean - Whether search is focused
- **Events:**
  - `searchChanged`: Emitted when search query changes
  - `userMenuClicked`: Emitted when user profile is clicked
- **API Binding:**
  - GET /users/profile - Get user profile data

**SidebarComponent:**
- **Purpose:** Navigation sidebar with menu items and quick actions
- **Props:**
  - `isCollapsed`: boolean - Whether sidebar is collapsed
  - `activeRoute`: string - Currently active route
- **State:**
  - `menuItems`: MenuItem[] - Navigation menu items
  - `isOpen`: boolean - Whether sidebar is open (mobile)
- **Events:**
  - `navigationClicked`: Emitted when navigation item is clicked
  - `quickActionClicked`: Emitted when quick action is triggered
- **API Binding:**
  - GET /components/metadata - Get component library metadata

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **Load Kanban Board:**
   - User navigates to Kanban page
   - KanbanBoardComponent initializes
   - GET /forms/sessions/{session_id} to load board state
   - Render columns and tasks

2. **Create New Task:**
   - User clicks "Create Task" button
   - TaskModalComponent opens
   - User fills form and submits
   - POST /documents/upload (if attachments)
   - PUT /forms/sessions/{session_id} to save task
   - Update board display

3. **Move Task Between Columns:**
   - User drags task card
   - TaskCardComponent emits drag events
   - KanbanColumnComponent handles drop
   - PUT /forms/sessions/{session_id} to update task status
   - Update UI state

4. **Upload Task Attachment:**
   - User adds file in TaskModalComponent
   - POST /documents/validate for quality check
   - POST /documents/upload for final upload
   - Associate document with task

5. **Search Tasks:**
   - User types in HeaderComponent search
   - Filter tasks locally or via API
   - Update KanbanBoardComponent display
