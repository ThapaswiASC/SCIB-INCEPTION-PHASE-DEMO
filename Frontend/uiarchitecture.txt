# UI COMPONENT ARCHITECTURE

## 1. JIRA REQUIREMENT SUMMARY

**Story Description:**
Create Angular component for Kanban board container with three-column layout at src/app/features/kanban/components/kanban-board/kanban-board.component.ts. Implement component class with @Component decorator and selector 'app-kanban-board'. Define template in kanban-board.component.html with three column containers using CSS Grid layout (grid-template-columns: repeat(3, 1fr)). Create component SCSS file with responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). Implement OnInit lifecycle hook to initialize column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', and aria-labelledby for each column header. Implement error handling with *ngIf directive to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

**Acceptance Criteria:**
- KanbanBoardComponent created with proper Angular structure and decorators
- Template renders three distinct column containers with proper semantic HTML
- CSS Grid layout implemented with responsive breakpoints for desktop, tablet, and mobile
- ARIA attributes added for accessibility compliance
- Error state template implemented with conditional rendering
- Component compiles without errors and passes linting checks

**UI Tasks:**
- Create Angular component structure
- Implement three-column CSS Grid layout
- Add responsive breakpoints
- Implement ARIA accessibility features
- Add error handling and conditional rendering
- Ensure component compilation and linting compliance

## 6. COMPONENT DEFINITIONS

### KanbanBoardComponent (Primary Requirement):
**Purpose:** Main container for three-column Kanban layout with CSS Grid
**Props:**
- columns: KanbanColumn[] - Array of column configurations
- loading: boolean - Loading state indicator
- error: string | null - Error message for display
- gridLayout: string - CSS Grid template (default: 'repeat(3, 1fr)')

**State:**
- columns: [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- draggedTask: Task | null - Currently dragged task
- errorState: boolean - Error handling state
- isLoading: boolean - Component loading state

**Events:**
- taskMoved: EventEmitter<{task: Task, fromColumn: string, toColumn: string}>
- taskSelected: EventEmitter<Task>
- columnMenuToggled: EventEmitter<string>
- errorOccurred: EventEmitter<string>

**API Binding:**
- GET /youth-accounts/{id}/dashboard - Fetch dashboard data for context

### KanbanColumnComponent:
**Purpose:** Individual column container within Kanban board
**Props:**
- columnId: string - Unique column identifier ('todo', 'inprogress', 'done')
- title: string - Column display title
- tasks: Task[] - Array of tasks in column
- taskCount: number - Number of tasks for accessibility
- indicator: string - Column color indicator

**State:**
- dragOver: boolean - Drag and drop state
- menuOpen: boolean - Column menu visibility
- tasksLoaded: boolean - Tasks loading state

**Events:**
- taskDropped: EventEmitter<{task: Task, columnId: string}>
- addTaskClicked: EventEmitter<string>
- menuToggled: EventEmitter<string>

**API Binding:**
- None (receives data from parent)

### TaskCardComponent:
**Purpose:** Individual task representation with drag/drop capability
**Props:**
- task: Task - Task data object
- draggable: boolean - Enable/disable drag functionality
- priority: 'high' | 'medium' | 'low' - Priority level
- assignee: User - Assigned user information

**State:**
- dragging: boolean - Current drag state
- selected: boolean - Selection state
- hovered: boolean - Hover state

**Events:**
- taskClicked: EventEmitter<Task>
- dragStarted: EventEmitter<Task>
- dragEnded: EventEmitter<void>

**API Binding:**
- None (receives data from parent)

### HeaderComponent:
**Purpose:** Top navigation bar with search and user actions
**Props:**
- user: User - Current user information
- notificationCount: number - Unread notifications
- searchPlaceholder: string - Search input placeholder

**State:**
- searchQuery: string - Current search input
- notificationsOpen: boolean - Notification panel state
- userMenuOpen: boolean - User menu visibility

**Events:**
- searchSubmitted: EventEmitter<string>
- notificationToggled: EventEmitter<void>
- userMenuToggled: EventEmitter<void>

**API Binding:**
- GET /youth-accounts/{id}/dashboard - User context and notifications

### SidebarComponent:
**Purpose:** Left navigation panel with menu items
**Props:**
- activeRoute: string - Current active page
- navigationItems: NavigationItem[] - Menu structure
- collapsed: boolean - Sidebar collapse state

**State:**
- activeItem: string - Currently selected menu item
- mobileOpen: boolean - Mobile sidebar visibility

**Events:**
- navigationClicked: EventEmitter<string>
- sidebarToggled: EventEmitter<boolean>

**API Binding:**
- None (static navigation structure)

### TaskModalComponent:
**Purpose:** Modal dialog for task creation and editing
**Props:**
- task: Task | null - Task to edit (null for new task)
- visible: boolean - Modal visibility state
- defaultColumn: string - Default status column
- mode: 'create' | 'edit' - Modal operation mode

**State:**
- formData: TaskFormData - Form input values
- saving: boolean - Save operation state
- formValid: boolean - Form validation state
- errors: ValidationErrors - Form validation errors

**Events:**
- taskSaved: EventEmitter<Task>
- modalClosed: EventEmitter<void>
- formChanged: EventEmitter<TaskFormData>

**API Binding:**
- POST /tasks - Create new task (missing API)
- PUT /tasks/{id} - Update existing task (missing API)

## 8. API SUMMARY

**Available APIs:**
- **GET /youth-accounts/{youthAccountId}/dashboard** в†’ Fetch youth account dashboard data
  - Returns: accountBalance, spendingLimit, currentWeekSpent, remainingLimit, currency, weekStartDate, accountStatus, lastUpdated
  - Headers: X-Request-ID (required), X-RateLimit-Remaining, X-RateLimit-Reset
  - Authentication: OAuth2 + JWT Bearer Token
  - Rate Limiting: Enforced with headers
  - Response Format: JSON with YouthAccountDashboard schema

**Missing APIs (Required for Kanban functionality):**
- GET /tasks в†’ Fetch task list for Kanban board
- POST /tasks в†’ Create new task
- PUT /tasks/{id} в†’ Update task
- DELETE /tasks/{id} в†’ Delete task
- PUT /tasks/{id}/move в†’ Move task between columns
- GET /users/current в†’ Get current user info
- GET /notifications в†’ Fetch notifications
- GET /tasks/search в†’ Search tasks

## 10. API COMPATIBILITY

**Compatible:**
- Dashboard API supports basic account information display
- Authentication and authorization flow is properly defined with OAuth2 + JWT
- Rate limiting and error handling are specified with proper headers
- Request/response format follows REST standards with OpenAPI 3.0.3
- Security compliance (PCI-DSS, GDPR, SOX, BSA)

**Missing:**
- **Task Management APIs**: No endpoints for CRUD operations on tasks
- **Task Movement API**: No endpoint for moving tasks between columns
- **User Profile API**: No endpoint for current user information
- **Notification API**: No endpoint for notification management
- **Search API**: No endpoint for task/user search functionality
- **File Upload API**: No endpoint for task attachments
- **Comment API**: No endpoint for task comments

**Transformations:**
- Dashboard data needs to be supplemented with task data from missing APIs
- User context from dashboard API needs to be extracted for header display
- Error responses need to be mapped to user-friendly messages
- Rate limiting headers need to be handled for user feedback
- Authentication tokens need to be managed and refreshed

## 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **View Kanban Board**
   - User clicks "Kanban Board" in sidebar
   - SidebarComponent.navigationClicked в†’ AppComponent
   - AppComponent routes to KanbanBoardComponent
   - KanbanBoardComponent.ngOnInit в†’ GET /youth-accounts/{id}/dashboard (available)
   - KanbanBoardComponent в†’ GET /tasks (missing API)
   - Display three columns with CSS Grid layout

2. **Create New Task**
   - User clicks "Add Task" button in column
   - AddTaskButtonComponent.clicked в†’ KanbanColumnComponent
   - KanbanColumnComponent.addTaskClicked в†’ KanbanBoardComponent
   - KanbanBoardComponent opens TaskModalComponent
   - User fills form and clicks "Save"
   - TaskModalComponent.taskSaved в†’ POST /tasks (missing API)
   - Refresh KanbanBoardComponent data and update column counts

3. **Move Task Between Columns (Drag & Drop)**
   - User drags TaskCardComponent
   - TaskCardComponent.dragStarted в†’ KanbanColumnComponent
   - User drops in different column
   - KanbanColumnComponent.taskDropped в†’ KanbanBoardComponent
   - KanbanBoardComponent в†’ PUT /tasks/{id}/move (missing API)
   - Update UI state, column counts, and visual feedback

4. **Search Tasks**
   - User types in HeaderComponent search bar
   - HeaderComponent.searchSubmitted в†’ AppComponent
   - AppComponent в†’ GET /tasks/search?query={query} (missing API)
   - Filter and highlight matching tasks in Kanban board

5. **View Task Details**
   - User clicks on TaskCardComponent
   - TaskCardComponent.taskClicked в†’ KanbanColumnComponent
   - KanbanColumnComponent в†’ KanbanBoardComponent
   - KanbanBoardComponent opens TaskModalComponent with task data
   - Display task details in modal with form fields

6. **Handle Errors**
   - API call fails or network error occurs
   - Service catches error and emits to component
   - KanbanBoardComponent.errorOccurred в†’ AppComponent
   - Display error message using *ngIf directive
   - Provide retry mechanism for user

7. **Keyboard Navigation**
   - User presses Tab to navigate between tasks
   - Focus moves between TaskCardComponents
   - User presses Enter on focused task
   - TaskCardComponent.taskClicked triggered
   - User presses Ctrl+M to move task
   - Keyboard task movement flow initiated
