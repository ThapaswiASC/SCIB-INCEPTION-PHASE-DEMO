6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main kanban board container with three-column layout as per Jira requirements
Props:
- columns: KanbanColumn[] (default: [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}])
- loading: boolean
- error: string | null
State:
- draggedTask: Task | null
- columnData: KanbanColumn[]
- isLoading: boolean
- errorMessage: string
Events:
- onTaskMove: (taskId: string, fromColumn: string, toColumn: string) => void
- onTaskCreate: (task: Task) => void
- onTaskUpdate: (task: Task) => void
- onTaskDelete: (taskId: string) => void
API Binding:
- GET /youth-accounts/{youthAccountId}/dashboard for task data
- POST /youth-accounts/{youthAccountId}/fund-transfer for task updates

KanbanColumnComponent:
Purpose: Individual column container for tasks
Props:
- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean
State:
- isDragOver: boolean
- taskCount: number
Events:
- onTaskDrop: (task: Task) => void
- onAddTask: () => void
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task card with drag-and-drop functionality
Props:
- task: Task
- draggable: boolean
- priority: 'high' | 'medium' | 'low'
- assignees: User[]
- tags: string[]
State:
- isDragging: boolean
- isSelected: boolean
Events:
- onDragStart: (task: Task) => void
- onDragEnd: () => void
- onClick: (task: Task) => void
API Binding:
- PUT /youth-accounts/{youthAccountId}/spending-limit for task updates

ButtonComponent (from library):
Purpose: Reusable button with multiple variants and accessibility features
Props:

- variant: 'default' | 'primary' | 'secondary' | 'success' | 'warning' | 'danger' | 'info'
- size: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
- iconName: string
- iconPosition: 'left' | 'right' | 'only'
- disabled: boolean
- processing: boolean
- pill: boolean
- outlined: boolean
State:
- isPressed: boolean
- isFocused: boolean
Events:
- userClick: () => void
API Binding:
- None (pure UI component)

ModalComponent:
Purpose: Overlay modal for task creation and editing

- isOpen: boolean

8. API SUMMARY

Available APIs:
- GET /youth-accounts/{youthAccountId}/dashboard в†’ Retrieve youth account dashboard data
- POST /youth-accounts/{youthAccountId}/fund-transfer в†’ Transfer funds (can be adapted for task creation)
- PUT /youth-accounts/{youthAccountId}/spending-limit в†’ Configure limits (can be adapted for task updates)
- GET /youth-accounts/{youthAccountId}/transactions в†’ Retrieve transaction history (can be adapted for task history)

Authentication:
- OAuth 2.0 with authorization code flow
- JWT Bearer token authentication
- Rate limiting: 1000 requests/hour per user

Request/Response Models:
- YouthAccountDashboard schema for dashboard data
- FundTransferRequest/Response for task operations
- SpendingLimitRequest/Response for task configuration
- TransactionHistoryResponse for task history
