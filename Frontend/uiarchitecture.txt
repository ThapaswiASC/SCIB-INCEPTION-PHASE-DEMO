### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- Purpose: Main container for three-column Kanban layout
- Props:
  - columns: KanbanColumn[]
  - loading: boolean
  - error: string | null
- State:
  - columns: [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
  - isLoading: boolean
  - errorMessage: string | null
- Events:
  - onCardMove: (cardId: string, fromColumn: string, toColumn: string) => void
  - onCardAdd: (columnId: string, cardData: any) => void
  - onCardEdit: (cardId: string, cardData: any) => void
- API Binding:
  - GET /kanban/boards/{boardId} в†’ Fetch board data
  - POST /kanban/cards в†’ Create new card
  - PUT /kanban/cards/{cardId} в†’ Update card
  - PUT /kanban/cards/{cardId}/move в†’ Move card between columns

**KanbanColumnComponent:**
- Purpose: Individual column container with header and cards
- Props:
  - column: KanbanColumn
  - cards: KanbanCard[]
  - allowDrop: boolean
- State:
  - isDragOver: boolean
  - isAddingCard: boolean
- Events:
  - onCardDrop: (cardId: string) => void
  - onAddCard: (cardData: any) => void
- API Binding: None (delegates to parent)

**KanbanCardComponent:**
- Purpose: Individual task card with drag/drop functionality
- Props:
  - card: KanbanCard
  - draggable: boolean
- State:
  - isDragging: boolean
  - isEditing: boolean
- Events:
  - onDragStart: (cardId: string) => void
  - onEdit: (cardData: any) => void
  - onDelete: (cardId: string) => void
- API Binding: None (delegates to parent)

**ErrorStateComponent:**
- Purpose: Display error messages when column configuration fails
- Props:
  - error: string
  - retry: () => void
- State: None
- Events:
  - onRetry: () => void
- API Binding: None

### 13. INTERACTION FLOWS

**User Action в†’ UI в†’ API:**

1. **Load Kanban Board**
   - User navigates to Kanban page
   - KanbanBoardComponent calls ngOnInit
   - GET /kanban/boards/{boardId} (missing API)
   - Display columns with cards

2. **Add New Card**
   - User clicks "Add Card" in column
   - Show card creation form
   - User submits card data
   - POST /kanban/cards (missing API)
   - Update local state and UI

3. **Move Card Between Columns**
   - User drags card to different column
   - onCardDrop event triggered
   - PUT /kanban/cards/{cardId}/move (missing API)
   - Update column states

4. **Edit Card**
   - User clicks on card
   - Show edit modal/form
   - User saves changes
   - PUT /kanban/cards/{cardId} (missing API)
   - Update card display

5. **Error Handling**
   - API call fails
   - Display ErrorStateComponent
   - User clicks retry
   - Retry failed operation
