6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for the three-column Kanban layout
Props:
- columns: KanbanColumn[] - Array of column configurations
- tasks: Task[] - Array of all tasks
- loading: boolean - Loading state
- error: string | null - Error message
State:
- selectedTask: Task | null - Currently selected task
- draggedTask: Task | null - Task being dragged
Events:
- taskMoved: EventEmitter<{taskId: string, fromColumn: string, toColumn: string}>
- taskSelected: EventEmitter<Task>
- addTaskClicked: EventEmitter<string> - Column ID where task should be added
API Binding:
- GET /api/tasks - Fetch all tasks
- POST /api/tasks - Create new task

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn - Column configuration
- tasks: Task[] - Tasks for this column
- allowDrop: boolean - Whether column accepts dropped tasks
State:
- isDropTarget: boolean - Visual state for drag operations
Events:
- taskDropped: EventEmitter<{task: Task, columnId: string}>
- addTaskClicked: EventEmitter<string>
API Binding:
- None (delegates to parent)

TaskCardComponent:
Purpose: Individual task display with all task information
Props:
- task: Task - Task data object
- draggable: boolean - Whether task can be dragged
- selected: boolean - Whether task is selected
State:
- isDragging: boolean - Drag state
- isHovered: boolean - Hover state
Events:
- taskClicked: EventEmitter<Task>
- taskDragStart: EventEmitter<Task>
- taskDragEnd: EventEmitter<void>
API Binding:
- PUT /api/tasks/{id} - Update task

HeaderComponent:
Purpose: Main application header with title and actions
Props:
- title: string - Page title
- subtitle: string - Page subtitle
State:
- None
Events:
- settingsClicked: EventEmitter<void>
- addTaskClicked: EventEmitter<void>
API Binding:
- None

SidebarComponent:
Purpose: Navigation sidebar with brand and menu
Props:
- navigationItems: NavigationItem[] - Menu items
- activeItem: string - Currently active menu item
State:
- collapsed: boolean - Sidebar collapse state
Events:
- navigationChanged: EventEmitter<string>
API Binding:
- None

ButtonComponent (from library):
Purpose: Reusable button with multiple variants
Props:
- label: string
- variant: 'default' | 'primary' | 'secondary'
- size: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
- disabled: boolean
- iconName: string
- iconPosition: 'left' | 'right' | 'only'
Events:
- userClick: EventEmitter<void>

AvatarComponent:
Purpose: User avatar display
Props:
- initials: string - User initials
- size: 'sm' | 'md' | 'lg'
- backgroundColor: string
State:
- None
Events:

TagComponent:

8. API SUMMARY

Available APIs:
- POST /api/tasks → Create new task
  - Request: CreateTaskRequest (title, description, status, priority, dueDate, assignedTo, tags)
  - Response: Task object with generated ID and timestamps
  - Authentication: JWT Bearer token required
  - Rate Limit: 100 requests per minute

- GET /health → Health check
  - Response: Service health status
  - Authentication: None required

API Features:
- JWT-based authentication with RBAC
- Comprehensive input validation
- Real-time synchronization via WebSocket
- Rate limiting and security controls
- Audit logging and compliance support

9. API TO UI MAPPING

Component → API:
- KanbanBoardComponent → POST /api/tasks (create new task)
- TaskCardComponent → PUT /api/tasks/{id} (update task - not available in current spec)
- AddTaskButtonComponent → POST /api/tasks (trigger task creation)

Data Flow:
1. User clicks "Add Task" → AddTaskButtonComponent emits event
2. KanbanBoardComponent receives event → Opens task creation modal
3. User submits task form → POST /api/tasks with CreateTaskRequest
4. API returns created Task → Update local state and UI
5. WebSocket notification → Real-time sync across sessions

10. API COMPATIBILITY

Compatible:
- Task creation via POST /api/tasks supports all required fields
- Authentication and authorization properly implemented
- Validation and error handling comprehensive
- Real-time synchronization available

Missing:
- GET /api/tasks - Fetch existing tasks (needed for initial load)
- PUT /api/tasks/{id} - Update existing tasks (needed for drag/drop)
- DELETE /api/tasks/{id} - Delete tasks (needed for task management)
- GET /api/tasks/{id} - Fetch single task details
- GET /api/users - Fetch users for assignment dropdown

Transformations:
- Task status mapping: API uses 'TODO', 'IN_PROGRESS', 'DONE' vs UI columns 'todo', 'inprogress', 'done'
- Priority mapping: API uses 'LOW', 'MEDIUM', 'HIGH', 'CRITICAL' vs UI visual indicators
- Date formatting: API uses ISO 8601 format, UI needs localized display

13. INTERACTION FLOWS

User Action → UI → API:

1. Create New Task:
   - Click "Add Task" button
   → Open task creation modal/form
   → Fill task details (title, description, priority, assignee, tags)
   → Submit form
   → Call POST /api/tasks
   → Update UI with new task
   → WebSocket notification to other users

2. View Task Details:
   - Click task card
   → Open task details popover/modal
   → Display full task information
   → Allow inline editing (future enhancement)

3. Move Task Between Columns:
   - Drag task card to different column
   → Update task status locally
   → Call PUT /api/tasks/{id} (not available in current API)
   → Update UI on success
   → WebSocket notification to other users

4. Filter/Search Tasks:
   - Type in search bar
   → Filter tasks locally
   → Highlight matching tasks
   → Update column counts

5. Navigate Between Views:
   - Click sidebar navigation item
   → Update active view
   → Load appropriate component
   → Update URL routing
