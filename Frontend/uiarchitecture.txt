6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout as per Jira requirements
Props:
- columns: KanbanColumn[] = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- loading: boolean = false
- error: string | null = null
State:
- draggedTask: Task | null = null
- activeColumn: string = 'todo' (for mobile)
Events:
- taskMoved: EventEmitter<{task: Task, fromColumn: string, toColumn: string}>
- taskCreated: EventEmitter<Task>
- columnError: EventEmitter<string>
API Binding:
- GET /api/youth-accounts/{youthAccountId}/dashboard (for task data)
- POST /api/youth-accounts/{youthAccountId}/fund-transfer (for task updates)

TaskCardComponent:
Purpose: Individual task display with drag-drop functionality
Props:
- task: Task (title, description, priority, assignee, dueDate, tags)
- draggable: boolean = true
State:
- isDragging: boolean = false
Events:
- taskClick: EventEmitter<Task>
- taskDragStart: EventEmitter<Task>
- taskDragEnd: EventEmitter<void>
API Binding:
- PUT /api/youth-accounts/{youthAccountId}/spending-limit (for task updates)

HeaderComponent:
Purpose: Top navigation with search, notifications, user menu
Props:
- user: User
- notifications: Notification[]
State:
- searchQuery: string = ''
- showUserMenu: boolean = false
Events:
- searchSubmit: EventEmitter<string>
- themeToggle: EventEmitter<void>
- logout: EventEmitter<void>
API Binding:
- GET /api/youth-accounts/{youthAccountId}/transactions (for notifications)

SidebarComponent:
Purpose: Left navigation with menu items and project links
Props:
- activeRoute: string
- projects: Project[]
State:
- isCollapsed: boolean = false
Events:
- routeChange: EventEmitter<string>
- projectSelect: EventEmitter<Project>
API Binding:
- No direct API binding

TaskModalComponent:
Purpose: Modal for creating/editing tasks
Props:
- isOpen: boolean = false
- task: Task | null = null
- defaultStatus: string = 'todo'
State:
- formData: TaskFormData
- isSubmitting: boolean = false
Events:
- taskSave: EventEmitter<Task>
- modalClose: EventEmitter<void>
API Binding:
- POST /api/youth-accounts/{youthAccountId}/fund-transfer (for task creation)

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard в†’ Fetch dashboard data including account balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer в†’ Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit в†’ Configure spending limit (can be repurposed for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions в†’ Get transaction history (can be repurposed for task history)

Authentication:
- OAuth 2.0 with JWT tokens
- Scopes: youth-account:read, youth-account:write, fund-transfer:execute
- Rate limiting: 100 requests per minute
