6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for the three-column Kanban layout
Props:
- columns: KanbanColumn[]
- loading: boolean
- error: string | null
State:
- selectedTask: Task | null
- draggedTask: Task | null
Events:
- onTaskCreate: (task: Task) => void
- onTaskUpdate: (task: Task) => void
- onTaskMove: (taskId: string, columnId: string) => void
API Binding:
- GET /api/youth-accounts/{id}/dashboard for initial data

SidebarNavigationComponent:
Purpose: Left navigation with brand and menu items
Props:
- activeView: string
- navigationItems: NavigationItem[]
State:
- isCollapsed: boolean (mobile)
Events:
- onViewChange: (view: string) => void

HeaderComponent:
Purpose: Top header with search and user actions
Props:
- user: User
- searchQuery: string
State:
- showNotifications: boolean
Events:
- onSearch: (query: string) => void
- onUserAction: (action: string) => void

KanbanColumnComponent:
Purpose: Individual column container (To Do, In Progress, Done)
Props:
- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean
State:
- isDropTarget: boolean
Events:
- onTaskAdd: (columnId: string) => void
- onTaskDrop: (task: Task, columnId: string) => void
API Binding:
- Column-specific task filtering

TaskCardComponent:
Purpose: Individual task card display
Props:
- task: Task
- draggable: boolean
- clickable: boolean
State:
- isSelected: boolean
- isDragging: boolean
Events:
- onTaskClick: (task: Task) => void
- onTaskDrag: (task: Task) => void

TaskModalComponent:
Purpose: Task creation and editing modal
Props:
- isOpen: boolean
- task: Task | null (null for create, Task for edit)
- assignees: User[]
State:
- formData: TaskFormData
- isSubmitting: boolean
- validationErrors: ValidationError[]
Events:
- onSubmit: (task: TaskFormData) => void
- onCancel: () => void
API Binding:
- POST /api/youth-accounts/{id}/fund-transfer for task creation
- PUT /api/youth-accounts/{id}/spending-limit for task updates

7. DATA FLOW / PROPS

Parent в†’ Child Mapping:
App в†’ KanbanBoardComponent (columns, tasks, user)
KanbanBoardComponent в†’ SidebarNavigationComponent (activeView, navigationItems)
KanbanBoardComponent в†’ HeaderComponent (user, searchQuery)
KanbanBoardComponent в†’ KanbanColumnComponent (column, tasks)
KanbanColumnComponent в†’ TaskCardComponent (task, draggable)

Data Flow:
1. App loads dashboard data from API
2. KanbanBoardComponent receives and distributes data to columns
3. User interactions bubble up through event handlers
4. State changes trigger re-renders of affected components

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard в†’ Fetch dashboard data with account balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer в†’ Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit в†’ Update spending limits (can be repurposed for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions в†’ Get transaction history (can be repurposed for task history)

9. API TO UI MAPPING

Component в†’ API:
- KanbanBoardComponent в†’ GET /api/youth-accounts/{id}/dashboard (initial load)
- TaskModalComponent в†’ POST /api/youth-accounts/{id}/fund-transfer (create task)
- TaskModalComponent в†’ PUT /api/youth-accounts/{id}/spending-limit (update task)
- TaskCardComponent в†’ GET /api/youth-accounts/{id}/transactions (task history)

Data Transformations:
- Dashboard API response maps to Kanban columns and tasks
- Fund transfer API repurposed for task creation with amount as priority
- Spending limit API repurposed for task updates
- Transaction history API provides audit trail for task changes

10. API COMPATIBILITY

Transformations:

- Transform transferAmount to task priority/effort points

   в†’ Drag TaskCardComponent

   в†’ PUT /api/youth-accounts/{id}/spending-limit
   в†’ Update column states

4. Search Tasks
   в†’ Type in SearchComponent
   в†’ HeaderComponent.onSearch()
   в†’ Filter local task data
   в†’ Update visible tasks

   в†’ Click TaskCardComponent
