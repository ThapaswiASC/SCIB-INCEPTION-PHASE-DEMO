6. COMPONENT DEFINITIONS

KanbanBoardComponent (PRIMARY JIRA REQUIREMENT):
Purpose: Main container for three-column kanban layout with CSS Grid
Props:
- columns: KanbanColumn[] = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- loading: boolean
- error: string | null
State:
- draggedTask: Task | null
- columnData: KanbanColumn[]
- isInitialized: boolean
Events:
- taskMoved: EventEmitter<TaskMoveEvent>
- taskSelected: EventEmitter<Task>
- columnConfigError: EventEmitter<string>
API Binding:
- GET /youth-accounts/{id}/dashboard for initial data
ARIA Attributes:
- role="region"
- aria-label="Kanban Board"
- aria-labelledby for each column header

KanbanColumnComponent:
Purpose: Individual column container (To Do, In Progress, Done)
Props:
- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean
State:
- isDragOver: boolean
- taskCount: number
Events:
- taskDropped: EventEmitter<DropEvent>
- addTaskClicked: EventEmitter<string>
API Binding:
- None (receives data from parent)
ARIA Attributes:
- role="region"
- aria-labelledby for column title

TaskCardComponent:
Purpose: Individual task display with drag/drop functionality
Props:
- task: Task
- draggable: boolean
- priority: 'high' | 'medium' | 'low'
State:
- isDragging: boolean
Events:
- taskClicked: EventEmitter<Task>
- dragStarted: EventEmitter<Task>
API Binding:
- None (receives data from parent)
ARIA Attributes:
- role="listitem"
- tabindex="0"
- aria-label with task details

HeaderComponent:
Purpose: Top navigation bar with search and user actions
Props:
- user: User
- notifications: Notification[]
State:
- searchQuery: string
- showNotifications: boolean
Events:
- searchSubmitted: EventEmitter<string>
- userMenuClicked: EventEmitter<void>
API Binding:
- GET /notifications for notification data
Components Used:
- AavaSearchBarComponent
- AavaAvatarsComponent
- AavaButtonComponent

SidebarComponent:
Purpose: Left navigation menu with accessibility features
Props:
- activeRoute: string
- navigationItems: NavItem[]
State:
- isCollapsed: boolean
Events:
- routeChanged: EventEmitter<string>
API Binding:
- None (static navigation)
ARIA Attributes:
- role="navigation"
- aria-label="Main navigation"

TaskModalComponent:
Purpose: Task creation/editing modal dialog
Props:
- task: Task | null
- isVisible: boolean
- mode: 'create' | 'edit'
State:
- formData: TaskFormData
- isSubmitting: boolean
Events:
- taskSaved: EventEmitter<Task>
- modalClosed: EventEmitter<void>
API Binding:
- POST /tasks for task creation
- PUT /tasks/{id} for task updates
Components Used:
- AavaButtonComponent
- FormInputComponent
- FormSelectComponent
- FormTextareaComponent

AnalyticsPanelComponent:
Purpose: Display team performance metrics
Props:
- metrics: PerformanceMetric[]
- loading: boolean
State:
- selectedMetric: string
Events:
- metricSelected: EventEmitter<string>
API Binding:
- GET /youth-accounts/{id}/dashboard for metrics data
Components Used:
- CardComponent
- BadgeComponent

13. INTERACTION FLOWS

User Action → UI → API:
- Load Kanban Board
  → KanbanBoardComponent.ngOnInit()
  → Call GET /youth-accounts/{id}/dashboard
  → Transform response to column data
  → Initialize columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
  → Render three columns with tasks
  → Handle error with *ngIf directive if column configuration fails

- Drag Task Between Columns
  → TaskCardComponent.onDragStart()
  → KanbanColumnComponent.onDrop()
  → Update local state
  → Call PUT /tasks/{id} (when available)
  → Update column counts

- Create New Task
  → AddTaskButtonComponent.onClick()
  → Open TaskModalComponent
  → Fill form and submit
  → Call POST /tasks (when available)
  → Refresh kanban board data

- Search Tasks
  → AavaSearchBarComponent.searchChange
  → HeaderComponent.onSearchSubmit()
  → Filter local task data
  → Update kanban board display

- Navigate Between Pages
  → SidebarComponent.routeChanged
  → AppComponent.onRouteChange
  → Show/hide pages with router

- Keyboard Navigation
  → Tab/Shift+Tab for element navigation
  → Enter/Space for activation
  → Arrow keys for menu navigation
  → Ctrl+M for task movement
  → Escape for modal close
