## UI COMPONENT ARCHITECTURE

1. JIRA REQUIREMENT SUMMARY

Story Description:
AS a parent or guardian I WANT to allocate and manage funds for my child's youth account SO THAT I can teach financial responsibility while maintaining visibility and control over spending

Acceptance Criteria:
- Scenario 1 вЂ“ Access youth account dashboard: Given the parent is logged into the banking web application When the parent navigates to the youth account section Then the system should display the youth account dashboard with current balance and recent activity
- Scenario 2 вЂ“ Allocate funds to youth account: Given the parent is on the youth account dashboard When the parent selects "Add Funds" Then the system should allow the parent to transfer funds from their primary account to the youth account
- Scenario 3 вЂ“ Set spending limit: Given the parent is viewing the youth account management section When the parent configures a weekly spending limit Then the system should save and display the limit on the dashboard
- Scenario 4 вЂ“ View youth spending activity: Given the parent is viewing the youth account dashboard When the parent selects the activity section Then the system should display recent transactions made from the youth account
- Scenario 5 вЂ“ Handle insufficient balance during fund allocation: Given the parent attempts to transfer funds When the selected parent account does not have sufficient balance Then the system should display an error message indicating insufficient funds

UI Tasks:
- SCIB-194: Design fund transfer interaction for youth account
- SCIB-195: Design youth account dashboard layout
- SCIB-196: Design spending limit configuration interface
- SCIB-197: Design youth account activity view

- PUT /api/youth-accounts/{youthAccountId}/spending-limit в†’ Update spending limit

- GET /api/youth-accounts/{youthAccountId}/transactions в†’ Get transaction history
- GET /api/youth-accounts/{youthAccountId}/validate в†’ Validate account access

Authentication:
- OAuth2 with scopes: youth-account:read, youth-account:write

Rate Limiting:
- Standard endpoints: 100 requests/minute
- Transaction endpoints: 10 requests/minute

13. INTERACTION FLOWS

User Action в†’ UI в†’ API:

1. View Dashboard:
   - User navigates to youth account section
   в†’ Load YouthAccountDashboard component
   в†’ Call GET /api/youth-accounts/{id}/dashboard
   в†’ Display account overview, spending limit, recent activity

2. Add Funds:
   - User clicks "Add Funds" button
   в†’ Open FundTransferModal
   в†’ User selects source account and enters amount
   в†’ User clicks "Transfer"
   в†’ Call POST /api/youth-accounts/{id}/fund-transfer
   в†’ Show success/error notification
   в†’ Refresh dashboard data

3. Set Spending Limit:
   - User clicks "Set Limit" or "Edit Limit"
   в†’ Open SpendingLimitModal
   в†’ User configures weekly limit
   в†’ User clicks "Save"
   в†’ Call PUT /api/youth-accounts/{id}/spending-limit
   в†’ Show success/error notification
   в†’ Update limit display

4. View Transaction History:
   - User clicks "View All Activity"
   в†’ Navigate to activity section
   в†’ Load TransactionTable component
   в†’ Call GET /api/youth-accounts/{id}/transactions
   в†’ Display paginated transaction list

5. Filter Transactions:
   - User applies filters (date range, type, etc.)
   в†’ Update TransactionFilters state
   в†’ Call GET /api/youth-accounts/{id}/transactions with filters
   в†’ Update transaction display

6. Handle Errors:
