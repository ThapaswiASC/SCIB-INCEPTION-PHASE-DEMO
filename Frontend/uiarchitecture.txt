### 6. COMPONENT DEFINITIONS

**KanbanBoardComponent:**
- **Purpose**: Main container for three-column kanban layout
- **Props**: 
  - columns: KanbanColumn[]
  - loading: boolean
  - error: string | null
- **State**:
  - draggedTask: Task | null
  - columnData: KanbanColumn[]
- **Events**:
  - taskMoved: EventEmitter<TaskMoveEvent>
  - taskCreated: EventEmitter<Task>
- **API Binding**:
  - GET /api/youth-accounts/{id}/dashboard
  - GET /api/youth-accounts/{id}/transactions

**TaskCardComponent:**
- **Purpose**: Individual task representation with drag-and-drop
- **Props**:
  - task: Task
  - draggable: boolean
  - compact: boolean
- **State**:
  - isDragging: boolean
  - isSelected: boolean
- **Events**:
  - taskClick: EventEmitter<Task>
  - taskDragStart: EventEmitter<DragEvent>
- **API Binding**: None (data passed from parent)

**HeaderComponent:**
- **Purpose**: Top navigation and user actions
- **Props**:
  - user: User
  - notifications: Notification[]
- **State**:
  - searchQuery: string
  - isUserMenuOpen: boolean
- **Events**:
  - searchSubmit: EventEmitter<string>
  - themeToggle: EventEmitter<void>
- **API Binding**: User profile and notifications APIs

**SidebarComponent (from library):**
- **Purpose**: Main navigation with collapsible functionality
- **Props**:
  - width: string
  - collapsedWidth: string
  - isCollapsed: boolean
  - menuItems: MenuItem[]
- **State**:
  - activeRoute: string
- **Events**:
  - collapseToggle: EventEmitter<boolean>
  - navigationClick: EventEmitter<MenuItem>
- **API Binding**: None (navigation only)

**TaskModalComponent:**
- **Purpose**: Task creation and editing dialog
- **Props**:
  - task: Task | null
  - isOpen: boolean
  - mode: 'create' | 'edit'
- **State**:
  - formData: TaskFormData
  - isSubmitting: boolean
- **Events**:
  - taskSave: EventEmitter<Task>
  - modalClose: EventEmitter<void>
- **API Binding**:
  - POST /api/tasks (create)
  - PUT /api/tasks/{id} (update)

### 8. API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Add funds
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limit
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history

**Authentication:**
- OAuth 2.0 with JWT tokens
- Scopes: youth-account:read, youth-account:write, fund-transfer:execute
- Rate limiting: 100 requests per minute

### 13. INTERACTION FLOWS

**User Action → UI → API:**

1. **View Dashboard**
   - User clicks "Dashboard" in sidebar
   - Navigate to DashboardComponent
   - Call GET /api/youth-accounts/{id}/dashboard
   - Display account balance, spending limits, recent transactions

2. **Create Task**
   - User clicks "Add Task" button
   - Open TaskModalComponent
   - User fills form and submits
   - Call POST /api/tasks (custom API)
   - Update kanban board state
   - Close modal and refresh view

3. **Move Task (Drag & Drop)**
   - User drags TaskCardComponent
   - Update UI state immediately
   - Call PUT /api/tasks/{id}/status
   - Handle success/error states
   - Revert on failure

4. **Fund Transfer**
   - User clicks "Add Funds" in dashboard
   - Open fund transfer modal
   - User enters amount and submits
   - Call POST /api/youth-accounts/{id}/fund-transfer
   - Update dashboard balance
   - Show success notification

5. **Update Spending Limit**
   - User navigates to settings
   - Modify spending limit field
   - Call PUT /api/youth-accounts/{id}/spending-limit
   - Update UI with new limit
   - Show confirmation message
