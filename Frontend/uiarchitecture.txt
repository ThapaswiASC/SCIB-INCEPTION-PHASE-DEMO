6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout
Props:
- columns: KanbanColumn[] - Array of column configurations
- tasks: Task[] - Array of task data
- loading: boolean - Loading state indicator
State:
- draggedTask: Task | null - Currently dragged task
- errorMessage: string | null - Error state message
Events:
- taskMoved: EventEmitter<TaskMoveEvent> - Task movement between columns
- taskCreated: EventEmitter<Task> - New task creation
- taskUpdated: EventEmitter<Task> - Task updates
API Binding:
- GET /api/youth-accounts/{id}/dashboard - Fetch dashboard data
- POST /api/youth-accounts/{id}/fund-transfer - Create fund transfer tasks

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn - Column configuration
- tasks: Task[] - Tasks for this column
- allowDrop: boolean - Enable drop functionality
State:
- isDropTarget: boolean - Drop target state
Events:
- taskDropped: EventEmitter<TaskDropEvent> - Task dropped in column
- addTaskClicked: EventEmitter<string> - Add task button clicked
API Binding:
- None (data flows from parent)

TaskCardComponent:
Purpose: Individual task display with drag functionality
Props:
- task: Task - Task data
- draggable: boolean - Enable drag functionality
State:
- isDragging: boolean - Drag state
Events:
- taskClicked: EventEmitter<Task> - Task selection
- taskDragStart: EventEmitter<Task> - Drag start
API Binding:
- PUT /api/youth-accounts/{id}/spending-limit - Update task limits

HeaderComponent:
Purpose: Top navigation bar with search and user menu
Props:
- user: User - Current user data
- notifications: Notification[] - User notifications
State:
- searchQuery: string - Search input value
- themeMode: 'light' | 'dark' - Current theme
Events:
- searchSubmitted: EventEmitter<string> - Search query submitted
- themeToggled: EventEmitter<string> - Theme change
- userMenuClicked: EventEmitter<void> - User menu interaction
API Binding:
- GET /api/youth-accounts/{id}/dashboard - User account data

ModalComponent:
Purpose: Reusable modal overlay for forms and dialogs
Props:
- isOpen: boolean - Modal visibility state
- title: string - Modal title
- size: 'sm' | 'md' | 'lg' - Modal size
State:
- isAnimating: boolean - Animation state
Events:
- modalClosed: EventEmitter<void> - Modal close event
- modalOpened: EventEmitter<void> - Modal open event
API Binding:
- None (content-agnostic)

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data with balance, limits, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds from parent to youth account
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Configure weekly spending limits
- GET /api/youth-accounts/{youthAccountId}/transactions → Get paginated transaction history with filtering

Authentication:
- OAuth 2.0 with JWT tokens
- Required scopes: youth-account:read, youth-account:write, fund-transfer:execute
- Rate limiting: 100 requests per minute

Data Models:
- DashboardResponse: account balance, spending limit, recent transactions
- FundTransferRequest/Response: transfer amount, confirmation details
- SpendingLimitRequest/Response: weekly limit configuration
- TransactionHistoryResponse: paginated transaction list with summary

13. INTERACTION FLOWS

User Action → UI → API:

1. View Dashboard:
   → Load DashboardPageComponent
   → Call GET /api/youth-accounts/{id}/dashboard
   → Display balance, limits, recent transactions

2. Create Task (Fund Transfer):
   → Click "Add Task" button
   → Open TaskCreationModal
   → Fill form and submit
   → Call POST /api/youth-accounts/{id}/fund-transfer
   → Update kanban board with new task
   → Show success notification

3. Move Task Between Columns:
   → Drag task card to different column
   → Update local state immediately
   → Call PUT /api/youth-accounts/{id}/spending-limit (if applicable)
   → Handle API response and error states

4. Configure Spending Limit:
   → Click "Settings" or limit configuration
   → Open SpendingLimitModal
   → Set new limit and submit
   → Call PUT /api/youth-accounts/{id}/spending-limit
   → Update UI with new limit
   → Show confirmation message

5. View Transaction History:
   → Navigate to "All Tasks" page
   → Call GET /api/youth-accounts/{id}/transactions
   → Display paginated transaction list
   → Apply filters and sorting as needed

6. Search and Filter:
   → Enter search query in header
   → Filter local data or call API with search parameters
   → Update displayed results
   → Maintain search state across navigation
