Purpose: Main container for three-column kanban layout

- columns: KanbanColumn[]

- error: string | null

- taskMoved: (task: Task, fromColumn: string, toColumn: string)

- GET /youth-accounts/{id}/dashboard for task data

Purpose: Individual column container with header and task list

- allowDrop: boolean

Events:

- addTaskClicked: (columnId: string)
API Binding:
- None (receives data from parent)

Purpose: Individual task display with drag functionality

8. API SUMMARY

Available APIs:
- GET /youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data including balance, spending limit, recent transactions
- POST /youth-accounts/{youthAccountId}/fund-transfer → Transfer funds from parent to youth account
- PUT /youth-accounts/{youthAccountId}/spending-limit → Configure weekly spending limit
- GET /youth-accounts/{youthAccountId}/transactions → Retrieve transaction history with filtering and pagination

Authentication:
- OAuth 2.0 with authorization code flow
- JWT Bearer token authentication
- Rate limiting: 1000 requests/hour per user

Request/Response Format:
- Content-Type: application/json
- Standard HTTP status codes
- Consistent error response format with code, message, timestamp, requestId

13. INTERACTION FLOWS

User Action → UI → API:

1. View Dashboard:
   → Load DashboardComponent
   → Call GET /youth-accounts/{id}/dashboard
   → Display account balance, spending limits, recent transactions

2. Create Task:
   → Click "Add Task" button
   → Open TaskFormModal
   → Fill form and submit
   → Call POST /youth-accounts/{id}/fund-transfer (adapted for task creation)
   → Update kanban board

3. Move Task:
   → Drag task card to different column
   → Update local state
   → Call PUT /youth-accounts/{id}/tasks/{taskId} (inferred API)
   → Confirm move or rollback on error

4. Configure Spending Limit:
   → Navigate to Settings
   → Update spending limit form
   → Call PUT /youth-accounts/{id}/spending-limit
   → Show success/error notification

5. View Transaction History:
   → Click on activity section
   → Call GET /youth-accounts/{id}/transactions
   → Display paginated transaction list

6. Search Tasks:
   → Type in search input
   → Filter local task data
   → Optionally call search API (not available)
