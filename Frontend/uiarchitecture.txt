6. COMPONENT DEFINITIONS

KanbanBoardComponent:
Purpose: Main container for three-column kanban layout with drag-and-drop functionality
Props:
- columns: KanbanColumn[] - Array of column configurations
- tasks: Task[] - Array of task data
- loading: boolean - Loading state indicator
State:
- draggedTask: Task | null - Currently dragged task
- errorMessage: string | null - Error state message
- mobileActiveColumn: string - Active column in mobile view
Events:
- taskMoved: EventEmitter<TaskMoveEvent> - Emitted when task is moved between columns
- taskCreated: EventEmitter<Task> - Emitted when new task is created
- taskUpdated: EventEmitter<Task> - Emitted when task is updated
API Binding:
- GET /api/youth-accounts/{id}/dashboard - Fetch dashboard data
- POST /api/youth-accounts/{id}/fund-transfer - Create new tasks (mapped)
- PUT /api/youth-accounts/{id}/spending-limit - Update task limits (mapped)

KanbanColumnComponent:
Purpose: Individual column container with header and task list
Props:
- column: KanbanColumn - Column configuration data
- tasks: Task[] - Tasks assigned to this column
- allowDrop: boolean - Whether column accepts dropped tasks
State:
- isDropTarget: boolean - Whether column is current drop target
- taskCount: number - Number of tasks in column
Events:
- taskDropped: EventEmitter<TaskDropEvent> - Emitted when task is dropped
- addTaskClicked: EventEmitter<string> - Emitted when add task button is clicked
API Binding:
- None (receives data from parent)

TaskCardComponent:
Purpose: Individual task display with drag functionality and task details
Props:
- task: Task - Task data object
- draggable: boolean - Whether task can be dragged
- compact: boolean - Compact display mode
State:
- isDragging: boolean - Whether task is being dragged
- isSelected: boolean - Whether task is selected
Events:
- taskClicked: EventEmitter<Task> - Emitted when task is clicked
- dragStarted: EventEmitter<Task> - Emitted when drag starts
- dragEnded: EventEmitter<Task> - Emitted when drag ends
API Binding:
- GET /api/youth-accounts/{id}/transactions - Fetch task history (mapped)

HeaderComponent:
Purpose: Top navigation bar with search, notifications, and user menu
Props:
- user: User - Current user information
- notifications: Notification[] - User notifications
- searchEnabled: boolean - Whether search is enabled
State:
- searchQuery: string - Current search query
- notificationCount: number - Unread notification count
Events:
- searchSubmitted: EventEmitter<string> - Emitted when search is performed
- themeToggled: EventEmitter<string> - Emitted when theme is changed
- userMenuClicked: EventEmitter<void> - Emitted when user menu is accessed
API Binding:
- None (receives data from parent)

SidebarComponent:
Purpose: Left navigation panel with menu items and project links
Props:
- navigationItems: NavigationItem[] - Menu items configuration
- currentPage: string - Currently active page
- collapsed: boolean - Whether sidebar is collapsed
State:
- isOpen: boolean - Mobile sidebar open state
Events:
- navigationClicked: EventEmitter<string> - Emitted when navigation item is clicked
- sidebarToggled: EventEmitter<boolean> - Emitted when sidebar is toggled
API Binding:
- None (static navigation)

ModalComponent:
Purpose: Reusable modal container for task creation and editing
Props:
- isOpen: boolean - Whether modal is visible
- title: string - Modal title
- size: 'sm' | 'md' | 'lg' - Modal size
State:
- isAnimating: boolean - Whether modal is animating
Events:
- modalClosed: EventEmitter<void> - Emitted when modal is closed
- modalOpened: EventEmitter<void> - Emitted when modal is opened
API Binding:
- None (content provided by parent)

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard в†’ Fetch dashboard data with balance and recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer в†’ Transfer funds (mapped to task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit в†’ Configure spending limits (mapped to task limits)
- GET /api/youth-accounts/{youthAccountId}/transactions в†’ Get transaction history (mapped to task history)
