        <div class="flow-description">
            <h3>Flow Sections Explained</h3>
            
            <h4>1. Dashboard Navigation (Start → ViewBoard)</h4>
            <ul>
                <li>User accesses the main dashboard</li>
                <li>System loads the kanban board with three columns: To Do, In Progress, Done</li>
                <li>User can navigate between Board, Analytics, and Issues sections</li>
            </ul>
            
            <h4>2. Task Creation Initiation (CreateTask → InitForm)</h4>
            <ul>
                <li>User clicks Add Task button from any kanban column</li>
                <li>Task creation modal opens with empty form</li>
                <li>Form initializes with all required fields</li>
            </ul>
            
            <h4>3. Form Field Completion (FillTitle → FormComplete)</h4>
            <ul>
                <li>Title input with real-time validation and character counter (max 255 chars)</li>
                <li>Description textarea with character counter (max 2000 chars)</li>
                <li>Priority selection from dropdown (LOW, MEDIUM, HIGH, CRITICAL)</li>
                <li>Assignee selection from user list or leave unassigned</li>
                <li>Due date picker with future date validation</li>
            </ul>
            
            <h4>4. Form Submission (SubmitForm → CloseModal)</h4>
            <ul>
                <li>Client-side validation of all fields</li>
                <li>Loading spinner during API call</li>
                <li>API call to POST /api/v1/tasks</li>
                <li>Success message and modal closure on success</li>
                <li>Error handling with retry option on failure</li>
            </ul>
            
            <h4>5. Task Management (ViewNewTask → RefreshBoard)</h4>
            <ul>
                <li>Created task appears in appropriate kanban column</li>
                <li>User can edit, move, or delete tasks</li>
                <li>Board refreshes to show updated task states</li>
                <li>Continuous workflow for task management</li>
            </ul>
        </div>
        
        <div class="flow-description">
            <h3>Technical Implementation Notes</h3>
            
            <h4>Components Involved:</h4>
            <ul>
                <li><strong>TaskCreationComponent:</strong> Main modal container with state management</li>
                <li><strong>TaskFormComponent:</strong> Reactive form with validation logic</li>
                <li><strong>TaskTitleInput:</strong> Title field with AavaTextboxComponent</li>
                <li><strong>TaskDescriptionInput:</strong> Description field with AavaTextareaComponent</li>
                <li><strong>TaskPrioritySelect:</strong> Custom dropdown for priority selection</li>
                <li><strong>TaskAssigneeSelect:</strong> User selection dropdown</li>
                <li><strong>TaskDueDatePicker:</strong> Date picker with AavaCalendarComponent</li>
            </ul>
            
            <h4>API Integration:</h4>
            <ul>
                <li><strong>POST /api/v1/tasks:</strong> Create new task with form data</li>
                <li><strong>GET /api/v1/users:</strong> Load available users for assignment</li>
                <li><strong>Error Handling:</strong> Standardized error responses with user feedback</li>
            </ul>
            
            <h4>Validation Rules:</h4>
            <ul>
                <li><strong>Title:</strong> Required, maximum 255 characters</li>
                <li><strong>Description:</strong> Optional, maximum 2000 characters</li>
                <li><strong>Priority:</strong> Required, must be valid enum value</li>
                <li><strong>Due Date:</strong> Required, must be future date</li>
                <li><strong>Assignee:</strong> Optional, must be valid user ID if selected</li>
            </ul>
