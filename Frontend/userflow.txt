            <h2>User Flow Description</h2>
            <p>This diagram represents the complete user interaction flow for the Kanban Board application. The flow covers all major user journeys and system interactions:</p>
            
            <h3>Main User Journeys:</h3>
            <ul>
                <li><strong>Board Loading:</strong> User opens the application, system checks authentication, loads board data, and displays the three-column layout (To Do, In Progress, Done)</li>
                <li><strong>Card Management:</strong> Users can add new cards, edit existing cards, delete cards, and move cards between columns</li>
                <li><strong>Drag and Drop:</strong> Users can drag cards from one column to another, with real-time API updates and error handling</li>
                <li><strong>Error Handling:</strong> System provides error states with retry options for failed operations</li>
                <li><strong>Authentication:</strong> Ensures users are authenticated before accessing board functionality</li>
            </ul>
            
            <h3>Key Interaction Points:</h3>
            <ul>
                <li><strong>Authentication Check:</strong> Validates user access before loading board data</li>
                <li><strong>API Integration:</strong> All CRUD operations (Create, Read, Update, Delete) are handled through REST API calls</li>
                <li><strong>Real-time Updates:</strong> UI updates immediately reflect user actions with API confirmation</li>
                <li><strong>Error Recovery:</strong> Failed operations provide retry mechanisms and clear error messaging</li>
                <li><strong>Responsive Design:</strong> Flow supports desktop, tablet, and mobile interactions</li>
            </ul>
            
            <h3>Component Interactions:</h3>
            <ul>
                <li><strong>KanbanBoardComponent:</strong> Main container managing overall board state and API calls</li>
                <li><strong>KanbanColumnComponent:</strong> Individual column containers handling card display and drop zones</li>
                <li><strong>KanbanCardComponent:</strong> Individual cards with drag functionality and edit capabilities</li>
                <li><strong>ErrorStateComponent:</strong> Error display with retry functionality for failed operations</li>
            </ul>
            
            <h3>API Endpoints Used:</h3>
            <ul>
                <li><strong>GET /kanban/boards/{boardId}:</strong> Load initial board data and columns</li>
                <li><strong>POST /kanban/cards:</strong> Create new cards in specified columns</li>
                <li><strong>PUT /kanban/cards/{cardId}:</strong> Update existing card information</li>
                <li><strong>PUT /kanban/cards/{cardId}/move:</strong> Move cards between columns</li>
                <li><strong>DELETE /kanban/cards/{cardId}:</strong> Remove cards from the board</li>
            </ul>
            
            <h3>Accessibility Features:</h3>
            <ul>
                <li><strong>ARIA Labels:</strong> All interactive elements include proper ARIA attributes</li>
                <li><strong>Keyboard Navigation:</strong> Full keyboard support for drag and drop operations</li>
                <li><strong>Screen Reader Support:</strong> Semantic HTML structure with role attributes</li>
                <li><strong>Focus Management:</strong> Proper focus handling during modal operations</li>
            </ul>
