            <h2>User Flow Description</h2>
            
            <div class="flow-section">
                <strong>1. Session Initialization:</strong>
                User opens the document upload page, system initializes a new form session and displays the header with progress tracking.
            </div>
            
            <div class="flow-section">
                <strong>2. Document Capture/Upload:</strong>
                User can choose between camera capture or file upload. Camera option requires permission and provides overlay guidance for proper document positioning.
            </div>
            
            <div class="flow-section">
                <strong>3. Quality Validation:</strong>
                System performs real-time quality assessment of the captured/uploaded document and provides feedback with improvement suggestions if needed.
            </div>
            
            <div class="flow-section">
                <strong>4. Multi-Step Form Navigation:</strong>
                User progresses through multiple steps (Document Upload → Personal Information → Review → Submission) with validation at each step.
            </div>
            
            <div class="flow-section">
                <strong>5. Review and Submission:</strong>
                User reviews all information before final submission, with options to edit any section if needed.
            </div>
            
            <div class="flow-section">
                <strong>6. Error Handling:</strong>
                System provides graceful error handling with retry options and fallback mechanisms for camera permissions and upload failures.
            </div>
            
            <div class="flow-section">
                <strong>7. Progress Tracking:</strong>
                Throughout the process, users can navigate between completed steps and track their progress via the header progress indicator.
            </div>
