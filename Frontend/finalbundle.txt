### COMPONENT DEFINITIONS

#### KanbanBoardComponent
**Purpose:** Main kanban board container with three-column layout
**Props:**
- columns: KanbanColumn[] - Array of column configurations
- tasks: Task[] - Array of all tasks
- loading: boolean - Loading state indicator
**State:**
- draggedTask: Task | null - Currently dragged task
- error: string | null - Error message for display
**Events:**
- taskMoved: (taskId: string, newColumnId: string) => void
- taskCreated: (task: Task) => void
- taskUpdated: (task: Task) => void
**API Binding:**
- GET /api/youth-accounts/{id}/dashboard for initial data

#### KanbanColumnComponent
**Purpose:** Individual column within kanban board
**Props:**
- column: KanbanColumn - Column configuration
- tasks: Task[] - Tasks for this column
- allowDrop: boolean - Whether column accepts drops
**State:**
- isDropTarget: boolean - Visual drop target state
**Events:**
- taskDropped: (taskId: string) => void
- addTaskClicked: () => void
**API Binding:** None (receives data from parent)

#### TaskCardComponent
**Purpose:** Individual task card with drag-and-drop functionality
**Props:**
- task: Task - Task data object
- draggable: boolean - Whether card is draggable
**State:**
- isDragging: boolean - Dragging state
**Events:**
- taskClicked: (taskId: string) => void
- taskDragStart: (taskId: string) => void
**API Binding:** None (receives data from parent)

#### HeaderComponent
**Purpose:** Application header with navigation and user controls
**Props:**
- user: User - Current user information
- notifications: Notification[] - User notifications
**State:**
- searchQuery: string - Current search input
- showUserMenu: boolean - User menu visibility
**Events:**
- searchSubmitted: (query: string) => void
- themeToggled: () => void
- userMenuToggled: () => void
**API Binding:** None (receives data from parent)

#### ModalComponent
**Purpose:** Reusable modal for task creation/editing
**Props:**
- isOpen: boolean - Modal visibility state
- title: string - Modal title
- task?: Task - Task data for editing
**State:**
- formData: TaskFormData - Form input values
- isSubmitting: boolean - Form submission state
**Events:**
- modalClosed: () => void
- taskSubmitted: (task: Task) => void
**API Binding:**
- POST /api/youth-accounts/{id}/fund-transfer for task creation

### DATA FLOW / PROPS

**Parent → Child Mapping:**
- AppComponent → HeaderComponent (user, notifications)

- KanbanPageComponent → KanbanBoardComponent (columns, tasks, loading)
- KanbanBoardComponent → KanbanColumnComponent (column, tasks, allowDrop)
- KanbanColumnComponent → TaskCardComponent (task, draggable)

**State Management Flow:**

### API SUMMARY

**Available APIs:**
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data with account balance, spending limit, recent transactions
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds (can be repurposed for task creation)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits (can be repurposed for task updates)
- GET /api/youth-accounts/{youthAccountId}/transactions → Get transaction history with pagination (can be repurposed for task history)

### API TO UI MAPPING

**Component → API:**
- KanbanBoardComponent → GET /api/youth-accounts/{id}/dashboard (fetch initial kanban data)
- TaskCardComponent → PUT /api/youth-accounts/{id}/spending-limit (update task status/details)
- ModalComponent → POST /api/youth-accounts/{id}/fund-transfer (create new tasks)
- TaskHistoryComponent → GET /api/youth-accounts/{id}/transactions (fetch task history)

**Data Transformation Required:**
- Dashboard API response needs mapping to kanban columns and tasks
- Fund transfer API needs adaptation for task creation workflow
- Spending limit API needs adaptation for task status updates
- Transaction history API needs mapping to task activity timeline
