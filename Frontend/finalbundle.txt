### 3.1 TaskCreationComponent

**Component Type:** Feature
**Mapped From A1:** TaskCreationComponent (JIRA Requirement)
**Purpose:** Create new tasks with reactive form validation and API integration
**Parent:** MainContentComponent
**Children:**
- AavaButtonComponent (Submit, Cancel)
- FormControl elements
- ValidationMessage components

**Dependencies:**
- @angular/forms (FormBuilder, Validators)
- TaskService
- @aava/play-core (AavaButtonComponent)

**Library Components Used:**
- AavaButtonComponent → Submit and Cancel actions

**TypeScript Specification**

**Inputs:**
- initialColumn: string (default column for new task)
- isVisible: boolean (modal visibility state)

**Outputs:**
- taskCreated: EventEmitter<Task> (emits created task)
- modalClosed: EventEmitter<void> (emits when modal closes)

**State:**
- taskForm: FormGroup (reactive form instance)
- isLoading: boolean (API call loading state)
- validationErrors: ValidationErrors (form validation errors)
- characterCounts: object (real-time character counters)

**Methods:**
- createTask() → Submits form and calls TaskService
- resetForm() → Clears form after successful submission
- validateField(fieldName: string) → Real-time field validation
- updateCharacterCount(field: string, value: string) → Updates character counters

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="task-creation-modal" *ngIf="isVisible">
  <div class="modal-overlay">
    <div class="modal-content">
      <div class="modal-header">
        <h2>Create New Task</h2>
        <button class="close-btn" (click)="closeModal()">×</button>
      </div>
      
      <form [formGroup]="taskForm" (ngSubmit)="createTask()">
        <div class="form-group">
          <label>Title *</label>
          <input formControlName="title" maxlength="255">
          <div class="character-counter">{{characterCounts.title}}/255</div>
          <div class="validation-error" *ngIf="taskForm.get('title')?.errors">
            Title is required
          </div>
        </div>
        
        <div class="form-group">
          <label>Description</label>
          <textarea formControlName="description" maxlength="2000"></textarea>
          <div class="character-counter">{{characterCounts.description}}/2000</div>
        </div>
        
        <div class="form-group">
          <label>Priority *</label>
          <select formControlName="priority">
            <option value="LOW">Low</option>
            <option value="MEDIUM">Medium</option>
            <option value="HIGH">High</option>
            <option value="CRITICAL">Critical</option>
          </select>
        </div>
        
        <div class="form-group">
          <label>Due Date *</label>
          <input type="date" formControlName="dueDate">
        </div>
        
        <div class="form-actions">
          <aava-button type="secondary" (click)="closeModal()">Cancel</aava-button>
          <aava-button type="primary" [loading]="isLoading" [disabled]="taskForm.invalid">
            Create Task
          </aava-button>
        </div>
      </form>
    </div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- position: fixed (modal overlay)
- display: flex, justify-content: center, align-items: center
- z-index: 1000 (modal layer)

**Spacing:**
- padding: 24px (modal content)
- gap: 16px (form groups)
- margin-bottom: 16px (form elements)

**Component Styles:**
- .task-creation-modal (full viewport overlay)
- .modal-content (centered dialog box)
- .form-group (field containers)
- .character-counter (right-aligned counters)
- .validation-error (error message styling)

**Responsive:**
- Mobile: full-width modal with reduced padding
- Tablet: max-width 600px
- Desktop: max-width 800px

**API INTEGRATION**

- createTask() → POST /api/v1/tasks → Create new task with form data
- validateTask() → Client-side validation matching API requirements

**ERROR HANDLING**

- API failure → Show error snackbar with retry option
- Validation error → Inline field-level error messages
- Network timeout → Show connection error message
- Form submission → Disable submit button during loading

**INTERACTION FLOW**

User Action → UI → API

- Click "Add Task" button → Open TaskCreationComponent modal → Initialize empty form
- Fill form fields → Real-time validation → Update character counters
- Submit form → Validate all fields → Call POST /api/v1/tasks → Show loading state
- API success → Emit taskCreated event → Close modal → Reset form
- API error → Show error message → Keep modal open for retry

### 3.2 KanbanBoardComponent

**Component Type:** Feature
**Mapped From A1:** KanbanBoardComponent
**Purpose:** Display and manage tasks in a three-column Kanban layout
**Parent:** MainContentComponent
**Children:**
- KanbanColumnComponent (3 instances)
- TaskCreationComponent
- EmptyStateComponent

**Dependencies:**
- TaskService
- DragDropModule (Angular CDK)

**Library Components Used:**

**TypeScript Specification**

**Inputs:**
- tasks: Task[] (all tasks data)
- boardConfig: BoardConfig (column configuration)

**Outputs:**
- taskMoved: EventEmitter<TaskMoveEvent> (task status changes)
- taskSelected: EventEmitter<Task> (task selection)

**State:**
- todoTasks: Task[] (filtered tasks)
- inProgressTasks: Task[] (filtered tasks)
- doneTasks: Task[] (filtered tasks)
- isLoading: boolean (data loading state)
- selectedTask: Task | null (currently selected task)

**Methods:**
- loadTasks() → Fetches tasks from API
- moveTask(task: Task, newStatus: string) → Updates task status
- filterTasksByStatus(status: string) → Filters tasks for columns
- openTaskCreation(columnType: string) → Opens creation modal

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-board">
  <div class="board-header">
    <h1>Task Board</h1>
    <aava-button type="primary" (click)="openTaskCreation('TODO')">
      Add Task
    </aava-button>
  </div>
  
  <div class="board-columns">
    <kanban-column 
      title="To Do" 
      [tasks]="todoTasks" 
      columnType="TODO"
      (taskMoved)="moveTask($event)"
      (addTaskClicked)="openTaskCreation($event)">
    </kanban-column>
    
    <kanban-column 
      title="In Progress" 
      [tasks]="inProgressTasks" 
      columnType="IN_PROGRESS"
      (taskMoved)="moveTask($event)">
    </kanban-column>
    
    <kanban-column 
      title="Done" 
      [tasks]="doneTasks" 
      columnType="COMPLETED"
      (taskMoved)="moveTask($event)">
    </kanban-column>
  </div>
  
  <task-creation 
    [isVisible]="showTaskCreation"
    [initialColumn]="selectedColumn"
    (taskCreated)="onTaskCreated($event)"
    (modalClosed)="closeTaskCreation()">
  </task-creation>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: grid
- grid-template-columns: 1fr 1fr 1fr (three equal columns)
- gap: 24px (column spacing)

**Spacing:**
- padding: 24px (board container)
- margin-bottom: 16px (board header)

**Component Styles:**
- .kanban-board (main container)
- .board-header (title and actions)
- .board-columns (grid layout)

**Responsive:**
- Mobile: single column stack
- Tablet: two columns
- Desktop: three columns

**API INTEGRATION**

- loadTasks() → GET /api/v1/tasks → Fetch all tasks for board display
- moveTask() → PUT /api/v1/tasks/{id} → Update task status

**ERROR HANDLING**

- API failure → Show error banner with retry option
- Task move failure → Revert UI change and show error
- Loading error → Show empty state with reload button

**INTERACTION FLOW**

User Action → UI → API

- Load board → Call GET /api/v1/tasks → Distribute tasks to columns
- Drag task between columns → Update UI optimistically → Call PUT /api/v1/tasks/{id}
- Click "Add Task" → Open TaskCreationComponent → Pass column context
