- Purpose: Display error messages when column configuration fails

### 3.1 KanbanBoardComponent

**Component Type:** Feature
**Mapped From A1:** Main Kanban board container with three-column layout
**Purpose:** Primary container component that manages the entire Kanban board with three columns (To Do, In Progress, Done)
**Parent:** App/Page Component
**Children:** KanbanColumnComponent (x3), ErrorStateComponent (conditional)

**Dependencies:**
- Angular CDK (Drag & Drop)
- KanbanService
- Shared UI Components

**Library Components Used:**
- awe-card → Board container styling
- Button → Action buttons
- Badge → Status indicators

**TypeScript Specification:**

**Inputs:**
- boardId: string (board identifier)
- permissions: BoardPermissions (user access rights)
- initialData: KanbanBoard | null (optional initial board data)

**Outputs:**
- onCardMove: EventEmitter<CardMoveEvent> (card moved between columns)
- onCardAdd: EventEmitter<CardAddEvent> (new card created)
- onCardEdit: EventEmitter<CardEditEvent> (card updated)
- onError: EventEmitter<string> (error occurred)

**State:**
- columns: KanbanColumn[] (three column configuration)
- isLoading: boolean (loading state)
- errorMessage: string | null (error state)
- draggedCard: KanbanCard | null (currently dragged card)

**Methods:**
- ngOnInit() → Initialize component and load board data
- loadBoardData() → Fetch board data from API
- handleCardMove(cardId: string, fromColumn: string, toColumn: string) → Process card movement
- handleCardAdd(columnId: string, cardData: any) → Add new card to column
- handleError(error: string) → Handle and display errors
- retryOperation() → Retry failed operations

**HTML STRUCTURE:**

```html
<div class="kanban-board-container" 
     role="region" 
     aria-label="Kanban Board" 
     [attr.aria-busy]="isLoading">
  
  <div class="kanban-board-header">
    <h1>Kanban Board</h1>
    <div class="board-actions">
      <button type="button" class="btn-primary">Add Card</button>
    </div>
  </div>
  
  <div class="kanban-columns-grid" 
       *ngIf="!errorMessage; else errorTemplate">
    <app-kanban-column 
      *ngFor="let column of columns; trackBy: trackByColumnId"
      [column]="column"
      [cards]="column.cards"
      [allowDrop]="!isLoading"
      (onCardDrop)="handleCardMove($event)"
      (onAddCard)="handleCardAdd($event)">
    </app-kanban-column>
  </div>
  
  <ng-template #errorTemplate>
    <app-error-state 
      [error]="errorMessage"
      (onRetry)="retryOperation()">
    </app-error-state>
  </ng-template>
  
</div>
```

**CSS SPECIFICATION:**

**Layout:**
- .kanban-board-container: flex column layout, full height
- .kanban-columns-grid: display: grid, grid-template-columns: repeat(3, 1fr), gap: 1rem

**Spacing:**
- padding: 1.5rem
- gap: 1rem between columns
- margin: 0.5rem for headers

**Component Styles:**
- .kanban-board-container: background: var(--surface-color), border-radius: 8px
- .kanban-board-header: flex justify-between, padding-bottom: 1rem
- .kanban-columns-grid: flex: 1, overflow: hidden

**Responsive:**
- Desktop (>1024px): 3 columns side by side
- Tablet (768px-1024px): 2 columns with horizontal scroll
- Mobile (<768px): 1 column with tab navigation

**API INTEGRATION:**
- ngOnInit() → GET /kanban/boards/{boardId} → Load initial board data
- handleCardAdd() → POST /kanban/cards → Create new card
- handleCardMove() → PUT /kanban/cards/{cardId}/move → Move card between columns

**ERROR HANDLING:**
- API failure → show ErrorStateComponent with retry option
- Network error → display offline message with retry
- Validation error → show inline error messages

**INTERACTION FLOW:**
**User Action → UI → API:**
- Page load → show loading spinner → GET board data → render columns
- Drag card → update UI optimistically → PUT move API → confirm/rollback
- Add card → show form modal → POST card data → update column
- Retry error → clear error state → retry failed operation

### 3.2 KanbanColumnComponent

**Component Type:** Feature
**Mapped From A1:** Individual column container with header and card list
**Purpose:** Represents a single column (To Do, In Progress, Done) with drag/drop functionality
**Parent:** KanbanBoardComponent
**Children:** KanbanCardComponent (multiple), AddCardComponent (inline form)

**Dependencies:**
- Angular CDK Drag & Drop
- Shared UI Components

**Library Components Used:**
- awe-card → Column container styling
- Button → Add card button
- Badge → Card count indicator

**TypeScript Specification:**

**Inputs:**
- column: KanbanColumn (column configuration)
- cards: KanbanCard[] (cards in this column)
- allowDrop: boolean (enable/disable drop functionality)

**Outputs:**
- onCardDrop: EventEmitter<CardDropEvent> (card dropped in column)
- onAddCard: EventEmitter<CardAddEvent> (add card to column)

**State:**
- isDragOver: boolean (drag over state)
- isAddingCard: boolean (show add card form)
- newCardData: Partial<KanbanCard> (new card form data)

**Methods:**
- onDragOver(event: CdkDragDrop) → Handle drag over events
- onCardDrop(event: CdkDragDrop) → Process card drop
- startAddCard() → Show add card form
- cancelAddCard() → Hide add card form
- submitNewCard() → Submit new card data

**HTML STRUCTURE:**

```html
<div class="kanban-column" 
     [class.drag-over]="isDragOver"
     cdkDropList
     [cdkDropListData]="cards"
     (cdkDropListDropped)="onCardDrop($event)">
  
  <div class="column-header" [attr.aria-labelledby]="column.id + '-header'">
    <h2 [id]="column.id + '-header'">{{ column.title }}</h2>
    <span class="card-count badge-neutral">{{ cards.length }}</span>
  </div>
  
  <div class="column-content" [attr.aria-label]="column.title + ' cards'">
    <app-kanban-card 
      *ngFor="let card of cards; trackBy: trackByCardId"
      [card]="card"
      [draggable]="allowDrop"
      cdkDrag>
    </app-kanban-card>
    
    <div class="add-card-section" *ngIf="!isAddingCard">
      <button type="button" 
              class="btn-ghost btn-small add-card-btn"
              (click)="startAddCard()">
        + Add Card
      </button>
    </div>
    
    <div class="add-card-form" *ngIf="isAddingCard">
      <input type="text" 
             [(ngModel)]="newCardData.title"
             placeholder="Card title"
             class="card-title-input">
      <div class="form-actions">
        <button type="button" class="btn-primary btn-small" (click)="submitNewCard()">Add</button>
        <button type="button" class="btn-ghost btn-small" (click)="cancelAddCard()">Cancel</button>
      </div>
    </div>
  </div>
  
</div>
```

**CSS SPECIFICATION:**

**Layout:**
- .kanban-column: flex column, min-height: 500px, flex: 1
- .column-content: flex: 1, overflow-y: auto, padding: 0.5rem

**Spacing:**
- padding: 1rem
- gap: 0.5rem between cards
- margin: 0.25rem for form elements

**Component Styles:**
- .kanban-column: background: var(--surface-light), border-radius: 8px, border: 1px solid var(--border-color)
- .column-header: flex justify-between, padding-bottom: 0.5rem, border-bottom: 1px solid var(--border-light)
- .drag-over: border-color: var(--primary-color), background: var(--primary-light)

**Responsive:**
- Desktop: fixed width columns
- Tablet: flexible width with min-width
- Mobile: full width stacked

**API INTEGRATION:**
- onAddCard() → Emit to parent → POST /kanban/cards → Add card to column
- onCardDrop() → Emit to parent → PUT /kanban/cards/{cardId}/move → Update card position

**ERROR HANDLING:**
- Card drop failure → revert UI state → show error message
- Add card failure → show inline error → keep form open

**INTERACTION FLOW:**
**User Action → UI → API:**
- Click "Add Card" → show form → enter data → submit → API call
- Drag card over column → highlight drop zone → drop → move API
- Scroll column → lazy load more cards if needed

### 3.3 KanbanCardComponent

**Component Type:** Feature
**Mapped From A1:** Individual task card with drag/drop and edit functionality
**Purpose:** Represents a single task card that can be dragged between columns and edited
**Parent:** KanbanColumnComponent
**Children:** None

**Dependencies:**
- Angular CDK Drag & Drop
- Shared UI Components

**Library Components Used:**
- awe-card → Card container styling
- Badge → Priority/status badges
- Button → Action buttons

**TypeScript Specification:**

**Inputs:**
- card: KanbanCard (card data)
- draggable: boolean (enable/disable drag)

**Outputs:**
- onEdit: EventEmitter<KanbanCard> (card edit requested)
- onDelete: EventEmitter<string> (card deletion requested)

**State:**
- isDragging: boolean (currently being dragged)
- isEditing: boolean (in edit mode)
- editData: Partial<KanbanCard> (edit form data)

**Methods:**
- onDragStart() → Set dragging state
- onDragEnd() → Clear dragging state
- startEdit() → Enter edit mode
- saveEdit() → Save card changes
- cancelEdit() → Cancel edit mode
- deleteCard() → Request card deletion

**HTML STRUCTURE:**

```html
<div class="kanban-card" 
     [class.dragging]="isDragging"
     [class.editing]="isEditing"
     [attr.aria-label]="'Card: ' + card.title"
     role="article">
  
  <div class="card-content" *ngIf="!isEditing">
    <div class="card-header">
      <h3 class="card-title">{{ card.title }}</h3>
      <div class="card-actions">
        <button type="button" 
                class="btn-ghost btn-small"
                (click)="startEdit()"
                aria-label="Edit card">
          Edit
        </button>
      </div>
    </div>
    
    <div class="card-body" *ngIf="card.description">
      <p class="card-description">{{ card.description }}</p>
    </div>
    
    <div class="card-footer">
      <span class="card-priority badge-{{ card.priority }}">{{ card.priority }}</span>
      <span class="card-assignee" *ngIf="card.assignee">{{ card.assignee }}</span>
    </div>
  </div>
  
  <div class="card-edit-form" *ngIf="isEditing">
    <input type="text" 
           [(ngModel)]="editData.title"
           class="edit-title-input"
           placeholder="Card title">
    <textarea [(ngModel)]="editData.description"
              class="edit-description-input"
              placeholder="Description"></textarea>
    <div class="edit-actions">
      <button type="button" class="btn-primary btn-small" (click)="saveEdit()">Save</button>
      <button type="button" class="btn-ghost btn-small" (click)="cancelEdit()">Cancel</button>
      <button type="button" class="btn-danger btn-small" (click)="deleteCard()">Delete</button>
    </div>
  </div>
  
</div>
```

**Layout:**

**Spacing:**

**Component Styles:**

**Responsive:**

**API INTEGRATION:**
- saveEdit() → Emit to parent → PUT /kanban/cards/{cardId} → Update card

**ERROR HANDLING:**
- Edit save failure → show inline error → keep edit mode

**INTERACTION FLOW:**
**User Action → UI → API:**
