## 3. UI COMPONENT SPECIFICATIONS (FROM AGENT-2)

### 3.1 KanbanBoardComponent

**Component Type:** Feature
**Purpose:** Main container component that manages the three-column Kanban board layout with drag-and-drop functionality
**Parent:** KanbanPageComponent
**Children:** KanbanColumnComponent (3 instances), TaskModalComponent

**TypeScript Specification:**

**Inputs:**
- columns: KanbanColumn[] (array of column configurations)
- tasks: Task[] (array of all tasks)
- loading: boolean (loading state indicator)
- error: string | null (error message)

**Outputs:**
- taskMoved: EventEmitter<TaskMoveEvent> (when task is moved between columns)
- taskCreated: EventEmitter<Task> (when new task is created)
- taskUpdated: EventEmitter<Task> (when task is updated)
- taskDeleted: EventEmitter<string> (when task is deleted, emits task ID)

**State:**
- draggedTask: Task | null (currently dragged task)
- columnData: KanbanColumn[] (processed column data with tasks)
- isLoading: boolean (component loading state)
- errorMessage: string | null (error display message)
- isTaskModalOpen: boolean (modal visibility state)
- selectedTask: Task | null (task being edited)

**Methods:**
- ngOnInit(): void → Initialize component and load data
- loadKanbanData(): void → Fetch initial kanban data
- onTaskDrop(event: CdkDragDrop<Task[]>): void → Handle task drop events
- openTaskModal(task?: Task): void → Open task creation/editing modal
- closeTaskModal(): void → Close task modal
- handleTaskSave(task: Task): void → Process task save operations
- handleError(error: any): void → Process and display errors

**HTML Structure:**
```html
<div class="kanban-board-container" 
     role="region" 
     aria-label="Kanban Board" 
     [attr.aria-busy]="isLoading">
  
  <div class="kanban-error" *ngIf="errorMessage" role="alert">
    <span>{{ errorMessage }}</span>
    <button (click)="loadKanbanData()">Retry</button>
  </div>
  
  <div class="kanban-grid" 
       cdkDropListGroup 
       *ngIf="!errorMessage">
    
    <app-kanban-column 
      *ngFor="let column of columnData; trackBy: trackByColumnId"
      [column]="column"
      [tasks]="getTasksForColumn(column.id)"
      [loading]="isLoading"
      (taskDropped)="onTaskDrop($event)"
      (addTaskClicked)="openTaskModal()"
      (taskClicked)="openTaskModal($event)">
    </app-kanban-column>
    
  </div>
  
  <app-task-modal
    [isOpen]="isTaskModalOpen"
    [task]="selectedTask"
    [mode]="selectedTask ? 'edit' : 'create'"
    (taskSaved)="handleTaskSave($event)"
    (modalClosed)="closeTaskModal()">
  </app-task-modal>
  
</div>
```

**CSS Specification:**
- .kanban-board-container: flex column, full height
- .kanban-grid: CSS Grid with grid-template-columns: repeat(3, 1fr)
- gap: 1.5rem between columns
- Responsive: Desktop (>1024px): 3 columns, Tablet (768px-1024px): 3 columns reduced padding, Mobile (<768px): single column stack

### 3.2 KanbanColumnComponent

**Component Type:** Feature
**Purpose:** Represents a single column in the Kanban board (To Do, In Progress, Done) with drag-drop functionality
**Parent:** KanbanBoardComponent
**Children:** TaskCardComponent (multiple instances), ButtonComponent (Add Task button)

**TypeScript Specification:**

**Inputs:**
- column: KanbanColumn (column configuration with id, title, color)
- tasks: Task[] (tasks assigned to this column)
- loading: boolean (loading state)
- allowDrop: boolean (whether column accepts drops)

**Outputs:**
- taskDropped: EventEmitter<CdkDragDrop<Task[]>> (when task is dropped in column)
- addTaskClicked: EventEmitter<string> (when add task button clicked, emits column ID)
- taskClicked: EventEmitter<Task> (when task card is clicked)
- menuToggled: EventEmitter<string> (when column menu is toggled)

**State:**
- isDragOver: boolean (drag over state for visual feedback)
- isLoading: boolean (column-specific loading)
- taskCount: number (computed task count)
- isMenuOpen: boolean (column menu visibility)

**Methods:**
- ngOnInit(): void → Initialize column
- onTaskDrop(event: CdkDragDrop<Task[]>): void → Handle task drops
- onAddTaskClick(): void → Emit add task event
- onTaskClick(task: Task): void → Emit task click event
- toggleMenu(): void → Toggle column menu
- getTaskCount(): number → Calculate task count

**HTML Structure:**
```html
<div class="kanban-column" 
     [class.drag-over]="isDragOver"
     [attr.aria-labelledby]="column.id + '-header'">
  
  <div class="column-header" [id]="column.id + '-header'">
    <h3 class="column-title">{{ column.title }}</h3>
    <span class="task-count" aria-label="Task count">{{ taskCount }}</span>
    <button class="column-menu-btn" 
            (click)="toggleMenu()"
            [attr.aria-expanded]="isMenuOpen">
      <i class="icon-more" aria-hidden="true"></i>
    </button>
  </div>
  
  <div class="column-content"
       cdkDropList
       [cdkDropListData]="tasks"
       (cdkDropListDropped)="onTaskDrop($event)"
       (cdkDropListEntered)="isDragOver = true"
       (cdkDropListExited)="isDragOver = false">
    
    <app-task-card
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="!loading"
      cdkDrag
      (taskClicked)="onTaskClick(task)">
    </app-task-card>
    
    <div class="empty-state" *ngIf="tasks.length === 0 && !loading">
      <p>No tasks in {{ column.title }}</p>
    </div>
    
    <div class="loading-state" *ngIf="loading">
      <div class="spinner" aria-label="Loading tasks"></div>
    </div>
    
  </div>
  
  <div class="column-footer">
    <app-button
      variant="ghost"
      size="small"
      iconName="plus"
      label="Add Task"
      (userClick)="onAddTaskClick()"
      [disabled]="loading">
    </app-button>
  </div>
  
</div>
```

**CSS Specification:**
- .kanban-column: flex column, min-height: 500px
- .column-content: flex-grow: 1, overflow-y: auto
- .column-header: flex row, justify-content: space-between
- .drag-over: border highlight and background change

### 3.3 TaskCardComponent

**Component Type:** Feature
**Purpose:** Displays individual task information with priority, assignee, and metadata in a draggable card format
**Parent:** KanbanColumnComponent
**Children:** UserAvatarComponent, ChipTagComponent (from library)

**TypeScript Specification:**

**Inputs:**
- task: Task (task data object)
- draggable: boolean (whether card can be dragged)
- priority: TaskPriority (high, medium, low)
- selected: boolean (selection state)

**Outputs:**
- taskClicked: EventEmitter<Task> (when card is clicked)
- dragStarted: EventEmitter<Task> (when drag begins)
- dragEnded: EventEmitter<Task> (when drag ends)
- menuClicked: EventEmitter<Task> (when task menu is clicked)

**State:**
- isDragging: boolean (current drag state)
- isSelected: boolean (selection state)
- isHovered: boolean (hover state)
- showMenu: boolean (menu visibility)

**Methods:**
- ngOnInit(): void → Initialize task card
- onCardClick(): void → Handle card click events
- onDragStart(): void → Handle drag start
- onDragEnd(): void → Handle drag end
- toggleMenu(): void → Toggle task menu
- getPriorityColor(): string → Get priority indicator color
- formatDueDate(): string → Format due date display

**HTML Structure:**
```html
<div class="task-card"
     [class.dragging]="isDragging"
     [class.selected]="isSelected"
     [class.high-priority]="task.priority === 'high'"
     (click)="onCardClick()"
     (mouseenter)="isHovered = true"
     (mouseleave)="isHovered = false"
     tabindex="0"
     role="button"
     [attr.aria-label]="'Task: ' + task.title">
  
  <div class="task-header">
    <div class="task-priority">
      <app-chip-tag
        [label]="task.priority"
        [variant]="getPriorityVariant(task.priority)"
        size="small">
      </app-chip-tag>
    </div>
    
    <button class="task-menu-btn"
            *ngIf="isHovered"
            (click)="toggleMenu(); $event.stopPropagation()"
            [attr.aria-expanded]="showMenu">
      <i class="icon-more" aria-hidden="true"></i>
    </button>
  </div>
  
  <div class="task-content">
    <h4 class="task-title">{{ task.title }}</h4>
    <p class="task-description" *ngIf="task.description">
      {{ task.description | truncate:100 }}
    </p>
  </div>
  
  <div class="task-metadata">
    <div class="task-assignee" *ngIf="task.assignee">
      <app-user-avatar
        [user]="task.assignee"
        size="small"
        [showTooltip]="true">
      </app-user-avatar>
    </div>
    
    <div class="task-due-date" *ngIf="task.dueDate">
      <i class="icon-calendar" aria-hidden="true"></i>
      <span>{{ formatDueDate() }}</span>
    </div>
    
    <div class="task-tags" *ngIf="task.tags?.length">
      <app-chip-tag
        *ngFor="let tag of task.tags | slice:0:2"
        [label]="tag"
        variant="outline"
        size="small">
      </app-chip-tag>
      <span *ngIf="task.tags.length > 2" class="more-tags">
        +{{ task.tags.length - 2 }}
      </span>
    </div>
  </div>
  
  <div class="drag-handle" *ngIf="draggable" aria-hidden="true">
    <i class="icon-drag"></i>
  </div>
  
</div>
```

**CSS Specification:**
- .task-card: flex column, border-radius: 8px, white background, subtle shadow
- .task-header: flex row, justify-content: space-between
- .task-metadata: flex row, align-items: center, gap: 0.5rem
- .dragging: increased shadow, slight rotation
- .high-priority: left border accent color

### 3.4 TaskModalComponent

**Component Type:** Feature
**Purpose:** Modal dialog for creating new tasks and editing existing tasks with form validation
**Parent:** KanbanBoardComponent
**Children:** DialogComponent (from library), ButtonComponent (from library), Form input components

**TypeScript Specification:**

**Inputs:**
- task: Task | null (task to edit, null for create mode)
- isOpen: boolean (modal visibility state)
- mode: 'create' | 'edit' (operation mode)
- columns: KanbanColumn[] (available columns for status selection)

**Outputs:**
- taskSaved: EventEmitter<Task> (when task is saved)
- modalClosed: EventEmitter<void> (when modal is closed)
- taskDeleted: EventEmitter<string> (when task is deleted, emits task ID)
- formChanged: EventEmitter<boolean> (when form validity changes)

**State:**
- taskForm: FormGroup (reactive form for task data)
- isSubmitting: boolean (form submission state)
- validationErrors: ValidationErrors (form validation errors)
- isDirty: boolean (form dirty state)
- showDeleteConfirm: boolean (delete confirmation state)

**Methods:**
- ngOnInit(): void → Initialize form and modal
- ngOnChanges(): void → Handle input changes
- initializeForm(): void → Setup reactive form
- onSave(): void → Handle form submission
- onCancel(): void → Handle modal cancellation
- onDelete(): void → Handle task deletion
- validateForm(): boolean → Validate form data
- resetForm(): void → Reset form to initial state

**HTML Structure:**
```html
<app-dialog
  [isOpen]="isOpen"
  [title]="mode === 'create' ? 'Create New Task' : 'Edit Task'"
  size="medium"
  (dialogClosed)="onCancel()"
  role="dialog"
  aria-modal="true">
  
  <div class="task-modal-content">
    
    <form [formGroup]="taskForm" 
          (ngSubmit)="onSave()"
          novalidate>
      
      <div class="form-section">
        <label for="task-title" class="form-label required">
          Task Title
        </label>
        <input
          id="task-title"
          type="text"
          formControlName="title"
          class="form-input"
          [class.error]="taskForm.get('title')?.invalid && taskForm.get('title')?.touched"
          placeholder="Enter task title"
          maxlength="100"
          required>
        <div class="error-message" 
             *ngIf="taskForm.get('title')?.invalid && taskForm.get('title')?.touched">
          Title is required
        </div>
      </div>
      
      <div class="form-section">
        <label for="task-description" class="form-label">
          Description
        </label>
        <textarea
          id="task-description"
          formControlName="description"
          class="form-textarea"
          placeholder="Enter task description"
          rows="4"
          maxlength="500">
        </textarea>
      </div>
      
      <div class="form-row">
        <div class="form-section">
          <label for="task-priority" class="form-label">
            Priority
          </label>
          <select
            id="task-priority"
            formControlName="priority"
            class="form-select">
            <option value="low">Low</option>
            <option value="medium">Medium</option>
            <option value="high">High</option>
          </select>
        </div>
        
        <div class="form-section">
          <label for="task-status" class="form-label">
            Status
          </label>
          <select
            id="task-status"
            formControlName="status"
            class="form-select">
            <option *ngFor="let column of columns" [value]="column.id">
              {{ column.title }}
            </option>
          </select>
        </div>
      </div>
      
      <div class="form-section">
        <label for="task-due-date" class="form-label">
          Due Date
        </label>
        <input
          id="task-due-date"
          type="date"
          formControlName="dueDate"
          class="form-input"
          [min]="today">
      </div>
      
    </form>
    
  </div>
  
  <div class="modal-actions" slot="footer">
    
    <div class="action-group-left">
      <app-button
        *ngIf="mode === 'edit'"
        variant="danger"
        size="medium"
        label="Delete"
        iconName="trash"
        (userClick)="onDelete()"
        [disabled]="isSubmitting">
      </app-button>
    </div>
    
    <div class="action-group-right">
      <app-button
        variant="secondary"
        size="medium"
        label="Cancel"
        (userClick)="onCancel()"
        [disabled]="isSubmitting">
      </app-button>
      
      <app-button
        variant="primary"
        size="medium"
        [label]="mode === 'create' ? 'Create Task' : 'Save Changes'"
        iconName="check"
        (userClick)="onSave()"
        [disabled]="taskForm.invalid || isSubmitting"
        [processing]="isSubmitting">
      </app-button>
    </div>
    
  </div>
  
</app-dialog>
```

**CSS Specification:**
- .task-modal-content: flex column, gap: 1.5rem

- Impact: Component errors could crash entire application sections.
