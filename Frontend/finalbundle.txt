## 3. UI COMPONENT SPECIFICATIONS (FROM AGENT-2)

### 3.1 TaskCreationComponent

**Component Type:** Feature

**Mapped From A1:** TaskCreationComponent (Primary Focus - NEW)

**Purpose:** Main component for creating new tasks with comprehensive form validation, reactive forms, and API integration

**Parent:** App Container / Kanban Board

**Children:**
- TaskFormComponent
- ValidationFeedbackComponent
- CharacterCounterComponent
- LoadingSpinnerComponent

**Dependencies:**
- Angular Reactive Forms
- TaskService
- FormBuilder
- Validators

**Library Components Used:**
- ButtonComponent → Primary/Secondary variants for form actions
- ModalComponent → Overlay container

**TypeScript Specification**

**Inputs:**
- isOpen: boolean (modal visibility state)
- onClose: () => void (modal close handler)
- onTaskCreated: (task: Task) => void (success callback)

**Outputs:**
- taskCreated: EventEmitter<Task> (emits created task)
- formClosed: EventEmitter<void> (emits when modal closes)
- validationChanged: EventEmitter<boolean> (emits form validity state)

**State:**
- formGroup: FormGroup (reactive form instance)
- isLoading: boolean (API call state)
- validationErrors: ValidationError[] (form validation errors)
- characterCounts: { title: number, description: number } (character counters)
- submitAttempted: boolean (tracks submission attempts)

**Methods:**
- initializeForm() → void (setup reactive form with validators)
- onSubmit() → void (handle form submission)
- resetForm() → void (reset form after successful submission)
- closeModal() → void (close modal and cleanup)
- updateCharacterCount(field: string, count: number) → void (update character counters)

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<modal-component [isOpen]="isOpen" (onClose)="closeModal()">
  <div class="task-creation-container">
    <div class="modal-header">
      <h2>Create New Task</h2>
      <button-component variant="secondary" (userClick)="closeModal()">Close</button-component>
    </div>
    
    <div class="modal-body">
      <task-form-component 
        [formGroup]="formGroup" 
        [isLoading]="isLoading"
        (formSubmit)="onSubmit()"
        (characterCountChanged)="updateCharacterCount($event.field, $event.count)">
      </task-form-component>
      
      <validation-feedback-component 
        [field]="formGroup.get('title')"
        fieldName="title"
        [validationRules]="titleValidationRules">
      </validation-feedback-component>
      
      <character-counter-component 
        [currentLength]="characterCounts.title"
        [maxLength]="255"
        fieldName="title">
      </character-counter-component>
    </div>
    
    <div class="modal-footer">
      <button-component 
        variant="secondary" 
        (userClick)="closeModal()"
        [disabled]="isLoading">Cancel</button-component>
      <button-component 
        variant="primary" 
        (userClick)="onSubmit()"
        [processing]="isLoading"
        [disabled]="!formGroup.valid">Create Task</button-component>
    </div>
    
    <loading-spinner-component 
      [isVisible]="isLoading"
      message="Creating task...">
    </loading-spinner-component>
  </div>
</modal-component>
```

**CSS SPECIFICATION**

**Layout:**
- display: flex, flex-direction: column
- modal-container: max-width 600px, centered
- form sections: gap 1.5rem

**Spacing:**
- padding: 2rem
- gap: 1rem between form fields
- margin: 0.5rem for validation messages

**Component Styles:**
- .task-creation-container: background glass morphism
- .modal-header: border-bottom, flex justify-between
- .modal-body: flex-1, overflow-y auto
- .modal-footer: border-top, flex justify-end gap 1rem

**Responsive:**
- @media (max-width: 768px): full-width modal
- @media (max-width: 480px): reduced padding

**API INTEGRATION**

- onSubmit() → POST /api/v1/tasks → Create new task with form data
- TaskService.createTask(taskData) → Handle API response and errors

**ERROR HANDLING**

- API failure → Display error snackbar with retry option
- Validation error → Show inline validation messages
- Network error → Show connection error message
- 401 Unauthorized → Redirect to login
- 403 Forbidden → Show permission error
- 429 Rate Limit → Show rate limit message with retry timer

**INTERACTION FLOW**

User Action → UI → API
- Click "Add Task" → Open modal → Initialize form → No API call
- Type in form fields → Real-time validation → Update character counters → No API call
- Click "Create Task" → Validate form → Show loading → POST /api/v1/tasks → Handle response
- Success response → Hide loading → Show success message → Reset form → Close modal → Emit taskCreated
- Error response → Hide loading → Show error message → Keep modal open → Focus error field
