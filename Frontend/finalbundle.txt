### Component Responsibilities:

**KanbanBoardComponent:**
- Purpose: Main container component managing the entire Kanban board
- Props: columns (KanbanColumn[]), loading (boolean), error (string)
- State: selectedTask, modalOpen, searchQuery
- Events: taskCreated, taskMoved, taskUpdated
- API Binding: Dashboard API for initial data load

**KanbanColumnComponent:**
- Purpose: Individual column container for tasks
- Props: column (KanbanColumn), tasks (Task[]), columnId (string)
- State: dragOver, taskCount
- Events: taskDropped, taskSelected, addTaskClicked
- API Binding: None (receives data from parent)

**TaskCardComponent:**
- Purpose: Individual task display card
- Props: task (Task), draggable (boolean), selectable (boolean)
- State: selected, dragState
- Events: taskClicked, taskDragStart, taskDragEnd
- API Binding: None (receives data from parent)

**TaskModalComponent:**
- Purpose: Modal for creating/editing tasks
- Props: isOpen (boolean), task (Task), mode ('create'|'edit')
- State: formData, validation, submitting
- Events: taskSaved, modalClosed
- API Binding: Fund Transfer API for task creation

### Data Flow:
**Parent → Child Mapping:**
- KanbanBoardComponent → KanbanColumnComponent (column data, tasks)
- KanbanBoardComponent → TaskModalComponent (modal state, task data)
- KanbanColumnComponent → TaskCardComponent (task data, interaction handlers)
- TaskCardComponent → AavaDefaultCardComponent (card content, styling)
- KanbanHeaderComponent → AavaSearchBarComponent (search props, events)

### Component Implementation Details:

#### KanbanBoardComponent

**Component Type:** Page
**Purpose:** Root component that orchestrates the entire Kanban board functionality
**Parent:** AppComponent
**Children:** KanbanHeaderComponent, KanbanSidebarComponent, KanbanContentComponent, TaskModalComponent

**TypeScript Specification:**

**Inputs:**
- youthAccountId: string
- initialColumns: KanbanColumn[]

**Outputs:**
- taskCreated: EventEmitter<Task>
- taskMoved: EventEmitter<{task: Task, fromColumn: string, toColumn: string}>
- taskUpdated: EventEmitter<Task>

**State:**
- columns: KanbanColumn[]
- selectedTask: Task | null
- modalOpen: boolean
- searchQuery: string
- loading: boolean
- error: string | null

**Methods:**
- ngOnInit() → Initialize component and load data
- loadDashboardData() → Fetch initial dashboard data
- onTaskCreated(task: Task) → Handle task creation
- onTaskMoved(event) → Handle task movement between columns
- openTaskModal(task?: Task) → Open task creation/edit modal
- closeTaskModal() → Close task modal

**HTML STRUCTURE:**
```html
<div class="kanban-app-container" role="application" aria-label="Kanban Board Application">
  <kanban-sidebar></kanban-sidebar>
  
  <div class="main-layout">
    <kanban-header 
      [searchQuery]="searchQuery"
      (searchChanged)="onSearchChanged($event)">
    </kanban-header>
    
    <kanban-content 
      [columns]="columns"
      [loading]="loading"
      [error]="error"
      (taskSelected)="openTaskModal($event)"
      (taskMoved)="onTaskMoved($event)">
    </kanban-content>
  </div>
  
  <task-modal 
    [isOpen]="modalOpen"
    [task]="selectedTask"
    (taskSaved)="onTaskCreated($event)"
    (modalClosed)="closeTaskModal()">
  </task-modal>
</div>
```

**CSS SPECIFICATION:**

**Layout:**
- display: grid
- grid-template-columns: 280px 1fr
- height: 100vh

**Spacing:**
- gap: 0
- padding: 0

**Component Styles:**
- .kanban-app-container: full viewport layout
- .main-layout: flex column layout

**Responsive:**
- Mobile (<768px): sidebar collapses, single column
- Tablet (768px-1024px): sidebar width 240px
- Desktop (>1024px): sidebar width 280px

**API INTEGRATION:**
- ngOnInit() → GET /api/youth-accounts/{youthAccountId}/dashboard → Load initial data
- onTaskCreated() → POST /api/youth-accounts/{youthAccountId}/fund-transfer → Create new task

**ERROR HANDLING:**
- API failure → show error message in content area
- Validation error → display inline validation messages
- Network error → show retry button with error snackbar

**Component Type:** Feature
**Purpose:** Manages individual Kanban column with drag-drop functionality

**TypeScript Specification:**

**Inputs:**

**Outputs:**
- taskDropped: EventEmitter<{task: Task, columnId: string}>

- addTaskClicked: EventEmitter<string>

**State:**
- dragOver: boolean
- taskCount: number
- expanded: boolean

**Methods:**
- onDrop(event: CdkDragDrop<Task[]>) → Handle task drop
- onDragEnter() → Handle drag enter
- onDragLeave() → Handle drag leave

**HTML STRUCTURE:**
```html
<div class="kanban-column" 
     [class.drag-over]="dragOver"
     cdkDropList

    <h3>{{ column.title }}</h3>

      variant="icon"
      (click)="onAddTask()"

    <task-card

      [task]="task"
      [draggable]="true"
      (taskClicked)="onTaskClick(task)"
      cdkDrag>
    </task-card>
  </div>
</div>
```

**Inputs:**

- onCancel() → Close modal without saving

```html
<aava-popup

          {{ validation.title }}
        </span>
      </div>
      
      <div class="form-field">
        <label for="task-description">Description</label>
        <textarea 
          id="task-description"
          formControlName="description"
          rows="3">
        </textarea>
      </div>
      
      <div class="form-field">
        <label for="task-priority">Priority</label>
        <select id="task-priority" formControlName="priority">
          <option value="low">Low</option>

        </select>
      </div>
      
      <div class="form-field">
        <label for="task-due-date">Due Date</label>
        <input 
          id="task-due-date"
          type="date"
          formControlName="dueDate">
      </div>
    </div>
    
    <div class="modal-actions">
      <aava-button
```
