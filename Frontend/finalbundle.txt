### Component Responsibilities:

**KanbanBoardComponent:**
- Purpose: Main container for three-column kanban layout
- Props: columns (array), loading (boolean), error (string)
- State: draggedTask, columnData, errorState
- Events: taskMoved, taskCreated, taskUpdated
- API Binding: GET /api/youth-accounts/{id}/dashboard

**KanbanColumnComponent:**
- Purpose: Individual column container with header and task list
- Props: columnId, title, tasks, count, status
- State: dragOver, expanded
- Events: taskDropped, addTaskClicked
- API Binding: None (data passed from parent)

**TaskCardComponent:**
- Purpose: Individual task display with drag-and-drop
- Props: task (object), draggable (boolean)
- State: dragging, selected
- Events: taskClicked, dragStart, dragEnd
- API Binding: None (data passed from parent)

**HeaderComponent:**
- Purpose: Top navigation with breadcrumb, search, and user menu
- Props: currentPage, user, notifications
- State: sidebarOpen, searchQuery
- Events: sidebarToggle, searchSubmit, themeToggle
- API Binding: None

**SidebarComponent:**
- Purpose: Left navigation menu with project links
- Props: currentPage, projects, isOpen
- State: activeSection, collapsed
- Events: navigationClick, sectionToggle
- API Binding: None

**TaskModalComponent:**
- Purpose: Modal for creating and editing tasks
- Props: isOpen, editMode, taskData
- State: formData, validation, submitting
- Events: taskSubmit, modalClose
- API Binding: POST /api/youth-accounts/{id}/fund-transfer (for task creation)

### Data Flow:
**Parent → Child Mapping:**
- App → Sidebar (currentPage, projects, isOpen)
- App → Header (user, notifications, currentPage)
- App → KanbanBoard (columns, loading, error)
- KanbanBoard → KanbanColumn (columnData, tasks, status)
- KanbanColumn → TaskCard (taskData, draggable)
- App → TaskModal (isOpen, mode, taskData)
