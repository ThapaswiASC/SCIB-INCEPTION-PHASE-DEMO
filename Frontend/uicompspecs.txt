## 4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

**Component Type:** Feature

**Mapped From A1:** KanbanBoardComponent (Main container for three-column Kanban layout)

**Purpose:** Main container component that manages the three-column Kanban board layout with drag-and-drop functionality, task management, and responsive design.

**Parent:** KanbanPageComponent

**Children:**
- KanbanColumnComponent (3 instances: Todo, InProgress, Done)
- MobileTabsComponent (mobile navigation)

**Dependencies:**
- Angular CDK Drag Drop
- KanbanService
- TaskService

**Library Components Used:**
- aava-button → Add task buttons
- aava-popup → Column action menus

**TypeScript Specification**

**Inputs:**
- columns: KanbanColumn[] (Array of column configurations)
- tasks: Task[] (Array of all tasks)
- loading: boolean (Loading state indicator)
- error: string | null (Error message display)

**Outputs:**
- taskMoved: EventEmitter<TaskMoveEvent> (Task moved between columns)
- taskCreated: EventEmitter<Task> (New task created)
- taskUpdated: EventEmitter<Task> (Task updated)
- columnUpdated: EventEmitter<KanbanColumn> (Column configuration changed)

**State:**
- draggedTask: Task | null (Currently dragged task)
- columnData: ColumnData[] (Column state with tasks)
- activeColumn: string (Active column for mobile)
- isLoading: boolean (Internal loading state)

**Methods:**
- initializeColumns() → void (Initialize column data structure)
- onTaskDrop(event: CdkDragDrop<Task[]>) → void (Handle task drop events)
- onAddTask(columnId: string) → void (Open task creation modal)
- onTaskEdit(task: Task) → void (Open task edit modal)
- switchMobileColumn(columnId: string) → void (Switch active column on mobile)

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-board" role="region" aria-label="Kanban Board">
  <div class="mobile-tabs" *ngIf="isMobile">
    <mobile-tabs 
      [columns]="columns" 
      [activeColumn]="activeColumn"
      (columnChanged)="switchMobileColumn($event)">
    </mobile-tabs>
  </div>
  
  <div class="kanban-grid" [class.mobile-single]="isMobile">
    <kanban-column 
      *ngFor="let column of columns; trackBy: trackByColumnId"
      [column]="column"
      [tasks]="getTasksForColumn(column.id)"
      [isActive]="!isMobile || activeColumn === column.id"
      [allowDrop]="true"
      (taskDropped)="onTaskDrop($event)"
      (addTaskClicked)="onAddTask($event)"
      (taskClicked)="onTaskEdit($event)">
    </kanban-column>
  </div>
  
  <div class="error-message" *ngIf="error" role="alert">
    {{ error }}
  </div>
  
  <div class="loading-overlay" *ngIf="loading">
    <div class="spinner"></div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1rem
- min-height: calc(100vh - 120px)

**Spacing:**
- padding: 1rem
- gap: 1rem between columns
- margin: 0 auto

**Component Styles:**
- .kanban-board: main container
- .kanban-grid: grid layout container
- .mobile-single: single column mobile view
- .error-message: error state styling
- .loading-overlay: loading state overlay

**Responsive:**
- Desktop (>1024px): Three columns visible
- Tablet (768px-1024px): Three columns with reduced spacing
- Mobile (<768px): Single column with tab navigation

**API INTEGRATION**

- initializeColumns() → GET /api/youth-accounts/{id}/dashboard → Fetch initial kanban data
- onTaskCreate() → POST /api/youth-accounts/{id}/fund-transfer → Create new task (mapped)
- onTaskUpdate() → PUT /api/youth-accounts/{id}/spending-limit → Update task limits (mapped)

**ERROR HANDLING**

- API failure → Display error message in error-message div
- Column configuration failure → Show fallback empty columns
- Drag drop failure → Revert task position with snackbar notification

**INTERACTION FLOW**

**User Action → UI → API**
- Load board → Initialize columns → GET /dashboard → Render columns with tasks
- Drag task → Update UI optimistically → PUT /spending-limit → Confirm or revert
- Add task → Open modal → Submit form → POST /fund-transfer → Add to column

### 4.2 KanbanColumnComponent

**Component Type:** Feature

**Mapped From A1:** KanbanColumnComponent (Individual column container with header and task list)

**Purpose:** Individual column component that displays column header, task count, and contains draggable task cards with drop zone functionality.

**Parent:** KanbanBoardComponent

**Children:**
- TaskCardComponent (multiple instances for tasks in column)

**Dependencies:**
- Angular CDK Drag Drop
- TaskCardComponent

**Library Components Used:**
- aava-button → Add task button, column actions
- aava-popup → Column menu dropdown

**TypeScript Specification**

**Inputs:**
- column: KanbanColumn (Column configuration and metadata)
- tasks: Task[] (Tasks assigned to this column)
- allowDrop: boolean (Whether column accepts task drops)
- isActive: boolean (Active state for mobile view)

**Outputs:**
- taskDropped: EventEmitter<TaskDropEvent> (Task dropped in this column)
- addTaskClicked: EventEmitter<string> (Add task button clicked)
- taskClicked: EventEmitter<Task> (Task card clicked)
- columnMenuClicked: EventEmitter<string> (Column menu action)

**State:**
- isDropTarget: boolean (Drop target highlight state)
- taskCount: number (Number of tasks in column)
- isMenuOpen: boolean (Column menu open state)

**Methods:**
- onDrop(event: CdkDragDrop<Task[]>) → void (Handle task drop)
- onDragEnter() → void (Highlight drop zone)
- onDragLeave() → void (Remove drop zone highlight)
- onAddTask() → void (Emit add task event)
- toggleMenu() → void (Toggle column menu)

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-column" 
     [class.active]="isActive"
     [class.drop-target]="isDropTarget"
     cdkDropList
     [cdkDropListData]="tasks"
     (cdkDropListDropped)="onDrop($event)">
  
  <div class="column-header" [attr.aria-labelledby]="column.id + '-title'">
    <h3 [id]="column.id + '-title'" class="column-title">
      {{ column.title }}
    </h3>
    <span class="task-count">{{ taskCount }}</span>
    
    <aava-popup [isOpen]="isMenuOpen" position="bottom-right">
      <aava-button 
        variant="ghost" 
        size="sm" 
        (click)="toggleMenu()"
        aria-label="Column menu">
        <icon name="more-vertical"></icon>
      </aava-button>
      
      <div slot="content" class="column-menu">
        <button (click)="onAddTask()">Add Task</button>
        <button (click)="onClearColumn()">Clear Column</button>
      </div>
    </aava-popup>
  </div>
  
  <div class="column-content">
    <task-card 
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="true"
      cdkDrag
      (taskClicked)="onTaskClicked($event)">
    </task-card>
    
    <div class="add-task-zone" *ngIf="tasks.length === 0">
      <aava-button 
        variant="dashed" 
        size="lg" 
        (click)="onAddTask()"
        class="add-first-task">
        <icon name="plus"></icon>
        Add your first task
      </aava-button>
    </div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: flex
- flex-direction: column
- min-height: 500px
- background: var(--column-bg)

**Spacing:**
- padding: 1rem
- gap: 0.75rem between tasks
- border-radius: 8px

**Component Styles:**
- .kanban-column: main column container
- .column-header: header with title and actions
- .column-content: scrollable task area
- .drop-target: drop zone highlight
- .add-task-zone: empty state with add button

**Responsive:**
- Desktop: Fixed width 320px
- Tablet: Flexible width with min 280px
- Mobile: Full width when active

**API INTEGRATION**

- No direct API calls (data flows from parent KanbanBoardComponent)

**ERROR HANDLING**

- Task drop failure → Show error toast and revert position
- Empty column state → Display add task prompt
- Loading state → Show skeleton placeholders for tasks

**INTERACTION FLOW**

**User Action → UI → API**
- Drop task → Update column → Emit taskDropped → Parent handles API
- Click add task → Emit addTaskClicked → Parent opens modal
- Click task → Emit taskClicked → Parent opens edit modal

### 4.3 TaskCardComponent

**Component Type:** Feature

**Mapped From A1:** TaskCardComponent (Individual draggable task item with metadata)

**Purpose:** Draggable task card component displaying task information, priority, assignee, and providing interaction handlers for editing and status changes.

**Parent:** KanbanColumnComponent

**Children:** None

**Dependencies:**
- Angular CDK Drag Drop
- UserAvatarComponent

**Library Components Used:**
- aava-button → Task action buttons
- aava-popup → Task quick actions menu

**TypeScript Specification**

**Inputs:**
- task: Task (Task data object)
- draggable: boolean (Whether task can be dragged)
- selected: boolean (Selection state for multi-select)

**Outputs:**
- taskClicked: EventEmitter<Task> (Task clicked for editing)
- taskDragStart: EventEmitter<Task> (Drag operation started)
- taskDragEnd: EventEmitter<Task> (Drag operation ended)
- quickActionClicked: EventEmitter<{action: string, task: Task}> (Quick action selected)

**State:**
- isDragging: boolean (Currently being dragged)
- isHovered: boolean (Mouse hover state)
- showQuickActions: boolean (Quick actions menu visibility)

**Methods:**
- onTaskClick() → void (Handle task click)
- onDragStart() → void (Handle drag start)
- onDragEnd() → void (Handle drag end)
- onQuickAction(action: string) → void (Handle quick action)

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="task-card" 
     [class.dragging]="isDragging"
     [class.selected]="selected"
     [class.hovered]="isHovered"
     (click)="onTaskClick()"
     (mouseenter)="isHovered = true"
     (mouseleave)="isHovered = false">
  
  <div class="task-header">
    <span class="task-id">#{{ task.id }}</span>
    <div class="task-priority" [class]="'priority-' + task.priority">
      <icon [name]="getPriorityIcon(task.priority)"></icon>
    </div>
  </div>
  
  <div class="task-content">
    <h4 class="task-title">{{ task.title }}</h4>
    <p class="task-description" *ngIf="task.description">
      {{ task.description | truncate:100 }}
    </p>
  </div>
  
  <div class="task-meta">
    <div class="task-tags" *ngIf="task.tags?.length">
      <span class="tag" *ngFor="let tag of task.tags">{{ tag }}</span>
    </div>
    
    <div class="task-footer">
      <user-avatar 
        *ngIf="task.assignee"
        [user]="task.assignee"
        size="sm">
      </user-avatar>
      
      <span class="task-date" *ngIf="task.dueDate">
        {{ task.dueDate | date:'MMM dd' }}
      </span>
      
      <aava-popup [isOpen]="showQuickActions" position="bottom-right">
        <aava-button 
          variant="ghost" 
          size="xs" 
          (click)="showQuickActions = !showQuickActions"
          class="quick-actions-trigger">
          <icon name="more-horizontal"></icon>
        </aava-button>
        
        <div slot="content" class="quick-actions">
          <button (click)="onQuickAction('edit')">Edit</button>
          <button (click)="onQuickAction('duplicate')">Duplicate</button>
          <button (click)="onQuickAction('delete')">Delete</button>
        </div>
      </aava-popup>
    </div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**

- flex-direction: column

**Spacing:**
- padding: 12px

**Component Styles:**

- .task-header: ID and priority section
- .task-content: title and description

- .priority-high/medium/low: priority indicators

**Responsive:**

**ERROR HANDLING**

**INTERACTION FLOW**

**User Action → UI → API**

### 4.4 HeaderComponent

**Component Type:** Layout

**Mapped From A1:** HeaderComponent (Top navigation bar with search and user menu)

**Purpose:** Application header providing navigation breadcrumbs, global search, notifications, theme toggle, and user menu functionality.

**Parent:** AppComponent

**Children:**
- SearchInputComponent
- UserAvatarComponent
- BreadcrumbComponent

**Dependencies:**
- AuthService
- ThemeService
- NotificationService

**Library Components Used:**
- aava-button → Menu toggle, theme toggle, notification button
- aava-popup → User menu dropdown, notifications panel

**TypeScript Specification**

**Inputs:**
- user: User (Current authenticated user)
- notifications: Notification[] (User notifications)
- searchEnabled: boolean (Search functionality enabled)
- breadcrumbs: BreadcrumbItem[] (Navigation breadcrumbs)

**Outputs:**
- searchChanged: EventEmitter<string> (Search query changed)
- themeToggled: EventEmitter<void> (Theme toggle clicked)
- userMenuToggled: EventEmitter<void> (User menu toggled)
- menuToggled: EventEmitter<void> (Mobile menu toggle)
- notificationClicked: EventEmitter<Notification> (Notification clicked)

**State:**
- searchQuery: string (Current search term)
- userMenuOpen: boolean (User menu visibility)
- notificationsOpen: boolean (Notifications panel visibility)
- unreadCount: number (Unread notifications count)

**Methods:**
- onSearchChange(query: string) → void (Handle search input)
- toggleUserMenu() → void (Toggle user menu)
- toggleNotifications() → void (Toggle notifications)
- onThemeToggle() → void (Handle theme change)
- onLogout() → void (Handle user logout)

### 4.5 ModalComponent

**Component Type:** Shared

**Mapped From A1:** ModalComponent (Reusable modal container with overlay)

**Purpose:** Reusable modal dialog component providing overlay, focus management, keyboard navigation, and content projection for various modal use cases.

**Parent:** Various (AppComponent, KanbanBoardComponent, etc.)

**Children:** Projected content (TaskFormComponent, ConfirmationComponent, etc.)

**Dependencies:**
- Angular CDK Overlay
- FocusTrapFactory

**Library Components Used:**
- aava-button → Close button, action buttons

**TypeScript Specification**

**Inputs:**
- isOpen: boolean (Modal visibility state)
- title: string (Modal title text)
- size: 'sm' | 'md' | 'lg' | 'xl' (Modal size variant)
- closable: boolean (Whether modal can be closed)
- closeOnBackdrop: boolean (Close on backdrop click)
- closeOnEscape: boolean (Close on escape key)

**Outputs:**
- closed: EventEmitter<void> (Modal closed event)
- opened: EventEmitter<void> (Modal opened event)
- backdropClicked: EventEmitter<void> (Backdrop clicked event)

**State:**
- isAnimating: boolean (Animation in progress)
- focusTrap: FocusTrap (Focus management)

**Methods:**
- open() → void (Open modal with animation)
- close() → void (Close modal with animation)
- onBackdropClick() → void (Handle backdrop click)
- onEscapeKey() → void (Handle escape key)
- trapFocus() → void (Initialize focus trap)

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="modal-overlay" 
     *ngIf="isOpen"
     [class.animating]="isAnimating"
     (click)="onBackdropClick()"
     role="dialog"
     aria-modal="true"
     [attr.aria-labelledby]="title ? 'modal-title' : null">
  
  <div class="modal-container" 
       [class]="'modal-' + size"
       (click)="$event.stopPropagation()"
       cdkTrapFocus>
    
    <div class="modal-header" *ngIf="title || closable">
      <h2 id="modal-title" class="modal-title" *ngIf="title">
        {{ title }}
      </h2>
      
      <aava-button 
        *ngIf="closable"
        variant="ghost" 
        size="sm"

        aria-label="Close modal">

    </div>
    
    <div class="modal-body">
      <ng-content></ng-content>
    </div>
    
    <div class="modal-footer" *ngIf="hasFooterContent">
      <ng-content select="[slot=footer]"></ng-content>
    </div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- position: fixed
- top: 0, left: 0, right: 0, bottom: 0
- display: flex
- justify-content: center
- align-items: center
- z-index: 1000

**Spacing:**
- .modal-sm: max-width 400px
- .modal-md: max-width 600px
- .modal-lg: max-width 800px
- .modal-xl: max-width 1200px

**Component Styles:**
- .modal-overlay: backdrop with blur
- .modal-container: white card with shadow
- .modal-header: title and close button
- .modal-body: scrollable content area
- .modal-footer: action buttons area

**Responsive:**
- Desktop: Fixed max-width based on size
- Mobile: Full width with margin, reduced height

**API INTEGRATION**

- No direct API calls (content agnostic)

**ERROR HANDLING**

**User Action → UI → API**
