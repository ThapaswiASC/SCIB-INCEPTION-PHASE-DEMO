### 4. COMPONENT IMPLEMENTATION

#### 4.1 KanbanBoardComponent

**Component Type:** Feature

**Mapped From A1:** Main container for three-column kanban layout

**Purpose:** Orchestrate the entire kanban board functionality with drag-and-drop, data management, and API integration

**Parent:** KanbanPageComponent

**Children:**
- KanbanColumnComponent (x3)
- TaskModalComponent
- MobileTabsComponent

**Dependencies:**
- @angular/cdk/drag-drop
- @aava/play-core (Button, Modal)
- KanbanService
- ApiService

**Library Components Used:**
- Button Component → Add Task actions
- Modal Component → Task creation dialog

**TypeScript Specification**

**Inputs:**
- loading: boolean (Loading state)
- error: string | null (Error message)

**Outputs:**
- taskCreated: EventEmitter<Task> (New task created)
- taskUpdated: EventEmitter<Task> (Task updated)
- taskMoved: EventEmitter<{task: Task, fromColumn: string, toColumn: string}> (Task moved between columns)

**State:**
- columns: Column[] (Column data with tasks)
- draggedTask: Task | null (Currently dragged task)
- selectedTask: Task | null (Selected task for editing)
- isModalOpen: boolean (Modal visibility)
- errorState: string | null (Error handling)

**Methods:**
- ngOnInit() → Initialize component and load data
- loadKanbanData() → Fetch initial kanban data
- onTaskDrop(event: CdkDragDrop) → Handle task drag-and-drop
- openTaskModal(task?: Task) → Open task creation/editing modal
- closeTaskModal() → Close task modal
- handleTaskSubmit(taskData: Task) → Process task creation/update
- handleError(error: any) → Error handling

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="page-header">
    <h1>Kanban Board</h1>
    <div class="header-actions">
      <Button variant="primary" (click)="openTaskModal()">Add Task</Button>
    </div>
  </div>
  
  <MobileTabsComponent *ngIf="isMobile" [columns]="columns"></MobileTabsComponent>
  
  <div class="kanban-grid" cdkDropListGroup>
    <KanbanColumnComponent 
      *ngFor="let column of columns" 
      [column]="column"
      [tasks]="column.tasks"
      (taskDropped)="onTaskDrop($event)"
      (addTaskClicked)="openTaskModal()">
    </KanbanColumnComponent>
  </div>
  
  <div *ngIf="errorState" class="error-message" role="alert">
    {{ errorState }}
  </div>
  
  <TaskModalComponent
    [isOpen]="isModalOpen"
    [task]="selectedTask"
    (taskSubmit)="handleTaskSubmit($event)"
    (modalClose)="closeTaskModal()">
  </TaskModalComponent>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1rem
- min-height: 100vh

**Spacing:**
- padding: 1rem
- gap: 1rem between columns

**Component Styles:**
- .kanban-board-container
- .page-header
- .kanban-grid
- .error-message

**Responsive:**
- Desktop (>1024px): 3-column grid
- Tablet (768px-1024px): 2-column grid with tabs
- Mobile (<768px): Single column with tab navigation

**API INTEGRATION**

- loadKanbanData() → GET /api/youth-accounts/{id}/dashboard → Load initial kanban data
- handleTaskSubmit() → POST /api/youth-accounts/{id}/fund-transfer → Create new task
- onTaskDrop() → PUT /api/youth-accounts/{id}/spending-limit → Update task status

**ERROR HANDLING**

- API failure → show error message in errorState
- Validation error → display inline validation messages
- Network error → show retry mechanism

**INTERACTION FLOW**

**User Action → UI → API**
- Load page → initialize component → GET /dashboard → display columns
- Drag task → update UI optimistically → PUT /spending-limit → confirm changes
- Click Add Task → open modal → submit form → POST /fund-transfer → refresh board

#### 4.2 KanbanColumnComponent

**Component Type:** Feature

**Mapped From A1:** Individual column container with header and task list

**Purpose:** Display column header, task list, and handle column-specific interactions

**Parent:** KanbanBoardComponent

**Children:**
- TaskCardComponent (multiple)
- Button Component (Add Task)

**Dependencies:**
- @angular/cdk/drag-drop
- @aava/play-core (Button)

**TypeScript Specification**

**Inputs:**
- column: Column (Column configuration)

- dragDisabled: boolean (Disable drag-and-drop)

**Outputs:**
- taskDropped: EventEmitter<CdkDragDrop<Task[]>> (Task dropped in column)
- addTaskClicked: EventEmitter<string> (Add task button clicked)
- taskClicked: EventEmitter<Task> (Task card clicked)

**State:**
- dragOver: boolean (Drag over state)
- expanded: boolean (Column expanded state)
- taskCount: number (Number of tasks)

**Methods:**
- onTaskDrop(event: CdkDragDrop) → Handle task drop
- onAddTaskClick() → Emit add task event
- onTaskClick(task: Task) → Handle task selection
- getColumnStatus() → Get column status indicator

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-column" [attr.aria-labelledby]="column.id + '-header'">
  <div class="column-header" [id]="column.id + '-header'">
    <div class="column-title">
      <h3>{{ column.title }}</h3>
      <StatusIndicatorComponent [status]="column.status"></StatusIndicatorComponent>
    </div>
    <div class="column-meta">
      <span class="task-count">{{ tasks.length }}</span>
      <Button variant="ghost" size="small" (click)="onAddTaskClick()">+</Button>
    </div>
  </div>
  
  <div class="column-content" 
       cdkDropList 
       [cdkDropListData]="tasks"
       (cdkDropListDropped)="onTaskDrop($event)">
    <TaskCardComponent
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="!dragDisabled"
      (taskClicked)="onTaskClick(task)"
      cdkDrag>
    </TaskCardComponent>
    
    <div *ngIf="tasks.length === 0" class="empty-column">
      <p>No tasks in {{ column.title }}</p>
      <Button variant="secondary" (click)="onAddTaskClick()">Add First Task</Button>
    </div>
  </div>
</div>
```

**Component Styles:**
- .kanban-column
- .column-header
- .column-content
- .empty-column

**Responsive:**
- Mobile: Full width with margin
- Tablet: Flex basis 50%
- Desktop: Flex basis 33.33%

- onTaskDrop() → Emit to parent → PUT /api/youth-accounts/{id}/spending-limit → Update task status
