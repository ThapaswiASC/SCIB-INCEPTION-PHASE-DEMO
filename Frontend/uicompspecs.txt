### 4. COMPONENT IMPLEMENTATION

#### 4.1 TaskCreationComponent

Component Type: Feature

Mapped From A1: TaskCreationComponent (NEW - Primary Focus)

Purpose: Create new tasks with comprehensive validation and API integration using Angular reactive forms

Parent: App Container / KanbanBoardComponent

Children:
- ButtonComponent (Submit, Cancel)
- ValidationMessageComponent
- CharacterCounterComponent
- LoadingSpinnerComponent

Dependencies:
- @angular/forms (FormBuilder, Validators)
- TaskService
- Custom Validators

Library Components Used:
- ButtonComponent → Primary/Secondary variants with loading states

TypeScript Specification:

Inputs:
- isVisible: boolean (modal visibility state)
- initialData: Partial<Task> (for editing existing tasks)
- parentStatus: TaskStatus (default status for new tasks)

Outputs:
- taskCreated: EventEmitter<Task> (emitted when task is successfully created)
- formClosed: EventEmitter<void> (emitted when modal is closed)
- validationChanged: EventEmitter<boolean> (emitted when form validation state changes)

State:
- taskForm: FormGroup (reactive form instance)
- isSubmitting: boolean (API call loading state)
- validationErrors: ValidationError[] (form validation errors)
- characterCounts: { title: number, description: number } (real-time character counts)

Methods:
- initializeForm() → void (setup form with validators)
- onSubmit() → void (handle form submission)
- updateCharacterCount(field: string, value: string) → void (update character counters)
- resetForm() → void (reset form after successful submission)
- closeModal() → void (close modal and emit event)

HTML STRUCTURE (PSEUDO CODE ONLY):

<div class="task-creation-modal" [class.visible]="isVisible">
  <div class="modal-overlay" (click)="closeModal()">
    <div class="modal-content" (click)="$event.stopPropagation()">
      <div class="modal-header">
        <h2>Create New Task</h2>
        <button class="close-btn" (click)="closeModal()">×</button>
      </div>
      
      <form [formGroup]="taskForm" (ngSubmit)="onSubmit()">
        <div class="form-group">
          <label for="title">Task Title *</label>
          <input type="text" id="title" formControlName="title" maxlength="255">
          <character-counter [current]="characterCounts.title" [max]="255"></character-counter>
          <validation-message [errors]="taskForm.get('title')?.errors"></validation-message>
        </div>
        
        <div class="form-group">
          <label for="description">Description</label>
          <textarea id="description" formControlName="description" maxlength="2000"></textarea>
          <character-counter [current]="characterCounts.description" [max]="2000"></character-counter>
          <validation-message [errors]="taskForm.get('description')?.errors"></validation-message>
        </div>
        
        <div class="form-group">
          <label for="priority">Priority *</label>
          <select id="priority" formControlName="priority">
            <option value="LOW">Low</option>
            <option value="MEDIUM">Medium</option>
            <option value="HIGH">High</option>
            <option value="CRITICAL">Critical</option>
          </select>
          <validation-message [errors]="taskForm.get('priority')?.errors"></validation-message>
        </div>
        
        <div class="form-group">
          <label for="dueDate">Due Date *</label>
          <input type="date" id="dueDate" formControlName="dueDate">
          <validation-message [errors]="taskForm.get('dueDate')?.errors"></validation-message>
        </div>
        
        <div class="form-actions">
          <button-component 
            [label]="'Cancel'" 
            [variant]="'secondary'" 
            (userClick)="closeModal()">
          </button-component>
          <button-component 
            [label]="'Create Task'" 
            [variant]="'primary'" 
            [processing]="isSubmitting" 
            [disabled]="taskForm.invalid || isSubmitting"
            type="submit">
          </button-component>
        </div>
      </form>
      
      <loading-spinner *ngIf="isSubmitting"></loading-spinner>
    </div>
  </div>
</div>

CSS SPECIFICATION:

Layout:
- position: fixed with full viewport overlay
- flexbox centering for modal content
- z-index: 1000 for modal layering

Spacing:
- modal-content: padding 24px, max-width 600px
- form-group: margin-bottom 16px
- form-actions: gap 12px, justify-content flex-end

Component Styles:
- .task-creation-modal: opacity transition for smooth show/hide
- .modal-overlay: background rgba(0,0,0,0.5)
- .form-group: display flex, flex-direction column
- input, textarea, select: border-radius 4px, padding 8px 12px

Responsive:
- @media (max-width: 768px): modal-content width 90vw, padding 16px

API INTEGRATION:
- onSubmit() → POST /tasks → TaskService.createTask(taskData)
- Success: emit taskCreated event, reset form, close modal
- Error: display validation errors, keep form open

ERROR HANDLING:
- API failure → show error snackbar notification
- Validation error → inline validation messages per field
- Network error → retry mechanism with user feedback

INTERACTION FLOW:

User Action → UI → API:
- Click "Add Task" button → open modal → initialize form
- Type in title field → real-time validation → update character counter
- Fill required fields → enable submit button → validate form
- Click "Create Task" → show loading spinner → call TaskService.createTask → POST /tasks
- API success → close modal → refresh kanban board → show success message

#### 4.2 KanbanBoardComponent

Component Type: Feature

Mapped From A1: KanbanBoardComponent

Purpose: Display and manage kanban board with three columns (To Do, In Progress, Done)

Parent: App Container

Children:
- KanbanColumnComponent (3 instances)
- TaskCreationComponent

Dependencies:
- TaskService
- Task Model

Library Components Used:
- ButtonComponent → Add Task buttons

TypeScript Specification:

Inputs:
- tasks: Task[] (list of all tasks)
- columns: KanbanColumn[] (column configurations)
- filters: TaskFilter (current filter settings)

Outputs:
- taskMoved: EventEmitter<TaskMoveEvent> (when task is moved between columns)
- taskCreated: EventEmitter<Task> (when new task is created)
- taskUpdated: EventEmitter<Task> (when task is updated)

State:
- isLoading: boolean (data loading state)
- selectedTask: Task | null (currently selected task)
- showCreateModal: boolean (task creation modal visibility)

Methods:
- loadTasks() → void (fetch tasks from API)
- onTaskMoved(event: TaskMoveEvent) → void (handle task movement)
- openCreateModal(status?: TaskStatus) → void (open task creation modal)
- onTaskCreated(task: Task) → void (handle new task creation)

HTML STRUCTURE (PSEUDO CODE ONLY):

<div class="kanban-board">
  <div class="board-header">
    <h1>Task Management</h1>
    <button-component 
      [label]="'Add Task'" 
      [variant]="'primary'" 
      (userClick)="openCreateModal()">
    </button-component>
  </div>
  
  <div class="kanban-columns">
    <kanban-column 
      *ngFor="let column of columns" 
      [title]="column.title"
      [status]="column.status"
      [tasks]="getTasksByStatus(column.status)"
      [taskCount]="getTaskCount(column.status)"
      (taskMoved)="onTaskMoved($event)"
      (addTaskClicked)="openCreateModal($event)">
    </kanban-column>
  </div>
  
  <task-creation-component
    [isVisible]="showCreateModal"
    [initialData]="{}"
    (taskCreated)="onTaskCreated($event)"
    (formClosed)="showCreateModal = false">
  </task-creation-component>
</div>

CSS SPECIFICATION:

Layout:
- display: flex, flex-direction: column
- kanban-columns: display grid, grid-template-columns repeat(3, 1fr), gap 16px

Spacing:
- board-header: padding 16px 0, justify-content space-between
- kanban-columns: padding 0 16px

Component Styles:
- .kanban-board: min-height 100vh, background #f5f5f5
- .board-header: display flex, align-items center

Responsive:
- @media (max-width: 1024px): grid-template-columns repeat(2, 1fr)
- @media (max-width: 768px): grid-template-columns 1fr

API INTEGRATION:
- loadTasks() → GET /tasks → TaskService.getTasks()
- onTaskMoved() → PUT /tasks/{id} → TaskService.updateTask()

ERROR HANDLING:
- API failure → show error notification
- Task move failure → revert UI state, show error message

INTERACTION FLOW:

User Action → UI → API:
- Page load → show loading spinner → call loadTasks() → GET /tasks
- Drag task between columns → update UI optimistically → call updateTask() → PUT /tasks/{id}
- Click "Add Task" → open TaskCreationComponent modal

#### 4.3 KanbanColumnComponent

Children:

Inputs:

#### 4.4 TaskCardComponent

Component Type: Feature

Mapped From A1: TaskCardComponent

Purpose: Display individual task with interactive elements and drag functionality

Parent: KanbanColumnComponent

- PriorityIndicatorComponent

- TaskTagComponent

Dependencies:

Library Components Used:

<div class="task-card" 
     [class.hovered]="isHovered"
     [class.focused]="isFocused"
     [draggable]="isDraggable"

    </div>
  </div>
  
  <div class="task-content">
    <h4 class="task-title">{{ task.title }}</h4>
    <p class="task-description" *ngIf="task.description">{{ task.description }}</p>
  </div>
  
  <div class="task-meta">
    <div class="task-tags">
      <task-tag *ngFor="let tag of task.tags" [tag]="tag"></task-tag>
    </div>
    <div class="task-footer">
      <assignee-avatar [assignee]="task.assignee"></assignee-avatar>
      <span class="due-date" [class.overdue]="isOverdue(task.dueDate)">
        {{ task.dueDate | date:'MMM dd' }}
      </span>
    </div>
  </div>
</div>

CSS SPECIFICATION:

Component Styles:
- .task-card: cursor pointer, transition all 0.2s
- .hovered: box-shadow 0 4px 8px rgba(0,0,0,0.15), transform translateY(-2px)
- .overdue: color #dc3545, font-weight bold

Responsive:
- @media (max-width: 768px): padding 8px

INTERACTION FLOW:

User Action → UI → API:
- Click task → emit taskClicked event → open task details

### 8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly (Kanban board with three columns)
- Do NOT create new components unnecessarily (reuse library components where possible)
- Prefer reusable library components (ButtonComponent from uicomplibraryplay)
- HTML must be pseudo-code ONLY (NOT real framework code)
- All form validation must be real-time with immediate feedback
- Character counters must update on every keystroke
- API calls must include proper error handling and loading states
- Form must reset after successful task creation
- Drag-and-drop functionality must include visual feedback
- All components must be accessible with proper ARIA labels
- Responsive design required for mobile and tablet viewports
- Follow Angular reactive forms best practices
- Implement proper TypeScript typing for all props and state
- Use consistent naming conventions across all components
- Ensure proper component lifecycle management
- Implement proper error boundaries for API failures
- Follow OpenAPI specification guidelines for API integration
- Maintain component reusability through proper abstraction
- Ensure zero-touch onboarding through self-explanatory component APIs
