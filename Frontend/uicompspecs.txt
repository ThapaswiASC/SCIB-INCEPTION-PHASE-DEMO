### 4.1 HeaderComponent

Component Type: Layout

Mapped From A1: HeaderComponent with search and user actions

Purpose: Application header with search functionality and user menu

Parent: AppComponent

Children:
- InputComponent (search)
- ButtonComponent (notifications, settings)
- AvatarComponent (user display)
- IconComponent (various icons)

Dependencies:
- @angular/core
- @angular/common
- SharedModule (for reusable components)

Library Components Used:
- InputComponent → Search input field
- ButtonComponent → Action buttons
- AvatarComponent → User avatar display
- IconComponent → Navigation and action icons

TypeScript Specification

Inputs:
- user: User (user profile data)
- searchPlaceholder: string (search input placeholder)
- showNotifications: boolean (toggle notification display)

Outputs:
- onSearch: EventEmitter<string> (search query payload)
- onNotificationClick: EventEmitter<void> (notification action)
- onUserMenuClick: EventEmitter<void> (user menu action)

State:
- searchQuery: string (current search input)
- notificationCount: number (unread notifications)
- isUserMenuOpen: boolean (menu visibility state)

Methods:
- handleSearch(query: string) → void (process search input)
- toggleUserMenu() → void (toggle user menu visibility)
- handleNotificationClick() → void (handle notification interactions)

HTML STRUCTURE (PSEUDO CODE ONLY)

<header class="app-header">
  <div class="search-container">
    <app-input 
      [placeholder]="searchPlaceholder"
      [value]="searchQuery"
      (valueChange)="handleSearch($event)">
    </app-input>
    <app-icon name="search"></app-icon>
  </div>
  
  <div class="header-actions">
    <app-button 
      variant="icon"
      (click)="handleNotificationClick()">
      <app-icon name="notifications"></app-icon>
      <app-badge [count]="notificationCount" *ngIf="notificationCount > 0"></app-badge>
    </app-button>
    
    <app-button variant="icon">
      <app-icon name="settings"></app-icon>
    </app-button>
    
    <div class="user-menu" (click)="toggleUserMenu()">
      <app-avatar [user]="user" size="sm"></app-avatar>
      <span class="user-name">{{user.name}}</span>
    </div>
  </div>
</header>

CSS SPECIFICATION

Layout:
- display: flex
- justify-content: space-between
- align-items: center
- padding: 1rem 2rem

Spacing:
- gap: 1rem between header sections
- padding: 0.5rem for action buttons

Component Styles:
- .app-header: main header container
- .search-container: flex container for search
- .header-actions: flex container for action buttons
- .user-menu: flex container for user info

Responsive:
- Mobile (<768px): hide user name, reduce padding
- Tablet (768px-1024px): compact spacing
- Desktop (>1024px): full layout

API INTEGRATION

- getUserProfile() → GET /users/profile → fetch user data
- getNotificationCount() → GET /notifications → fetch notification count

ERROR HANDLING

- API failure → show default user avatar and name
- Network error → disable search functionality temporarily
- Validation error → show inline search validation

INTERACTION FLOW

User Action → UI → API
- Type in search → update searchQuery → emit onSearch event
- Click notifications → open notification panel → fetch notifications
- Click user menu → toggle dropdown → show user options

### 4.3 KanbanBoardComponent

Component Type: Feature

Mapped From A1: KanbanBoardComponent with three-column layout

Purpose: Main kanban board container with drag-and-drop functionality

Parent: KanbanPageComponent

Children:
- KanbanColumnComponent (three instances)
- TaskModalComponent (task editing)

Dependencies:
- @angular/core
- @angular/cdk/drag-drop
- KanbanModule

Library Components Used:
- KanbanColumnComponent → Column containers
- TaskModalComponent → Task editing modal

TypeScript Specification

Inputs:
- columns: KanbanColumn[] (column configuration)
- tasks: Task[] (all tasks data)
- loading: boolean (loading state)

Outputs:
- onTaskMove: EventEmitter<{taskId: string, targetColumn: string}> (task movement payload)
- onTaskCreate: EventEmitter<string> (column ID for new task)
- onTaskEdit: EventEmitter<string> (task ID for editing)

State:
- draggedTask: Task | null (currently dragged task)
- columnData: KanbanColumn[] (processed column data)
- error: string | null (error state)
- isModalOpen: boolean (task modal visibility)
- selectedTask: Task | null (task being edited)

Methods:
- ngOnInit() → void (initialize component)
- loadBoardData() → void (fetch kanban data)
- handleTaskDrop(event: CdkDragDrop) → void (process drag-drop)
- openTaskModal(task?: Task) → void (open task editing modal)
- closeTaskModal() → void (close modal)

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board" 
     role="region" 
     aria-label="Kanban Board"
     [class.loading]="loading">
  
  <div class="board-header">
    <h2>Kanban Board</h2>
    <app-button variant="primary" (click)="openTaskModal()">
      Add Task
    </app-button>
  </div>
  
  <div class="kanban-container" 
       cdkDropListGroup>
    <app-kanban-column 
      *ngFor="let column of columnData"
      [column]="column"
      [tasks]="getTasksForColumn(column.id)"
      (onTaskDrop)="handleTaskDrop($event)"
      (onAddTask)="openTaskModal()"
      (onTaskEdit)="openTaskModal($event)">
    </app-kanban-column>
  </div>
  
  <div class="error-message" *ngIf="error">
    <app-icon name="error"></app-icon>
    <span>{{error}}</span>
  </div>
  
  <app-task-modal 
    [task]="selectedTask"
    [isOpen]="isModalOpen"
    [mode]="selectedTask ? 'edit' : 'create'"
    (onSave)="handleTaskSave($event)"
    (onCancel)="closeTaskModal()"
    (onClose)="closeTaskModal()">
  </app-task-modal>
</div>

CSS SPECIFICATION

Layout:
- display: flex
- flex-direction: column
- height: 100%
- .kanban-container: display: grid, grid-template-columns: repeat(3, 1fr), gap: 1rem

Spacing:
- padding: 1rem
- gap: 1rem between columns

Component Styles:
- .kanban-board: main container
- .board-header: flex header with title and actions
- .kanban-container: grid layout for columns
- .error-message: error display styling

Responsive:
- Mobile (<768px): single column stack, horizontal scroll
- Tablet (768px-1024px): two columns visible, scroll for third
- Desktop (>1024px): all three columns visible

API INTEGRATION

- loadBoardData() → GET /kanban/board → fetch board configuration and tasks
- moveTask(taskId, columnId) → PUT /tasks/{id}/move → update task column

ERROR HANDLING

- API failure → show error message with retry option
- Network error → show offline indicator
- Validation error → show inline validation messages

INTERACTION FLOW

User Action → UI → API
- Load board → show loading state → GET /kanban/board → update columnData
- Drag task → update UI immediately → PUT /tasks/{id}/move → confirm or revert
- Add task → open modal → fill form → POST /tasks → refresh board

### 4.4 KanbanColumnComponent

Component Type: Feature

Mapped From A1: KanbanColumnComponent for individual columns

Purpose: Individual kanban column with task list and drop zone

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)
- AddTaskButtonComponent (add task action)

Dependencies:
- @angular/core
- @angular/cdk/drag-drop

- ButtonComponent → Add task button

Inputs:
- column: KanbanColumn (column configuration)

- allowDrop: boolean (enable drop functionality)

Outputs:
- onTaskDrop: EventEmitter<CdkDragDrop<Task[]>> (task drop payload)
- onAddTask: EventEmitter<void> (add task action)

State:
- isDragOver: boolean (drag over state)
- taskCount: number (number of tasks in column)

Methods:
- drop(event: CdkDragDrop<Task[]>) → void (handle task drop)
- addTask() → void (trigger add task)
- editTask(task: Task) → void (trigger task edit)

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-column">
  <div class="column-header" [attr.aria-labelledby]="column.id + '-title'">
    <h3 [id]="column.id + '-title'" class="column-title">{{column.title}}</h3>
    <app-badge [count]="taskCount" variant="secondary"></app-badge>

  <div class="column-content"
       cdkDropList
       [cdkDropListData]="tasks"
       [id]="column.id"
       (cdkDropListDropped)="drop($event)"
       [class.drag-over]="isDragOver">
    
    <app-task-card 
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="true"
      cdkDrag
      (onEdit)="editTask(task)">
    </app-task-card>
    
    <div class="empty-state" *ngIf="tasks.length === 0">
      <app-icon name="empty-tasks"></app-icon>
      <span>No tasks in {{column.title}}</span>

  </div>
  
  <div class="column-footer">
    <app-button 
      variant="secondary" 
      size="sm"
      (click)="addTask()">
      <app-icon name="add"></app-icon>
      Add Task
    </app-button>
  </div>
</div>

- flex-direction: column

- background: #f8f9fa

Spacing:
- padding: 1rem
- gap: 0.5rem between tasks

Component Styles:
- .kanban-column: main column container
- .column-header: flex header with title and count
- .column-content: scrollable task area
- .empty-state: centered empty state
- .drag-over: drag over visual feedback

Responsive:
- Mobile (<768px): full width, stack vertically
- Tablet (768px-1024px): flexible width
