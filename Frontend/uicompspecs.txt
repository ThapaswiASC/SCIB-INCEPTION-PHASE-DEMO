### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main kanban board container with three-column layout

Purpose: Primary container for kanban board functionality with drag-and-drop support

Parent: KanbanPageComponent

Children:
- KanbanColumnComponent (3 instances)
- MobileTabsComponent (responsive view)

Dependencies:
- Angular CDK Drag Drop
- RxJS for state management
- API service for data operations

Library Components Used:
- Button Component → Add task actions
- Popup Component → Quick task preview

TypeScript Specification

Inputs:
- columns: KanbanColumn[] (column configuration)
- loading: boolean (loading state)
- error: string | null (error message)
- allowDragDrop: boolean (enable/disable drag functionality)

Outputs:
- taskMoved: EventEmitter<TaskMoveEvent> (task position change)
- taskCreated: EventEmitter<Task> (new task creation)
- taskUpdated: EventEmitter<Task> (task modification)
- columnUpdated: EventEmitter<KanbanColumn> (column changes)

State:
- draggedTask: Task | null (currently dragged task)
- columnData: KanbanColumn[] (internal column state)
- isMobileView: boolean (responsive state)
- selectedColumn: string | null (mobile column selection)

Methods:
- ngOnInit() → Initialize component and load data
- loadKanbanData() → Fetch initial kanban board data
- onTaskDrop(event: CdkDragDrop<Task[]>) → Handle task drop events
- onTaskCreate(columnId: string) → Open task creation modal
- onTaskUpdate(task: Task) → Handle task updates
- toggleMobileColumn(columnId: string) → Switch mobile view columns

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="mobile-tabs" *ngIf="isMobileView">
    <mobile-tabs-component 
      [columns]="columns" 
      [selectedColumn]="selectedColumn"
      (columnSelected)="toggleMobileColumn($event)">
    </mobile-tabs-component>
  </div>
  
  <div class="kanban-grid" [class.mobile-single-column]="isMobileView">
    <kanban-column-component 
      *ngFor="let column of columnData; trackBy: trackByColumnId"
      [column]="column"
      [tasks]="column.tasks"
      [allowDrop]="allowDragDrop"
      [isVisible]="!isMobileView || selectedColumn === column.id"
      (taskDropped)="onTaskDrop($event)"
      (addTaskClicked)="onTaskCreate(column.id)"
      cdkDropList
      [cdkDropListData]="column.tasks"
      [cdkDropListConnectedTo]="getConnectedDropLists()">
    </kanban-column-component>
  </div>
  
  <div class="error-state" *ngIf="error" role="alert">
    <p>{{ error }}</p>
    <button-component 
      variant="primary" 
      size="sm"
      (click)="loadKanbanData()">Retry</button-component>
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; }
- .mobile-single-column { grid-template-columns: 1fr; }

Spacing:
- .kanban-board-container { padding: 1rem; gap: 1rem; }
- .kanban-grid { gap: 1rem; }

Component Styles:
- .kanban-board-container { min-height: 100vh; background: var(--bg-secondary); }
- .error-state { text-align: center; padding: 2rem; }

Responsive:
- @media (max-width: 768px) { .kanban-grid { grid-template-columns: 1fr; } }
- @media (min-width: 1024px) { .kanban-grid { grid-template-columns: repeat(3, minmax(300px, 1fr)); } }

API INTEGRATION

- loadKanbanData() → GET /api/youth-accounts/{id}/dashboard → Load initial kanban board data
- createTask() → POST /api/youth-accounts/{id}/fund-transfer → Create new task (repurposed endpoint)
- updateTaskStatus() → PUT /api/youth-accounts/{id}/spending-limit → Update task status (repurposed endpoint)

ERROR HANDLING

- API failure → Display error message with retry button
- Drag drop failure → Revert task position and show snackbar
- Column configuration error → Show fallback three-column layout

INTERACTION FLOW

User Action → UI → API
- Load board → ngOnInit() → GET /api/dashboard → Display columns and tasks
- Drag task → onTaskDrop() → PUT /api/spending-limit → Update task status
- Add task → onTaskCreate() → Open modal → POST /api/fund-transfer → Refresh board

        [priority]="task.priority"

      </task-card-component>
    </div>
    
    <div class="add-task-section">
      <button-component 
        variant="outline" 
        size="sm"

  </div>

Component → API:
- KanbanBoardComponent → GET /api/youth-accounts/{id}/dashboard (load kanban board data)
- TaskFormComponent → POST /api/youth-accounts/{id}/fund-transfer (create new task - repurposed endpoint)
- TaskFormComponent → PUT /api/youth-accounts/{id}/spending-limit (update task status - repurposed endpoint)
- DashboardPageComponent → GET /api/youth-accounts/{id}/dashboard (load dashboard metrics)
- TaskHistoryComponent → GET /api/youth-accounts/{id}/transactions (load task history - repurposed endpoint)

API Transformations:
- Fund transfer amount → Task priority (high: >1000, medium: 100-1000, low: <100)
- Spending limit → Task deadline (limit date becomes due date)
- Transaction description → Task title and description
- Account balance → Project completion percentage
- Transaction status → Task status (pending→todo, completed→done, processing→in-progress)
