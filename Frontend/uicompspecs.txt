### 4.1 KanbanDashboard (Page Component)

Component Type: Page

Mapped From A1: Main page container with three-column Kanban layout

Purpose: Root page component that orchestrates the entire Kanban board experience

Parent: App Router

Children:
- HeaderComponent
- SidebarComponent
- KanbanBoardComponent
- AddTaskModalComponent
- TaskDetailsModalComponent

Dependencies:
- React hooks (useState, useEffect, useContext)
- WebSocket connection for real-time updates
- API service layer

Library Components Used:
- AavaButtonComponent → Add Task button
- AavaPopoverComponent → Help tooltips

TypeScript Specification

Inputs:
- boardId: string (from route parameters)
- userId: string (from authentication context)

Outputs:
- onTaskCreate: (task: Task) => void
- onTaskUpdate: (taskId: string, updates: Partial<Task>) => void
- onTaskDelete: (taskId: string) => void

State:
- tasks: Task[]
- columns: Column[]
- isLoading: boolean
- selectedTask: Task | null
- showAddModal: boolean
- showDetailsModal: boolean

Methods:
- loadBoardData() → fetches initial board state
- handleTaskDragEnd() → handles drag and drop operations
- openAddTaskModal() → shows task creation modal
- openTaskDetails() → shows task details modal

HTML STRUCTURE (PSEUDO CODE ONLY)

<div className="kanban-dashboard">
  <HeaderComponent 
    searchValue={searchTerm}
    onSearch={handleSearch}
    userProfile={currentUser}
  />
  
  <div className="dashboard-content">
    <SidebarComponent 
      filters={activeFilters}
      onFilterChange={handleFilterChange}
    />
    
    <KanbanBoardComponent 
      columns={columns}
      tasks={filteredTasks}
      onTaskMove={handleTaskMove}
      onTaskClick={openTaskDetails}
    />
  </div>
  
  {showAddModal && (
    <AddTaskModalComponent 
      onSubmit={handleTaskCreate}
      onClose={closeAddModal}
    />
  )}
  
  {showDetailsModal && selectedTask && (
    <TaskDetailsModalComponent 
      task={selectedTask}
      onUpdate={handleTaskUpdate}
      onDelete={handleTaskDelete}
      onClose={closeDetailsModal}
    />
  )}
</div>

CSS SPECIFICATION

Layout:
- display: grid
- grid-template-rows: auto 1fr
- grid-template-columns: 250px 1fr
- height: 100vh

Spacing:
- gap: 0
- padding: 0

Component Styles:
- .kanban-dashboard: full viewport container
- .dashboard-content: main content grid

Responsive:
- @media (max-width: 768px): sidebar collapses to overlay
- @media (max-width: 480px): single column layout

API INTEGRATION

- loadBoardData() → GET /api/boards/{boardId} → fetch board configuration and tasks
- handleTaskCreate() → POST /api/boards/{boardId}/tasks → create new task
- handleTaskUpdate() → PUT /api/boards/{boardId}/tasks/{taskId} → update task
- handleTaskMove() → POST /api/boards/{boardId}/tasks/batch → batch update task positions
- WebSocket connection → /ws/boards/{boardId} → real-time task updates

ERROR HANDLING

- API failure → show error snackbar with retry option
- WebSocket disconnect → show connection status indicator
- Validation error → inline form validation messages
- Network timeout → graceful degradation with cached data

INTERACTION FLOW

User Action → UI → API
- Click "Add Task" → open AddTaskModalComponent → submit → POST /api/tasks
- Drag task card → update UI optimistically → POST /api/tasks/batch
- Click task card → open TaskDetailsModalComponent → edit → PUT /api/tasks/{id}
- Search tasks → filter local state → debounced GET /api/tasks?search=query

### 4.3 KanbanBoardComponent (Feature Component)

Component Type: Feature

Mapped From A1: Main three-column Kanban board with drag-and-drop

Purpose: Renders the core Kanban board with columns and task cards

Parent: KanbanDashboard

Children:
- KanbanColumnComponent (3 instances: To Do, In Progress, Done)
- KanbanCardComponent (multiple instances per column)

Dependencies:
- React DnD for drag and drop
- Task state management

Library Components Used:
- AavaButtonComponent → column action buttons
- AavaPopoverComponent → column settings menu

TypeScript Specification

Inputs:
- columns: Column[]
- tasks: Task[]
- isLoading: boolean

Outputs:
- onTaskMove: (taskId: string, newColumnId: string, newPosition: number) => void
- onTaskClick: (task: Task) => void
- onColumnUpdate: (columnId: string, updates: Partial<Column>) => void

State:
- draggedTask: Task | null
- dragOverColumn: string | null

Methods:
- handleDragStart() → initiates task drag operation
- handleDragOver() → handles drag over column
- handleDrop() → completes task move operation
- getTasksByColumn() → filters tasks by column

HTML STRUCTURE (PSEUDO CODE ONLY)

<div className="kanban-board">
  <div className="board-columns">
    {columns.map(column => (
      <KanbanColumnComponent 
        key={column.id}
        column={column}
        tasks={getTasksByColumn(column.id)}
        onTaskMove={onTaskMove}
        onTaskClick={onTaskClick}
        isDragOver={dragOverColumn === column.id}
      />
    ))}
  </div>
  
  {isLoading && (
    <div className="board-loading">
      <LoadingSpinner />
    </div>
  )}
</div>

CSS SPECIFICATION

Layout:
- display: flex
- gap: 24px
- padding: 24px
- overflow-x: auto

Spacing:
- min-height: calc(100vh - 64px)
- padding: 24px

Component Styles:
- .kanban-board: main board container
- .board-columns: flex container for columns
- .board-loading: centered loading overlay

Responsive:
- @media (max-width: 768px): horizontal scroll for columns
- @media (max-width: 480px): single column stack layout

API INTEGRATION

- handleDrop() → POST /api/boards/{boardId}/tasks/batch → batch update task positions
- onColumnUpdate() → PUT /api/boards/{boardId}/columns/{columnId} → update column settings

ERROR HANDLING

- Drag operation failure → revert task position, show error
- Column update error → show inline error message
- Task load failure → show empty state with retry

INTERACTION FLOW

User Action → UI → API
- Drag task between columns → optimistic UI update → POST /api/tasks/batch
- Click task card → open task details → navigate to task view
- Update column settings → immediate UI update → PUT /api/columns/{id}

### 4.5 KanbanCardComponent (Feature Component)

Component Type: Feature

Mapped From A1: Individual task card with drag functionality

Purpose: Renders a single task card with task information and interactions

Parent: KanbanColumnComponent

Children:
- AavaAvatarsComponent (assignee)
- AavaButtonComponent (quick actions)

Dependencies:
- Drag and drop functionality
- Task data model

Library Components Used:
- AavaAvatarsComponent → task assignee display
- AavaButtonComponent → quick action buttons
- AavaPopoverComponent → task preview on hover

TypeScript Specification

Inputs:
- task: Task
- index: number
- isDragging: boolean

Outputs:
- onClick: (task: Task) => void
- onQuickEdit: (taskId: string) => void

State:
- isHovered: boolean
- showQuickActions: boolean

Methods:
- handleCardClick() → opens task details
- handleQuickEdit() → enables inline editing
- formatDueDate() → formats task due date
- getPriorityColor() → returns priority indicator color

HTML STRUCTURE (PSEUDO CODE ONLY)

<div 
  className={`kanban-card ${isDragging ? 'dragging' : ''} priority-${task.priority}`}
  onClick={() => onClick(task)}
  onMouseEnter={() => setIsHovered(true)}
  onMouseLeave={() => setIsHovered(false)}
>
  <div className="card-header">
    <span className="task-id">#{task.id}</span>
    <span className={`priority-indicator priority-${task.priority}`} />
  </div>
  
  <div className="card-content">
    <h4 className="task-title">{task.title}</h4>
    <p className="task-description">{task.description}</p>
  </div>
  
  <div className="card-footer">
    <div className="task-meta">
      {task.dueDate && (
        <span className="due-date">
          {formatDueDate(task.dueDate)}
        </span>
      )}
      
      {task.tags.map(tag => (
        <span key={tag} className="task-tag">{tag}</span>
      ))}
    </div>
    
    <div className="task-assignee">
      {task.assignee && (
        <AavaAvatarsComponent 
          user={task.assignee}
          size="small"
          showTooltip={true}
        />
      )}
    </div>
  </div>
  
  {isHovered && (
    <div className="quick-actions">
      <AavaButtonComponent 
        variant="ghost"
        size="small"
        icon="edit"
        onClick={(e) => {
          e.stopPropagation();
          onQuickEdit(task.id);
        }}
      />
    </div>
  )}
</div>

CSS SPECIFICATION

Layout:
- display: flex
- flex-direction: column
- padding: 12px
- border-radius: 6px
- margin-bottom: 8px

Spacing:
- gap: 8px
- min-height: 120px

Component Styles:
- .kanban-card: card container with shadow
- .card-header: flex header with ID and priority
- .card-content: main task information
- .card-footer: meta information and assignee
- .dragging: reduced opacity during drag
- .priority-high: red border accent
- .priority-medium: yellow border accent
- .priority-low: green border accent

Responsive:
- @media (max-width: 480px): reduced padding, smaller text

API INTEGRATION

- onClick() → triggers parent to open task details modal
- onQuickEdit() → enables inline editing mode

ERROR HANDLING

- Card click failure → show error toast
- Quick edit error → revert changes, show error

INTERACTION FLOW

User Action → UI → API
- Click card → open TaskDetailsModalComponent → GET /api/tasks/{id}
- Drag card → visual feedback → POST /api/tasks/batch on drop
- Quick edit → inline editing → PUT /api/tasks/{id}

8. NOTES & CONSTRAINTS

- HTML must follow Agent-1 structure strictly - three-column Kanban layout with accessibility features
- Do NOT create new components unnecessarily - maximize reuse of AavaButtonComponent, AavaAvatarsComponent, AavaSearchBarComponent, AavaPopoverComponent
- Prefer reusable library components over custom implementations
- HTML must be pseudo-code ONLY (NOT real framework code)
- Implement drag-and-drop using React DnD library for accessibility compliance
- WebSocket integration required for real-time collaboration features
- All API calls must include proper error handling and loading states
- Responsive design must support mobile-first approach
- ARIA attributes required for accessibility compliance
- Performance optimization through virtual scrolling for large task lists
- Component state management should use React Context for global state
- CSS Grid layout required for main dashboard structure
- Flexbox for component-level layouts
- Task cards must support keyboard navigation
- Color coding for task priorities and status indicators
- Batch API operations for performance optimization
- Optimistic UI updates for better user experience
