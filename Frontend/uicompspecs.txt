4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for three-column Kanban layout

Purpose: Primary container component that manages the overall Kanban board state and coordinates between columns

Parent: AppComponent

Children:
- KanbanColumnComponent (3 instances)
- KanbanErrorStateComponent
- KanbanHeaderComponent

Dependencies:
- KanbanService
- KanbanApiService
- Angular CDK Drag Drop

Library Components Used:
- CardComponent → Container wrapper
- ButtonComponent → Action buttons
- ProgressBarComponent → Loading states

TypeScript Specification

Inputs:
- boardId: string (board identifier)
- permissions: Permission[] (user permissions)
- loading: boolean (loading state)

Outputs:
- onColumnUpdate: EventEmitter<{columnId: string, data: any}> (column update payload)
- onCardMove: EventEmitter<{cardId: string, fromColumn: string, toColumn: string}> (card move payload)
- onError: EventEmitter<string> (error message payload)

State:
- columns: KanbanColumn[] (current column data)
- isLoading: boolean (loading indicator)
- errorMessage: string | null (error state)
- draggedCard: KanbanCard | null (currently dragged card)

Methods:
- ngOnInit() → Initialize board data
- loadBoardData() → Fetch board from API
- handleCardMove(event: CdkDragDrop<KanbanCard[]>) → Process card movement
- handleError(error: string) → Handle error states
- refreshBoard() → Reload board data

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="kanban-header">
    <h1 class="board-title">{{boardTitle}}</h1>
    <div class="board-actions">
      <button-component type="primary" (click)="refreshBoard()">Refresh</button-component>
    </div>
  </div>
  
  <div class="kanban-columns" cdkDropListGroup>
    <kanban-column-component 
      *ngFor="let column of columns; trackBy: trackByColumnId"
      [columnData]="column"
      [cards]="column.cards"
      [allowDrop]="!isLoading"
      (onCardDrop)="handleCardMove($event)"
      (onCardAdd)="handleCardAdd($event)"
      cdkDropList
      [cdkDropListData]="column.cards">
    </kanban-column-component>
  </div>
  
  <kanban-error-state-component 
    *ngIf="errorMessage" 
    [message]="errorMessage"
    (onRetry)="loadBoardData()">
  </kanban-error-state-component>
  
  <div class="loading-overlay" *ngIf="isLoading">
    <progress-bar-component mode="indeterminate"></progress-bar-component>
  </div>
</div>

CSS SPECIFICATION

Layout:
- display: grid
- grid-template-columns: repeat(3, 1fr)
- grid-gap: 1rem
- min-height: 100vh

Spacing:
- padding: 1rem
- gap: 1rem between columns
- margin: 0 auto for centering

Component Styles:
- .kanban-board-container: main grid container
- .kanban-columns: three-column grid layout
- .kanban-header: flex header with title and actions
- .loading-overlay: absolute positioned overlay

Responsive:
- Desktop (>1024px): 3 columns, full width
- Tablet (768px-1024px): 2 columns, stacked layout
- Mobile (<768px): 1 column, vertical stack

API INTEGRATION

- loadBoardData() → GET /kanban/boards/{boardId} → Fetch initial board state
- handleCardMove() → PUT /kanban/cards/{cardId}/move → Update card position
- refreshBoard() → GET /kanban/boards/{boardId} → Reload board data

ERROR HANDLING

- API failure → show error state component with retry option
- Network timeout → display connection error message
- Invalid board ID → redirect to board selection
- Permission denied → show access denied message

INTERACTION FLOW

User Action → UI → API
- Load board → ngOnInit() → GET /kanban/boards/{boardId} → populate columns
- Drag card → handleCardMove() → PUT /kanban/cards/{cardId}/move → update UI optimistically
- Refresh board → click refresh → GET /kanban/boards/{boardId} → reload data

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: Individual column container with drag-drop support

Purpose: Manages individual column state, card rendering, and drop zone functionality

Parent: KanbanBoardComponent

Children:
- KanbanCardComponent (multiple instances)
- AddCardButtonComponent

Dependencies:
- Angular CDK Drag Drop
- KanbanService

Library Components Used:
- CardComponent → Column container
- ButtonComponent → Add card button
- BadgeComponent → Card count indicator

TypeScript Specification

Inputs:
- columnData: KanbanColumn (column configuration)
- cards: KanbanCard[] (cards in column)
- allowDrop: boolean (enable drop functionality)
- maxCards: number (WIP limit)

Outputs:
- onCardDrop: EventEmitter<{cardId: string, columnId: string}> (card drop payload)
- onCardAdd: EventEmitter<{columnId: string, cardData: Partial<KanbanCard>}> (new card payload)
- onColumnCollapse: EventEmitter<string> (column collapse payload)

State:
- isDragOver: boolean (drag over state)
- isExpanded: boolean (column expansion state)
- isAddingCard: boolean (card creation state)
- cardCount: number (current card count)

Methods:
- onDragOver(event: CdkDragOver) → Handle drag over events
- onDrop(event: CdkDragDrop<KanbanCard[]>) → Process card drops
- toggleExpansion() → Expand/collapse column
- startAddCard() → Initialize card creation
- cancelAddCard() → Cancel card creation

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-column" 
     [class.drag-over]="isDragOver"
     [class.collapsed]="!isExpanded"
     cdkDropList
     [cdkDropListData]="cards"
     (cdkDropListDropped)="onDrop($event)">
  
  <div class="column-header" [attr.aria-labelledby]="'column-' + columnData.id">
    <h2 [id]="'column-' + columnData.id" class="column-title">
      {{columnData.title}}
    </h2>
    <badge-component [count]="cardCount" [maxCount]="maxCards"></badge-component>
    <button-component 
      type="ghost" 
      icon="collapse"
      (click)="toggleExpansion()"
      [attr.aria-expanded]="isExpanded">
    </button-component>
  </div>
  
  <div class="column-content" *ngIf="isExpanded">
    <div class="cards-container">
      <kanban-card-component
        *ngFor="let card of cards; trackBy: trackByCardId"
        [cardData]="card"
        [draggable]="allowDrop"
        [editable]="true"
        cdkDrag
        [cdkDragData]="card">
      </kanban-card-component>
    </div>
    
    <div class="add-card-section">
      <add-card-button-component
        *ngIf="!isAddingCard"
        (click)="startAddCard()"
        [disabled]="cardCount >= maxCards">
      </add-card-button-component>
      
      <div class="add-card-form" *ngIf="isAddingCard">
        <input-component
          placeholder="Enter card title"
          [(ngModel)]="newCardTitle"
          (keyup.enter)="createCard()"
          (keyup.escape)="cancelAddCard()">
        </input-component>
        <div class="form-actions">
          <button-component type="primary" (click)="createCard()">Add</button-component>
          <button-component type="secondary" (click)="cancelAddCard()">Cancel</button-component>
        </div>
      </div>
    </div>
  </div>
</div>

CSS SPECIFICATION

Layout:
- display: flex
- flex-direction: column
- min-height: 500px
- flex: 1

Spacing:
- padding: 1rem
- margin-bottom: 1rem between cards
- gap: 0.5rem in header

Component Styles:
- .kanban-column: flex column container with border
- .column-header: flex header with title and controls
- .cards-container: scrollable card list
- .add-card-section: fixed bottom section
- .drag-over: highlight style for drop target

Responsive:
- Desktop: fixed width 300px
- Tablet: flexible width with min 250px
- Mobile: full width stacked

API INTEGRATION

- onDrop() → PUT /kanban/cards/{cardId}/move → Update card column
- createCard() → POST /kanban/cards → Create new card in column
- loadColumnCards() → GET /kanban/columns/{columnId}/cards → Refresh column data

ERROR HANDLING

- Card creation failure → show inline error message
- Drop operation failure → revert card position
- Column load error → display retry option
- WIP limit exceeded → prevent card drops

INTERACTION FLOW

User Action → UI → API
- Drop card → onDrop() → PUT /kanban/cards/{cardId}/move → update card position
- Add card → createCard() → POST /kanban/cards → add to column
- Collapse column → toggleExpansion() → save preference locally

### 4.3 KanbanCardComponent

Component Type: Feature

Mapped From A1: Individual card with content and actions

Purpose: Displays card content, handles editing, and provides drag functionality

Parent: KanbanColumnComponent

Children:
- None (leaf component)

Dependencies:
- Angular CDK Drag Drop
- KanbanService

Library Components Used:
- CardComponent → Card container
- ButtonComponent → Action buttons
- InputComponent → Edit mode input
- BadgeComponent → Priority/status indicators
- AvatarComponent → Assignee display

TypeScript Specification

Inputs:
- cardData: KanbanCard (card information)
- draggable: boolean (enable drag functionality)
- editable: boolean (allow editing)
- compact: boolean (compact display mode)

Outputs:
- onCardEdit: EventEmitter<{cardId: string, data: Partial<KanbanCard>}> (edit payload)
- onCardDelete: EventEmitter<string> (delete payload)
- onDragStart: EventEmitter<string> (drag start payload)
- onCardClick: EventEmitter<string> (card click payload)

State:
- isEditing: boolean (edit mode state)
- isDragging: boolean (drag state)
- isHovered: boolean (hover state)
- editableTitle: string (temporary edit value)

Methods:
- startEdit() → Enter edit mode
- saveEdit() → Save card changes
- cancelEdit() → Cancel editing
- deleteCard() → Delete card with confirmation
- onDragStart() → Handle drag start
- onDragEnd() → Handle drag end

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-card"
     [class.editing]="isEditing"
     [class.dragging]="isDragging"
     [class.compact]="compact"
     cdkDrag
     [cdkDragData]="cardData"
     (cdkDragStarted)="onDragStart()"
     (cdkDragEnded)="onDragEnd()"
     (click)="onCardClick.emit(cardData.id)">
  
  <div class="card-header">
    <div class="card-title" *ngIf="!isEditing">
      {{cardData.title}}
    </div>
    <input-component
      *ngIf="isEditing"
      [(ngModel)]="editableTitle"
      (keyup.enter)="saveEdit()"
      (keyup.escape)="cancelEdit()"
      class="title-input">
    </input-component>
    
    <div class="card-actions">
      <button-component
        *ngIf="!isEditing && editable"
        type="ghost"
        icon="edit"
        size="small"
        (click)="startEdit(); $event.stopPropagation()">
      </button-component>
      
      <button-component
        *ngIf="!isEditing"
        type="ghost"
        icon="delete"
        size="small"
        (click)="deleteCard(); $event.stopPropagation()">
      </button-component>
    </div>
  </div>
  
  <div class="card-content" *ngIf="!compact">
    <div class="card-description" *ngIf="cardData.description">
      {{cardData.description}}
    </div>
    
    <div class="card-metadata">
      <badge-component
        *ngIf="cardData.priority"
        [type]="cardData.priority"
        [text]="cardData.priority">
      </badge-component>
      
      <div class="card-assignee" *ngIf="cardData.assignee">
        <avatar-component
          [user]="cardData.assignee"
          size="small">
        </avatar-component>
      </div>
      
      <div class="card-due-date" *ngIf="cardData.dueDate">
        {{cardData.dueDate | date:'short'}}
      </div>
    </div>
  </div>
  
  <div class="edit-actions" *ngIf="isEditing">
    <button-component type="primary" size="small" (click)="saveEdit()">Save</button-component>
    <button-component type="secondary" size="small" (click)="cancelEdit()">Cancel</button-component>
  </div>
</div>

CSS SPECIFICATION

Layout:
- display: block
- position: relative
- cursor: grab (when draggable)
- cursor: grabbing (when dragging)

Spacing:
- padding: 0.75rem
- margin-bottom: 0.5rem
- border-radius: 4px

Component Styles:
- .kanban-card: card container with shadow and border
- .card-header: flex header with title and actions
- .card-content: content area with metadata
- .card-metadata: flex row with badges and assignee
- .editing: edit mode styling
- .dragging: drag state styling

Responsive:
- Desktop: full card content displayed
- Tablet: compact mode with essential info
- Mobile: minimal card display

API INTEGRATION

- saveEdit() → PUT /kanban/cards/{cardId} → Update card data
- deleteCard() → DELETE /kanban/cards/{cardId} → Remove card
- loadCardDetails() → GET /kanban/cards/{cardId} → Fetch full card data

ERROR HANDLING

- Save failure → revert changes and show error
- Delete failure → show error message
- Network error → show retry option
- Validation error → highlight invalid fields

INTERACTION FLOW

User Action → UI → API
- Edit card → startEdit() → saveEdit() → PUT /kanban/cards/{cardId} → update display
- Delete card → deleteCard() → confirm → DELETE /kanban/cards/{cardId} → remove from UI
- Drag card → onDragStart() → visual feedback → drop triggers parent update

### 4.4 HeaderComponent

Component Type: Layout

Mapped From A1: Application header with navigation and user controls

Purpose: Provides global navigation, search, notifications, and user menu

Parent: AppComponent

Children:
- SearchFieldComponent
- NotificationIconComponent
- UserAvatarComponent

Dependencies:
- AuthService
- NotificationService
- Router

Library Components Used:
- SearchFieldComponent → Search input
- AvatarComponent → User avatar
- BadgeComponent → Notification count
- ButtonComponent → Action buttons
- ModalComponent → User menu dropdown

TypeScript Specification

Inputs:
- user: User (current user data)
- notifications: Notification[] (user notifications)
- searchEnabled: boolean (enable search functionality)
- brandLogo: string (application logo URL)

Outputs:
- onSearch: EventEmitter<string> (search query payload)
- onNotificationClick: EventEmitter<void> (notification click payload)
- onUserMenuClick: EventEmitter<void> (user menu payload)
- onLogout: EventEmitter<void> (logout payload)

State:
- isSearchFocused: boolean (search focus state)
- notificationCount: number (unread notifications)
- isUserMenuOpen: boolean (user menu state)
- searchQuery: string (current search term)

Methods:
- onSearchInput(query: string) → Handle search input
- toggleNotifications() → Open notification panel
- toggleUserMenu() → Open/close user menu
- logout() → Handle user logout
- navigateToProfile() → Navigate to user profile

HTML STRUCTURE (PSEUDO CODE ONLY)

<header class="app-header" role="banner">

### 4.5 SidebarComponent

Component Type: Layout

Mapped From A1: Navigation sidebar with menu items

Purpose: Provides application navigation with collapsible menu structure

Parent: AppComponent

Children:
- NavigationMenuComponent
- BrandComponent
- SidebarFooterComponent

Dependencies:
- Router
- NavigationService

Library Components Used:
- ButtonComponent → Menu toggle and items
- BadgeComponent → Menu item indicators
- DividerComponent → Section separators
- IconComponent → Menu icons

TypeScript Specification

Inputs:
- menuItems: MenuItem[] (navigation items)
- collapsed: boolean (sidebar collapse state)
- activeRoute: string (current active route)
- userPermissions: Permission[] (user access permissions)

Outputs:
- onMenuItemClick: EventEmitter<string> (menu item payload)
- onToggleCollapse: EventEmitter<boolean> (collapse state payload)
- onSectionExpand: EventEmitter<string> (section expand payload)

State:
- isCollapsed: boolean (collapse state)
- expandedSections: string[] (expanded menu sections)
- hoveredItem: string | null (hovered menu item)
- sidebarWidth: number (current width)

Methods:
- toggleCollapse() → Toggle sidebar collapse
- expandSection(sectionId: string) → Expand menu section
- collapseSection(sectionId: string) → Collapse menu section
- navigateToItem(itemId: string) → Navigate to menu item
- isItemActive(route: string) → Check if item is active

HTML STRUCTURE (PSEUDO CODE ONLY)

<aside class="app-sidebar"
       [class.collapsed]="isCollapsed"
       [style.width.px]="sidebarWidth"
       role="navigation"
       aria-label="Main Navigation">
  
  <div class="sidebar-header">
    <div class="brand-section" *ngIf="!isCollapsed">
      <img src="/assets/logo.svg" alt="Logo" class="sidebar-logo">
      <span class="app-name">Kanban App</span>
    </div>
    
    <button-component
      type="ghost"
      icon="menu"
      size="small"
      (click)="toggleCollapse()"
      [attr.aria-expanded]="!isCollapsed"
      class="collapse-toggle">
    </button-component>
  </div>
  
  <nav class="sidebar-nav">
    <ul class="nav-menu" role="menubar">
      <li *ngFor="let item of menuItems; trackBy: trackByItemId" 
          class="nav-item"
          [class.active]="isItemActive(item.route)"
          [class.has-children]="item.children?.length > 0"
          role="none">
        
        <button-component
          type="ghost"
          fullWidth="true"
          [class.active]="isItemActive(item.route)"
          (click)="item.children ? expandSection(item.id) : navigateToItem(item.id)"
          role="menuitem"
          [attr.aria-expanded]="expandedSections.includes(item.id)">
          
          <icon-component [name]="item.icon" size="small"></icon-component>
          <span class="nav-label" *ngIf="!isCollapsed">{{item.label}}</span>
          
          <badge-component
            *ngIf="item.badge && !isCollapsed"
            [count]="item.badge.count"
            [type]="item.badge.type">
          </badge-component>
          
          <icon-component
            *ngIf="item.children && !isCollapsed"
            name="chevron-down"
            size="small"
            [class.rotated]="expandedSections.includes(item.id)"
            class="expand-icon">
          </icon-component>
        </button-component>
        
        <ul *ngIf="item.children && expandedSections.includes(item.id) && !isCollapsed"
            class="nav-submenu"
            role="menu">
          <li *ngFor="let child of item.children"
              class="nav-subitem"
              [class.active]="isItemActive(child.route)"
              role="none">
            <button-component
              type="ghost"
              fullWidth="true"
              [class.active]="isItemActive(child.route)"
              (click)="navigateToItem(child.id)"
              role="menuitem">
              <span class="nav-sublabel">{{child.label}}</span>
            </button-component>
          </li>
        </ul>
      </li>
    </ul>
  </nav>
  
  <div class="sidebar-footer" *ngIf="!isCollapsed">
    <divider-component></divider-component>
    <div class="version-info">
      <span class="version-label">Version 1.0.0</span>
    </div>
  </div>
</aside>

CSS SPECIFICATION

Layout:
- display: flex
- flex-direction: column
- position: fixed
- left: 0
- top: 0
- height: 100vh
