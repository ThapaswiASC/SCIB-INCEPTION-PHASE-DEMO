4. COMPONENT IMPLEMENTATION

### 4.1 TaskCreationComponent

Component Type: Feature

Mapped From A1: TaskCreationComponent (Primary Focus from Jira)

Purpose: Create new tasks with comprehensive validation and API integration using Angular reactive forms

Parent: MainContentComponent

Children:
- FormFieldComponent (title, description, priority, due date)
- ButtonComponent (submit, cancel)
- LoadingSpinnerComponent (during API call)
- ToastComponent (success/error messages)

Dependencies:
- FormBuilder (Angular reactive forms)
- Validators (Angular form validation)
- TaskService (API integration)
- ModalComponent (wrapper)

Library Components Used:
- ModalComponent → Overlay modal with backdrop
- FormFieldComponent → Input, Select, Textarea with validation
- ButtonComponent → Primary and secondary actions
- LoadingSpinnerComponent → API call loading state
- ToastComponent → Success/error notifications

TypeScript Specification

Inputs:
- isOpen: boolean (modal visibility)
- onClose: () => void (close handler)
- onTaskCreated: (task: Task) => void (success callback)

Outputs:
- taskCreated (Task payload)
- modalClosed (void)
- validationError (ValidationError payload)

State:
- taskForm: FormGroup (reactive form with title, description, priority, dueDate)
- isLoading: boolean (submission state)
- validationErrors: ValidationError[] (form errors)
- characterCounts: { title: number, description: number } (real-time counters)
- showCharacterCounters: boolean (display state)

Methods:
- initializeForm() → Setup FormBuilder with validators
- onSubmit() → Form submission with validation
- onCancel() → Modal close with confirmation
- onFieldChange(field: string, value: any) → Real-time validation and character counting
- resetForm() → Clear form after successful submission
- handleApiError(error: any) → Error handling and display

HTML STRUCTURE (PSEUDO CODE ONLY)

<modal [isOpen]="isOpen" [title]="'Create New Task'" (onClose)="onCancel()">
  <form [formGroup]="taskForm" (ngSubmit)="onSubmit()">
    <div class="form-section">
      <form-field 
        label="Task Title" 
        type="text" 
        formControlName="title"
        [required]="true"
        [maxLength]="255"
        [error]="getFieldError('title')"
        [characterCount]="characterCounts.title"
        (onChange)="onFieldChange('title', $event)">
      </form-field>
      
      <form-field 
        label="Description" 
        type="textarea" 
        formControlName="description"
        [maxLength]="2000"
        [error]="getFieldError('description')"
        [characterCount]="characterCounts.description"
        (onChange)="onFieldChange('description', $event)">
      </form-field>
      
      <form-field 
        label="Priority" 
        type="select" 
        formControlName="priority"
        [required]="true"
        [options]="priorityOptions"
        [error]="getFieldError('priority')">
      </form-field>
      
      <form-field 
        label="Due Date" 
        type="date" 
        formControlName="dueDate"
        [required]="true"
        [minDate]="tomorrow"
        [error]="getFieldError('dueDate')">
      </form-field>
    </div>
    
    <div class="form-actions">
      <button type="button" variant="secondary" (click)="onCancel()">Cancel</button>
      <button type="submit" variant="primary" [disabled]="taskForm.invalid || isLoading">
        <loading-spinner *ngIf="isLoading"></loading-spinner>
        {{ isLoading ? 'Creating...' : 'Create Task' }}
      </button>
    </div>
  </form>
</modal>

CSS SPECIFICATION

Layout:
- .form-section { display: flex; flex-direction: column; gap: 1.5rem; }
- .form-actions { display: flex; justify-content: flex-end; gap: 1rem; margin-top: 2rem; }

Spacing:
- padding: 1.5rem (modal content)
- gap: 1.5rem (form fields)
- margin-top: 2rem (action buttons)

Component Styles:
- .character-counter { font-size: 0.875rem; color: var(--text-secondary); }
- .validation-error { color: var(--error-color); font-size: 0.875rem; }
- .form-field-required::after { content: '*'; color: var(--error-color); }

Responsive:
- @media (max-width: 768px) { modal width: 95vw; padding: 1rem; }

API INTEGRATION

- onSubmit() → POST /api/tasks → Create new task with form data
- TaskService.createTask(taskData) → Returns Observable<Task>

ERROR HANDLING

- API failure → show toast notification with error message
- Validation error → inline field validation messages
- Network timeout → retry mechanism with user notification
- 422 Unprocessable Entity → map server validation errors to form fields

INTERACTION FLOW

User Action → UI → API
- Click "Add Task" button → open TaskCreationComponent modal → initialize form
- Type in form fields → real-time validation → character counter updates
- Submit form → show loading spinner → POST /api/tasks → success toast + close modal + refresh board
- Cancel action → confirmation dialog → close modal without saving

### 4.2 KanbanBoardComponent

Component Type: Feature

Mapped From A1: KanbanBoardComponent

Purpose: Display and manage task workflow columns with drag-and-drop functionality

Parent: MainContentComponent

Children:
- TaskCardComponent (for each task)
- LoadingSpinnerComponent (initial load)
- ButtonComponent (column actions)

Dependencies:
- TaskService (API integration)
- DragDropModule (Angular CDK)

Library Components Used:
- CardComponent → Column containers
- TaskCardComponent → Individual task display
- LoadingSpinnerComponent → Loading states
- ButtonComponent → Column actions

TypeScript Specification

Inputs:
- tasks: Task[] (task data)
- columns: Column[] (board configuration)
- onTaskMove: (taskId: string, newStatus: string) => void

Outputs:
- taskMoved (TaskMoveEvent payload)
- taskClicked (string taskId payload)
- columnUpdated (Column payload)

State:
- draggedTask: Task | null
- columnTasks: Map<string, Task[]>
- isLoading: boolean
- dragOverColumn: string | null

Methods:
- loadTasks() → Fetch tasks from API
- onDragStart(task: Task) → Initialize drag operation
- onDragOver(column: string) → Handle drag over column
- onDrop(taskId: string, newStatus: string) → Handle task drop
- onTaskClick(taskId: string) → Navigate to task detail
- groupTasksByStatus() → Organize tasks by column

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board">
  <div class="board-header">
    <h2>Task Board</h2>
    <button variant="primary" (click)="openTaskCreation()">Add Task</button>
  </div>
  
  <div class="kanban-columns">
    <div class="kanban-column" *ngFor="let column of columns" 
         [class.drag-over]="dragOverColumn === column.id"
         (dragover)="onDragOver(column.id)"
         (drop)="onDrop($event, column.status)">
      
      <div class="column-header">
        <h3>{{ column.title }}</h3>
        <badge [variant]="column.color">{{ getTaskCount(column.status) }}</badge>
      </div>
      
      <div class="column-content">
        <task-card 
          *ngFor="let task of getTasksByStatus(column.status)"
          [task]="task"
          [draggable]="true"
          (click)="onTaskClick(task.id)"
          (dragstart)="onDragStart(task)">
        </task-card>
        
        <div class="empty-column" *ngIf="getTasksByStatus(column.status).length === 0">
          <p>No tasks in {{ column.title.toLowerCase() }}</p>
        </div>
      </div>
    </div>
  </div>
  
  <loading-spinner *ngIf="isLoading" class="board-loading"></loading-spinner>
</div>

CSS SPECIFICATION

Layout:
- .kanban-board { display: flex; flex-direction: column; height: 100%; }
- .kanban-columns { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1.5rem; flex: 1; }
- .kanban-column { display: flex; flex-direction: column; min-height: 500px; }

Spacing:
- padding: 1.5rem (board container)
- gap: 1.5rem (between columns)
- margin-bottom: 1rem (column headers)

Component Styles:
- .drag-over { background-color: var(--primary-light); border: 2px dashed var(--primary); }
- .column-header { display: flex; justify-content: space-between; align-items: center; }
- .empty-column { text-align: center; color: var(--text-secondary); padding: 2rem; }

Responsive:
- @media (max-width: 1024px) { grid-template-columns: repeat(2, 1fr); }
- @media (max-width: 768px) { grid-template-columns: 1fr; }

API INTEGRATION

- loadTasks() → GET /api/tasks → Load all tasks for board display
- onDrop() → PUT /api/tasks/{id} → Update task status on drag and drop

ERROR HANDLING

- API failure → show toast notification, retry mechanism
- Drag operation failure → revert UI state, show error message
- Network timeout → offline mode indication

INTERACTION FLOW

User Action → UI → API
- Load board → GET /api/tasks → display tasks in columns
- Drag task to new column → optimistic UI update → PUT /api/tasks/{id} → confirm or revert
- Click task card → navigate to TaskDetailComponent → GET /api/tasks/{id}

### 4.6 ModalComponent

Component Type: Shared

Mapped From A1: ModalComponent

Purpose: Reusable modal wrapper with backdrop, accessibility, and animation support

Parent: Various components (TaskCreationComponent, etc.)

Children:
- ButtonComponent (close button)
- Content projection (ng-content)

Dependencies:

- Focus management utilities

Library Components Used:
- ButtonComponent → Close button
- OverlayModule → Modal backdrop and positioning

TypeScript Specification

Inputs:
- isOpen: boolean
- title: string
- size: 'sm' | 'md' | 'lg' | 'xl'
- closable: boolean
- closeOnBackdrop: boolean
- closeOnEscape: boolean

- close() → Hide modal with animation and focus restoration
- onBackdropClick() → Handle backdrop click if enabled
- onEscapeKey() → Handle escape key if enabled
- trapFocus() → Manage focus within modal
- restoreFocus() → Return focus to previous element
