4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main kanban board container with three-column layout

Purpose: Orchestrates the entire kanban board functionality with drag-and-drop support

Parent: AppComponent

Children:
- KanbanColumnComponent (3 instances)
- TaskModalComponent

Dependencies:
- KanbanService
- TaskModel
- Play UI components

Library Components Used:
- Card → Container wrapper
- Button → Action buttons

TypeScript Specification

Inputs:
- boardId: string
- userId: string

Outputs:
- taskCreated: EventEmitter<Task>
- taskUpdated: EventEmitter<Task>
- taskDeleted: EventEmitter<string>

State:
- tasks: Task[]
- columns: Column[]
- isLoading: boolean
- error: string | null

Methods:
- ngOnInit() → Initialize board data
- loadTasks() → Fetch tasks from API
- onTaskDrop(event: CdkDragDrop) → Handle drag-and-drop
- openTaskModal(task?: Task) → Open task creation/editing modal

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container">
  <div class="kanban-header">
    <h2>TaskFlow Board</h2>
    <play-button (click)="openTaskModal()">Add Task</play-button>
  </div>
  
  <div class="kanban-columns" cdkDropListGroup>
    <app-kanban-column 
      *ngFor="let column of columns" 
      [column]="column"
      [tasks]="getTasksByColumn(column.id)"
      (taskMoved)="onTaskDrop($event)">
    </app-kanban-column>
  </div>
  
  <app-task-modal 
    [(visible)]="showTaskModal"
    [task]="selectedTask"
    (taskSaved)="onTaskSaved($event)">
  </app-task-modal>
</div>

CSS SPECIFICATION

Layout:
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1rem

Spacing:
- padding: 1.5rem
- margin: 0 auto

Component Styles:
- .kanban-board-container: max-width 1200px
- .kanban-columns: display flex, gap 1rem

Responsive:
- @media (max-width: 768px): single column layout
- @media (min-width: 1024px): three column layout

API INTEGRATION

- loadTasks() → GET /api/tasks → Fetch all tasks for board
- createTask() → POST /api/tasks → Create new task
- updateTask() → PUT /api/tasks/{id} → Update existing task
- deleteTask() → DELETE /api/tasks/{id} → Remove task

ERROR HANDLING

- API failure → show error snackbar
- Validation error → inline form validation
- Network timeout → retry mechanism

INTERACTION FLOW

User Action → UI → API
- Drag task → update column → PUT /api/tasks/{id}
- Click "Add Task" → open modal → POST /api/tasks
- Edit task → open modal with data → PUT /api/tasks/{id}

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: Individual column container for tasks

Purpose: Displays tasks in a specific column with drag-and-drop support

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)

Dependencies:
- TaskModel
- CDK Drag Drop

Library Components Used:
- Card → Column container

TypeScript Specification

Inputs:
- column: Column
- tasks: Task[]

Outputs:
- taskMoved: EventEmitter<CdkDragDrop<Task[]>>

State:
- draggedTask: Task | null

Methods:
- onTaskDrop(event: CdkDragDrop) → Handle task drop
- getTaskCount() → Return number of tasks in column

HTML STRUCTURE (PSEUDO CODE ONLY)

<play-card class="kanban-column">
  <div class="column-header">
    <h3>{{ column.title }}</h3>
    <span class="task-count">{{ getTaskCount() }}</span>
  </div>
  
  <div class="column-content" 
       cdkDropList 
       [cdkDropListData]="tasks"
       (cdkDropListDropped)="onTaskDrop($event)">
    
    <app-task-card 
      *ngFor="let task of tasks" 
      [task]="task"
      cdkDrag>
    </app-task-card>
  </div>
</play-card>

CSS SPECIFICATION

Layout:
- display: flex
- flex-direction: column
- min-height: 500px

Spacing:
- padding: 1rem
- gap: 0.5rem

Component Styles:
- .kanban-column: background #f5f5f5, border-radius 8px
- .column-content: flex-grow 1, overflow-y auto

Responsive:
- @media (max-width: 768px): full width

API INTEGRATION

- onTaskDrop() → PUT /api/tasks/{id} → Update task column

ERROR HANDLING

- Drop failure → revert task position
- API error → show error message

INTERACTION FLOW

User Action → UI → API
- Drop task in column → update task status → API call

### 4.5 TaskModalComponent

Component Type: Feature

Mapped From A1: Task creation and editing modal

Purpose: Provides form interface for creating and editing tasks

Parent: KanbanBoardComponent

Children: None

Dependencies:
- TaskModel
- FormBuilder
- Validators

Library Components Used:
- Modal → Container
- Form Input → Text fields
- Button → Actions
- Dropdown → Status selection

TypeScript Specification

Inputs:
- visible: boolean
- task: Task | null

Outputs:
- taskSaved: EventEmitter<Task>
- modalClosed: EventEmitter<void>

State:
- taskForm: FormGroup
- isSubmitting: boolean
- validationErrors: string[]

Methods:
- ngOnInit() → Initialize form
- buildForm() → Create reactive form
- onSubmit() → Handle form submission
- onCancel() → Close modal without saving
- validateForm() → Check form validity

HTML STRUCTURE (PSEUDO CODE ONLY)

<play-modal [(visible)]="visible" 
            title="{{ task ? 'Edit Task' : 'Create Task' }}">
  
  <form [formGroup]="taskForm" (ngSubmit)="onSubmit()">
    <play-form-input 
      label="Title"
      formControlName="title"
      [error]="getFieldError('title')">
    </play-form-input>
    
    <play-form-input 
      label="Description"
      type="textarea"
      formControlName="description">
    </play-form-input>
    
    <play-dropdown 
      label="Status"
      formControlName="status"
      [options]="statusOptions">
    </play-dropdown>
    
    <play-form-input 
      label="Due Date"
      type="date"
      formControlName="dueDate">
    </play-form-input>
  </form>
  
  <div modal-footer>
    <play-button variant="secondary" 
                 (click)="onCancel()">Cancel</play-button>
    <play-button type="submit" 
                 [loading]="isSubmitting"
                 [disabled]="taskForm.invalid">
      {{ task ? 'Update' : 'Create' }}
    </play-button>
  </div>
</play-modal>

CSS SPECIFICATION

Layout:
- Modal width: 500px
- Form layout: vertical stack

Spacing:
- Form fields: margin-bottom 1rem
- Modal padding: 2rem

Component Styles:
- .task-form: display flex, flex-direction column
- .form-actions: display flex, justify-content flex-end, gap 1rem

Responsive:
- @media (max-width: 768px): full-width modal

API INTEGRATION

- onSubmit() → POST /api/tasks → Create new task
- onSubmit() → PUT /api/tasks/{id} → Update existing task

ERROR HANDLING

- Form validation → inline field errors
- API failure → show error message
- Network timeout → retry option

INTERACTION FLOW

User Action → UI → API
- Fill form → validate → submit → API call → close modal
- Cancel → close modal without saving

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly with three-column kanban layout
- Do NOT create new components unnecessarily - maximize reuse of Play UI components
- Prefer reusable library components (Button, Card, Avatar, Modal, Form Input)
- HTML must be pseudo-code ONLY (NOT real framework code)
- Implement ARIA accessibility features as specified in DEMO-1060
- Support responsive breakpoints: mobile (<768px), tablet (768px-1024px), desktop (>1024px)
- Use CSS Grid for main layout and Flexbox for component internals
- Implement drag-and-drop functionality using Angular CDK
- Follow Angular reactive forms pattern for task creation/editing
- Ensure proper error handling and loading states
- Implement proper TypeScript typing for all data models
- Use Angular services for API communication and state management
