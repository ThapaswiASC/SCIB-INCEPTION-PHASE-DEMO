### 4.3 KanbanBoardComponent

Component Type: Feature

Mapped From A1: KanbanBoardComponent with three-column layout and drag-and-drop

Purpose: Main Kanban board with drag-and-drop functionality for task management

Parent: KanbanBoardPageComponent

Children:
- TaskCardComponent (multiple instances)
- ButtonComponent (from uicomplibraryplay) - add task buttons
- IconComponent (from uicomplibraryplay) - column icons

Dependencies:
- TaskCardComponent (custom)
- ButtonComponent from uicomplibraryplay
- IconComponent from uicomplibraryplay
- Drag and drop library
- WebSocket service

Library Components Used:
- ButtonComponent → Add task buttons, column actions
- IconComponent → Column status indicators

TypeScript Specification:

Inputs:
- boardId: string
- columns: Column[]
- tasks: Task[]
- onTaskMove: (taskId: string, columnId: string, position: number) => void
- onTaskCreate: (columnId: string) => void
- onTaskClick: (taskId: string) => void

Outputs:
- taskMoved (payload: { taskId: string, fromColumn: string, toColumn: string, position: number })
- taskCreated (payload: { columnId: string })
- taskSelected (payload: { taskId: string })
- boardUpdated (payload: { boardId: string, timestamp: Date })

State:
- draggedTask: Task | null
- columnTasks: Map<string, Task[]>
- loading: boolean
- error: string | null
- dragOverColumn: string | null

Methods:
- handleDragStart(task: Task) → void
- handleDragEnd() → void
- handleDrop(columnId: string, position: number) → void
- handleTaskClick(taskId: string) → void
- handleAddTask(columnId: string) → void
- refreshBoard() → void

HTML STRUCTURE (PSEUDO CODE ONLY):

<div className="kanban-board">
  <div className="board-header">
    <h2>Kanban Board</h2>
    <ButtonComponent
      variant="primary"
      onClick={refreshBoard}
      disabled={loading}
    >
      <IconComponent name="refresh" />
      Refresh
    </ButtonComponent>
  </div>
  
  <div className="board-columns">
    {columns.map(column => (
      <div
        key={column.id}
        className={`board-column ${dragOverColumn === column.id ? 'drag-over' : ''}`}
        onDragOver={handleDragOver}
        onDrop={(e) => handleDrop(column.id, getDropPosition(e))}
      >
        <div className="column-header">
          <div className="column-title">
            <IconComponent name={column.icon} />
            <h3>{column.name}</h3>
            <span className="task-count">{columnTasks.get(column.id)?.length || 0}</span>
          </div>
          
          <ButtonComponent
            variant="ghost"
            size="small"
            onClick={() => handleAddTask(column.id)}
            ariaLabel={`Add task to ${column.name}`}
          >
            <IconComponent name="plus" />
          </ButtonComponent>
        </div>
        
        <div className="column-content">
          {columnTasks.get(column.id)?.map((task, index) => (
            <TaskCardComponent
              key={task.id}
              task={task}
              draggable={true}
              onDragStart={() => handleDragStart(task)}
              onClick={() => handleTaskClick(task.id)}
              isDragging={draggedTask?.id === task.id}
            />
          ))}
          
          {columnTasks.get(column.id)?.length === 0 && (
            <div className="empty-column">
              <IconComponent name="empty-state" />
              <p>No tasks in {column.name}</p>
              <ButtonComponent
                variant="secondary"
                onClick={() => handleAddTask(column.id)}
              >
                Add First Task
              </ButtonComponent>
            </div>
          )}
        </div>
      </div>
    ))}
  </div>
</div>

CSS SPECIFICATION:

Layout:
- display: flex
- flex-direction: column
- height: 100%
- overflow: hidden

Spacing:
- gap: 1rem between columns
- padding: 1rem for board container
- margin: 0.5rem between tasks

Component Styles:
- .kanban-board: height: 100%, display: flex, flex-direction: column
- .board-columns: display: grid, grid-template-columns: repeat(3, 1fr), gap: 1rem, flex: 1, overflow: hidden
- .board-column: background-color, border-radius, padding: 1rem, display: flex, flex-direction: column
- .column-content: flex: 1, overflow-y: auto, min-height: 200px
- .drag-over: border: 2px dashed primary color, background: highlight color

Responsive:
- Mobile (<768px): Single column layout, horizontal scroll
- Tablet (768px-1024px): Two columns visible, scroll for third
- Desktop (>1024px): All three columns visible

API INTEGRATION:

- loadBoardTasks() → GET /api/boards/{boardId}/tasks → Initial task loading
- moveTask(taskId, columnId, position) → PUT /api/tasks/{taskId}/move → Task movement
- createTask(columnId) → POST /api/boards/{boardId}/tasks → Task creation
- WebSocket connection → wss://api.scib.platform.com/ws/boards/{boardId} → Real-time updates

ERROR HANDLING:

- API failure → show error toast, revert optimistic updates
- WebSocket disconnect → show connection status, attempt reconnection
- Drag operation failure → revert task position, show error message
- Task creation error → show validation errors in modal

INTERACTION FLOW:

User Action → UI → API
- Drag task to new column → optimistic update → PUT /api/tasks/{taskId}/move → WebSocket broadcast
- Click add task → open TaskModalComponent → POST /api/boards/{boardId}/tasks → refresh board
- Click task card → open TaskModalComponent with task details → GET /api/tasks/{taskId}
- WebSocket receives update → update local state → re-render affected columns
