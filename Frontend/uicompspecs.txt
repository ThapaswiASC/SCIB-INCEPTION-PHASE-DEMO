### 4. COMPONENT IMPLEMENTATION

#### 4.1 KanbanBoardComponent

Component Type: Page

Mapped From A1: Main container component managing the entire Kanban board

Purpose: Root component that orchestrates the entire Kanban board functionality

Parent: AppComponent

Children:
- KanbanHeaderComponent
- KanbanSidebarComponent
- KanbanContentComponent
- TaskModalComponent

Dependencies:
- KanbanService
- TaskService
- Angular Common Module
- Angular Material

Library Components Used:
- AavaPopupComponent → TaskModalComponent

TypeScript Specification:

Inputs:
- youthAccountId: string
- initialColumns: KanbanColumn[]

Outputs:
- taskCreated: EventEmitter<Task>
- taskMoved: EventEmitter<{task: Task, fromColumn: string, toColumn: string}>
- taskUpdated: EventEmitter<Task>

State:
- columns: KanbanColumn[]
- selectedTask: Task | null
- modalOpen: boolean
- searchQuery: string
- loading: boolean
- error: string | null

Methods:
- ngOnInit() → Initialize component and load data
- loadDashboardData() → Fetch initial dashboard data
- onTaskCreated(task: Task) → Handle task creation
- onTaskMoved(event) → Handle task movement between columns
- openTaskModal(task?: Task) → Open task creation/edit modal
- closeTaskModal() → Close task modal

HTML STRUCTURE (PSEUDO CODE ONLY):

<div class="kanban-app-container" role="application" aria-label="Kanban Board Application">
  <kanban-sidebar></kanban-sidebar>
  
  <div class="main-layout">
    <kanban-header 
      [searchQuery]="searchQuery"
      (searchChanged)="onSearchChanged($event)">
    </kanban-header>
    
    <kanban-content 
      [columns]="columns"
      [loading]="loading"
      [error]="error"
      (taskSelected)="openTaskModal($event)"
      (taskMoved)="onTaskMoved($event)">
    </kanban-content>
  </div>
  
  <task-modal 
    [isOpen]="modalOpen"
    [task]="selectedTask"
    (taskSaved)="onTaskCreated($event)"
    (modalClosed)="closeTaskModal()">
  </task-modal>
</div>

CSS SPECIFICATION:

Layout:
- display: grid
- grid-template-columns: 280px 1fr
- height: 100vh

Spacing:
- gap: 0
- padding: 0

Component Styles:
- .kanban-app-container: full viewport layout
- .main-layout: flex column layout

Responsive:
- Mobile (<768px): sidebar collapses, single column
- Tablet (768px-1024px): sidebar width 240px
- Desktop (>1024px): sidebar width 280px

API INTEGRATION:
- ngOnInit() → GET /api/youth-accounts/{youthAccountId}/dashboard → Load initial data
- onTaskCreated() → POST /api/youth-accounts/{youthAccountId}/fund-transfer → Create new task

ERROR HANDLING:
- API failure → show error message in content area
- Validation error → display inline validation messages
- Network error → show retry button with error snackbar

INTERACTION FLOW:

User Action → UI → API:
- Page Load → Initialize component → GET /dashboard → Display columns and tasks
- Click "Add Task" → Open TaskModalComponent → User fills form → POST /fund-transfer
- Drag task between columns → Update UI optimistically → PUT /spending-limit

#### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: Individual column container for tasks

Purpose: Manages individual Kanban column with drag-drop functionality

Parent: KanbanContentComponent

Children:
- ColumnHeaderComponent
- TaskListComponent
- TaskCardComponent (multiple)

Dependencies:
- Angular CDK Drag Drop
- TaskService

Library Components Used:
- AavaButtonComponent → Add task button

TypeScript Specification:

Inputs:
- column: KanbanColumn
- tasks: Task[]
- columnId: string
- allowDrop: boolean

Outputs:
- taskDropped: EventEmitter<{task: Task, columnId: string}>
- taskSelected: EventEmitter<Task>
- addTaskClicked: EventEmitter<string>

State:
- dragOver: boolean
- taskCount: number
- expanded: boolean

Methods:
- onDrop(event: CdkDragDrop<Task[]>) → Handle task drop
- onDragEnter() → Handle drag enter
- onDragLeave() → Handle drag leave
- onTaskClick(task: Task) → Handle task selection
- onAddTask() → Emit add task event

HTML STRUCTURE (PSEUDO CODE ONLY):

<div class="kanban-column" 
     [class.drag-over]="dragOver"
     cdkDropList
     [cdkDropListData]="tasks"
     (cdkDropListDropped)="onDrop($event)"
     role="region"
     [aria-labelledby]="column.id + '-header'">
  
  <div class="column-header" [id]="column.id + '-header'">
    <h3>{{ column.title }}</h3>
    <span class="task-count">{{ taskCount }}</span>
    <aava-button 
      variant="icon"
      (click)="onAddTask()"
      aria-label="Add task to {{ column.title }}">
      <i class="icon-plus"></i>
    </aava-button>
  </div>
  
  <div class="task-list">
    <task-card 
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="true"
      (taskClicked)="onTaskClick(task)"
      cdkDrag>
    </task-card>
  </div>
</div>

CSS SPECIFICATION:

Layout:
- display: flex
- flex-direction: column
- min-height: 500px
- width: 100%

Spacing:
- padding: 16px
- gap: 12px

Component Styles:
- .kanban-column: background #f8f9fa, border-radius 8px
- .column-header: flex layout with space-between
- .task-list: flex column with gap
- .drag-over: border color change, background highlight

Responsive:
- Mobile: full width, vertical stacking
- Tablet: min-width 300px
- Desktop: flex-grow 1

API INTEGRATION:
- onDrop() → PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update task status

ERROR HANDLING:
- Drop failure → revert task position, show error message
- Task load error → show placeholder with retry option

INTERACTION FLOW:

User Action → UI → API:
- Drag task from another column → onDrop() → Update task status → API call
- Click task → onTaskClick() → Emit to parent → Open task modal
- Click add button → onAddTask() → Emit to parent → Open creation modal

#### 4.3 TaskCardComponent

Component Type: Feature

Mapped From A1: Individual task display card

Purpose: Display individual task with metadata and interaction capabilities

Parent: KanbanColumnComponent

Children:
- TaskPriorityIndicator
- TaskMetaComponent

Dependencies:
- Angular CDK Drag Drop
- DatePipe

Library Components Used:
- AavaDefaultCardComponent → Main card container
- AavaCardHeaderComponent → Task title area
- AavaCardContentComponent → Task description
- AavaCardFooterComponent → Task metadata
- AavaTagComponent → Task tags

TypeScript Specification:

Inputs:
- task: Task
- draggable: boolean
- selectable: boolean
- compact: boolean

Outputs:
- taskClicked: EventEmitter<Task>
- taskDragStart: EventEmitter<Task>
- taskDragEnd: EventEmitter<Task>

State:
- selected: boolean
- dragState: 'idle' | 'dragging' | 'preview'
- hovered: boolean

Methods:
- onClick() → Emit task clicked event
- onDragStart() → Handle drag start
- onDragEnd() → Handle drag end
- getPriorityColor() → Return priority color class
- formatDueDate() → Format due date display

HTML STRUCTURE (PSEUDO CODE ONLY):

<aava-default-card 
  class="task-card"
  [class.selected]="selected"
  [class.dragging]="dragState === 'dragging'"
  (click)="onClick()"
  (cdkDragStarted)="onDragStart()"
  (cdkDragEnded)="onDragEnd()"
  role="button"
  [attr.aria-label]="'Task: ' + task.title">
  
  <aava-card-header>
    <div class="priority-indicator" [class]="getPriorityColor()"></div>
    <h4 class="task-title">{{ task.title }}</h4>
  </aava-card-header>
  
  <aava-card-content>
    <p class="task-description">{{ task.description }}</p>
    
    <div class="task-tags">
      <aava-tag 
        *ngFor="let tag of task.tags"
        [label]="tag.name"
        [color]="tag.color">
      </aava-tag>
    </div>
  </aava-card-content>
  
  <aava-card-footer>
    <div class="assignee-avatar">
      <img [src]="task.assignee?.avatar" [alt]="task.assignee?.name">
    </div>
    <span class="due-date">{{ formatDueDate(task.dueDate) }}</span>
  </aava-card-footer>
</aava-default-card>

CSS SPECIFICATION:

Layout:
- display: block
- margin-bottom: 12px
- cursor: pointer

Spacing:
- padding: 16px (inherited from AavaDefaultCardComponent)
- gap: 8px between elements

Component Styles:
- .task-card: border-radius 6px, box-shadow, hover effects
- .priority-indicator: 4px width colored border
- .task-title: font-weight 600, line-height 1.4
- .task-description: color #666, font-size 14px
- .task-tags: flex wrap layout
- .assignee-avatar: 24px circle
- .selected: border color primary, box-shadow enhanced
- .dragging: opacity 0.8, transform scale(1.02)

Responsive:
- Mobile: full width, reduced padding
- Tablet: min-width 280px
- Desktop: hover effects enabled

API INTEGRATION:
- onClick() → No direct API calls (handled by parent)

ERROR HANDLING:
- Missing task data → show placeholder content
- Image load error → show default avatar

INTERACTION FLOW:

User Action → UI → API:
- Click task → onClick() → Emit to parent → Open task details modal
- Start drag → onDragStart() → Visual feedback → Prepare for drop
- End drag → onDragEnd() → Reset visual state → Parent handles API

#### 4.4 TaskModalComponent

Component Type: Feature

Mapped From A1: Modal for creating/editing tasks

Purpose: Handle task creation and editing with form validation

Parent: KanbanBoardComponent

Children:
- TaskFormComponent

Dependencies:
- Angular Reactive Forms
- Angular Material Dialog
- TaskService

Library Components Used:
- AavaPopupComponent → Modal container
- AavaButtonComponent → Action buttons

TypeScript Specification:

Inputs:
- isOpen: boolean
- task: Task | null
- mode: 'create' | 'edit'

Outputs:
- taskSaved: EventEmitter<Task>
- modalClosed: EventEmitter<void>

State:
- formData: TaskFormData
- validation: ValidationErrors
- submitting: boolean
- formValid: boolean

Methods:
- ngOnInit() → Initialize form
- onSubmit() → Handle form submission
- onCancel() → Close modal without saving
- validateForm() → Validate form data
- resetForm() → Reset form to initial state

HTML STRUCTURE (PSEUDO CODE ONLY):

<aava-popup 
  [isOpen]="isOpen"
  [title]="mode === 'create' ? 'Create Task' : 'Edit Task'"
  (closed)="onCancel()"
  role="dialog"
  aria-modal="true">
  
  <form [formGroup]="taskForm" (ngSubmit)="onSubmit()" novalidate>
    <div class="form-content">
      <div class="form-field">
        <label for="task-title">Title *</label>
        <input 
          id="task-title"
          type="text"
          formControlName="title"
          [class.error]="validation.title"
          aria-describedby="title-error">
        <span id="title-error" class="error-message" *ngIf="validation.title">
          {{ validation.title }}
        </span>
      </div>
      
      <div class="form-field">
        <label for="task-description">Description</label>
        <textarea 
          id="task-description"
          formControlName="description"
          rows="3">
        </textarea>
      </div>
      
      <div class="form-field">
        <label for="task-priority">Priority</label>
        <select id="task-priority" formControlName="priority">
          <option value="low">Low</option>
          <option value="medium">Medium</option>
          <option value="high">High</option>
        </select>
      </div>
      
      <div class="form-field">
        <label for="task-due-date">Due Date</label>
        <input 
          id="task-due-date"
          type="date"
          formControlName="dueDate">
      </div>
    </div>
    
    <div class="modal-actions">
      <aava-button 
        variant="secondary"
        type="button"
        (click)="onCancel()"
        [disabled]="submitting">
        Cancel
      </aava-button>
      
      <aava-button 
        variant="primary"
        type="submit"
        [disabled]="!formValid || submitting"
        [loading]="submitting">
        {{ mode === 'create' ? 'Create Task' : 'Update Task' }}
      </aava-button>
    </div>
  </form>
</aava-popup>

CSS SPECIFICATION:

Layout:
- .form-content: display flex, flex-direction column, gap 20px
- .modal-actions: display flex, justify-content flex-end, gap 12px

Spacing:
- padding: 24px
- gap: 20px between form fields

Component Styles:
- .form-field: margin-bottom 16px
- label: font-weight 500, margin-bottom 4px
- input, textarea, select: full width, padding 8px 12px
- .error: border-color red
- .error-message: color red, font-size 12px

Responsive:
- Mobile: full screen modal, adjusted padding
- Tablet: max-width 500px
- Desktop: max-width 600px

API INTEGRATION:
- onSubmit() → POST /api/youth-accounts/{youthAccountId}/fund-transfer → Create task
- onSubmit() → PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update task

ERROR HANDLING:
- API failure → show error message above form
- Validation error → show inline field errors
- Network error → show retry option

INTERACTION FLOW:

User Action → UI → API:
- Open modal → Initialize form → Load existing data if editing
- Fill form → Real-time validation → Enable/disable submit button
- Submit form → Validate → API call → Close modal on success
- Cancel → Close modal → Discard changes

### 8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components (AavaDefaultCardComponent, AavaButtonComponent, etc.)
- HTML must be pseudo-code ONLY (NOT real framework code)
- Implement proper ARIA attributes for accessibility
- Use CSS Grid for Kanban columns layout
- Implement responsive breakpoints: mobile (<768px), tablet (768px-1024px), desktop (>1024px)
- Follow Angular OnInit lifecycle hook pattern
- Implement proper error handling with *ngIf directives
- Use Angular Material or custom CSS for styling
- Ensure component compiles without errors and passes linting checks
