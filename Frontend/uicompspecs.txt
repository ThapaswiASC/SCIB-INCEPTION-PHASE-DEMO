4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for the three-column kanban board layout

Purpose: Orchestrates the entire kanban board functionality including drag-and-drop, task management, and API integration

Parent: MainLayoutComponent

Children:
- KanbanColumnComponent (3 instances)
- TaskModalComponent
- ButtonComponent (Add Task, Export Report)
- SearchFieldComponent (in board controls)

Dependencies:
- KanbanService
- YouthAccountService
- DragDropModule

Library Components Used:
- ButtonComponent → Add Task, Export Report buttons
- SearchFieldComponent → Task search functionality
- LoadingSpinnerComponent → Loading states

TypeScript Specification

Inputs:
- youthAccountId: string
- initialColumns: KanbanColumn[]
- loading: boolean

Outputs:
- taskCreated: EventEmitter<Task>
- taskUpdated: EventEmitter<Task>
- taskMoved: EventEmitter<{taskId: string, targetColumn: string}>

State:
- columns: KanbanColumn[] = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- tasks: Task[] = []
- draggedTask: Task | null = null
- filterCriteria: FilterOptions = {}
- isModalOpen: boolean = false
- selectedTask: Task | null = null
- error: string | null = null

Methods:
- ngOnInit() → Initialize component and load data
- loadDashboardData() → Fetch account and transaction data
- onTaskMove(taskId: string, targetColumn: string) → Handle task movement
- onTaskCreate(task: Task) → Handle new task creation
- onTaskUpdate(task: Task) → Handle task updates
- openTaskModal(task?: Task) → Open task creation/edit modal
- closeTaskModal() → Close task modal
- applyFilters(criteria: FilterOptions) → Apply task filters

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="board-header">
    <div class="breadcrumbs">
      <app-breadcrumb [items]="breadcrumbItems"></app-breadcrumb>
    </div>
    <div class="page-title">
      <h1>Kanban Board</h1>
      <p class="subtitle">Manage your tasks and workflow</p>
    </div>
    <div class="action-buttons">
      <app-button variant="primary" (click)="openTaskModal()">Add Task</app-button>
      <app-button variant="secondary">Export Report</app-button>
    </div>
  </div>
  
  <div class="board-controls">
    <app-search-field 
      placeholder="Search tasks..."
      (searchChange)="onSearchChange($event)">
    </app-search-field>
    <div class="filter-groups">
      <!-- Filter controls -->
    </div>
  </div>
  
  <div class="kanban-grid" *ngIf="!error; else errorTemplate">
    <app-kanban-column 
      *ngFor="let column of columns"
      [column]="column"
      [tasks]="getTasksForColumn(column.id)"
      [allowDrop]="true"
      (taskDropped)="onTaskMove($event.taskId, column.id)"
      (addTaskClicked)="openTaskModal()">
    </app-kanban-column>
  </div>
  
  <ng-template #errorTemplate>
    <div class="error-state" role="alert">
      <p>{{ error }}</p>
      <app-button (click)="loadDashboardData()">Retry</app-button>
    </div>
  </ng-template>
</div>

<app-task-modal
  [isOpen]="isModalOpen"
  [task]="selectedTask"
  [columns]="columns"
  (taskSubmitted)="onTaskSubmit($event)"
  (modalClosed)="closeTaskModal()">
</app-task-modal>

CSS SPECIFICATION

Layout:
- .kanban-board-container: flex column layout
- .kanban-grid: display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem;
- .board-header: flex row with space-between alignment
- .board-controls: flex row with gap for filters

Spacing:
- Container padding: 1.5rem
- Grid gap: 1rem between columns
- Header margin-bottom: 1rem

Component Styles:
- .kanban-board-container: full height container
- .board-header: sticky header with shadow
- .action-buttons: flex gap 0.5rem
- .error-state: centered content with error styling

Responsive:
- Desktop (>1024px): 3 columns, full sidebar
- Tablet (768px-1024px): 3 columns, collapsed sidebar
- Mobile (<768px): 1 column, hidden sidebar

API INTEGRATION

- loadDashboardData() → GET /api/youth-accounts/{id}/dashboard → Load account balance and spending limit
- loadTransactions() → GET /api/youth-accounts/{id}/transactions → Load transaction history as tasks
- createFundTransferTask() → POST /api/youth-accounts/{id}/fund-transfer → Create fund allocation task
- updateSpendingLimit() → PUT /api/youth-accounts/{id}/spending-limit → Update spending limit task

ERROR HANDLING

- API failure → show error template with retry button
- Validation error → display inline error messages
- Network timeout → show connection error with retry
- Invalid data → fallback to empty state

INTERACTION FLOW

User Action → UI → API
- Load page → ngOnInit() → loadDashboardData() → GET /dashboard → display data
- Add task → click Add Task → open modal → submit form → POST /fund-transfer → refresh tasks
- Move task → drag task → onTaskMove() → update local state → sync with backend
- Search tasks → type in search → filter tasks locally → update display

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: Individual column container with header and task list

Purpose: Represents a single column in the kanban board with drag-and-drop functionality

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)
- ButtonComponent (Add Task)
- BadgeComponent (task count)

Dependencies:
- DragDropModule
- CdkDragDrop

Library Components Used:
- ButtonComponent → Add task button
- BadgeComponent → Task count display
- IconComponent → Column actions

TypeScript Specification

Inputs:
- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean = true

Outputs:
- taskDropped: EventEmitter<{taskId: string, sourceColumn: string}>
- addTaskClicked: EventEmitter<string>

State:
- isDragOver: boolean = false
- isCollapsed: boolean = false

Methods:
- onDrop(event: CdkDragDrop<Task[]>) → Handle task drop
- onDragEnter() → Set drag over state
- onDragLeave() → Clear drag over state
- onAddTask() → Emit add task event
- toggleCollapse() → Toggle column collapse state

<div class="kanban-column" 
     [class.drag-over]="isDragOver"

    <div class="column-title" [attr.aria-labelledby]="column.id + '-header'">

  </div>

      <app-task-card
        *ngFor="let task of tasks; trackBy: trackByTaskId"

      <app-button

Spacing:
- Column padding: 1rem
- Task gap: 0.75rem

- .kanban-column: border, border-radius, background

INTERACTION FLOW

User Action → UI → Parent
- Drop task → onDrop() → emit taskDropped → parent handles API
- Add task → click Add Task → emit addTaskClicked → parent opens modal
- Click task → onTaskClick() → emit taskClicked → parent opens edit modal
