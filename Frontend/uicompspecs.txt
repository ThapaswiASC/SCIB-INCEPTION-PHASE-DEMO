### 4. COMPONENT IMPLEMENTATION

#### 4.1 KanbanBoardComponent

**Component Type:** Feature

**Mapped From A1:** Main kanban container with three-column layout

**Purpose:** Primary container for kanban board with drag-and-drop functionality

**Parent:** MainContentComponent

**Children:**
- KanbanColumnComponent (3 instances)
- MobileTabsComponent
- ModalComponent

**Dependencies:**
- KanbanService
- TaskService
- Angular CDK Drag Drop

**Library Components Used:**
- aava-button → Add task buttons
- aava-popup → Column options menu

**TypeScript Specification**

**Inputs:**
- youthAccountId: string
- initialColumns: KanbanColumn[]
- loading: boolean

**Outputs:**
- taskMoved: EventEmitter<{task: Task, fromColumn: string, toColumn: string}>
- taskCreated: EventEmitter<Task>
- taskUpdated: EventEmitter<Task>

**State:**
- columns: KanbanColumn[]
- draggedTask: Task | null
- isLoading: boolean
- error: string | null
- selectedTask: Task | null
- isModalOpen: boolean

**Methods:**
- ngOnInit() → Initialize component and load data
- loadKanbanData() → Fetch initial kanban data
- onTaskDrop(event: CdkDragDrop<Task[]>) → Handle drag-and-drop
- openTaskModal(columnId?: string) → Open task creation modal
- closeTaskModal() → Close modal and refresh data

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="page-header">
    <h1>Kanban Board</h1>
    <aava-button (click)="openTaskModal()">Add Task</aava-button>
  </div>
  
  <div class="mobile-tabs" *ngIf="isMobile">
    <mobile-tabs [columns]="columns"></mobile-tabs>
  </div>
  
  <div class="kanban-grid" cdkDropListGroup>
    <kanban-column 
      *ngFor="let column of columns" 
      [column]="column"
      [tasks]="column.tasks"
      (taskDropped)="onTaskDrop($event)"
      (addTaskClicked)="openTaskModal(column.id)">
    </kanban-column>
  </div>
  
  <modal-component 
    [isOpen]="isModalOpen"
    [mode]="selectedTask ? 'edit' : 'create'"
    [task]="selectedTask"
    (taskSubmitted)="onTaskSubmitted($event)"
    (modalClosed)="closeTaskModal()">
  </modal-component>
  
  <div *ngIf="error" class="error-message" role="alert">
    {{ error }}
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1rem
- min-height: 100vh

**Spacing:**
- padding: 1rem
- gap: 1rem between columns

**Component Styles:**
- .kanban-board-container
- .kanban-grid
- .page-header
- .error-message

**Responsive:**
- Desktop (>1024px): 3-column grid
- Tablet (768px-1024px): 2-column grid with tabs
- Mobile (<768px): single column with tabs

**API INTEGRATION**

- loadKanbanData() → GET /api/youth-accounts/{id}/dashboard → Fetch initial kanban data
- onTaskSubmitted() → POST /api/youth-accounts/{id}/fund-transfer → Create new task
- updateTaskStatus() → PUT /api/youth-accounts/{id}/spending-limit → Update task status

**ERROR HANDLING**

- API failure → show error message with retry option
- Validation error → inline form validation messages
- Network timeout → show offline indicator

**INTERACTION FLOW**

**User Action → UI → API**
- Load page → ngOnInit → GET /dashboard → render columns
- Drag task → onTaskDrop → PUT /spending-limit → update UI
- Add task → openTaskModal → form submit → POST /fund-transfer → refresh data

**Outputs:**

**State:**

**Methods:**

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-column" 
     [attr.aria-labelledby]="column.id + '-header'">

       cdkDropList
```

#### 4.5 ModalComponent

**Component Type:** Shared

**Mapped From A1:** Task creation and editing modal overlay

**Purpose:** Modal dialog for task creation, editing, and detailed view

**Parent:** KanbanBoardComponent

**Children:** None

**Dependencies:**
- ReactiveFormsModule
- TaskService

**Library Components Used:**
- aava-button → Action buttons
- aava-dropdown → Priority and status selectors

**TypeScript Specification**

**Inputs:**
- isOpen: boolean
- mode: 'create' | 'edit' | 'view'
- task?: Task
- columnId?: string

**Outputs:**
- taskSubmitted: EventEmitter<Task>
- modalClosed: EventEmitter<void>

**State:**
- taskForm: FormGroup
- isSubmitting: boolean
- validationErrors: {[key: string]: string}

**Methods:**
- ngOnInit() → Initialize form
- onSubmit() → Handle form submission
- onCancel() → Close modal without saving
- validateForm() → Validate form inputs

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="modal-overlay" *ngIf="isOpen" (click)="onCancel()">
  <div class="modal-content" (click)="$event.stopPropagation()" role="dialog" aria-modal="true">
    <div class="modal-header">
      <h2>{{ mode === 'create' ? 'Create Task' : 'Edit Task' }}</h2>
      <button class="close-btn" (click)="onCancel()" aria-label="Close">&times;</button>
    </div>
    
    <form [formGroup]="taskForm" (ngSubmit)="onSubmit()" class="modal-body">
      <div class="form-group">
        <label for="task-title">Title *</label>
        <input id="task-title" formControlName="title" type="text" required>
        <div class="error-message" *ngIf="validationErrors['title']">{{ validationErrors['title'] }}</div>
      </div>
      
      <div class="form-group">
        <label for="task-description">Description</label>
        <textarea id="task-description" formControlName="description" rows="3"></textarea>
      </div>
      
      <div class="form-row">
        <div class="form-group">
          <label for="task-priority">Priority</label>
          <aava-dropdown 
            id="task-priority"
            formControlName="priority"
            [options]="priorityOptions">
          </aava-dropdown>
        </div>
        
        <div class="form-group">
          <label for="task-assignee">Assignee</label>
          <aava-dropdown 
            id="task-assignee"
            formControlName="assignee"
            [options]="assigneeOptions">
          </aava-dropdown>
        </div>
      </div>
      
      <div class="form-group">
        <label for="task-tags">Tags</label>
        <input id="task-tags" formControlName="tags" type="text" placeholder="Enter tags separated by commas">
      </div>
    </form>
    
    <div class="modal-footer">
      <aava-button variant="secondary" (click)="onCancel()">Cancel</aava-button>
      <aava-button 
        variant="primary" 
        (click)="onSubmit()"
        [disabled]="taskForm.invalid || isSubmitting">
        {{ isSubmitting ? 'Saving...' : (mode === 'create' ? 'Create Task' : 'Update Task') }}
      </aava-button>
    </div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- position: fixed
- top: 0, left: 0
- width: 100vw, height: 100vh
- display: flex
- justify-content: center
- align-items: center

**Spacing:**
- modal-content: max-width 500px
- padding: 1.5rem
- gap: 1rem between form groups

**Component Styles:**
- .modal-overlay
- .modal-content
- .modal-header, .modal-body, .modal-footer
- .form-group, .form-row
- .error-message

**Responsive:**
- Mobile: full-screen modal
- Tablet/Desktop: centered dialog

**API INTEGRATION**

- onSubmit() → POST /api/youth-accounts/{id}/fund-transfer → Create task
- onSubmit() → PUT /api/youth-accounts/{id}/spending-limit → Update task

**ERROR HANDLING**

- Form validation → inline error messages
- API failure → show error notification
- Network timeout → retry mechanism

**INTERACTION FLOW**

**User Action → UI → API**
- Open modal → initialize form with task data
- Submit form → validate → API call → emit success → close modal
- Cancel → close modal without saving

### 8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly with three-column CSS Grid layout
- Do NOT create new components unnecessarily - reuse library components where possible
- Prefer reusable library components (aava-button, aava-popup, aava-autocomplete, aava-dropdown)
- HTML must be pseudo-code ONLY (NOT real framework code)
- Implement proper ARIA attributes for accessibility compliance
- Follow responsive design patterns: desktop (>1024px), tablet (768px-1024px), mobile (<768px)
- Use Angular CDK Drag Drop for task movement functionality
- Implement OnInit lifecycle hook for component initialization
- Add proper error handling with *ngIf directives for error states
- Use OnPush change detection strategy for better performance
- Implement trackBy functions for *ngFor loops to optimize rendering
- Follow Angular Material design patterns where applicable
- Ensure all components compile without errors and pass linting checks
- API endpoints are adapted from youth account management system
- Transform API responses to match kanban task data structure
- Implement proper TypeScript interfaces for type safety
- Use reactive forms for modal form handling
- Add proper loading states and error boundaries
- Implement keyboard navigation for accessibility
- Use semantic HTML elements for better screen reader support
