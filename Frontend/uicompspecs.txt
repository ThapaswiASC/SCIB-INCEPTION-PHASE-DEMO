### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main kanban board with three-column layout

Purpose: Display and manage tasks across To Do, In Progress, and Done columns with drag-and-drop functionality

Parent: KanbanPage

Children:
- TaskCardComponent (multiple instances)
- Button (Play UI - Add Task)
- SearchField (Play UI - Filter tasks)

Dependencies:
- React DnD for drag and drop
- Play UI components
- Task management API

Library Components Used:
- Button → Add Task actions
- SearchField → Task filtering
- Card → Column containers

TypeScript Specification:

Inputs:
- tasks: Task[] (Array of task objects)
- onTaskMove: (taskId: string, newStatus: string) => void
- onTaskCreate: (task: Task) => void
- onTaskUpdate: (taskId: string, updates: Partial<Task>) => void
- onTaskDelete: (taskId: string) => void

Outputs:
- taskMoved (payload: {taskId: string, fromColumn: string, toColumn: string})
- taskCreated (payload: Task)
- taskUpdated (payload: {taskId: string, updates: Partial<Task>})
- taskDeleted (payload: {taskId: string})

State:
- filteredTasks: Task[]
- searchQuery: string
- draggedTask: Task | null
- isLoading: boolean
- error: string | null

Methods:
- handleTaskDrop(taskId: string, newStatus: string) → Updates task status
- handleSearch(query: string) → Filters tasks
- handleTaskCreate() → Opens task creation modal
- refreshTasks() → Fetches latest tasks from API

HTML STRUCTURE (PSEUDO CODE ONLY):

<div className="kanban-board">
  <div className="kanban-header">
    <SearchField placeholder="Filter tasks..." onSearch={handleSearch} />
    <Button variant="primary" onClick={handleTaskCreate}>Add Task</Button>
  </div>
  
  <div className="kanban-columns">
    <div className="kanban-column" data-status="todo">
      <div className="column-header">
        <h3>To Do</h3>
        <Badge count={todoTasks.length} />
      </div>
      <div className="column-content">
        {todoTasks.map(task => 
          <TaskCardComponent key={task.id} task={task} onUpdate={handleTaskUpdate} />
        )}
      </div>
    </div>
    
    <div className="kanban-column" data-status="inprogress">
      <div className="column-header">
        <h3>In Progress</h3>
        <Badge count={inProgressTasks.length} />
      </div>
      <div className="column-content">
        {inProgressTasks.map(task => 
          <TaskCardComponent key={task.id} task={task} onUpdate={handleTaskUpdate} />
        )}
      </div>
    </div>
    
    <div className="kanban-column" data-status="done">
      <div className="column-header">
        <h3>Done</h3>
        <Badge count={doneTasks.length} />
      </div>
      <div className="column-content">
        {doneTasks.map(task => 
          <TaskCardComponent key={task.id} task={task} onUpdate={handleTaskUpdate} />
        )}
      </div>
    </div>
  </div>
</div>

CSS SPECIFICATION:

Layout:
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1rem
- min-height: 100vh

Spacing:
- padding: 1rem
- column gap: 1rem
- card margin: 0.5rem 0

Component Styles:
- .kanban-board: main container with grid layout
- .kanban-column: flex column with overflow handling
- .column-header: flex row with title and badge
- .column-content: scrollable area for tasks

Responsive:
- Desktop (>1024px): 3 columns
- Tablet (768px-1024px): 2 columns, scroll horizontally
- Mobile (<768px): 1 column, stack vertically

API INTEGRATION:
- getTasks() → GET /api/tasks → Fetch all tasks
- createTask(task) → POST /api/tasks → Create new task
- updateTask(id, updates) → PUT /api/tasks/{id} → Update task
- deleteTask(id) → DELETE /api/tasks/{id} → Delete task

ERROR HANDLING:
- API failure → show error snackbar
- Network timeout → retry mechanism
- Validation error → inline error messages
- Drag drop failure → revert to original position

INTERACTION FLOW:
User Action → UI → API
- Drag task → update UI optimistically → PUT /api/tasks/{id}
- Search tasks → filter local state → no API call
- Add task → open modal → POST /api/tasks → refresh list

### 4.2 TaskCardComponent

Component Type: Feature

Mapped From A1: Individual task card within kanban columns

Purpose: Display task information and provide interaction controls

Parent: KanbanBoardComponent

Children:
- Avatar (Play UI - Assignee)
- Badge (Play UI - Priority/Status)
- Button (Play UI - Actions)

Dependencies:
- Play UI components
- Date formatting utilities

Library Components Used:
- Avatar → Task assignee
- Badge → Priority indicator
- Button → Edit/Delete actions
- Card → Container wrapper

TypeScript Specification:

Inputs:
- task: Task (Task object with id, title, description, assignee, priority, dueDate)
- onUpdate: (taskId: string, updates: Partial<Task>) => void
- onDelete: (taskId: string) => void
- isDragging: boolean

Outputs:
- taskClicked (payload: Task)
- taskUpdated (payload: {taskId: string, updates: Partial<Task>})
- taskDeleted (payload: {taskId: string})

State:
- isHovered: boolean
- showActions: boolean

Methods:
- handleClick() → Opens task details
- handleEdit() → Opens edit modal
- handleDelete() → Confirms and deletes task

HTML STRUCTURE (PSEUDO CODE ONLY):

<Card className="task-card" draggable onDragStart={handleDragStart}>
  <div className="task-header">
    <div className="task-title">{task.title}</div>
    <Badge variant={getPriorityVariant(task.priority)}>{task.priority}</Badge>
  </div>
  
  <div className="task-description">
    {task.description}
  </div>
  
  <div className="task-footer">
    <div className="task-assignee">
      <Avatar src={task.assignee.avatar} size="small" />
      <span>{task.assignee.name}</span>
    </div>
    
    <div className="task-actions">
      <Button variant="ghost" size="small" onClick={handleEdit}>Edit</Button>
      <Button variant="ghost" size="small" onClick={handleDelete}>Delete</Button>
    </div>
  </div>
  
  {task.dueDate && (
    <div className="task-due-date">
      Due: {formatDate(task.dueDate)}
    </div>
  )}
</Card>

CSS SPECIFICATION:

Layout:
- display: flex
- flex-direction: column
- gap: 0.5rem

Spacing:
- padding: 1rem
- margin: 0.5rem 0

Component Styles:
- .task-card: rounded corners, shadow, hover effects
- .task-header: flex row with title and priority
- .task-footer: flex row with assignee and actions

Responsive:
- Mobile: stack elements vertically
- Desktop: maintain flex layout

API INTEGRATION:
- updateTask(id, updates) → PUT /api/tasks/{id} → Update task details
- deleteTask(id) → DELETE /api/tasks/{id} → Remove task

ERROR HANDLING:
- Update failure → revert changes, show error
- Delete failure → show confirmation error
- Network error → retry option

INTERACTION FLOW:
User Action → UI → API
- Click card → open task details modal → no API call
- Edit task → open edit form → PUT /api/tasks/{id}
- Delete task → confirm dialog → DELETE /api/tasks/{id}

### 4.4 SidebarComponent

Component Type: Layout

Mapped From A1: Navigation sidebar with menu items and filters

Purpose: Provide navigation between different views and filtering options

Parent: KanbanPage

Children:
- Button (Play UI - Navigation items)
- Badge (Play UI - Count indicators)

Dependencies:
- Router for navigation
- Filter state management
- Play UI components

Library Components Used:
- Button → Navigation menu items
- Badge → Count indicators for filters

TypeScript Specification:

Inputs:
- activeRoute: string
- taskCounts: {todo: number, inProgress: number, done: number}
- onNavigate: (route: string) => void
- onFilterChange: (filter: FilterOptions) => void

Outputs:
- navigationChanged (payload: {route: string})
- filterApplied (payload: FilterOptions)

State:
- isCollapsed: boolean
- activeFilters: FilterOptions

Methods:
- handleNavigation(route) → Changes active route
- handleFilterToggle(filter) → Applies/removes filter
- handleSidebarToggle() → Collapses/expands sidebar

<aside className="sidebar">
