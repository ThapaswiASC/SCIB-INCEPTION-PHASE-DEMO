### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for three-column Kanban board layout

Purpose: Primary container managing kanban board state, task operations, and column interactions

Parent: MainContainerComponent

Children:
- KanbanColumnComponent (3 instances)
- CreateTaskModalComponent
- AddTaskButtonComponent

Dependencies:
- TaskService (API operations)
- KanbanService (board state)
- AuthService (user context)

Library Components Used:
- Card → Board container
- Button → Action buttons
- Modal → Task creation
- Badge → Status indicators

TypeScript Specification

Inputs:
- boardId: string (board identifier)
- userId: string (current user)
- permissions: Permission[] (user permissions)

Outputs:
- onTaskCreated: EventEmitter<Task> (new task created)
- onTaskMoved: EventEmitter<TaskMoveEvent> (task moved between columns)
- onBoardUpdated: EventEmitter<Board> (board configuration changed)

State:
- columns: KanbanColumn[] (column configuration)
- tasks: Task[] (all tasks)
- loading: boolean (loading state)
- error: string | null (error message)
- selectedTask: Task | null (currently selected task)
- draggedTask: Task | null (task being dragged)
- showCreateModal: boolean (modal visibility)

Methods:
- ngOnInit() → Initialize board data
- loadTasks() → Fetch tasks from API
- createTask(task: CreateTaskRequest) → Create new task
- moveTask(taskId: string, columnId: string) → Move task between columns
- openCreateModal() → Show task creation modal
- closeCreateModal() → Hide task creation modal

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="board-header">
    <h1>Kanban Board</h1>
    <Button variant="primary" (click)="openCreateModal()">Add Task</Button>
  </div>
  
  <div class="board-content" *ngIf="!loading && !error">
    <div class="columns-container">
      <KanbanColumnComponent 
        *ngFor="let column of columns" 
        [column]="column"
        [tasks]="getTasksForColumn(column.id)"
        (onTaskDrop)="moveTask($event.taskId, column.id)"
        (onAddTask)="openCreateModal()"
      />
    </div>
  </div>
  
  <div class="error-state" *ngIf="error">
    <p>{{ error }}</p>
    <Button (click)="loadTasks()">Retry</Button>
  </div>
  
  <div class="loading-state" *ngIf="loading">
    <p>Loading board...</p>
  </div>
  
  <CreateTaskModalComponent
    [isOpen]="showCreateModal"
    [columns]="columns"
    (onSubmit)="createTask($event)"
    (onCancel)="closeCreateModal()"
  />
</div>

CSS SPECIFICATION

Layout:
- display: grid
- grid-template-rows: auto 1fr
- height: 100vh
- .columns-container: display: grid, grid-template-columns: repeat(3, 1fr), gap: 1rem

Spacing:
- padding: 1rem
- gap: 1rem between columns
- margin: 0.5rem for headers

Component Styles:
- .kanban-board-container: background: var(--surface-color)
- .board-header: display: flex, justify-content: space-between, align-items: center
- .error-state: text-align: center, color: var(--error-color)

Responsive:
- Desktop (>1024px): 3 columns side by side
- Tablet (768px-1024px): 2 columns, third wraps
- Mobile (<768px): single column stack

API INTEGRATION

- loadTasks() → GET /api/v1/tasks → Load all tasks for board
- createTask() → POST /api/v1/tasks → Create new task
- moveTask() → PUT /api/v1/tasks/{id} → Update task column

ERROR HANDLING

- API failure → show error message with retry button
- Network timeout → show connection error
- Validation error → display inline validation messages
- Task creation failure → show snackbar notification

INTERACTION FLOW

User Action → UI → API
- Click "Add Task" → open CreateTaskModalComponent → submit form → POST /api/v1/tasks → refresh board
- Drag task → update UI optimistically → PUT /api/v1/tasks/{id} → confirm or rollback
- Load board → show loading state → GET /api/v1/tasks → populate columns

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: Individual column container with header and task list

Purpose: Display column header, task list, and handle drop operations

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)
- AddTaskButtonComponent

Dependencies:
- DragDropService (drag and drop)

Library Components Used:
- Card → Column container
- Button → Add task button
- Badge → Task count indicator

TypeScript Specification

Inputs:
- column: KanbanColumn (column configuration)
- tasks: Task[] (tasks in this column)
- maxTasks: number | null (maximum tasks allowed)
- allowDrop: boolean (whether drops are allowed)

Outputs:
- onTaskDrop: EventEmitter<TaskDropEvent> (task dropped in column)
- onAddTask: EventEmitter<string> (add task to column)
- onColumnConfig: EventEmitter<string> (configure column)

State:
- isDropTarget: boolean (currently accepting drops)
- taskCount: number (number of tasks)
- isOverLimit: boolean (exceeds max tasks)

Methods:
- onDragOver(event: DragEvent) → Handle drag over
- onDrop(event: DragEvent) → Handle task drop
- addTask() → Emit add task event
- configureColumn() → Open column configuration

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-column" 
     [class.drop-target]="isDropTarget"
     [class.over-limit]="isOverLimit"
     (dragover)="onDragOver($event)"
     (drop)="onDrop($event)">
  
  <div class="column-header" [attr.aria-labelledby]="column.id + '-header'">
    <h2 [id]="column.id + '-header'">{{ column.title }}</h2>
    <Badge [text]="taskCount.toString()" variant="secondary" />
    <Button variant="ghost" size="small" (click)="configureColumn()">⚙️</Button>
  </div>
  
  <div class="column-content">
    <TaskCardComponent
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="true"
      (onTaskClick)="selectTask($event)"
      (onDragStart)="startDrag($event)"
    />
    
    <AddTaskButtonComponent
      [columnId]="column.id"
      (onClick)="addTask()"
    />
  </div>
  
  <div class="drop-indicator" *ngIf="isDropTarget">
    Drop task here
  </div>
</div>

CSS SPECIFICATION

Layout:
- display: flex
- flex-direction: column
- min-height: 500px
- max-height: calc(100vh - 200px)
- overflow-y: auto

Spacing:
- padding: 1rem
- gap: 0.5rem between tasks
- margin-bottom: 1rem

Component Styles:
- .kanban-column: background: var(--column-bg), border-radius: 8px, border: 1px solid var(--border-color)
- .column-header: display: flex, justify-content: space-between, align-items: center
- .drop-target: border-color: var(--primary-color), background: var(--drop-highlight)
- .over-limit: border-color: var(--warning-color)

Responsive:
- Desktop: fixed width 300px
- Tablet: flexible width with min 250px
- Mobile: full width

API INTEGRATION

- None (receives data from parent KanbanBoardComponent)

ERROR HANDLING

- Drop validation → show error if invalid drop
- Task limit exceeded → visual warning and prevent drop
- Column configuration error → show error message

INTERACTION FLOW

User Action → UI → API
- Drag task over column → highlight drop zone → validate drop target
- Drop task → emit onTaskDrop → parent handles API call
- Click add task → emit onAddTask → parent opens modal
- Configure column → emit onColumnConfig → open configuration modal
