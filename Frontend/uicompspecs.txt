### 8. NOTES & CONSTRAINTS

**Implementation Guidelines:**
- HTML must follow Agent-1 structure strictly (flex layout with 260px sidebar)
- Do NOT create new components unnecessarily - maximize Play UI component reuse
- Prefer reusable library components over custom implementations
- HTML must be pseudo-code ONLY (not full framework code)
- CSS should include only key classes and layout rules, not full stylesheets

**Technical Constraints:**
- Angular framework with @Component decorator as per Jira requirements
- CSS Grid layout for three-column Kanban structure
- ARIA attributes required for accessibility compliance
- Responsive breakpoints: mobile (<768px), tablet (768-1024px), desktop (>1024px)
- Drag-and-drop functionality using React DnD or Angular CDK

**Performance Considerations:**
- Implement virtual scrolling for large task lists
- Use React.memo or Angular OnPush for task card optimization
- Debounce search input to reduce API calls
- Implement optimistic updates for drag-and-drop operations

**Accessibility Requirements:**
- Semantic HTML structure with proper heading hierarchy
- ARIA labels for drag-and-drop interactions
- Keyboard navigation support for all interactive elements
- Screen reader announcements for task status changes
- High contrast mode support

**Error Handling Strategy:**
- API failures: Show user-friendly error messages with retry options
- Network issues: Implement offline mode with local storage
- Validation errors: Inline form validation with clear error messages
- Drag-and-drop errors: Revert UI state and show appropriate feedback

**Security Considerations:**
- Input sanitization for task titles and descriptions
- Authentication checks before API calls
- CSRF protection for state-changing operations
- XSS prevention in user-generated content
