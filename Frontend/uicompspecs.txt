4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for the three-column kanban layout

Purpose: Orchestrates the entire kanban board functionality including drag-and-drop, task management, and column coordination

Parent: KanbanPageComponent

Children:
- KanbanColumnComponent (3 instances)
- TaskCreationModalComponent
- TaskDetailModalComponent

Dependencies:
- KanbanService
- TaskService
- DragDropModule

Library Components Used:
- Card → Container wrapper
- LoadingSpinner → Loading states

TypeScript Specification

Inputs:
- boardId: string (board identifier)
- initialColumns: KanbanColumn[] (initial column configuration)
- readonly: boolean (disable editing)

Outputs:
- taskMoved: EventEmitter<TaskMoveEvent> (task position changes)
- taskCreated: EventEmitter<Task> (new task creation)
- taskUpdated: EventEmitter<Task> (task modifications)
- taskDeleted: EventEmitter<string> (task removal)

State:
- columns: KanbanColumn[] (current column data)
- tasks: Task[] (all tasks in board)
- draggedTask: Task | null (currently dragged task)
- isLoading: boolean (loading state)
- error: string | null (error messages)
- showCreateModal: boolean (modal visibility)
- showDetailModal: boolean (detail modal visibility)
- selectedTask: Task | null (selected task for editing)

Methods:
- ngOnInit() → Initialize board data
- loadBoardData() → Fetch columns and tasks
- onTaskDrop(event: CdkDragDrop<Task[]>) → Handle drag-and-drop
- openCreateModal(columnId: string) → Show task creation modal
- openTaskDetail(task: Task) → Show task detail modal
- handleTaskCreate(task: Task) → Process new task creation
- handleTaskUpdate(task: Task) → Process task updates
- handleTaskDelete(taskId: string) → Process task deletion

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board" role="region" aria-label="Kanban Board">
  <div class="board-header">
    <h2>Kanban Board</h2>
    <button (click)="refreshBoard()">Refresh</button>
  </div>
  
  <div class="columns-container" cdkDropListGroup>
    <kanban-column 
      *ngFor="let column of columns" 
      [column]="column"
      [tasks]="getTasksForColumn(column.id)"
      [allowDrop]="!readonly"
      (taskDrop)="onTaskDrop($event)"
      (addTask)="openCreateModal(column.id)">
    </kanban-column>
  </div>
  
  <task-creation-modal
    [isOpen]="showCreateModal"
    [columnId]="selectedColumnId"
    (taskCreated)="handleTaskCreate($event)"
    (cancelled)="showCreateModal = false">
  </task-creation-modal>
  
  <task-detail-modal
    [isOpen]="showDetailModal"
    [task]="selectedTask"
    (taskUpdated)="handleTaskUpdate($event)"
    (taskDeleted)="handleTaskDelete($event)"
    (cancelled)="showDetailModal = false">
  </task-detail-modal>
  
  <div *ngIf="isLoading" class="loading-overlay">
    <loading-spinner></loading-spinner>
  </div>
  
  <div *ngIf="error" class="error-message" role="alert">
    {{ error }}
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-board { display: flex; flex-direction: column; height: 100vh; }
- .columns-container { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; flex: 1; }

Spacing:
- padding: 1rem
- gap: 1rem between columns

Component Styles:
- .board-header { display: flex; justify-content: space-between; align-items: center; }
- .loading-overlay { position: absolute; top: 0; left: 0; right: 0; bottom: 0; }
- .error-message { background: var(--error-bg); color: var(--error-text); padding: 1rem; }

Responsive:
- @media (max-width: 768px) { .columns-container { grid-template-columns: 1fr; } }
- @media (min-width: 769px) and (max-width: 1024px) { .columns-container { grid-template-columns: repeat(2, 1fr); } }

API INTEGRATION

- loadBoardData() → GET /forms/sessions/{sessionId} → Load board state
- handleTaskCreate() → PUT /forms/sessions/{sessionId} → Save new task
- handleTaskUpdate() → PUT /forms/sessions/{sessionId} → Update task
- handleTaskDelete() → PUT /forms/sessions/{sessionId} → Remove task

ERROR HANDLING

- API failure → show error message in error-message div
- Validation error → display inline validation messages
- Network timeout → show retry button with error message

INTERACTION FLOW

User Action → UI → API
- Load board → ngOnInit → GET /forms/sessions/{sessionId} → populate columns and tasks
- Drag task → onTaskDrop → PUT /forms/sessions/{sessionId} → update task position
- Click add task → openCreateModal → show TaskCreationModalComponent
- Click task → openTaskDetail → show TaskDetailModalComponent

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: Individual column container with header and task list

Purpose: Manages individual kanban column display, task list, and drop zone functionality

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)
- AddTaskButtonComponent

Dependencies:
- CdkDropList
- CdkDrag

Library Components Used:
- Card → Column container
- Badge → Task count indicator

TypeScript Specification

Inputs:
- column: KanbanColumn (column configuration)
- tasks: Task[] (tasks in this column)
- allowDrop: boolean (enable/disable drop functionality)

Outputs:
- taskDrop: EventEmitter<CdkDragDrop<Task[]>> (task dropped in column)
- addTask: EventEmitter<string> (add task button clicked)
- taskClick: EventEmitter<Task> (task card clicked)

State:
- isDragOver: boolean (drag over state)
- taskCount: number (number of tasks in column)
- isCollapsed: boolean (column collapsed state)

Methods:
- ngOnInit() → Initialize column
- onTaskDrop(event: CdkDragDrop<Task[]>) → Handle task drop
- onAddTaskClick() → Emit add task event
- onTaskCardClick(task: Task) → Emit task click event
- toggleCollapse() → Toggle column collapse state

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-column" 
     [class.drag-over]="isDragOver"
     cdkDropList
     [cdkDropListData]="tasks"
     (cdkDropListDropped)="onTaskDrop($event)">
  
  <div class="column-header">
    <h3 [id]="column.id + '-header'" class="column-title">
      {{ column.title }}
    </h3>
    <badge [count]="taskCount" variant="secondary"></badge>
    <button class="collapse-btn" (click)="toggleCollapse()">
      {{ isCollapsed ? 'Expand' : 'Collapse' }}
    </button>
  </div>
  
  <div class="column-content" [class.collapsed]="isCollapsed">
    <div class="tasks-list" 
         role="list" 
         [attr.aria-labelledby]="column.id + '-header'">
      
      <task-card
        *ngFor="let task of tasks; trackBy: trackByTaskId"
        [task]="task"
        [draggable]="allowDrop"
        (taskClick)="onTaskCardClick(task)"
        cdkDrag
        role="listitem">
      </task-card>
      
      <div *ngIf="tasks.length === 0" class="empty-state">
        <p>No tasks in this column</p>
      </div>
    </div>
    
    <add-task-button
      [columnId]="column.id"
      (addTask)="onAddTaskClick()"
      [disabled]="!allowDrop">
    </add-task-button>
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-column { display: flex; flex-direction: column; min-height: 400px; }
- .column-content { flex: 1; display: flex; flex-direction: column; }
- .tasks-list { flex: 1; overflow-y: auto; }

Spacing:
- padding: 1rem
- gap: 0.5rem between tasks

Component Styles:
- .column-header { display: flex; align-items: center; justify-content: space-between; }
- .drag-over { background-color: var(--drop-zone-bg); border: 2px dashed var(--primary-color); }
- .collapsed .column-content { height: 0; overflow: hidden; }
- .empty-state { text-align: center; color: var(--text-muted); padding: 2rem; }

Responsive:
- @media (max-width: 768px) { .kanban-column { min-height: 300px; } }

API INTEGRATION

- No direct API calls (managed by parent KanbanBoardComponent)

ERROR HANDLING

- Invalid drop → show visual feedback and revert
- Empty column state → display helpful empty state message

INTERACTION FLOW

User Action → UI → API
- Drop task → onTaskDrop → emit to parent → parent handles API call
- Click add task → onAddTaskClick → emit to parent → parent opens modal
- Click task card → onTaskCardClick → emit to parent → parent opens detail modal

### 4.3 TaskCardComponent

Component Type: Feature

Mapped From A1: Individual task display with metadata and actions

Purpose: Displays task information including title, assignee, priority, and provides interaction capabilities

Parent: KanbanColumnComponent

Children: None

Dependencies:
- CdkDrag

Library Components Used:
- Card → Task container
- Avatar → Assignee avatar
- Badge → Priority indicator

TypeScript Specification

Inputs:
- task: Task (task data)
- draggable: boolean (enable drag functionality)
- clickable: boolean (enable click interaction)
- selected: boolean (highlight as selected)

Outputs:
- taskClick: EventEmitter<Task> (task clicked)
- taskEdit: EventEmitter<Task> (edit action triggered)

State:
- isDragging: boolean (currently being dragged)
- isHovered: boolean (mouse hover state)

Methods:
- onTaskClick() → Emit task click event
- onEditClick(event: Event) → Emit edit event and stop propagation
- getPriorityColor() → Return priority-based color
- getAssigneeInitials() → Extract initials from assignee name

HTML STRUCTURE (PSEUDO CODE ONLY)

<card class="task-card"
      [class.dragging]="isDragging"
      [class.selected]="selected"
      [class.clickable]="clickable"
      (click)="onTaskClick()"
      (mouseenter)="isHovered = true"
      (mouseleave)="isHovered = false">
  
  <div class="task-header">
    <span class="task-id">{{ task.id }}</span>
    <badge [variant]="getPriorityColor()" [text]="task.priority"></badge>
  </div>
  
  <div class="task-content">
    <h4 class="task-title">{{ task.title }}</h4>
    <p class="task-description" *ngIf="task.description">
      {{ task.description | truncate:100 }}
    </p>
  </div>
  
  <div class="task-footer">
    <div class="task-assignee" *ngIf="task.assignee">
      <avatar 
        [initials]="getAssigneeInitials()"
        [title]="task.assignee.name"
        size="small">
      </avatar>
      <span class="assignee-name">{{ task.assignee.name }}</span>
    </div>
    
    <div class="task-actions" *ngIf="isHovered">
      <button class="edit-btn" 
              (click)="onEditClick($event)"
              aria-label="Edit task">
        Edit
      </button>
    </div>
  </div>
  
  <div class="task-metadata">
    <span class="due-date" *ngIf="task.dueDate">
      Due: {{ task.dueDate | date:'short' }}
    </span>
    <span class="created-date">
      Created: {{ task.createdAt | date:'short' }}
    </span>
  </div>
</card>

User Action → UI → API
- Click task → onTaskClick → emit to parent → parent opens detail modal

### 4.4 TaskCreationModalComponent

Component Type: Feature

Mapped From A1: Modal for creating new tasks

Purpose: Provides form interface for creating new tasks with validation and submission

Parent: KanbanBoardComponent

Children: None

Dependencies:
- ReactiveFormsModule
- ValidationService

Library Components Used:
- Modal → Container wrapper
- Input → Form fields
- Select → Dropdown selections
- Textarea → Description field
- Button → Action buttons

TypeScript Specification

Inputs:
- isOpen: boolean (modal visibility)
- columnId: string (target column for new task)
- availableAssignees: User[] (assignee options)
- initialData: Partial<Task> (pre-filled data)

Outputs:
- taskCreated: EventEmitter<Task> (new task created)
- cancelled: EventEmitter<void> (modal cancelled)

State:
- taskForm: FormGroup (reactive form)
- validationErrors: ValidationError[] (form validation errors)
- isSubmitting: boolean (submission state)
- showAdvanced: boolean (advanced options visibility)

Methods:
- ngOnInit() → Initialize form
- buildForm() → Create reactive form structure
- onSubmit() → Handle form submission
- onCancel() → Handle modal cancellation
- validateForm() → Perform form validation
- resetForm() → Reset form to initial state
- toggleAdvanced() → Show/hide advanced options
