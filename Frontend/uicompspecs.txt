4. COMPONENT IMPLEMENTATION

### 4.1 TaskCreationFormComponent

Component Type: Feature

Mapped From A1: TaskCreationFormComponent (Main)

Purpose: Main form component for creating tasks with Angular reactive forms, real-time validation, and API integration

Parent: KanbanBoardComponent

Children:
- FormFieldComponent (title, description, priority, due date)
- ValidationMessageComponent (error display)
- CharacterCounterComponent (title and description counters)
- LoadingSpinnerComponent (API call loading state)
- ButtonComponent (Create Task, Cancel)

Dependencies:
- @angular/forms (FormBuilder, Validators)
- TaskService (API integration)
- ValidationService (custom validation rules)

Library Components Used:
- ButtonComponent → Create Task and Cancel buttons
- DropdownComponent → Priority selection
- DatePickerComponent → Due date selection

TypeScript Specification

Inputs:
- isVisible: boolean (modal visibility)
- initialData: Partial<TaskFormData> (pre-populate form)

Outputs:
- taskCreated: EventEmitter<Task> (successful task creation)
- formCancelled: EventEmitter<void> (form cancellation)

State:
- taskForm: FormGroup (reactive form instance)
- isLoading: boolean (API call state)
- validationErrors: ValidationErrors (form validation errors)
- characterCounts: { title: number, description: number } (character counters)

Methods:
- initializeForm() → void (setup reactive form with validators)
- onSubmit() → void (handle form submission)
- onCancel() → void (handle form cancellation)
- validateField(fieldName: string) → boolean (field-level validation)
- updateCharacterCount(field: string, value: string) → void (update counters)
- resetForm() → void (reset form to initial state)

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="task-creation-modal" [class.visible]="isVisible">
  <div class="modal-backdrop" (click)="onCancel()"></div>
  <div class="modal-content">
    <div class="modal-header">
      <h2>Create New Task</h2>
      <button class="close-btn" (click)="onCancel()">×</button>
    </div>
    
    <form [formGroup]="taskForm" (ngSubmit)="onSubmit()" class="task-form">
      <FormFieldComponent
        label="Task Title"
        type="text"
        formControlName="title"
        [required]="true"
        [maxLength]="255">
      </FormFieldComponent>
      
      <CharacterCounterComponent
        [currentLength]="characterCounts.title"
        [maxLength]="255">
      </CharacterCounterComponent>
      
      <FormFieldComponent
        label="Description"
        type="textarea"
        formControlName="description"
        [maxLength]="2000">
      </FormFieldComponent>
      
      <CharacterCounterComponent
        [currentLength]="characterCounts.description"
        [maxLength]="2000">
      </CharacterCounterComponent>
      
      <FormFieldComponent
        label="Priority"
        type="dropdown"
        formControlName="priority"
        [required]="true"
        [options]="priorityOptions">
      </FormFieldComponent>
      
      <FormFieldComponent
        label="Due Date"
        type="date"
        formControlName="dueDate"
        [required]="true">
      </FormFieldComponent>
      
      <ValidationMessageComponent
        *ngIf="taskForm.invalid && taskForm.touched"
        [message]="getFormErrorMessage()"
        type="error">
      </ValidationMessageComponent>
      
      <div class="form-actions">
        <ButtonComponent
          label="Cancel"
          variant="secondary"
          (userClick)="onCancel()">
        </ButtonComponent>
        
        <ButtonComponent
          label="Create Task"
          variant="primary"
          [disabled]="taskForm.invalid"
          [processing]="isLoading"
          (userClick)="onSubmit()">
        </ButtonComponent>
      </div>
    </form>
    
    <LoadingSpinnerComponent
      [isVisible]="isLoading"
      message="Creating task...">
    </LoadingSpinnerComponent>
  </div>
</div>

CSS SPECIFICATION

Layout:
- .task-creation-modal: fixed positioning, full viewport overlay
- .modal-content: centered flex container, max-width 600px
- .task-form: flex column layout with gap spacing

Spacing:
- Modal padding: 2rem
- Form field gaps: 1.5rem
- Button group gap: 1rem

Component Styles:
- .modal-backdrop: rgba(0,0,0,0.5) overlay
- .modal-content: white background, border-radius 8px, box-shadow
- .form-actions: flex row, justify-content space-between

Responsive:
- Mobile: full-width modal, reduced padding
- Tablet: 90% width, maintained spacing
- Desktop: fixed 600px width

API INTEGRATION

- onSubmit() → POST /tasks → TaskService.createTask(taskData)
- Form validation → Real-time validation using Angular validators
- Error handling → Display API errors via ValidationMessageComponent

ERROR HANDLING

- API failure → Show error message in ValidationMessageComponent
- Network timeout → Display "Connection timeout" message
- Validation errors → Inline field validation messages
- Form submission errors → Global form error message

INTERACTION FLOW

User Action → UI → API
- Click "Add Task" button → Open modal → Initialize reactive form
- Enter title → Update form control → Trigger validation → Update character counter
- Enter description → Update form control → Trigger validation → Update character counter
- Select priority → Update dropdown → Trigger required validation
- Select due date → Update date picker → Trigger future date validation
- Click "Create Task" → Validate form → Show loading → Call TaskService.createTask() → Handle response → Close modal on success

### 4.2 FormFieldComponent

Component Type: Shared

Mapped From A1: FormFieldComponent

Purpose: Reusable form field wrapper with validation display and accessibility features

Parent: TaskCreationFormComponent

Children:
- Native input/textarea/select elements
- ValidationMessageComponent (for field-level errors)

Dependencies:
- @angular/forms (ControlValueAccessor)

Library Components Used:
- InputFieldComponent → Text inputs
- TextAreaComponent → Description field
- DropdownComponent → Priority selection

TypeScript Specification

Inputs:
- label: string (field label)
- type: 'text' | 'textarea' | 'dropdown' | 'date' (field type)
- required: boolean (required field indicator)
- maxLength: number (character limit)
- placeholder: string (placeholder text)
- options: any[] (dropdown options)

Outputs:
- valueChange: EventEmitter<any> (value change events)
- validationChange: EventEmitter<boolean> (validation state changes)

State:
- value: any (current field value)
- isValid: boolean (validation state)
- isTouched: boolean (user interaction state)
- errorMessage: string (current error message)

Methods:
- onValueChange(value: any) → void (handle value changes)
- validateField() → boolean (perform field validation)
- getErrorMessage() → string (get current error message)

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="form-field" [class.error]="!isValid && isTouched">
  <label class="field-label" [class.required]="required">
    {{ label }}
  </label>
  
  <div class="field-wrapper">
    <input *ngIf="type === 'text'"
           class="field-input"
           [value]="value"
           [placeholder]="placeholder"
           [maxlength]="maxLength"
           (input)="onValueChange($event.target.value)"
           (blur)="isTouched = true" />
    
    <textarea *ngIf="type === 'textarea'"
              class="field-textarea"
              [value]="value"
              [placeholder]="placeholder"
              [maxlength]="maxLength"
              (input)="onValueChange($event.target.value)"
              (blur)="isTouched = true">
    </textarea>
    
    <select *ngIf="type === 'dropdown'"
            class="field-select"
            [value]="value"
            (change)="onValueChange($event.target.value)"
            (blur)="isTouched = true">
      <option value="">Select {{ label }}</option>
      <option *ngFor="let option of options" [value]="option.value">
        {{ option.label }}
      </option>
    </select>
    
    <input *ngIf="type === 'date'"
           type="date"
           class="field-date"
           [value]="value"
           (change)="onValueChange($event.target.value)"
           (blur)="isTouched = true" />
  </div>
  
  <ValidationMessageComponent
    *ngIf="!isValid && isTouched"
    [message]="errorMessage"
    type="error">
  </ValidationMessageComponent>
</div>

CSS SPECIFICATION

Layout:
- .form-field: flex column layout
- .field-wrapper: relative positioning for validation styling

Spacing:
- Label margin-bottom: 0.5rem
- Field padding: 0.75rem
- Error message margin-top: 0.25rem

Component Styles:
- .field-label.required::after: "*" content, red color
- .field-input, .field-textarea, .field-select: border, border-radius, focus styles
- .form-field.error: red border color

Responsive:
- Mobile: full-width fields
- Desktop: consistent field sizing

API INTEGRATION

- No direct API calls
- Validation rules applied based on API schema requirements

ERROR HANDLING

- Field validation errors → Display inline error messages
- Required field validation → Show "This field is required"
- Max length validation → Show "Maximum X characters allowed"

INTERACTION FLOW

User Action → UI
- Focus field → Add focus styling
- Enter text → Update value → Trigger validation
- Blur field → Set touched state → Show validation errors if invalid

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly (Kanban board with task creation modal)
- Do NOT create new components unnecessarily (reuse library components where possible)
- Prefer reusable library components (ButtonComponent, DropdownComponent, etc.)
- HTML must be pseudo-code ONLY (NOT real framework code)
- Real-time validation must be implemented using Angular reactive forms
- Character counters must update on every keystroke
- Form must reset after successful submission
- Loading states must be shown during all API calls
- Error handling must cover both API errors and validation errors
- Form must be accessible with proper ARIA labels and keyboard navigation
- Component architecture must support future enhancements (assignee selection, tags, etc.)
- All validation rules must match API schema requirements
- Authentication via JWT Bearer tokens must be supported
- Form must handle network timeouts and connection errors gracefully
