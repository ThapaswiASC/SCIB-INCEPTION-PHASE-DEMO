## 4. COMPONENT IMPLEMENTATION

### 4.1 YouthAccountDashboard

Component Type: Feature

Mapped From A1: YouthAccountDashboard (Main container component for youth account management)

Purpose: Main container component that orchestrates the youth account dashboard functionality, manages state, and coordinates API calls

Parent: YouthAccountPage

Children:
- DashboardHeader
- AccountOverviewCard
- FundTransferSection
- SpendingLimitSection
- ActivitySection

Dependencies:
- YouthAccountService
- FundTransferService
- TransactionService
- Snackbar component

Library Components Used:
- Button → Action buttons for refresh and settings
- Snackbar → Success/error notifications

TypeScript Specification

Inputs:
- youthAccountId: string (Account identifier)
- parentAccountId: string (Parent account identifier)

Outputs:
- onRefresh: () => void (Dashboard refresh event)
- onError: (error: string) => void (Error handling event)

State:
- dashboardData: DashboardData (Complete dashboard information)
- loading: boolean (Loading state indicator)
- error: string (Error message state)
- showFundTransferModal: boolean (Fund transfer modal visibility)
- showSpendingLimitModal: boolean (Spending limit modal visibility)

Methods:
- loadDashboardData() → Fetch dashboard data from API
- handleRefresh() → Refresh dashboard data
- openFundTransferModal() → Show fund transfer modal
- openSpendingLimitModal() → Show spending limit modal
- handleError(error: string) → Process and display errors

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="youth-account-dashboard">
  <DashboardHeader 
    title="Youth Account Dashboard"
    subtitle="Manage your child's account"
    [actions]="headerActions"
  />
  
  <div class="dashboard-content">
    <div class="overview-section">
      <AccountOverviewCard 
        [balance]="dashboardData.balance"
        [currency]="dashboardData.currency"
        [spendingLimit]="dashboardData.spendingLimit"
        [accountId]="youthAccountId"
        (onRefresh)="handleRefresh()"
      />
    </div>
    
    <div class="actions-section">
      <div class="fund-transfer-section">
        <Button 
          variant="primary"
          (click)="openFundTransferModal()"
          text="Add Funds"
        />
      </div>
      
      <div class="spending-limit-section">
        <Button 
          variant="secondary"
          (click)="openSpendingLimitModal()"
          text="Configure Limit"
        />
      </div>
    </div>
    
    <div class="activity-section">
      <ActivityTimeline 
        [youthAccountId]="youthAccountId"
        [pageSize]="10"
      />
    </div>
  </div>
  
  <FundTransferModal 
    [isOpen]="showFundTransferModal"
    [youthAccountId]="youthAccountId"
    [parentAccounts]="dashboardData.parentAccounts"
    (onTransfer)="handleFundTransfer($event)"
    (onClose)="closeFundTransferModal()"
  />
  
  <SpendingLimitConfigModal 
    [isOpen]="showSpendingLimitModal"
    [youthAccountId]="youthAccountId"
    [currentLimit]="dashboardData.spendingLimit"
    (onSave)="handleSpendingLimitUpdate($event)"
    (onClose)="closeSpendingLimitModal()"
  />
  
  <Snackbar 
    [message]="notificationMessage"
    [type]="notificationType"
    [isVisible]="showNotification"
  />
</div>

CSS SPECIFICATION

Layout:
- display: flex; flex-direction: column; gap: 24px;
- Container uses full viewport height with proper spacing

Spacing:
- padding: 24px; gap: 16px between sections
- margin-bottom: 32px for main sections

Component Styles:
- .youth-account-dashboard (main container)
- .dashboard-content (content grid layout)
- .overview-section, .actions-section, .activity-section (section containers)

Responsive:
- Mobile: Single column layout, reduced padding (16px)
- Tablet: Two-column grid for actions section
- Desktop: Full grid layout with optimal spacing

API INTEGRATION

- loadDashboardData() → GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch complete dashboard data
- handleFundTransfer() → POST /api/youth-accounts/{youthAccountId}/fund-transfer → Process fund transfers
- handleSpendingLimitUpdate() → PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limits

ERROR HANDLING

- API failure → show error snackbar with retry option
- Network timeout → show "Connection timeout" message
- Validation error → show inline validation messages
- Insufficient funds → show specific error message in fund transfer modal

INTERACTION FLOW

User Action → UI → API
- Page load → loadDashboardData() → GET dashboard API → display data
- Click "Add Funds" → open FundTransferModal → user fills form → POST fund-transfer API
- Click "Configure Limit" → open SpendingLimitConfigModal → user sets limit → PUT spending-limit API
- Refresh action → loadDashboardData() → GET dashboard API → update display

### 4.3 FundTransferModal

Component Type: Feature

Mapped From A1: FundTransferModal (Handle fund transfer from parent to youth account)

Purpose: Provide interface for parents to transfer funds from their accounts to youth account with validation and confirmation

Parent: YouthAccountDashboard

Children:
- Modal (container)
- Textbox (form inputs)
- Button (action buttons)
- Dropdown (account selection)

Dependencies:
- Modal component
- Form validation utilities
- FundTransferService
- Currency formatting

Library Components Used:
- Modal → Dialog container
- Textbox → Form input fields
- Button → Submit and cancel actions
- Dropdown → Source account selection

TypeScript Specification

Inputs:
- isOpen: boolean (Modal visibility state)
- youthAccountId: string (Target account ID)
- parentAccounts: Account[] (Available source accounts)

Outputs:
- onTransfer: (transferData: FundTransferRequest) => void (Transfer submission event)
- onClose: () => void (Modal close event)
- onSuccess: (result: FundTransferResponse) => void (Successful transfer event)

State:
- transferAmount: number (Transfer amount)
- sourceAccountId: string (Selected source account)
- description: string (Transfer description)
- processing: boolean (Transfer processing state)
- validationErrors: ValidationError[] (Form validation errors)
- confirmationStep: boolean (Confirmation dialog state)

Methods:
- validateForm() → Validate transfer form data
- handleSubmit() → Process transfer submission
- handleConfirm() → Confirm and execute transfer
- resetForm() → Clear form data
- formatAccountDisplay(account: Account) → Format account for dropdown

HTML STRUCTURE (PSEUDO CODE ONLY)

<Modal 
  [isOpen]="isOpen"
  title="Add Funds to Youth Account"
  size="medium"
  (onClose)="handleClose()"
>
  <div class="fund-transfer-form" *ngIf="!confirmationStep">
    <div class="form-section">
      <label class="form-label">Transfer From</label>
      <Dropdown 
        [options]="parentAccountOptions"
        [(value)]="sourceAccountId"
        placeholder="Select source account"
        [error]="getFieldError('sourceAccount')"
      />
    </div>
    
    <div class="form-section">
      <label class="form-label">Amount</label>
      <Textbox 
        type="number"
        [(value)]="transferAmount"
        placeholder="0.00"
        prefix="$"
        [error]="getFieldError('amount')"
      />
    </div>
    
    <div class="form-section">
      <label class="form-label">Description (Optional)</label>
      <Textbox 
        [(value)]="description"
        placeholder="What's this for?"
        maxLength="100"
      />
    </div>
    
    <div class="available-balance" *ngIf="selectedAccount">
      <span class="balance-label">Available Balance:</span>
      <span class="balance-amount">{{formatCurrency(selectedAccount.balance)}}</span>
    </div>
  </div>
  
  <div class="confirmation-step" *ngIf="confirmationStep">
    <div class="confirmation-details">
      <h4>Confirm Transfer</h4>
      <div class="transfer-summary">
        <div class="summary-row">
          <span class="label">From:</span>
          <span class="value">{{selectedAccount.displayName}}</span>
        </div>
        <div class="summary-row">
          <span class="label">To:</span>
          <span class="value">Youth Account</span>
        </div>
        <div class="summary-row">
          <span class="label">Amount:</span>
          <span class="value amount">{{formatCurrency(transferAmount)}}</span>
        </div>
        <div class="summary-row" *ngIf="description">
          <span class="label">Description:</span>
          <span class="value">{{description}}</span>
        </div>
      </div>
    </div>
  </div>
  
  <div class="modal-actions">
    <Button 
      variant="secondary"
      text="Cancel"
      (click)="handleClose()"
      [disabled]="processing"
    />
    <Button 
      variant="primary"
      [text]="confirmationStep ? 'Confirm Transfer' : 'Continue'"
      (click)="confirmationStep ? handleConfirm() : handleSubmit()"
      [loading]="processing"
      [disabled]="!isFormValid()"
    />
  </div>
</Modal>

CSS SPECIFICATION

Layout:
- Modal content with padding: 24px
- Form sections with gap: 16px
- Two-column layout for confirmation details

Spacing:
- margin-bottom: 16px for form sections
- gap: 8px between form elements

Component Styles:
- .fund-transfer-form (form container)
- .form-section (individual form fields)
- .confirmation-step (confirmation display)
- .transfer-summary (summary details)
- .modal-actions (button container)

Responsive:
- Mobile: Single column, full-width inputs
- Tablet: Optimized spacing and button sizes
- Desktop: Standard modal width with proper proportions

API INTEGRATION

- handleConfirm() → POST /api/youth-accounts/{youthAccountId}/fund-transfer → Execute fund transfer
- Account validation → GET /api/accounts/{accountId}/balance → Verify sufficient funds

ERROR HANDLING

- Insufficient funds → show error message "Insufficient funds in selected account"
- Network error → show "Transfer failed, please try again" message
- Validation errors → show inline field-specific error messages
- API timeout → show "Request timeout, please retry" message

INTERACTION FLOW

User Action → UI → API
- Open modal → load parent accounts → display form
- Select source account → validate balance → show available funds
- Enter amount → validate against balance → enable/disable continue button
- Click continue → show confirmation step → display transfer summary
- Click confirm → POST transfer API → show success/error message → close modal

Component Type: Feature

Mapped From A1: SpendingLimitConfigModal (Configure weekly spending limits)

Parent: YouthAccountDashboard

Children:
- Modal (container)
- Textbox (limit input)
- DatePicker (effective date)
- Button (action buttons)

Dependencies:
- Modal component
- Form validation utilities
- Date utilities
- Currency formatting

Library Components Used:
- Modal → Dialog container
- Textbox → Limit amount input
- Button → Save and cancel actions
- DatePicker → Effective date selection

- isOpen: boolean (Modal visibility state)

Outputs:
- onSave: (limitData: SpendingLimitRequest) => void (Save limit event)

      variant="primary"
      text="Save Limit"
      (click)="handleSave()"
      [loading]="processing"
      [disabled]="!isFormValid()"
    />
  </div>
</Modal>

### 4.5 ActivityTimeline

Component Type: Feature

Mapped From A1: ActivityTimeline (Display transaction history and activity)

Purpose: Display paginated transaction history with filtering capabilities and detailed transaction information

Parent: YouthAccountDashboard

Children:
- Timeline (container)
- TransactionCard (individual transactions)
- Button (load more)
- SearchBar (transaction search)

Dependencies:
- Timeline component
- TransactionCard component
- SearchBar component
- TransactionService
- Date utilities

Library Components Used:
- SearchBar → Transaction search and filtering
- Button → Load more transactions action
- Timeline → Transaction timeline container

TypeScript Specification

Inputs:
- youthAccountId: string (Account identifier)
- pageSize: number (Number of transactions per page)

Outputs:
- onLoadMore: () => void (Load more transactions event)
- onFilter: (filters: TransactionFilters) => void (Apply filters event)

State:
- transactions: Transaction[] (Transaction list)
- pagination: PaginationInfo (Pagination state)
- loading: boolean (Loading state)
- filters: TransactionFilters (Applied filters)
- searchQuery: string (Search query)

Methods:
- loadTransactions() → Fetch transaction data
- handleLoadMore() → Load additional transactions
- handleSearch(query: string) → Filter transactions by search
- handleFilter(filters: TransactionFilters) → Apply transaction filters
- formatTransactionDate(date: Date) → Format date for display

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="activity-timeline">
  <div class="timeline-header">
    <h3 class="section-title">Recent Activity</h3>
    <SearchBar 
      placeholder="Search transactions..."
      [(value)]="searchQuery"
      (onSearch)="handleSearch($event)"
    />
  </div>
  
  <div class="filter-section">
    <div class="filter-chips">
      <Button 
        variant="ghost"
        text="All"
        [active]="!filters.type"
        (click)="clearFilters()"
      />
      <Button 
        variant="ghost"
        text="Deposits"
        [active]="filters.type === 'deposit'"
        (click)="filterByType('deposit')"
      />
      <Button 
        variant="ghost"
        text="Purchases"
        [active]="filters.type === 'purchase'"
        [class.active]="filters.type === 'purchase'"
        (click)="filterByType('purchase')"
      />
    </div>
  </div>
  
  <Timeline class="transaction-timeline">
    <TransactionCard 
      *ngFor="let transaction of transactions"
      [transaction]="transaction"
      [showCategory]="true"
      (onExpand)="handleTransactionExpand($event)"
    />
    
    <div class="timeline-loading" *ngIf="loading">
      <span class="loading-text">Loading transactions...</span>
    </div>
    
    <div class="timeline-empty" *ngIf="!loading && transactions.length === 0">
      <span class="empty-text">No transactions found</span>
      <span class="empty-subtext">Transactions will appear here once activity begins</span>
    </div>
  </Timeline>
  
  <div class="timeline-actions" *ngIf="pagination.hasMore">
    <Button 
      variant="secondary"
      text="Load More"
      (click)="handleLoadMore()"
      [loading]="loading"
    />
  </div>
</div>

CSS SPECIFICATION

Layout:
- Flex column layout with gap: 16px
- Timeline with vertical scroll for large lists

Spacing:
- padding: 16px for timeline container
- margin-bottom: 12px between transactions

Component Styles:
- .activity-timeline (main container)
- .timeline-header (title and search)
- .filter-section (filter chips)
- .transaction-timeline (scrollable timeline)
- .timeline-actions (load more button)

Responsive:
