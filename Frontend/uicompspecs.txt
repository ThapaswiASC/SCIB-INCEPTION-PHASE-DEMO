4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for three-column Kanban layout

Purpose: Orchestrates the entire Kanban board with drag-and-drop functionality, real-time updates, and state management

Parent: MainContentComponent

Children:
- KanbanColumnComponent (3 instances)
- TaskFormComponent (conditional)

Dependencies:
- KanbanService
- WebSocketService
- TaskService

Library Components Used:
- Button → Add Task actions
- Popup → Error messages and confirmations

TypeScript Specification

Inputs:
- boardId: string
- initialColumns: KanbanColumn[]
- initialTasks: Task[]

Outputs:
- taskCreated: EventEmitter<Task> (emitted when new task created)
- taskMoved: EventEmitter<{taskId: string, fromColumn: string, toColumn: string}> (drag-drop events)
- taskUpdated: EventEmitter<Task> (task modifications)

State:
- loading: boolean
- error: string | null
- draggedTask: Task | null
- columns: KanbanColumn[]
- tasks: Task[]
- showTaskForm: boolean
- selectedColumnId: string | null

Methods:
- ngOnInit() → Initialize board data and WebSocket connection
- loadTasks() → Fetch tasks from API
- handleTaskDrop(event: DragEvent) → Process drag-and-drop operations
- openTaskForm(columnId: string) → Show task creation form
- handleTaskSubmit(task: Task) → Process new task creation

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board" role="region" aria-label="Kanban Board">
  <div class="board-header">
    <h2>Kanban Board</h2>
    <Button variant="primary" (click)="openTaskForm(null)">Add Task</Button>
  </div>
  
  <div class="board-grid" [style.grid-template-columns]="'repeat(3, 1fr)'">
    <KanbanColumnComponent 
      *ngFor="let column of columns" 
      [column]="column" 
      [tasks]="getTasksForColumn(column.id)"
      [allowDrop]="true"
      (taskDropped)="handleTaskDrop($event)"
      (addTaskClicked)="openTaskForm($event)">
    </KanbanColumnComponent>
  </div>
  
  <TaskFormComponent 
    *ngIf="showTaskForm"
    [columnId]="selectedColumnId"
    [users]="availableUsers"
    (taskSubmitted)="handleTaskSubmit($event)"
    (formCancelled)="closeTaskForm()">
  </TaskFormComponent>
  
  <Popup *ngIf="error" [message]="error" type="error"></Popup>
</div>

CSS SPECIFICATION

Layout:
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1.5rem
- min-height: 100vh

Spacing:
- padding: 2rem
- gap: 1.5rem between columns

Component Styles:
- .kanban-board: main container with glassmorphism background
- .board-grid: CSS Grid layout for columns
- .board-header: flex layout for title and actions

Responsive:
- Desktop (>1024px): 3 columns, full spacing
- Tablet (768px-1024px): 2 columns, reduced spacing
- Mobile (<768px): 1 column, minimal spacing

API INTEGRATION

- loadTasks() → GET /api/tasks → Load initial board data
- createTask() → POST /api/tasks → Create new task
- WebSocket connection → wss://api.scib.com/ws/boards/{boardId} → Real-time updates

ERROR HANDLING

- API failure → show error popup with retry option
- WebSocket disconnect → show connection status indicator
- Validation error → inline form validation messages

INTERACTION FLOW

User Action → UI → API
- Load board → show loading state → GET /api/tasks → render columns and tasks
- Add task → open TaskFormComponent → validate → POST /api/tasks → update UI optimistically
- Drag task → update local state → call update API (when available) → broadcast via WebSocket

### 4.2 KanbanColumnComponent

Component Type: Feature

TypeScript Specification

- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean

Outputs:

State:

Methods:

  </div>

    <TaskCardComponent 
      *ngFor="let task of tasks; trackBy: trackByTaskId"

      [draggable]="true"
      [interactive]="true"
      (taskClicked)="onTaskClick($event)"

      [disabled]="false"

Responsive:
- Desktop: full width with margins
- Tablet: reduced padding
- Mobile: full width, minimal padding

API INTEGRATION

- None (receives data from parent component)

ERROR HANDLING

- Drop validation → prevent invalid drops
- Task rendering errors → show placeholder cards

INTERACTION FLOW

User Action → UI → API
- Drag task over column → highlight drop zone → validate drop target
- Drop task → update local state → emit taskDropped event → parent handles API call
- Click add task → emit addTaskClicked event → parent opens task form
