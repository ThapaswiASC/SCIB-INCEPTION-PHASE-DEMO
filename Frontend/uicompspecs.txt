4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: KanbanBoardComponent - Main container for three-column kanban layout

Purpose: Main container component that manages the kanban board with three columns and handles task operations

Parent: AppComponent

Children:
- KanbanColumnComponent (3 instances)
- CreateTaskModalComponent
- BulkEditModalComponent

Dependencies:
- TaskService
- NotificationService

Library Components Used:
- Card Component → Container wrapper
- Button Component → Action buttons
- Modal Component → Task creation dialog

TypeScript Specification

Inputs:
- columns: KanbanColumn[] (column configuration)
- tasks: Task[] (task data)
- loading: boolean (loading state)
- error: string | null (error state)

Outputs:
- onTaskCreate: EventEmitter<CreateTaskRequest> (task creation event)
- onTaskUpdate: EventEmitter<{taskId: string, updates: Partial<Task>}> (task update event)
- onTaskMove: EventEmitter<{taskId: string, columnId: string}> (task move event)

State:
- selectedTasks: string[] (selected task IDs)
- draggedTask: Task | null (currently dragged task)
- showCreateModal: boolean (modal visibility)

Methods:
- ngOnInit() → Initialize component and load data
- loadTasks() → Fetch tasks from API
- handleTaskCreate(task: CreateTaskRequest) → Create new task
- handleTaskMove(taskId: string, columnId: string) → Move task between columns

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board" role="region" aria-label="Kanban Board">
  <div class="kanban-header">
    <h2>Task Board</h2>
    <app-button (click)="openCreateModal()">Add Task</app-button>
  </div>
  
  <div class="kanban-columns" [style.grid-template-columns]="'repeat(3, 1fr)'">
    <app-kanban-column 
      *ngFor="let column of columns" 
      [column]="column"
      [tasks]="getTasksForColumn(column.id)"
      [allowDrop]="true"
      (onTaskDrop)="handleTaskMove($event.taskId, $event.columnId)"
      (onTaskSelect)="handleTaskSelect($event)">
    </app-kanban-column>
  </div>
  
  <app-create-task-modal
    [visible]="showCreateModal"
    [users]="users"
    (onSubmit)="handleTaskCreate($event)"
    (onCancel)="closeCreateModal()">
  </app-create-task-modal>
  
  <div *ngIf="error" class="error-message" role="alert">
    {{ error }}
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-board: display: flex, flex-direction: column
- .kanban-columns: display: grid, grid-template-columns: repeat(3, 1fr), gap: 1rem

Spacing:
- padding: 1.5rem
- gap: 1rem between columns

Component Styles:
- .kanban-header: flex layout with space-between
- .error-message: error styling with red background

Responsive:
- Desktop (>1024px): 3 columns side by side
- Tablet (768px-1024px): 2 columns with scroll
- Mobile (<768px): 1 column stacked

API INTEGRATION

- loadTasks() → GET /api/v1/tasks → Fetch initial task data
- handleTaskCreate() → POST /api/v1/tasks → Create new task
- handleTaskUpdate() → PUT /api/v1/tasks/{id} → Update task (future implementation)

ERROR HANDLING

- API failure → show error message in error-message div
- Validation error → display inline validation messages
- Network timeout → show retry button with notification

INTERACTION FLOW

User Action → UI → API
- Click "Add Task" → open CreateTaskModalComponent → submit form → POST /api/v1/tasks
- Drag task between columns → update task status → PUT /api/v1/tasks/{id}
- Select multiple tasks → enable bulk operations → show BulkEditModalComponent

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: KanbanColumnComponent - Individual column container with header and task list

Purpose: Represents a single column in the kanban board with drag-drop functionality

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)

Dependencies:
- DragDropService

Library Components Used:
- Card Component → Column container
- Badge Component → Task count indicator

TypeScript Specification

Inputs:
- column: KanbanColumn (column configuration)
- tasks: Task[] (tasks for this column)
- allowDrop: boolean (drop permission)

Outputs:
- onTaskDrop: EventEmitter<{taskId: string, columnId: string}> (task drop event)
- onTaskSelect: EventEmitter<string> (task selection event)

State:
- isDropTarget: boolean (drop target state)
- taskCount: number (number of tasks)

Methods:
- onDragOver(event: DragEvent) → Handle drag over events
- onDrop(event: DragEvent) → Handle task drop
- getTaskCount() → Calculate task count

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-column" 
     [attr.aria-labelledby]="column.id + '-header'"
     role="region">
  <div class="column-header" [id]="column.id + '-header'">
    <h3>{{ column.title }}</h3>
    <app-badge [count]="taskCount" variant="secondary"></app-badge>
  </div>
  
  <div class="column-content"
       (dragover)="onDragOver($event)"
       (drop)="onDrop($event)"
       [class.drop-target]="isDropTarget">
    <app-task-card
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [selected]="false"
      [draggable]="true"
      (onTaskClick)="handleTaskClick($event)"
      (onTaskSelect)="handleTaskSelect($event)">
    </app-task-card>
    
    <div *ngIf="tasks.length === 0" class="empty-column">
      <p>No tasks in {{ column.title }}</p>
    </div>
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-column: display: flex, flex-direction: column, min-height: 400px
- .column-content: flex: 1, overflow-y: auto

Spacing:
- padding: 1rem
- margin-bottom: 0.5rem between tasks

Component Styles:
- .column-header: border-bottom, padding-bottom: 0.5rem
- .drop-target: border with dashed style, background highlight
- .empty-column: centered text with muted color

Responsive:
- Mobile: full width with vertical scroll
- Tablet/Desktop: fixed width with scroll

API INTEGRATION

- None (receives data from parent KanbanBoardComponent)

ERROR HANDLING

- Invalid drop → show error notification
- Task loading failure → display placeholder with retry option

INTERACTION FLOW

User Action → UI → API
- Drag task over column → highlight drop target → visual feedback
- Drop task in column → emit onTaskDrop event → parent handles API call
- Click task → emit onTaskSelect event → parent handles selection
