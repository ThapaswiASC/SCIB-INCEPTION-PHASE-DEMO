### 4. COMPONENT IMPLEMENTATION

#### 4.1 KanbanBoardComponent

**Component Type:** Page

**Mapped From A1:** Main container component for the Kanban board with three-column layout

**Purpose:** Primary container managing the entire Kanban board functionality with drag-drop support

**Parent:** App Component

**Children:**
- HeaderComponent
- SidebarComponent
- PageHeaderComponent
- BoardControlsComponent
- KanbanContainerComponent
- TaskModalComponent

**Dependencies:**
- Angular CDK Drag Drop
- Angular Material Components
- RxJS for state management
- Kanban Service

**Library Components Used:**
- LoadingComponent → Shared loading spinner
- ErrorStateComponent → Error display component
- ModalComponent → Base modal wrapper

**TypeScript Specification**

**Inputs:**
- youthAccountId: string (Account identifier)
- initialColumns: KanbanColumn[] (Initial column configuration)
- permissions: UserPermissions (User access rights)

**Outputs:**
- onTaskMove: EventEmitter<TaskMoveEvent> (Task moved between columns)
- onTaskCreate: EventEmitter<Task> (New task created)
- onTaskUpdate: EventEmitter<Task> (Task updated)
- onError: EventEmitter<string> (Error occurred)

**State:**
- columns: KanbanColumn[] (Current column data)
- loading: boolean (Loading state)
- error: string | null (Error message)
- selectedFilters: FilterState (Active filters)
- draggedTask: Task | null (Currently dragged task)
- modalOpen: boolean (Task modal visibility)
- currentTask: Task | null (Task being edited)

**Methods:**
- ngOnInit() → Initialize component and load data
- loadKanbanData() → Fetch initial board data
- onTaskDrop(event: CdkDragDrop) → Handle task drop events
- openTaskModal(task?: Task) → Open task creation/editing modal
- closeTaskModal() → Close task modal
- handleTaskSubmit(task: Task) → Process task creation/update
- applyFilters(filters: FilterState) → Apply filtering to tasks
- handleError(error: any) → Process and display errors

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <app-header 
    [user]="currentUser" 
    [notifications]="notifications"
    (onSearch)="handleSearch($event)">
  </app-header>
  
  <div class="main-layout">
    <app-sidebar 
      [menuItems]="navigationItems"
      [activeItem]="activeMenuItem"
      (onMenuItemClick)="handleNavigation($event)">
    </app-sidebar>
    
    <main class="content-area">
      <app-page-header 
        [breadcrumbs]="breadcrumbItems"
        [title]="pageTitle"
        [subtitle]="pageSubtitle">
        <app-button 
          variant="primary" 
          (click)="openTaskModal()"
          class="add-task-btn">
          Add Task
        </app-button>
      </app-page-header>
      
      <app-board-controls 
        [filters]="availableFilters"
        (onFilterChange)="applyFilters($event)">
      </app-board-controls>
      
      <div class="kanban-container" 
           cdkDropListGroup
           [class.loading]="loading">
        <app-kanban-column 
          *ngFor="let column of columns; trackBy: trackByColumnId"
          [column]="column"
          [tasks]="getTasksForColumn(column.id)"
          [allowDrop]="!loading"
          (onTaskDrop)="onTaskDrop($event)"
          (onAddTask)="openTaskModal()">
        </app-kanban-column>
      </div>
      
      <app-loading-spinner *ngIf="loading"></app-loading-spinner>
      <app-error-state *ngIf="error" [message]="error"></app-error-state>
    </main>
  </div>
  
  <app-task-modal 
    [open]="modalOpen"
    [task]="currentTask"
    [columns]="columns"
    (onSubmit)="handleTaskSubmit($event)"
    (onCancel)="closeTaskModal()">
  </app-task-modal>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: grid
- grid-template-areas: "header header" "sidebar main"
- grid-template-columns: 280px 1fr
- grid-template-rows: 64px 1fr

**Spacing:**
- padding: 0
- gap: 0
- main content padding: 24px

**Component Styles:**
- .kanban-board-container: full viewport height
- .main-layout: grid layout for sidebar and content
- .content-area: flex column with gap
- .kanban-container: CSS grid for columns

**Responsive:**
- Desktop (>1024px): Full three-column layout
- Tablet (768px-1024px): Collapsible sidebar, scrollable columns
- Mobile (<768px): Single column view, drawer sidebar

**API INTEGRATION**

- loadKanbanData() → GET /api/youth-accounts/{id}/dashboard → Load initial board data
- handleTaskSubmit() → POST /api/youth-accounts/{id}/fund-transfer → Create new task (adapted)
- onTaskMove() → PUT /api/youth-accounts/{id}/spending-limit → Update task status (adapted)
- loadTransactionHistory() → GET /api/youth-accounts/{id}/transactions → Load task history

**ERROR HANDLING**

- API failure → Display error state component with retry option
- Validation error → Show inline validation messages
- Network timeout → Show connection error with offline indicator
- Permission denied → Redirect to access denied page

**INTERACTION FLOW**

**User Action → UI → API**
- Page load → Show loading spinner → GET dashboard API → Display board data
- Click "Add Task" → Open task modal → Fill form → POST task API → Update UI
- Drag task → Visual feedback → Drop → PUT update API → Sync state
- Apply filters → Update UI immediately → Optional API call for server filtering

#### 4.2 KanbanColumnComponent

**Component Type:** Feature

**Mapped From A1:** Individual column container (To Do, In Progress, Done)

**Purpose:** Manages individual Kanban column with drag-drop zone and task list

**Parent:** KanbanBoardComponent

**Children:**
- ColumnHeaderComponent
- TaskCardComponent (multiple)
- AddTaskButtonComponent
- DropZoneComponent

**Dependencies:**
- Angular CDK Drag Drop
- Task Card Component
- Badge Component

**Library Components Used:**
- BadgeComponent → Column task count
- ButtonComponent → Add task and column actions
- CardComponent → Column container styling

**TypeScript Specification**

**Inputs:**
- column: KanbanColumn (Column configuration and data)
- tasks: Task[] (Tasks assigned to this column)
- allowDrop: boolean (Enable/disable drop functionality)
- maxTasks: number (Maximum tasks per column)

**Outputs:**
- onTaskDrop: EventEmitter<TaskDropEvent> (Task dropped in column)
- onAddTask: EventEmitter<string> (Add task button clicked)
- onColumnAction: EventEmitter<ColumnAction> (Column header actions)

**State:**
- dragOver: boolean (Drag over state for visual feedback)
- expanded: boolean (Column expanded/collapsed state)
- taskCount: number (Current number of tasks)
- isOverLimit: boolean (Exceeds maximum task limit)

**Methods:**
- ngOnInit() → Initialize column state
- onDragEnter(event) → Handle drag enter events
- onDragLeave(event) → Handle drag leave events
- onDrop(event) → Process task drop
- toggleExpanded() → Toggle column expansion
- getTasksByPriority() → Sort tasks by priority
- canAcceptDrop(task: Task) → Validate drop eligibility

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-column" 
     [class.drag-over]="dragOver"
     [class.expanded]="expanded"
     [attr.aria-labelledby]="column.id + '-header'">
  
  <div class="column-header" [id]="column.id + '-header'">
    <h3 class="column-title">{{ column.title }}</h3>
    <app-badge 
      [count]="taskCount"
      [variant]="getCountVariant()"
      class="task-count">
    </app-badge>
    <app-button 
      variant="ghost" 
      size="small"
      (click)="toggleExpanded()"
      class="expand-btn">
      <app-icon [name]="expanded ? 'collapse' : 'expand'"></app-icon>
    </app-button>
  </div>
  
  <div class="column-body" 
       cdkDropList
       [cdkDropListData]="tasks"
       [cdkDropListDisabled]="!allowDrop"
       (cdkDropListDropped)="onDrop($event)"
       (cdkDropListEntered)="onDragEnter($event)"
       (cdkDropListExited)="onDragLeave($event)">
    
    <app-task-card 
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="allowDrop"
      cdkDrag
      [cdkDragData]="task"
      class="task-item">
    </app-task-card>
    
    <app-button 
      variant="dashed"
      size="large"
      (click)="onAddTask.emit(column.id)"
      class="add-task-btn"
      [disabled]="isOverLimit">
      <app-icon name="plus"></app-icon>
      Add Task
    </app-button>
    
    <div class="drop-zone-indicator" 
         *ngIf="dragOver && canAcceptCurrentDrag">
      Drop task here
    </div>
  </div>
  
  <div class="column-footer" *ngIf="column.showFooter">
    <span class="task-limit-info">{{ taskCount }}/{{ maxTasks }} tasks</span>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: flex
- flex-direction: column
- min-height: 500px
- width: 100%

**Spacing:**
- padding: 16px
- gap: 12px
- margin-bottom: 24px

**Component Styles:**
- .kanban-column: card styling with border radius
- .column-header: flex layout with space-between
- .column-body: flex-1 with scroll overflow
- .drag-over: highlighted border and background

**Responsive:**
- Desktop: Fixed width columns in grid
- Tablet: Scrollable horizontal layout
- Mobile: Full width stacked columns

**API INTEGRATION**

- onDrop() → Triggers parent component API calls
- Task validation → Uses column rules and limits
- No direct API calls (data flows through parent)

**ERROR HANDLING**

- Invalid drop → Show error toast and revert
- Task limit exceeded → Disable drop and show warning
- Column load failure → Show empty state with retry

**INTERACTION FLOW**

**User Action → UI → API**
- Drag task over column → Show drop zone → Highlight column
- Drop task → Validate drop → Update parent → API call
- Click add task → Emit event → Parent opens modal
- Toggle expand → Update local state → Save preference

#### 4.3 TaskCardComponent

**Component Type:** Feature

**Mapped From A1:** Individual task card with drag/drop functionality

**Purpose:** Displays individual task information with interactive elements

**Parent:** KanbanColumnComponent

**Children:**

**TypeScript Specification**

**Inputs:**
- task: Task (Task data object)
- draggable: boolean (Enable drag functionality)
- compact: boolean (Compact display mode)
- showActions: boolean (Show quick action buttons)

**Outputs:**
- onTaskClick: EventEmitter<Task> (Task clicked for editing)

**Methods:**

        size="small"

- AvatarComponent → User avatar display
- BadgeComponent → Notification count badge
- ButtonComponent → Action buttons

**TypeScript Specification**

**Inputs:**
- user: User (Current user information)
- notifications: Notification[] (User notifications)
- searchEnabled: boolean (Enable search functionality)
- showNotifications: boolean (Show notification icon)

**Outputs:**
- onSearch: EventEmitter<string> (Search query entered)
- onNotificationClick: EventEmitter<void> (Notification icon clicked)
- onUserMenuClick: EventEmitter<string> (User menu action)
- onLogoClick: EventEmitter<void> (Logo clicked)

**State:**
- searchQuery: string (Current search query)
- notificationCount: number (Unread notification count)
- userMenuOpen: boolean (User dropdown menu state)
- searchFocused: boolean (Search field focus state)

**Methods:**
- ngOnInit() → Initialize header state
- onSearchInput(query: string) → Handle search input
- onSearchSubmit() → Submit search query
- toggleUserMenu() → Toggle user dropdown
- handleNotificationClick() → Open notification panel
- onLogoClick() → Navigate to home
- getNotificationCount() → Calculate unread notifications
