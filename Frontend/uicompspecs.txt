### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for three-column Kanban layout

Purpose: Orchestrates the entire Kanban board functionality including task management, column layout, and drag-drop operations

Parent: MainLayoutComponent

Children:
- KanbanColumnComponent (3 instances)
- CreateTaskModalComponent
- TaskDetailComponent

Dependencies:
- TaskService (API calls)
- DragDropService (drag operations)
- NotificationService (user feedback)

Library Components Used:
- Grid → Layout container
- Button → Action buttons
- Modal → Task creation overlay

TypeScript Specification

Inputs:
- loading: boolean (loading state)
- error: string | null (error message)
- user: User (current user context)

Outputs:
- onTaskCreate: (task: CreateTaskRequest) => void (task creation event)
- onTaskUpdate: (taskId: string, updates: Partial<Task>) => void (task update event)
- onTaskDelete: (taskId: string) => void (task deletion event)

State:
- columns: KanbanColumn[] (column configuration)
- tasks: Task[] (all tasks)
- selectedTask: Task | null (currently selected task)
- draggedTask: Task | null (task being dragged)
- showCreateModal: boolean (modal visibility)
- columnConfig: ColumnConfig (column settings)

Methods:
- initializeBoard() → void (setup initial state)
- handleTaskMove(taskId: string, newColumnId: string) → void (drag-drop handler)
- openCreateModal() → void (show task creation)
- closeCreateModal() → void (hide task creation)
- refreshTasks() → Promise<void> (reload task data)

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container">
  <div class="board-header">
    <h1>Kanban Board</h1>
    <Button variant="primary" onClick={openCreateModal}>Add Task</Button>
  </div>
  
  <Grid className="kanban-columns" columns={3}>
    <KanbanColumnComponent 
      column={todoColumn} 
      tasks={todoTasks}
      onTaskDrop={handleTaskMove}
    />
    <KanbanColumnComponent 
      column={inProgressColumn} 
      tasks={inProgressTasks}
      onTaskDrop={handleTaskMove}
    />
    <KanbanColumnComponent 
      column={doneColumn} 
      tasks={doneTasks}
      onTaskDrop={handleTaskMove}
    />
  </Grid>
  
  {showCreateModal && (
    <CreateTaskModalComponent 
      visible={showCreateModal}
      onSubmit={handleTaskCreate}
      onCancel={closeCreateModal}
    />
  )}
</div>

CSS SPECIFICATION

Layout:
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1rem
- min-height: 100vh

Spacing:
- padding: 1.5rem
- gap: 1rem between columns

Component Styles:
- .kanban-board-container: main container
- .board-header: flex layout for title and actions
- .kanban-columns: grid layout for columns

Responsive:
- Desktop (>1024px): 3 columns
- Tablet (768px-1024px): 2 columns, scroll horizontal
- Mobile (<768px): 1 column, stack vertically

API INTEGRATION

- initializeBoard() → GET /api/v1/tasks → fetch all tasks
- handleTaskCreate() → POST /api/v1/tasks → create new task
- handleTaskMove() → PUT /api/v1/tasks/{id} → update task column
- refreshTasks() → GET /api/v1/tasks → reload task data

ERROR HANDLING

- API failure → show error notification via NotificationService
- Validation error → display inline form errors
- Network timeout → show retry button with exponential backoff
- Drag operation failure → revert task position and show error

INTERACTION FLOW

User Action → UI → API
- Click "Add Task" → open CreateTaskModalComponent → submit form → POST /api/v1/tasks
- Drag task between columns → update task position → PUT /api/v1/tasks/{id}
- Click task card → open TaskDetailComponent → GET /api/v1/tasks/{id}

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: Individual column container with header and task list

Purpose: Displays tasks for a specific column (To Do, In Progress, Done) with drag-drop capabilities

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)
- AddTaskButtonComponent

Dependencies:
- DragDropService (drop zone handling)

Library Components Used:
- Card → Column container
- Button → Add task action
- Badge → Task count indicator

TypeScript Specification

Inputs:
- column: KanbanColumn (column configuration)
- tasks: Task[] (tasks in this column)
- allowDrop: boolean (drag-drop enabled)
- maxTasks: number (optional task limit)

Outputs:
- onTaskDrop: (taskId: string, columnId: string) => void (drop event)
- onAddTask: (columnId: string) => void (add task event)
- onTaskSelect: (taskId: string) => void (task selection event)

State:
- isDropTarget: boolean (drop zone active)
- taskCount: number (number of tasks)
- isCollapsed: boolean (column collapsed state)

Methods:
- handleDragOver(event: DragEvent) → void (drag over handler)
- handleDrop(event: DragEvent) → void (drop handler)
- toggleCollapse() → void (expand/collapse column)
- addTask() → void (trigger add task)

HTML STRUCTURE (PSEUDO CODE ONLY)

<Card className="kanban-column" role="region" aria-label={column.title}>
  <div className="column-header">
    <h2 id={`column-${column.id}`}>{column.title}</h2>
    <Badge variant="secondary">{taskCount}</Badge>
    <Button variant="ghost" onClick={toggleCollapse}>
      <Icon name={isCollapsed ? 'expand' : 'collapse'} />
    </Button>
  </div>
  
  <div className="column-content" 
       onDragOver={handleDragOver}
       onDrop={handleDrop}
       aria-labelledby={`column-${column.id}`}>
    {tasks.map(task => (
      <TaskCardComponent 
        key={task.id}
        task={task}
        draggable={true}
        onSelect={onTaskSelect}
      />
    ))}
    
    <AddTaskButtonComponent 
      columnId={column.id}
      onClick={addTask}
    />
  </div>
</Card>

CSS SPECIFICATION

Layout:
- display: flex
- flex-direction: column
- min-height: 500px
- max-height: 80vh
- overflow-y: auto

Spacing:
- padding: 1rem
- gap: 0.75rem between tasks

Component Styles:
- .kanban-column: column container with border
- .column-header: flex layout for title and actions
- .column-content: scrollable task area
- .drop-target: highlight when drag over

Responsive:
- Mobile: reduce padding to 0.5rem
- Tablet: adjust max-height to 70vh

API INTEGRATION

- No direct API calls (receives data from parent)
- Delegates task operations to parent component

ERROR HANDLING

- Invalid drop operation → show error message and revert
- Task limit exceeded → prevent drop and show warning
- Column load failure → display error state with retry

INTERACTION FLOW

User Action → UI → API
- Drop task → validate drop zone → call parent onTaskDrop → PUT /api/v1/tasks/{id}
- Click add task → call parent onAddTask → open CreateTaskModalComponent
- Click task → call parent onTaskSelect → navigate to TaskDetailComponent
