4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Page

Mapped From A1: Main Kanban board container with three-column layout

Purpose: Root component managing the entire Kanban board functionality

Parent: AppComponent

Children:
- HeaderComponent
- KanbanColumnComponent (3 instances)
- TaskModalComponent
- LoadingSpinnerComponent
- ErrorMessageComponent

Dependencies:
- Angular CDK Drag Drop
- RxJS for state management
- UI Library components

Library Components Used:
- Card → Container wrapper
- Button → Action buttons
- Modal → Task dialogs

TypeScript Specification

Inputs:
- boardId: string
- userId: string

Outputs:
- taskCreated: EventEmitter<Task>
- taskUpdated: EventEmitter<Task>
- taskDeleted: EventEmitter<string>

State:
- tasks: Task[]
- columns: Column[]
- isLoading: boolean
- error: string | null
- selectedTask: Task | null
- showTaskModal: boolean

Methods:
- loadTasks() → void
- createTask(task: Task) → void
- updateTask(task: Task) → void
- deleteTask(taskId: string) → void
- onDrop(event: CdkDragDrop) → void
- openTaskModal(task?: Task) → void
- closeTaskModal() → void

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container">
  <HeaderComponent 
    [user]="currentUser"
    (searchChanged)="onSearchChanged($event)"
    (profileClicked)="onProfileClicked()">
  </HeaderComponent>
  
  <main class="kanban-board" cdkDropListGroup>
    <KanbanColumnComponent 
      *ngFor="let column of columns"
      [column]="column"
      [tasks]="getTasksByColumn(column.id)"
      (taskClicked)="openTaskModal($event)"
      (taskDropped)="onDrop($event)">
    </KanbanColumnComponent>
  </main>
  
  <TaskModalComponent
    *ngIf="showTaskModal"
    [task]="selectedTask"
    (taskSaved)="onTaskSaved($event)"
    (modalClosed)="closeTaskModal()">
  </TaskModalComponent>
  
  <LoadingSpinnerComponent *ngIf="isLoading"></LoadingSpinnerComponent>
  <ErrorMessageComponent *ngIf="error" [message]="error"></ErrorMessageComponent>
</div>

CSS SPECIFICATION

Layout:
- .kanban-board { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; }
- .kanban-board-container { height: 100vh; display: flex; flex-direction: column; }

Spacing:
- padding: 1rem
- gap: 1rem between columns

Component Styles:
- .kanban-board-container
- .kanban-board
- .column-header
- .task-card

Responsive:
- Desktop (>1024px): 3 columns
- Tablet (768px-1024px): 2 columns
- Mobile (<768px): 1 column, vertical scroll

API INTEGRATION

- loadTasks() → GET /api/tasks → Fetch all tasks for board
- createTask() → POST /api/tasks → Create new task
- updateTask() → PUT /api/tasks/{id} → Update existing task
- deleteTask() → DELETE /api/tasks/{id} → Remove task

ERROR HANDLING

- API failure → show error message component
- Network timeout → retry mechanism with user notification
- Validation error → inline form validation messages

INTERACTION FLOW

User Action → UI → API
- Drag task → update column → PUT /api/tasks/{id}
- Click task → open modal → display task details
- Create task → submit form → POST /api/tasks

### 4.2 KanbanColumnComponent

Component Type: Layout

Mapped From A1: Individual column container with drag/drop functionality

Purpose: Manages tasks within a specific column (To Do, In Progress, Done)

Parent: KanbanBoardComponent

Children:
- KanbanCardComponent (multiple instances)

Dependencies:
- Angular CDK Drag Drop
- UI Library Card component

Library Components Used:
- Card → Column container
- Button → Add task button

TypeScript Specification

Inputs:
- column: Column
- tasks: Task[]

Outputs:
- taskClicked: EventEmitter<Task>
- taskDropped: EventEmitter<CdkDragDrop>
- addTaskClicked: EventEmitter<string>

State:
- draggedTask: Task | null

Methods:
- onTaskClick(task: Task) → void
- onDrop(event: CdkDragDrop) → void
- onAddTask() → void

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-column" 
     cdkDropList 
     [cdkDropListData]="tasks"
     (cdkDropListDropped)="onDrop($event)">
  <div class="column-header">
    <h3>{{ column.title }}</h3>
    <span class="task-count">{{ tasks.length }}</span>
    <Button 
      variant="primary" 
      size="small"
      (click)="onAddTask()">
      Add Task
    </Button>
  </div>
  
  <div class="column-content">
    <KanbanCardComponent
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      (clicked)="onTaskClick(task)"
      cdkDrag>
    </KanbanCardComponent>
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-column { display: flex; flex-direction: column; min-height: 500px; }
- .column-content { flex: 1; overflow-y: auto; }

Spacing:
- padding: 1rem
- gap: 0.5rem between cards

Component Styles:
- .kanban-column
- .column-header
- .column-content
- .task-count

Responsive:
- Mobile: full width, stack vertically
- Tablet/Desktop: equal column widths

API INTEGRATION

- onDrop() → PUT /api/tasks/{id} → Update task column
- onAddTask() → Trigger parent task creation modal

ERROR HANDLING

- Drop failure → revert task position
- API error → show error toast

INTERACTION FLOW

User Action → UI → API
- Drop task → update task column → API call
- Click add → open task modal → create task

### 4.4 TaskModalComponent

Component Type: Feature

Mapped From A1: Modal for task creation and editing

Purpose: Handle task form operations (create/edit)

Parent: KanbanBoardComponent

Children:
- DocumentUploadComponent

Dependencies:
- Angular Reactive Forms
- UI Library components

Library Components Used:
- Modal → Dialog container
- Form controls → Input fields
- Button → Action buttons

TypeScript Specification

Inputs:
- task: Task | null
- isVisible: boolean

Outputs:
- taskSaved: EventEmitter<Task>
- modalClosed: EventEmitter<void>

State:
- taskForm: FormGroup
- isSubmitting: boolean
- validationErrors: string[]

Methods:
- initForm() → void
- onSubmit() → void
- onCancel() → void
- validateForm() → boolean

HTML STRUCTURE (PSEUDO CODE ONLY)

<Modal 
  [isOpen]="isVisible"
  [title]="task ? 'Edit Task' : 'Create Task'"
  (closed)="onCancel()">
  
  <form [formGroup]="taskForm" (ngSubmit)="onSubmit()">
    <div class="form-group">
      <label for="title">Title *</label>
      <input 
        type="text" 
        id="title"
        formControlName="title"
        placeholder="Enter task title">
      <span class="error" *ngIf="taskForm.get('title')?.errors?.['required']">Title is required</span>
    </div>
    
    <div class="form-group">
      <label for="description">Description</label>
      <textarea 
        id="description"
        formControlName="description"
        placeholder="Enter task description">
      </textarea>
    </div>
    
    <div class="form-row">
      <div class="form-group">
        <label for="priority">Priority</label>
        <select id="priority" formControlName="priority">
          <option value="low">Low</option>
          <option value="medium">Medium</option>
          <option value="high">High</option>
        </select>
      </div>
      
      <div class="form-group">
        <label for="dueDate">Due Date</label>
        <input 
          type="date" 
          id="dueDate"
          formControlName="dueDate">
      </div>
    </div>
    
    <DocumentUploadComponent
      (fileUploaded)="onFileUploaded($event)">
    </DocumentUploadComponent>
    
    <div class="modal-actions">
      <Button 
        type="button" 
        variant="secondary"
        (click)="onCancel()">
        Cancel
      </Button>
      <Button 
        type="submit" 
        variant="primary"
        [disabled]="taskForm.invalid || isSubmitting"
        [loading]="isSubmitting">
        {{ task ? 'Update' : 'Create' }} Task
      </Button>
    </div>
  </form>
</Modal>

CSS SPECIFICATION

Layout:
- .form-group { margin-bottom: 1rem; }
- .form-row { display: flex; gap: 1rem; }

Spacing:
- padding: 1.5rem
- gap: 1rem between form elements

Component Styles:
- .form-group
- .form-row
- .modal-actions
- .error

Responsive:
- Mobile: single column form
- Desktop: two-column for priority/date

API INTEGRATION

- onSubmit() → POST /api/tasks (create) or PUT /api/tasks/{id} (update)
- File upload → POST /api/documents

ERROR HANDLING

- Form validation → inline error messages
- API failure → show error notification
- File upload error → display upload status

INTERACTION FLOW

User Action → UI → API
- Submit form → validate → API call → close modal
- Upload file → validate → API call → attach to task

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly with three-column CSS Grid layout
- Do NOT create new components unnecessarily - maximize reuse of UI library components
- Prefer reusable library components (Button, Card, Avatar, Modal, Form controls)
- HTML must be pseudo-code ONLY (NOT real framework code)
- Implement proper ARIA attributes for accessibility compliance (WCAG 2.1 AA)
- Use Angular CDK Drag Drop for task movement functionality
- Implement responsive breakpoints: Desktop (>1024px), Tablet (768px-1024px), Mobile (<768px)
- Follow Angular best practices for component structure and naming
- Implement proper error handling and loading states
- Use reactive forms for task creation/editing
- Implement debounced search functionality
- Ensure proper TypeScript typing for all props and state
- Follow security best practices for file uploads and API calls
- Implement proper keyboard navigation for accessibility
- Use CSS Grid for main layout, Flexbox for component layouts
- Implement proper drag and drop visual feedback
- Follow consistent naming conventions across all components
- Implement proper form validation with user-friendly error messages
- Use trackBy functions for ngFor loops to optimize performance
- Implement proper cleanup in component destroy lifecycle
- Follow Angular style guide for file organization and naming
