### 4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: KanbanBoardComponent - Main container for three-column Kanban layout

Purpose: Renders the main Kanban board with three columns (To Do, In Progress, Done) and handles drag-and-drop functionality

Parent: KanbanBoardPage

Children:
- KanbanColumnComponent (3 instances)
- TaskModalComponent

Dependencies:
- React DnD for drag-and-drop
- API service for data fetching

Library Components Used:
- Button → Add Task button

TypeScript Specification:

Inputs:
- columns: KanbanColumn[]
- loading: boolean
- error: string

Outputs:
- onTaskMoved(taskId: string, sourceColumn: string, targetColumn: string)
- onTaskCreated(task: Task)
- onTaskUpdated(task: Task)
- onTaskDeleted(taskId: string)

State:
- draggedTask: Task | null
- activeColumn: string
- isModalOpen: boolean
- modalMode: 'create' | 'edit'
- selectedTask: Task | null

Methods:
- handleTaskDrop(taskId: string, columnId: string) → Updates task status
- openTaskModal(mode: string, task?: Task) → Opens modal for task creation/editing
- closeTaskModal() → Closes task modal
- loadTasks() → Fetches tasks from API

HTML STRUCTURE (PSEUDO CODE ONLY):

<div className="kanban-board">
  <div className="kanban-header">
    <h1>Kanban Board</h1>
    <Button onClick={openCreateModal}>Add Task</Button>
  </div>
  
  <div className="kanban-columns" role="application" aria-label="Kanban board">
    {columns.map(column => (
      <KanbanColumnComponent 
        key={column.id}
        column={column}
        tasks={getTasksForColumn(column.id)}
        onTaskDrop={handleTaskDrop}
        onAddTask={openTaskModal}
      />
    ))}
  </div>
  
  {isModalOpen && (
    <TaskModalComponent
      isOpen={isModalOpen}
      mode={modalMode}
      task={selectedTask}
      onSave={handleTaskSave}
      onClose={closeTaskModal}
    />
  )}
</div>

CSS SPECIFICATION:

Layout:
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1rem
- min-height: 100vh

Spacing:
- padding: 1rem
- gap: 1rem between columns

Component Styles:
- .kanban-board: main container
- .kanban-columns: grid layout for columns
- .kanban-header: flex layout for title and actions

Responsive:
- Desktop (>1024px): 3 columns side by side
- Tablet (768px-1024px): 2 columns with scroll
- Mobile (<768px): 1 column with tabs

API INTEGRATION:
- loadTasks() → GET /api/youth-accounts/{id}/dashboard → Fetch initial task data
- handleTaskSave() → POST /api/youth-accounts/{id}/fund-transfer → Create new task
- handleTaskUpdate() → PUT /api/youth-accounts/{id}/spending-limit → Update task status

ERROR HANDLING:
- API failure → show error snackbar with retry option
- Validation error → inline form validation messages
- Network error → offline mode indicator

INTERACTION FLOW:

User Action → UI → API
- Click "Add Task" → open TaskModalComponent → submit form → POST /api/fund-transfer
- Drag task card → update visual state → drop in column → PUT /api/spending-limit
- Load board → show loading spinner → GET /api/dashboard → render tasks

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: KanbanColumnComponent - Individual column container (To Do, In Progress, Done)

Purpose: Renders individual Kanban column with tasks and handles drop zone functionality

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)

Dependencies:
- React DnD for drop zone
- Task filtering logic

Library Components Used:
- Button → Add task to column button

TypeScript Specification:

Inputs:
- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean

Outputs:
- onTaskDropped(taskId: string, columnId: string)
- onAddTaskClicked(columnId: string)

State:
- isDropTarget: boolean
- taskCount: number
- isCollapsed: boolean

Methods:
- handleDrop(item: DragItem) → Processes task drop
- handleAddTask() → Triggers add task for this column
- toggleCollapse() → Collapses/expands column

HTML STRUCTURE (PSEUDO CODE ONLY):

<div className="kanban-column" data-column-id={column.id}>
  <div className="column-header">
    <h2>{column.title}</h2>
    <span className="task-count">{taskCount}</span>
    <Button size="small" onClick={handleAddTask}>+</Button>
  </div>
  
  <div className="column-content" 
       role="region" 
       aria-label={`${column.title} tasks`}
       aria-dropeffect="move">
    {tasks.map(task => (
      <TaskCardComponent
        key={task.id}
        task={task}
        draggable={true}
        onDragStart={handleTaskDragStart}
      />
    ))}
    
    {tasks.length === 0 && (
      <div className="empty-column">
        <p>No tasks in {column.title}</p>
        <Button onClick={handleAddTask}>Add First Task</Button>
      </div>
    )}
  </div>
</div>

CSS SPECIFICATION:

Layout:
- display: flex
- flex-direction: column
- min-height: 500px
- flex: 1

Spacing:
- padding: 1rem
- margin-bottom: 1rem

Component Styles:
- .kanban-column: column container with border
- .column-header: flex layout for title and actions
- .column-content: scrollable task area
- .empty-column: centered empty state

Responsive:
- Mobile: full width with horizontal scroll
- Tablet: 50% width
- Desktop: 33.33% width

API INTEGRATION:
- None (receives data from parent KanbanBoardComponent)

ERROR HANDLING:
- Drop validation → prevent invalid drops with visual feedback
- Task loading error → show column-level error message

INTERACTION FLOW:

User Action → UI → API
- Drop task → validate drop zone → update column state → notify parent
- Click add task → trigger parent add task method → open modal
- Scroll tasks → lazy load more tasks if needed

### 4.4 TaskModalComponent

Component Type: Feature

Mapped From A1: TaskModalComponent - Create/edit task dialog

Purpose: Modal dialog for creating new tasks or editing existing tasks

Parent: KanbanBoardComponent

Children: None

Dependencies:
- Form validation library
- Modal/Dialog component

Library Components Used:
- Button → Save, Cancel, Delete buttons
- Dropdown → Priority, Status, Assignee dropdowns
- Autocomplete → Tag selection

TypeScript Specification:

Inputs:
- isOpen: boolean
- task: Task | null
- mode: 'create' | 'edit'

Outputs:
- onTaskSaved(task: Task)
- onModalClosed()

State:
- formData: TaskFormData
- isSubmitting: boolean
- validationErrors: ValidationError[]
- isDirty: boolean

Methods:
- handleSubmit() → Validates and saves task
- handleCancel() → Closes modal with confirmation if dirty
- handleFieldChange(field: string, value: any) → Updates form data
- validateForm() → Validates all form fields

HTML STRUCTURE (PSEUDO CODE ONLY):

<div className="modal-overlay" onClick={handleOverlayClick}>
  <div className="task-modal" role="dialog" aria-labelledby="modal-title">
    <div className="modal-header">
      <h2 id="modal-title">{mode === 'create' ? 'Create Task' : 'Edit Task'}</h2>
      <Button variant="ghost" onClick={handleCancel}>×</Button>
    </div>
    
    <form className="modal-content" onSubmit={handleSubmit}>
      <div className="form-group">
        <label htmlFor="task-title">Title *</label>
        <input 
          id="task-title"
          type="text"
          value={formData.title}
          onChange={handleTitleChange}
          required
        />
        {validationErrors.title && (
          <span className="error">{validationErrors.title}</span>
        )}
      </div>
      
      <div className="form-group">
        <label htmlFor="task-description">Description</label>
        <textarea 
          id="task-description"
          value={formData.description}
          onChange={handleDescriptionChange}
        />
      </div>
      
      <div className="form-row">
        <div className="form-group">
          <label htmlFor="task-priority">Priority</label>
          <Dropdown
            id="task-priority"
            options={priorityOptions}
            value={formData.priority}
            onChange={handlePriorityChange}
          />
        </div>
        
        <div className="form-group">
          <label htmlFor="task-assignee">Assignee</label>
          <Autocomplete
            id="task-assignee"
            options={userOptions}
            value={formData.assignee}
            onChange={handleAssigneeChange}
          />
        </div>
      </div>
      
      <div className="form-group">
        <label htmlFor="task-tags">Tags</label>
        <Autocomplete
          id="task-tags"
          multiple={true}
          options={tagOptions}
          value={formData.tags}
          onChange={handleTagsChange}
        />
      </div>
    </form>
    
    <div className="modal-footer">
      <Button variant="secondary" onClick={handleCancel}>Cancel</Button>
      {mode === 'edit' && (
        <Button variant="danger" onClick={handleDelete}>Delete</Button>
      )}
      <Button 
        variant="primary" 
        onClick={handleSubmit}
        disabled={isSubmitting}
      >
        {isSubmitting ? 'Saving...' : 'Save Task'}
      </Button>
    </div>
  </div>
</div>

Layout:
- position: fixed
- top: 0, left: 0, right: 0, bottom: 0
- display: flex
- justify-content: center
- align-items: center

Spacing:
- modal padding: 2rem
- form group margin: 1rem

Component Styles:
- .modal-overlay: semi-transparent background
- .task-modal: centered modal container
- .modal-header: flex layout with title and close
- .modal-content: form layout
- .modal-footer: flex layout for actions

Responsive:
- Mobile: full screen modal
- Desktop: centered modal with max-width

API INTEGRATION:
- handleSubmit() → POST /api/youth-accounts/{id}/fund-transfer → Create new task
- handleSubmit() → PUT /api/youth-accounts/{id}/spending-limit → Update existing task
- handleDelete() → DELETE /api/youth-accounts/{id}/task → Delete task

ERROR HANDLING:
- API failure → show error message in modal
- Validation error → inline field validation
- Network error → retry mechanism

INTERACTION FLOW:

User Action → UI → API
- Fill form and submit → validate fields → POST/PUT API → close modal on success

- Click delete → confirm action → DELETE API → close modal

### 8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components (Button, Dropdown, Autocomplete, Carousel)
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- All components must include proper ARIA attributes for accessibility
- Error handling must be consistent across all components
- API transformations required to map youth account APIs to task management
- Responsive design must support mobile, tablet, and desktop breakpoints
- Drag-and-drop functionality requires React DnD or similar library
- Form validation must be implemented for task creation/editing
- Search functionality should include autocomplete suggestions
- Theme switching (light/dark mode) should be supported
- Navigation state must be persisted across page refreshes
