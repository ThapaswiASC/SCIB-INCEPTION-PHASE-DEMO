### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main kanban board container requirement

Purpose: Main container for three-column kanban layout with drag/drop functionality

Parent: AppComponent

Children:
- KanbanColumnComponent (3 instances)
- TaskModalComponent
- ErrorStateComponent

Dependencies:
- KanbanService
- TaskService
- DragDropModule

Library Components Used:
- None directly → Uses child components that consume library components

TypeScript Specification

Inputs:
- youthAccountId: string
- initialColumns: KanbanColumn[]

Outputs:
- taskMoved: EventEmitter<TaskMoveEvent> (payload: {taskId, fromColumn, toColumn})
- taskCreated: EventEmitter<Task> (payload: Task)
- errorOccurred: EventEmitter<string> (payload: error message)

State:
- columns: KanbanColumn[] = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- loading: boolean = false
- error: string | null = null
- draggedTask: Task | null = null

Methods:
- ngOnInit() → Initialize component and load data
- loadKanbanData() → Fetch kanban board data from API
- onTaskMoved(event: TaskMoveEvent) → Handle task drag/drop between columns
- onTaskCreated(task: Task) → Handle new task creation
- onAddTaskClicked(columnId: string) → Open task creation modal
- handleError(error: any) → Process and display errors

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="kanban-header">
    <h2>Kanban Board</h2>
    <div class="board-actions">
      <!-- Action buttons -->
    </div>
  </div>
  
  <div class="kanban-grid" *ngIf="!loading && !error">
    <app-kanban-column 
      *ngFor="let column of columns" 
      [column]="column"
      [tasks]="column.cards"
      [columnId]="column.id"
      (taskDropped)="onTaskMoved($event)"
      (addTaskClicked)="onAddTaskClicked($event)"
      [attr.aria-labelledby]="column.id + '-header'">
    </app-kanban-column>
  </div>
  
  <app-error-state 
    *ngIf="error" 
    [message]="error"
    (retry)="loadKanbanData()">
  </app-error-state>
  
  <app-task-modal
    [isOpen]="isModalOpen"
    [defaultStatus]="selectedColumnId"
    (taskSaved)="onTaskCreated($event)"
    (modalClosed)="closeModal()">
  </app-task-modal>
</div>

CSS SPECIFICATION

Layout:
- .kanban-grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; }
- .kanban-board-container { height: 100vh; display: flex; flex-direction: column; }

Spacing:
- padding: 1rem
- gap: 1rem between columns

Component Styles:
- .kanban-header { display: flex; justify-content: space-between; align-items: center; }
- .board-actions { display: flex; gap: 0.5rem; }

Responsive:
- Desktop (>1024px): 3 columns side by side
- Tablet (768px-1024px): 2 columns with scroll
- Mobile (<768px): 1 column with tabs

API INTEGRATION

- loadKanbanData() → GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch initial kanban data
- onTaskCreated() → POST /api/youth-accounts/{youthAccountId}/fund-transfer → Create new task
- onTaskMoved() → PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update task status

ERROR HANDLING

- API failure → show error state component with retry option
- Network timeout → display "Connection lost" message
- Validation error → show inline error messages
- Drag/drop failure → revert task position and show snackbar

INTERACTION FLOW

User Action → UI → API
- Load page → ngOnInit() → GET dashboard API → populate columns
- Drag task → update local state → PUT API call → confirm or rollback
- Click "Add Task" → open modal → submit form → POST API → update board
- Error occurs → display error state → user clicks retry → reload data
