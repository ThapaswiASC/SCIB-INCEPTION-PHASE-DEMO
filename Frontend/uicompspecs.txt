### 4.1 TaskCreationComponent

Component Type: Feature

Mapped From A1: TaskCreationComponent (Main container for task creation form)

Purpose: Main modal container for task creation with form validation and API integration

Parent: KanbanBoardComponent

Children:
- TaskFormComponent
- LoadingSpinnerComponent

Dependencies:
- TaskService
- FormBuilder
- Validators

Library Components Used:
- Modal → Container for form display
- Button → Close and action buttons

TypeScript Specification

Inputs:
- isOpen: boolean (Modal visibility state)
- onClose: () => void (Close modal callback)
- onTaskCreated: (task: Task) => void (Success callback)

Outputs:
- taskCreated: EventEmitter<Task> (Emits created task)
- formClosed: EventEmitter<void> (Emits on modal close)

State:
- formData: TaskFormData (Current form data)
- isLoading: boolean (API call state)
- validationErrors: ValidationError[] (Form errors)
- isSubmitting: boolean (Submission state)

Methods:
- openModal() → Opens task creation modal
- closeModal() → Closes modal and resets form
- handleFormSubmit(data: TaskFormData) → Processes form submission
- handleApiError(error: ApiError) → Handles API errors

HTML STRUCTURE (PSEUDO CODE ONLY)

<modal [isOpen]="isOpen" (onClose)="closeModal()">
  <modal-header>
    <h2>Create New Task</h2>
    <button (click)="closeModal()">×</button>
  </modal-header>
  
  <modal-body>
    <task-form-component 
      [initialData]="formData"
      [isLoading]="isLoading"
      (formSubmit)="handleFormSubmit($event)"
      (formCancel)="closeModal()">
    </task-form-component>
  </modal-body>
  
  <loading-spinner *ngIf="isLoading"></loading-spinner>
</modal>

CSS SPECIFICATION

Layout:
- position: fixed (Modal overlay)
- display: flex (Center modal)
- z-index: 1000 (Above other content)

Spacing:
- padding: 24px (Modal content)
- gap: 16px (Between sections)

Component Styles:
- .task-creation-modal (Main modal container)
- .modal-overlay (Background overlay)
- .modal-content (Content container)

Responsive:
- Mobile: Full screen modal
- Tablet: 80% width, centered
- Desktop: 600px max-width, centered

API INTEGRATION

- handleFormSubmit() → POST /tasks → Create new task
- handleApiError() → Error handling for API failures

ERROR HANDLING

- API failure → Display error snackbar with retry option
- Validation error → Show inline field errors
- Network error → Show connection error message

INTERACTION FLOW

User Action → UI → API

- Click "Add Task" → Open modal → Initialize empty form
- Fill form fields → Real-time validation → Update character counters
- Submit form → Show loading → POST /tasks → Handle response
- Success → Close modal → Emit taskCreated event → Reset form
- Error → Show error message → Keep modal open

### 4.2 TaskFormComponent

Component Type: Feature

Mapped From A1: TaskFormComponent (Form container with reactive forms)

Purpose: Reactive form container with validation and real-time feedback

Parent: TaskCreationComponent

Children:
- TitleInputComponent
- DescriptionInputComponent
- PrioritySelectComponent
- DueDatePickerComponent
- FormActionsComponent

Dependencies:
- FormBuilder
- Validators
- CustomValidators

Library Components Used:
- Form → Reactive form container

TypeScript Specification

Inputs:
- initialData: Partial<TaskFormData> (Pre-filled form data)
- isLoading: boolean (Loading state from parent)
- onSubmit: (data: TaskFormData) => void (Submit callback)
- onCancel: () => void (Cancel callback)

Outputs:
- formSubmit: EventEmitter<TaskFormData> (Valid form data)
- formCancel: EventEmitter<void> (Cancel action)
- formChange: EventEmitter<TaskFormData> (Form value changes)

State:
- taskForm: FormGroup (Reactive form group)
- isDirty: boolean (Form modification state)
- isValid: boolean (Form validation state)
- touchedFields: Set<string> (Touched field tracking)

Methods:
- initializeForm() → Creates reactive form with validators
- validateForm() → Runs form validation
- resetForm() → Resets form to initial state
- handleFieldChange(field: string, value: any) → Processes field changes

HTML STRUCTURE (PSEUDO CODE ONLY)

<form [formGroup]="taskForm" (ngSubmit)="onFormSubmit()">
  <div class="form-grid">
    <title-input-component
      formControlName="title"
      [maxLength]="255"
      [required]="true">
    </title-input-component>
    
    <description-input-component
      formControlName="description"
      [maxLength]="2000">
    </description-input-component>
    
    <priority-select-component
      formControlName="priority"
      [options]="priorityOptions"
      [required]="true">
    </priority-select-component>
    
    <due-date-picker-component
      formControlName="dueDate"
      [minDate]="today"
      [required]="true">
    </due-date-picker-component>
  </div>
  
  <form-actions-component
    [isValid]="taskForm.valid"
    [isLoading]="isLoading"
    (submit)="onFormSubmit()"
    (cancel)="onFormCancel()">
  </form-actions-component>
</form>

CSS SPECIFICATION

Layout:
- display: grid (Form field layout)
- grid-template-columns: 1fr (Single column)
- gap: 20px (Between form fields)

Spacing:
- padding: 20px (Form container)
- margin-bottom: 16px (Between sections)

Component Styles:
- .task-form (Main form container)
- .form-grid (Grid layout for fields)
- .form-section (Grouped form fields)

Responsive:
- Mobile: Single column layout
- Tablet: Two column for shorter fields
- Desktop: Optimized field widths

API INTEGRATION

- No direct API calls (Pure form component)
- Validates against API schema requirements

ERROR HANDLING

- Field validation → Show inline error messages
- Form submission → Prevent if invalid
- Required field → Highlight missing fields

INTERACTION FLOW

User Action → UI → API

- Field input → Real-time validation → Update UI state
- Form submit → Validate all fields → Emit form data
- Validation error → Show field errors → Prevent submission

### 4.3 TitleInputComponent

Component Type: Shared

Mapped From A1: TitleInputComponent (Task title input with validation)

Purpose: Task title input field with character counter and validation

Parent: TaskFormComponent

Children:
- CharacterCounterComponent

Dependencies:
- Validators

Library Components Used:
- Textbox → Input field with validation states

TypeScript Specification

Inputs:
- value: string (Current input value)
- maxLength: number (255 characters max)
- required: boolean (Required field flag)
- placeholder: string (Placeholder text)
- disabled: boolean (Disabled state)

Outputs:
- valueChange: EventEmitter<string> (Input value changes)
- validationChange: EventEmitter<ValidationState> (Validation state)

State:
- characterCount: number (Current character count)
- validationState: 'default' | 'error' | 'success' (Validation state)
- errorMessage: string (Current error message)

Methods:
- onInputChange(value: string) → Handles input changes
- validateInput(value: string) → Validates input value
- updateCharacterCount(value: string) → Updates character counter

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="title-input-container">
  <label for="task-title">Task Title *</label>
  
  <textbox
    id="task-title"
    [value]="value"
    [placeholder]="placeholder"
    [maxLength]="maxLength"
    [required]="required"
    [disabled]="disabled"
    [validationState]="validationState"
    (valueChange)="onInputChange($event)">
  </textbox>
  
  <div class="input-footer">
    <span class="error-message" *ngIf="validationState === 'error'">
      {{ errorMessage }}
    </span>
    
    <character-counter
      [current]="characterCount"
      [max]="maxLength"
      [showWarning]="characterCount > maxLength * 0.9">
    </character-counter>
  </div>
</div>

CSS SPECIFICATION

Layout:
- display: flex (Column layout)
- flex-direction: column

Spacing:
- margin-bottom: 16px (Between components)
- gap: 8px (Between label and input)

Component Styles:
- .title-input-container (Main container)
- .input-footer (Counter and error area)
- .error-message (Error text styling)

Responsive:
- Full width on all breakpoints
- Consistent spacing across devices

API INTEGRATION

- None (Pure input component)

ERROR HANDLING

- Required validation → "Title is required"
- Max length → "Title cannot exceed 255 characters"
- Empty input → Show required error

INTERACTION FLOW

User Action → UI → API

- Type in field → Update character count → Validate input
- Exceed max length → Show warning → Prevent further input
- Clear field → Show required error → Update validation state

Inputs:

- closeCalendar() → Closes date picker calendar

HTML STRUCTURE (PSEUDO CODE ONLY)

### 4.7 FormActionsComponent

Component Type: Shared

Mapped From A1: FormActionsComponent (Submit and cancel buttons)

Purpose: Form action buttons with loading states

Parent: TaskFormComponent

Children:
- LoadingSpinnerComponent

Dependencies:
- None

Library Components Used:
- Button → Primary button for submit
- Button → Secondary button for cancel

TypeScript Specification

Inputs:
- isValid: boolean (Form validation state)
- isLoading: boolean (Loading state)
- submitText: string (Submit button text, default "Create Task")
- cancelText: string (Cancel button text, default "Cancel")
- disabled: boolean (Disabled state)

Outputs:
- submit: EventEmitter<void> (Submit button click)
- cancel: EventEmitter<void> (Cancel button click)

State:
- isSubmitDisabled: boolean (Submit button state)

Methods:
- onSubmitClick() → Handles submit button click
- onCancelClick() → Handles cancel button click
- getSubmitButtonState() → Determines submit button state

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="form-actions">
  <button
    type="button"
    variant="secondary"
    [disabled]="isLoading"
    (click)="onCancelClick()">
    {{ cancelText }}
  </button>
  
  <button
    type="submit"
    variant="primary"
    [disabled]="!isValid || isLoading"
    (click)="onSubmitClick()">
    
    <loading-spinner 
      *ngIf="isLoading" 
      size="small" 
      color="white">
    </loading-spinner>
    
    <span *ngIf="!isLoading">{{ submitText }}</span>
    <span *ngIf="isLoading">Creating...</span>
  </button>
</div>

CSS SPECIFICATION

Layout:
- display: flex (Horizontal layout)
- justify-content: flex-end (Right alignment)
- gap: 12px (Between buttons)

Spacing:
- margin-top: 24px (Above form fields)
- padding: 16px 0 (Vertical padding)

Component Styles:
- .form-actions (Main container)
- .submit-loading (Loading state)
- .button-group (Button alignment)

Responsive:
- Stack vertically on mobile
- Horizontal on tablet and desktop
- Full width buttons on mobile

API INTEGRATION

- None (Pure action component)

ERROR HANDLING

- Disabled submit when form invalid
- Prevent double submission during loading

INTERACTION FLOW

User Action → UI → API

- Click submit → Validate form → Emit submit event
- Click cancel → Emit cancel event → Close form
- Form invalid → Disable submit button
- Loading state → Show spinner → Disable buttons
