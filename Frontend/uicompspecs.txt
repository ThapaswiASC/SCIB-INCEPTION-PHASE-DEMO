### 4. COMPONENT IMPLEMENTATION

#### 4.1 KanbanBoardComponent

**Component Type:** Feature

**Mapped From A1:** KanbanBoardComponent (main container for three-column Kanban layout)

**Purpose:** Main container for three-column Kanban layout with drag/drop functionality

**Parent:** AppComponent

**Children:**
- KanbanColumnComponent (3 instances)
- MobileTabsComponent
- LoadingSpinnerComponent

**Dependencies:**
- KanbanService
- DragDropModule
- CommonModule

**Library Components Used:**
- Button → Add Task functionality
- LoadingSpinner → Loading states

**TypeScript Specification**

**Inputs:**
- columns: KanbanColumn[] (array of column data)
- loading: boolean (loading state)
- error: string | null (error message)

**Outputs:**
- taskMoved: EventEmitter<TaskMoveEvent> (task moved between columns)
- taskCreated: EventEmitter<Task> (new task created)
- taskUpdated: EventEmitter<Task> (task updated)
- taskDeleted: EventEmitter<string> (task deleted by ID)

**State:**
- draggedTask: Task | null (currently dragged task)
- columnData: KanbanColumn[] (column data with tasks)
- isLoading: boolean (loading state)
- errorMessage: string | null (error message)
- isMobileView: boolean (responsive state)

**Methods:**
- ngOnInit() → Initialize component and load data
- loadKanbanData() → Fetch kanban board data
- onTaskDrop(event: CdkDragDrop) → Handle task drop events
- onAddTask() → Open task creation modal
- onTaskEdit(task: Task) → Open task editing modal
- handleError(error: any) → Process and display errors

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-board-container" [attr.aria-label]="'Kanban board with ' + columns.length + ' columns'">
  <div class="mobile-tabs" *ngIf="isMobileView">
    <app-mobile-tabs [columns]="columnData" (columnSelected)="onMobileColumnSelect($event)"></app-mobile-tabs>
  </div>
  
  <div class="kanban-grid" cdkDropListGroup>
    <app-kanban-column 
      *ngFor="let column of columnData; trackBy: trackByColumnId"
      [column]="column"
      [tasks]="column.tasks"
      [allowDrop]="!isLoading"
      (taskDropped)="onTaskDrop($event)"
      (addTaskClicked)="onAddTask($event)">
    </app-kanban-column>
  </div>
  
  <div class="loading-overlay" *ngIf="isLoading">
    <app-loading-spinner></app-loading-spinner>
  </div>
  
  <div class="error-state" *ngIf="errorMessage" role="alert">
    <p>{{ errorMessage }}</p>
    <app-button variant="primary" (click)="loadKanbanData()">Retry</app-button>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: grid
- grid-template-columns: repeat(3, 1fr)
- gap: 1rem
- min-height: 100vh

**Spacing:**
- padding: 1rem
- gap: 1rem between columns

**Component Styles:**
- .kanban-board-container: main container
- .kanban-grid: CSS Grid layout
- .loading-overlay: centered loading state
- .error-state: error display with retry

**Responsive:**
- Desktop (>1024px): 3-column grid
- Tablet (768px-1024px): 2-column grid with scrollable third
- Mobile (<768px): single column with tabs

**API INTEGRATION**

- loadKanbanData() → GET /api/youth-accounts/{id}/dashboard → Fetch initial kanban data
- onTaskCreate() → POST /api/youth-accounts/{id}/fund-transfer → Create new task
- onTaskUpdate() → PUT /api/youth-accounts/{id}/spending-limit → Update task status

**ERROR HANDLING**

- API failure → show error state with retry button
- Network timeout → display "Connection lost" message
- Validation error → show inline validation messages
- Drag/drop failure → revert task position and show snackbar

**INTERACTION FLOW**

**User Action → UI → API**
- Drag task → update visual state → drop in column → call PUT API → update state
- Click "Add Task" → open modal → submit form → call POST API → refresh board
- Load board → show loading spinner → call GET API → populate columns

#### 4.2 KanbanColumnComponent

**Component Type:** Feature

**Mapped From A1:** KanbanColumnComponent (individual column container)

**Purpose:** Individual column container (To Do, In Progress, Done) with drop zone

**Parent:** KanbanBoardComponent

**Children:**
- TaskCardComponent (multiple instances)
- Button (Add Task)

**Dependencies:**
- DragDropModule
- CommonModule

**Library Components Used:**
- Button → Add task to column

**TypeScript Specification**

**Inputs:**
- column: KanbanColumn (column configuration)
- tasks: Task[] (tasks in this column)
- allowDrop: boolean (enable/disable drop)

**Outputs:**
- taskDropped: EventEmitter<TaskDropEvent> (task dropped in column)
- addTaskClicked: EventEmitter<string> (add task clicked with column ID)

**State:**
- isDragOver: boolean (drag over state)
- taskCount: number (number of tasks)
- isCollapsed: boolean (column collapsed state)

**Methods:**
- onTaskDrop(event: CdkDragDrop) → Handle task drop
- onAddTask() → Emit add task event
- toggleCollapse() → Toggle column collapse
- trackByTaskId(index: number, task: Task) → Track function for *ngFor

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-column" 
     [class.drag-over]="isDragOver"
     [class.collapsed]="isCollapsed"
     cdkDropList
     [cdkDropListData]="tasks"
     (cdkDropListDropped)="onTaskDrop($event)">
  
  <div class="column-header">
    <h3 class="column-title">{{ column.title }}</h3>
    <span class="task-count">{{ taskCount }}</span>
    <app-button variant="ghost" size="small" (click)="toggleCollapse()">
      {{ isCollapsed ? 'Expand' : 'Collapse' }}
    </app-button>
  </div>
  
  <div class="column-content" [hidden]="isCollapsed">
    <app-task-card
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="allowDrop"
      cdkDrag>
    </app-task-card>
    
    <div class="add-task-zone">
      <app-button variant="secondary" (click)="onAddTask()">
        + Add Task
      </app-button>
    </div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: flex
- flex-direction: column
- min-height: 500px
- background: #f8f9fa

**Spacing:**
- padding: 1rem
- gap: 0.5rem between tasks

**Component Styles:**
- .kanban-column: main column container
- .column-header: flex header with title and count
- .column-content: scrollable task area
- .drag-over: highlight when dragging over

**Responsive:**
- Mobile: full width with margin
- Tablet/Desktop: flex-grow within grid

**API INTEGRATION**

- None (receives data from parent)

**ERROR HANDLING**

- Drop failure → revert visual state
- Invalid task data → show placeholder

**INTERACTION FLOW**

**User Action → UI → API**
- Drop task → highlight drop zone → validate drop → emit event to parent
- Click add task → emit column ID to parent → parent opens modal

#### 4.4 HeaderComponent

**Component Type:** Layout

**Mapped From A1:** HeaderComponent (top navigation)

**Purpose:** Top navigation with breadcrumb, search, and user menu

**Children:**

            placeholder="Add tags..."
            (keyup.enter)="addTag($event)"
            class="form-input">
        </div>
        <div class="tag-list">
          <app-task-tag 
            *ngFor="let tag of formData.get('tags')?.value"
            [tag]="tag"
            [removable]="true"
            (removed)="removeTag(tag)">
          </app-task-tag>
        </div>
      </div>
    </form>
    
    <div class="modal-footer">
      <app-button variant="secondary" (click)="onCancel()" [disabled]="isSubmitting">
        Cancel
      </app-button>
      <app-button 
        variant="primary" 
        (click)="onSubmit()"
        [disabled]="formData.invalid || isSubmitting"
        [loading]="isSubmitting">
        {{ mode === 'create' ? 'Create Task' : 'Update Task' }}
      </app-button>
    </div>
  </div>
</div>
