### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for the three-column kanban layout with drag-and-drop functionality

Purpose: Orchestrates the entire kanban board experience, manages task state, and handles drag-and-drop operations

Parent: KanbanPageComponent

Children:
- KanbanColumnComponent (3 instances)
- MobileTabsComponent (mobile view)

Dependencies:
- Angular CDK Drag Drop
- KanbanService
- TaskService

Library Components Used:
- ButtonComponent → Add Task button

TypeScript Specification

Inputs:
- columns: KanbanColumn[] (column configuration)
- tasks: Task[] (all tasks data)
- loading: boolean (loading state)
- error: string (error message)

Outputs:
- taskMoved: EventEmitter<TaskMoveEvent> (when task moved between columns)
- taskCreated: EventEmitter<Task> (when new task created)
- taskUpdated: EventEmitter<Task> (when task updated)

State:
- draggedTask: Task | null (currently dragged task)
- columnData: KanbanColumn[] (processed column data with tasks)
- isMobile: boolean (responsive state)

Methods:
- ngOnInit() → Initialize component and load data
- loadKanbanData() → Fetch initial kanban data
- onTaskDrop(event: CdkDragDrop<Task[]>) → Handle task drop events
- onAddTask(columnId: string) → Open task creation modal
- updateTaskStatus(taskId: string, newStatus: string) → Update task status

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="kanban-header">
    <h1>Kanban Board</h1>
    <app-button variant="primary" (click)="onAddTask()">Add Task</app-button>
  </div>
  
  <div class="mobile-tabs" *ngIf="isMobile">
    <app-mobile-tabs [columns]="columnData" (columnSelected)="onColumnSelected($event)"></app-mobile-tabs>
  </div>
  
  <div class="kanban-columns" cdkDropListGroup>
    <app-kanban-column 
      *ngFor="let column of columnData" 
      [column]="column" 
      [tasks]="getTasksForColumn(column.id)"
      (taskDropped)="onTaskDrop($event)"
      (addTaskClicked)="onAddTask($event)">
    </app-kanban-column>
  </div>
  
  <div class="error-state" *ngIf="error">
    <p role="alert">{{error}}</p>
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-board-container { display: flex; flex-direction: column; height: 100vh; }
- .kanban-columns { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; flex: 1; }

Spacing:
- padding: 1rem
- gap: 1rem between columns

Component Styles:
- .kanban-header { display: flex; justify-content: space-between; align-items: center; }
- .error-state { background: #fee; color: #c33; padding: 1rem; border-radius: 4px; }

Responsive:
- Desktop (>1024px): 3-column grid
- Tablet (768px-1024px): 2-column grid with horizontal scroll
- Mobile (<768px): Single column with tabs

API INTEGRATION

- loadKanbanData() → GET /api/youth-accounts/{id}/dashboard → Load initial kanban data
- updateTaskStatus() → PUT /api/youth-accounts/{id}/spending-limit → Update task status

ERROR HANDLING

- API failure → show error message in error-state div
- Drag drop failure → revert task position and show snackbar
- Column configuration failure → display fallback layout

INTERACTION FLOW

User Action → UI → API
- Drag task → update UI optimistically → call API to persist → revert on failure
- Click Add Task → open modal → submit form → API call → refresh board
- Mobile column switch → update visible column → no API call
