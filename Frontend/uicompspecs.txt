### 4. COMPONENT IMPLEMENTATION

#### 4.1 KanbanBoardComponent

**Component Type:** Page

**Mapped From A1:** Main container for Kanban board application

**Purpose:** Root component that orchestrates the entire Kanban board functionality with three-column layout

**Parent:** AppComponent

**Children:**
- HeaderComponent
- SidebarComponent
- PageHeaderComponent
- KanbanContainerComponent
- TaskModalComponent

**Dependencies:**
- @angular/core
- @angular/common
- KanbanService
- TaskService

**Library Components Used:**
- HeaderComponent → app-header
- SidebarComponent → app-sidebar
- ButtonComponent → app-button

**TypeScript Specification**

**Inputs:**
- youthAccountId: string
- permissions: UserPermissions
- initialColumns: KanbanColumn[]

**Outputs:**
- taskCreated: EventEmitter<Task> (payload: Task)
- taskUpdated: EventEmitter<Task> (payload: Task)
- taskDeleted: EventEmitter<string> (payload: taskId)
- columnChanged: EventEmitter<ColumnChangeEvent> (payload: ColumnChangeEvent)

**State:**
- columns: KanbanColumn[]
- tasks: Task[]
- loading: boolean
- error: string | null
- selectedTask: Task | null
- draggedTask: Task | null
- filterOptions: FilterOptions
- user: User
- notifications: Notification[]

**Methods:**
- ngOnInit() → Initialize component and load data
- loadDashboardData() → Fetch initial dashboard data
- onTaskMove(taskId: string, targetColumn: string) → Handle task movement
- onTaskCreate(task: Task) → Handle new task creation
- onTaskUpdate(task: Task) → Handle task updates
- onTaskDelete(taskId: string) → Handle task deletion
- openTaskModal(task?: Task) → Open task creation/editing modal
- closeTaskModal() → Close task modal

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-app-container" role="application" aria-label="Kanban Board Application">
  <app-sidebar 
    [menuItems]="menuItems" 
    [activeItem]="activeMenuItem" 
    [collapsed]="sidebarCollapsed"
    (menuItemClick)="onMenuItemClick($event)"
    (toggleCollapse)="onSidebarToggle()">
  </app-sidebar>
  
  <div class="main-layout">
    <app-header 
      [user]="user" 
      [notifications]="notifications"
      (search)="onSearch($event)"
      (notificationClick)="onNotificationClick()"
      (userMenuClick)="onUserMenuClick()">
    </app-header>
    
    <main class="content-area">
      <app-page-header 
        [breadcrumbs]="breadcrumbs"
        [title]="pageTitle"
        [subtitle]="pageSubtitle">
      </app-page-header>
      
      <app-board-controls 
        [filterOptions]="filterOptions"
        (filterChange)="onFilterChange($event)"
        (addTask)="openTaskModal()">
      </app-board-controls>
      
      <app-kanban-container 
        [columns]="columns"
        [tasks]="tasks"
        [loading]="loading"
        [error]="error"
        (taskMove)="onTaskMove($event.taskId, $event.targetColumn)"
        (taskClick)="openTaskModal($event)"
        (addTask)="openTaskModal()">
      </app-kanban-container>
    </main>
  </div>
  
  <app-task-modal 
    [isOpen]="showTaskModal"
    [task]="selectedTask"
    [mode]="taskModalMode"
    (submit)="onTaskSubmit($event)"
    (cancel)="closeTaskModal()"
    (close)="closeTaskModal()">
  </app-task-modal>
  
  <div *ngIf="error" class="error-container" role="alert">
    <p>{{ error }}</p>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: grid
- grid-template-columns: auto 1fr
- grid-template-rows: auto 1fr
- height: 100vh

**Spacing:**
- gap: 0
- padding: 0
- margin: 0

**Component Styles:**
- .kanban-app-container: full viewport layout
- .main-layout: flex column layout
- .content-area: flex-grow with padding
- .error-container: error state styling

**Responsive:**
- Desktop (>1024px): Full sidebar visible
- Tablet (768px-1024px): Collapsible sidebar
- Mobile (<768px): Overlay sidebar

**API INTEGRATION**

- loadDashboardData() → GET /api/youth-accounts/{id}/dashboard → Load initial dashboard data
- loadTransactions() → GET /api/youth-accounts/{id}/transactions → Load transaction history as tasks
- validateAccess() → GET /api/youth-accounts/{id}/validate → Validate user access

**ERROR HANDLING**

- API failure → show error message in error-container
- Network timeout → display retry button
- Validation error → show inline validation messages
- Column configuration failure → display fallback layout

**INTERACTION FLOW**

**User Action → UI → API**
- Page load → initialize component → GET /api/youth-accounts/{id}/dashboard
- Drag task → update UI optimistically → sync with backend
- Create task → open modal → submit form → POST /api/youth-accounts/{id}/fund-transfer

#### 4.2 KanbanColumnComponent

**Component Type:** Feature

**Mapped From A1:** Individual column container (Todo, In Progress, Done)

**Purpose:** Renders a single Kanban column with header, task cards, and drop zone functionality

**Parent:** KanbanContainerComponent

**Children:**
- ColumnHeaderComponent
- TaskCardComponent (multiple)
- AddTaskButtonComponent

**Dependencies:**
- @angular/core
- @angular/cdk/drag-drop

**Library Components Used:**
- ButtonComponent → app-button
- BadgeComponent → app-badge

**TypeScript Specification**

**Inputs:**
- column: KanbanColumn
- tasks: Task[]
- allowDrop: boolean
- draggedTask: Task | null

**Outputs:**
- taskDrop: EventEmitter<CdkDragDrop<Task[]>> (payload: drop event)
- taskClick: EventEmitter<Task> (payload: Task)
- addTask: EventEmitter<string> (payload: columnId)

**State:**
- isDragOver: boolean
- isLoading: boolean
- taskCount: number

**Methods:**
- ngOnInit() → Initialize column
- onDrop(event: CdkDragDrop<Task[]>) → Handle task drop
- onDragOver(event: DragEvent) → Handle drag over
- onTaskClick(task: Task) → Handle task selection
- onAddTask() → Emit add task event

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-column" 
     role="region" 
     [attr.aria-label]="column.title + ' column'"
     [attr.aria-labelledby]="column.id + '-header'"
     cdkDropList
     [cdkDropListData]="tasks"
     (cdkDropListDropped)="onDrop($event)">
  
  <div class="column-header" [id]="column.id + '-header'">
    <h3 class="column-title">{{ column.title }}</h3>
    <app-badge [count]="taskCount" variant="secondary"></app-badge>
    <app-button 
      variant="ghost" 
      size="small" 
      icon="more-vertical"
      (click)="onColumnMenu()">
    </app-button>
  </div>
  
  <div class="column-body" 
       [class.drag-over]="isDragOver"
       [class.loading]="isLoading">
    
    <app-task-card 
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="true"
      [selected]="task.id === selectedTaskId"
      (click)="onTaskClick(task)"
      cdkDrag>
    </app-task-card>
    
    <app-button 
      class="add-task-button"
      variant="dashed"
      size="medium"
      icon="plus"
      (click)="onAddTask()">
      Add Task
    </app-button>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: flex
- flex-direction: column
- min-height: 500px
- width: 100%

**Spacing:**
- padding: 16px
- gap: 12px
- margin: 0 8px

**Component Styles:**
- .kanban-column: card-like container with border
- .column-header: flex row with space-between
- .column-body: flex column with scroll
- .drag-over: highlight border when dragging

**Responsive:**
- Desktop: fixed width columns
- Tablet: flexible width
- Mobile: full width stacked

**API INTEGRATION**

- None (delegates to parent component)

**ERROR HANDLING**

- Task load failure → show empty state message
- Drop operation failure → revert task position
- Network error → show retry option

**INTERACTION FLOW**

**User Action → UI → API**
- Drop task → update column → emit to parent → API sync
- Click add task → emit event → parent opens modal
- Click task → emit selection → parent handles detail view

#### 4.3 TaskCardComponent

**Component Type:** Feature

**Mapped From A1:** Individual task card with drag/drop functionality

**Purpose:** Displays task information in a draggable card format with metadata

**Parent:** KanbanColumnComponent

**Children:**
- None (leaf component)

**Dependencies:**
- @angular/core
- @angular/cdk/drag-drop

**Library Components Used:**
- CardComponent → app-card
- BadgeComponent → app-badge
- AvatarComponent → app-avatar

**TypeScript Specification**

**Inputs:**
- task: Task
- draggable: boolean
- selected: boolean

**Outputs:**
- taskClick: EventEmitter<Task> (payload: Task)
- taskEdit: EventEmitter<Task> (payload: Task)
- taskDelete: EventEmitter<string> (payload: taskId)

**State:**
- isDragging: boolean
- isHovered: boolean

**Methods:**
- ngOnInit() → Initialize card
- onDragStart(event: DragEvent) → Handle drag start
- onDragEnd(event: DragEvent) → Handle drag end
- onClick() → Handle card click
- onEdit() → Handle edit action
- onDelete() → Handle delete action

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<app-card 
  class="task-card"
  [class.selected]="selected"
  [class.dragging]="isDragging"
  [draggable]="draggable"
  (click)="onClick()"
  (dragstart)="onDragStart($event)"
  (dragend)="onDragEnd($event)">
  
  <div class="task-header">
    <h4 class="task-title">{{ task.title }}</h4>
    <app-badge 
      *ngIf="task.priority"
      [variant]="getPriorityVariant(task.priority)"
      [text]="task.priority">
    </app-badge>
  </div>
  
  <p class="task-description" *ngIf="task.description">
    {{ task.description | truncate:100 }}
  </p>
  
  <div class="task-metadata">
    <div class="task-tags" *ngIf="task.tags?.length">
      <app-badge 
        *ngFor="let tag of task.tags"
        [text]="tag"
        variant="outline"
        size="small">
      </app-badge>
    </div>
    
    <div class="task-footer">
      <div class="task-assignee" *ngIf="task.assignee">
        <app-avatar 
          [src]="task.assignee.avatar"
          [name]="task.assignee.name"
          size="small">
        </app-avatar>
      </div>
      
      <div class="task-actions">
        <app-button 
          variant="ghost"
          size="small"
          icon="edit"
          (click)="onEdit(); $event.stopPropagation()">
        </app-button>
        <app-button 
          variant="ghost"
          size="small"
          icon="trash"
          (click)="onDelete(); $event.stopPropagation()">
        </app-button>
      </div>
    </div>
  </div>
</app-card>
```

**CSS SPECIFICATION**

**Layout:**
- display: block
- width: 100%
- margin-bottom: 12px

**Spacing:**
- padding: 16px
- gap: 8px

**Component Styles:**
- .task-card: elevated card with hover effects
- .task-header: flex row with space-between
- .task-footer: flex row with space-between
- .selected: highlighted border
- .dragging: reduced opacity

**Responsive:**
- All breakpoints: consistent card layout

**API INTEGRATION**

- None (data passed via props)

**ERROR HANDLING**

- Missing task data → show placeholder content

**INTERACTION FLOW**

**User Action → UI → API**

#### 4.4 TaskModalComponent

**Component Type:** Feature

**Mapped From A1:** Modal for creating/editing tasks

**Purpose:** Provides form interface for task creation and editing with validation

**Parent:** KanbanBoardComponent

**Children:**
- FormInputComponents
- ButtonComponents

**Dependencies:**
- @angular/core
- @angular/forms
- @angular/material/dialog

**Library Components Used:**
- ModalComponent → app-modal
- ButtonComponent → app-button
- FormInputComponent → app-form-input
- SelectComponent → app-select

**TypeScript Specification**

**Inputs:**
- isOpen: boolean
- task: Task | null
- mode: 'create' | 'edit'
- columns: KanbanColumn[]

**Outputs:**
- taskSubmit: EventEmitter<TaskFormData> (payload: TaskFormData)
- modalCancel: EventEmitter<void> (payload: void)
- modalClose: EventEmitter<void> (payload: void)

**State:**
- taskForm: FormGroup
- isSubmitting: boolean
- validationErrors: ValidationError[]
- selectedColumn: string

**Methods:**
- ngOnInit() → Initialize form
- ngOnChanges() → Handle input changes
- initializeForm() → Setup reactive form
- onSubmit() → Handle form submission
- onCancel() → Handle cancel action
- onClose() → Handle modal close
- validateForm() → Custom form validation

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<app-modal 
  [isOpen]="isOpen"
  [title]="mode === 'create' ? 'Create Task' : 'Edit Task'"
  size="medium"
  (close)="onClose()">
  
  <form [formGroup]="taskForm" (ngSubmit)="onSubmit()" class="task-form">
    
    <div class="form-section">
      <app-form-input 
        label="Task Title"
        formControlName="title"
        placeholder="Enter task title"
        [required]="true"
        [error]="getFieldError('title')">
      </app-form-input>
      
      <app-form-input 
        label="Description"
        formControlName="description"
        type="textarea"
        placeholder="Enter task description"
        rows="3">
      </app-form-input>
    </div>
    
    <div class="form-row">
      <app-select 
        label="Column"
        formControlName="columnId"
        [options]="columnOptions"
        [required]="true"
        [error]="getFieldError('columnId')">
      </app-select>
      
      <app-select 
        label="Priority"
        formControlName="priority"
        [options]="priorityOptions">
      </app-select>
    </div>
    
    <div class="form-section">
      <app-form-input 
        label="Tags"
        formControlName="tags"
        placeholder="Enter tags (comma separated)">
      </app-form-input>
      
      <app-select 
        label="Assignee"
        formControlName="assigneeId"
        [options]="assigneeOptions">
      </app-select>
    </div>
    
    <div class="form-actions">
      <app-button 
        type="button"
        variant="secondary"
        (click)="onCancel()"
        [disabled]="isSubmitting">
        Cancel
      </app-button>
      
      <app-button 
        type="submit"
        variant="primary"
        [loading]="isSubmitting"
        [disabled]="taskForm.invalid">
        {{ mode === 'create' ? 'Create Task' : 'Update Task' }}
      </app-button>
    </div>
  </form>
</app-modal>
```

**CSS SPECIFICATION**

**Layout:**

**Responsive:**

**ERROR HANDLING**

### 8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly with three-column CSS Grid layout
- Do NOT create new components unnecessarily - reuse library components
- Prefer reusable library components from uicomponentlibrary
- HTML must be pseudo-code ONLY (NOT real framework code)
- Implement ARIA attributes for accessibility compliance
- Add responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px)
- Error handling with *ngIf directive for conditional rendering
- Component must compile without errors and pass linting checks
- Map financial operations (fund transfers, spending limits) to Kanban tasks
- Implement drag-and-drop functionality with CDK
- Use Angular Material or custom CSS for styling with design tokens
- Add keyboard navigation support for accessibility
- Implement optimistic updates for better user experience
- Cache frequently accessed data for performance
- Follow Angular best practices for component architecture and lifecycle management
