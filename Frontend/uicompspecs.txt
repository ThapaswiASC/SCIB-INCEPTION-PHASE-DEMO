### 4. COMPONENT IMPLEMENTATION

#### 4.1 TaskCreationComponent

**Component Type:** Feature

**Mapped From A1:** TaskCreationComponent

**Purpose:** Create new tasks with comprehensive form validation and API integration

**Parent:** AppComponent / KanbanBoardComponent

**Children:**
- AavaTextbox (title, description)
- AavaButton (submit, cancel)
- PriorityBadgeComponent (priority selector)
- UserAvatarComponent (assignee selector)

**Dependencies:**
- ReactiveFormsModule
- TaskService
- FormBuilder
- Validators

**Library Components Used:**
- AavaTextbox → Form input fields
- AavaButton → Submit and cancel actions
- AavaDefaultCard → Modal container

**TypeScript Specification**

**Inputs:**
- initialData: Partial<Task>
- isVisible: boolean
- loading: boolean

**Outputs:**
- taskCreated: EventEmitter<Task> (emitted when task is successfully created)
- modalClosed: EventEmitter<void> (emitted when modal is closed)
- formValidationChanged: EventEmitter<boolean> (emitted when form validity changes)

**State:**
- taskForm: FormGroup
- validationErrors: ValidationError[]
- isSubmitting: boolean
- characterCounts: { title: number, description: number }

**Methods:**
- initializeForm() → void (sets up reactive form with validators)
- onSubmit() → void (handles form submission)
- resetForm() → void (resets form to initial state)
- updateCharacterCount(field: string) → void (updates character counters)
- closeModal() → void (closes modal and emits event)

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="task-creation-modal" *ngIf="isVisible">
  <div class="modal-overlay" (click)="closeModal()">
    <div class="modal-content" (click)="$event.stopPropagation()">
      <div class="modal-header">
        <h2>Create New Task</h2>
        <button class="close-btn" (click)="closeModal()">×</button>
      </div>
      
      <form [formGroup]="taskForm" (ngSubmit)="onSubmit()">
        <div class="form-group">
          <aava-textbox
            label="Task Title"
            formControlName="title"
            [maxLength]="255"
            placeholder="Enter task title"
            [showCharacterCount]="true">
          </aava-textbox>
        </div>
        
        <div class="form-group">
          <aava-textbox
            label="Description"
            formControlName="description"
            [maxLength]="2000"
            [multiline]="true"
            placeholder="Enter task description"
            [showCharacterCount]="true">
          </aava-textbox>
        </div>
        
        <div class="form-group">
          <label>Priority</label>
          <select formControlName="priority">
            <option value="LOW">Low</option>
            <option value="MEDIUM">Medium</option>
            <option value="HIGH">High</option>
            <option value="CRITICAL">Critical</option>
          </select>
        </div>
        
        <div class="form-group">
          <aava-textbox
            label="Due Date"
            formControlName="dueDate"
            type="date"
            [min]="tomorrow">
          </aava-textbox>
        </div>
        
        <div class="modal-actions">
          <aava-button
            variant="secondary"
            (click)="closeModal()"
            [disabled]="isSubmitting">
            Cancel
          </aava-button>
          <aava-button
            variant="primary"
            type="submit"
            [disabled]="taskForm.invalid || isSubmitting"
            [loading]="isSubmitting">
            Create Task
          </aava-button>
        </div>
      </form>
    </div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- position: fixed for modal overlay
- display: flex with justify-content: center, align-items: center
- z-index: 1000 for modal layering

**Spacing:**
- padding: 24px for modal content
- gap: 16px between form groups
- margin: 8px for form elements

**Component Styles:**
- .task-creation-modal (full viewport overlay)
- .modal-overlay (backdrop with blur effect)
- .modal-content (glassmorphism card effect)
- .form-group (consistent spacing)
- .modal-actions (button alignment)

**Responsive:**
- Mobile: max-width: 90vw, full-height scrollable
- Tablet: max-width: 600px
- Desktop: max-width: 800px

**API INTEGRATION**

- createTask() → POST /api/v1/tasks → Creates new task with form data
- Form validation aligns with CreateTaskRequest schema
- Maps form values to API payload structure

**ERROR HANDLING**

- API failure → show error snackbar with retry option
- Validation error → inline field-specific error messages
- Network timeout → display connection error message
- 400 Bad Request → show validation errors from API response

**INTERACTION FLOW**

**User Action → UI → API**
- Click "Add Task" → open modal → initialize form
- Fill form fields → real-time validation → character count updates
- Submit form → validation check → loading state → API call → success/error handling

#### 4.2 KanbanBoardComponent

**Component Type:** Page

**Mapped From A1:** KanbanBoardComponent

**Purpose:** Main kanban board container managing three columns and task flow

**Parent:** AppComponent

**Children:**
- KanbanColumnComponent (To Do, In Progress, Done)

**Inputs:**

**Outputs:**

- taskCreated: EventEmitter<Task>

**State:**

- inProgressTasks: Task[]
- doneTasks: Task[]
- showTaskCreationModal: boolean
- selectedColumn: string

**Methods:**
- loadTasks() → void
- onTaskMoved(event: TaskMoveEvent) → void
- openTaskCreation(column: string) → void

- filterTasksByStatus(status: string) → Task[]

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-board">
  <div class="board-header">
    <h1>Project Board</h1>
    <div class="board-actions">
      <aava-button variant="primary" (click)="openTaskCreation('TODO')">
        Add Task
      </aava-button>
    </div>
  </div>
  
  <div class="kanban-columns">
    <kanban-column
      title="To Do"
      [tasks]="todoTasks"
      [taskCount]="todoTasks.length"
      status="TODO"
      (taskMoved)="onTaskMoved($event)"
      (addTaskClicked)="openTaskCreation('TODO')">
    </kanban-column>
    
    <kanban-column
      title="In Progress"
      [tasks]="inProgressTasks"
      [taskCount]="inProgressTasks.length"
      status="IN_PROGRESS"
      (taskMoved)="onTaskMoved($event)"
      (addTaskClicked)="openTaskCreation('IN_PROGRESS')">
    </kanban-column>
    
    <kanban-column
      title="Done"
      [tasks]="doneTasks"
      [taskCount]="doneTasks.length"
      status="DONE"
      (taskMoved)="onTaskMoved($event)"
      (addTaskClicked)="openTaskCreation('DONE')">
    </kanban-column>
  </div>
  
  <task-creation
    [isVisible]="showTaskCreationModal"
    [loading]="loading"
    (taskCreated)="onTaskCreated($event)"
    (modalClosed)="showTaskCreationModal = false">
  </task-creation>
```

**CSS SPECIFICATION**

**Layout:**
- display: grid with grid-template-columns: repeat(3, 1fr)
- gap: 24px between columns
- min-height: 100vh for full viewport

**Spacing:**
- padding: 24px for board container
- margin: 16px for board header

**Component Styles:**
- .kanban-board (main container)
- .board-header (title and actions)
- .kanban-columns (grid layout)

**Responsive:**
- Mobile: single column stack
- Tablet: two columns
- Desktop: three columns

**API INTEGRATION**

- loadTasks() → GET /api/v1/tasks → Loads all tasks for board
- updateTaskStatus() → PATCH /api/v1/tasks/{id} → Updates task status on move

**ERROR HANDLING**

- Task load failure → show error message with retry
- Task move failure → revert UI state and show error

**INTERACTION FLOW**

**User Action → UI → API**
- Load board → fetch tasks → distribute to columns
- Drag task → update UI → API call → confirm/revert
- Click add task → open modal → create task → refresh board

#### 4.3 KanbanColumnComponent

**Component Type:** Feature

**Mapped From A1:** KanbanColumnComponent

**Purpose:** Display and manage tasks within a specific column with drag-drop support

**Parent:** KanbanBoardComponent

**Children:**
- TaskCardComponent (multiple)
- AavaButton (add task)

**Dependencies:**
- DragDropModule
- CommonModule

**Library Components Used:**
- AavaDefaultCard → Column container
- AavaButton → Add task button

**TypeScript Specification**

**Inputs:**
- title: string
- tasks: Task[]
- taskCount: number
- status: string
- allowDrop: boolean

**Outputs:**
- taskMoved: EventEmitter<TaskMoveEvent>
- taskSelected: EventEmitter<Task>
- addTaskClicked: EventEmitter<string>

**State:**
- draggedTask: Task | null
- isDropTarget: boolean
- isCollapsed: boolean

**Methods:**
- onDrop(event: CdkDragDrop<Task[]>) → void
- onTaskDragStart(task: Task) → void
- onTaskDragEnd() → void
- toggleCollapse() → void

**HTML STRUCTURE (PSEUDO CODE ONLY)**

```html
<div class="kanban-column" [class.drop-target]="isDropTarget">
  <div class="column-header">
    <h3>{{ title }}</h3>
    <span class="task-count">{{ taskCount }}</span>
    <button class="collapse-btn" (click)="toggleCollapse()">
      {{ isCollapsed ? '▶' : '▼' }}
    </button>
  </div>
  
  <div class="column-content" [class.collapsed]="isCollapsed"
       cdkDropList
       [cdkDropListData]="tasks"
       (cdkDropListDropped)="onDrop($event)">
    
    <task-card
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [isDraggable]="true"
      cdkDrag
      (taskClicked)="taskSelected.emit(task)"
      (cdkDragStarted)="onTaskDragStart(task)"
      (cdkDragEnded)="onTaskDragEnd()">
    </task-card>
    
    <div class="add-task-area">
      <aava-button
        variant="ghost"
        (click)="addTaskClicked.emit(status)"
        class="add-task-btn">
        + Add Task
      </aava-button>
    </div>
  </div>
</div>
```

**CSS SPECIFICATION**

**Layout:**
- display: flex with flex-direction: column
- min-height: 500px
- flex: 1 for equal column widths

**Spacing:**
- padding: 16px for column content
- gap: 12px between task cards

**Component Styles:**
- .kanban-column (column container with glassmorphism)
- .column-header (title and count display)
- .column-content (scrollable task area)
- .drop-target (highlight during drag)

**Responsive:**
- Mobile: full width with horizontal scroll
- Tablet/Desktop: fixed width columns

**API INTEGRATION**

- Task status updates handled by parent component
- No direct API calls

**ERROR HANDLING**

- Drag operation failure → revert task position
- Invalid drop target → prevent drop and show feedback

**INTERACTION FLOW**

**User Action → UI → API**
- Drag task → visual feedback → drop → emit event to parent
- Click add task → emit column status → parent opens modal
- Click task → emit task selection → parent handles detail view

### 8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components (AavaButton, AavaTextbox, AavaDefaultCard, AavaSidebar)
- HTML must be pseudo-code ONLY (NOT real framework code)
- All form validation must align with API constraints (title maxLength 200, description maxLength 2000)
- Priority enum must match API values (LOW, MEDIUM, HIGH, CRITICAL)
- Due date validation must ensure future dates only
- Error handling must cover all API error responses (400, 401, 403, 404, 409, 429, 500, 503)
- Real-time character counters required for text fields
- Loading states must be implemented for all API operations
- Form reset functionality required after successful submission
- Drag-and-drop functionality required for task management
- Responsive design required for mobile, tablet, and desktop
- Glassmorphism design effects must be maintained
- Accessibility standards must be followed
- OnPush change detection strategy recommended for performance
- Service worker implementation recommended for offline support
