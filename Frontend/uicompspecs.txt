### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for three-column kanban layout

Purpose: Orchestrate kanban board functionality with drag-and-drop, task management, and column organization

Parent: DashboardComponent

Children:
- KanbanColumnComponent (3 instances)
- TaskModalComponent

Dependencies:
- KanbanService
- TaskService
- DragDropModule

Library Components Used:
- Card → Container wrapper
- Button → Add task actions

TypeScript Specification

Inputs:
- columns: KanbanColumn[] (column configuration)
- loading: boolean (loading state)
- error: string | null (error message)

Outputs:
- onTaskMove: EventEmitter<{taskId: string, fromColumn: string, toColumn: string}> (task movement)
- onTaskCreate: EventEmitter<Task> (new task creation)
- onTaskUpdate: EventEmitter<Task> (task updates)

State:
- draggedTask: Task | null (currently dragged task)
- columnData: KanbanColumn[] (internal column state)
- showTaskModal: boolean (modal visibility)
- selectedTask: Task | null (task being edited)

Methods:
- ngOnInit() → Initialize component and load data
- loadColumns() → Fetch column configuration
- onTaskDrop(event: CdkDragDrop) → Handle task drop events
- openTaskModal(task?: Task) → Open task creation/edit modal
- closeTaskModal() → Close task modal

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container" role="region" aria-label="Kanban Board">
  <div class="board-header">
    <h1>Kanban Board</h1>
    <Button variant="primary" (click)="openTaskModal()">Add Task</Button>
  </div>
  
  <div class="kanban-grid" cdkDropListGroup>
    <KanbanColumnComponent 
      *ngFor="let column of columnData" 
      [column]="column"
      [tasks]="column.tasks"
      [allowDrop]="true"
      (onTaskDrop)="onTaskDrop($event)"
      (onAddTask)="openTaskModal()"
    />
  </div>
  
  <TaskModalComponent
    [visible]="showTaskModal"
    [task]="selectedTask"
    [mode]="selectedTask ? 'edit' : 'create'"
    (onSubmit)="onTaskSubmit($event)"
    (onCancel)="closeTaskModal()"
  />
  
  <div *ngIf="error" class="error-message" role="alert">
    {{ error }}
  </div>
</div>

CSS SPECIFICATION

Layout:
- display: flex, flex-direction: column
- .kanban-grid: display: grid, grid-template-columns: repeat(3, 1fr), gap: 1rem

Spacing:
- padding: 1.5rem
- gap: 1rem between columns

Component Styles:
- .kanban-board-container: background: #f5f5f5, min-height: 100vh
- .board-header: display: flex, justify-content: space-between, align-items: center
- .error-message: background: #fee, color: #c53030, padding: 0.75rem, border-radius: 0.375rem

Responsive:
- Desktop (>1024px): 3 columns, full grid
- Tablet (768px-1024px): 2 columns, scroll horizontal
- Mobile (<768px): 1 column, stack vertically

API INTEGRATION

- loadColumns() → GET /forms/sessions/{id} → Load kanban data
- onTaskCreate() → POST /forms/sessions → Create new task session
- onTaskMove() → PUT /forms/sessions/{id} → Update task status

ERROR HANDLING

- API failure → show error message in .error-message div
- Validation error → highlight invalid fields in modal
- Network timeout → show retry button with exponential backoff

INTERACTION FLOW

User Action → UI → API
- Load board → ngOnInit() → GET /forms/sessions/{id} → display columns
- Create task → click Add Task → open modal → submit → POST /forms/sessions
- Move task → drag task card → drop in column → PUT /forms/sessions/{id}

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly with three-column CSS Grid layout
- Do NOT create new components unnecessarily - reuse library components where possible
- Prefer reusable library components (Button, Card, Avatar, Modal, FormInput) over custom implementations
- HTML must be pseudo-code ONLY (NOT real framework code)
- Implement proper ARIA attributes for accessibility compliance
- Use Angular CDK for drag-and-drop functionality
- Form validation must be implemented using Angular Reactive Forms
- Error handling must include both API failures and form validation errors
- Responsive design must support desktop (>1024px), tablet (768px-1024px), and mobile (<768px) breakpoints
- All components must follow Angular component lifecycle hooks (OnInit, OnDestroy)
- Use TypeScript interfaces for type safety
- Implement proper event emitters for parent-child communication
- CSS Grid layout is mandatory for kanban board (grid-template-columns: repeat(3, 1fr))
- Drag-and-drop must work across all three columns
- Task modal must support both create and edit modes
- API integration must use the available form sessions endpoints
- Component folder structure must follow Angular feature module organization
