4. COMPONENT IMPLEMENTATION

### 4.1 KanbanBoardComponent

Component Type: Feature

Mapped From A1: Main container for three-column kanban layout

Purpose: Orchestrates the entire kanban board with drag-and-drop functionality and responsive design

Parent: KanbanPageComponent

Children:
- KanbanColumnComponent (3 instances)
- TaskModalComponent

Dependencies:
- KanbanService
- DragDropModule
- CommonModule

Library Components Used:
- ButtonComponent → Add task actions

TypeScript Specification

Inputs:
- columns: KanbanColumn[] = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]
- loading: boolean = false
- error: string | null = null

Outputs:
- taskMove: EventEmitter<{taskId: string, fromColumn: string, toColumn: string}> (payload)
- taskCreate: EventEmitter<{task: Task, column: string}> (payload)

State:
- draggedTask: Task | null = null
- activeColumn: string = 'todo'
- isModalOpen: boolean = false
- selectedTask: Task | null = null

Methods:
- ngOnInit() → Initialize component and load data
- onTaskDrop(event: CdkDragDrop<Task[]>) → Handle drag-and-drop events
- openTaskModal(column?: string) → Open task creation modal
- closeTaskModal() → Close task modal
- onMobileColumnSwitch(columnId: string) → Switch active column on mobile

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-board-container">
  <div class="mobile-tabs" *ngIf="isMobile">
    <button *ngFor="let column of columns" 
            [class.active]="activeColumn === column.id"
            (click)="onMobileColumnSwitch(column.id)">
      {{column.title}} ({{column.cards.length}})
    </button>
  </div>
  
  <div class="kanban-board" 
       cdkDropListGroup
       [class.mobile-view]="isMobile">
    <app-kanban-column 
      *ngFor="let column of columns"
      [column]="column"
      [tasks]="column.cards"
      [isActive]="!isMobile || activeColumn === column.id"
      [cdkDropListData]="column.cards"
      (taskDrop)="onTaskDrop($event)"
      (addTask)="openTaskModal(column.id)">
    </app-kanban-column>
  </div>
  
  <app-task-modal
    [isOpen]="isModalOpen"
    [task]="selectedTask"
    [defaultColumn]="activeColumn"
    (submit)="onTaskSubmit($event)"
    (close)="closeTaskModal()">
  </app-task-modal>
  
  <div *ngIf="error" class="error-state" role="alert">
    <p>{{error}}</p>
    <app-button label="Retry" variant="primary" (userClick)="loadData()"></app-button>
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-board { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1rem; }
- .mobile-view { grid-template-columns: 1fr; }

Spacing:
- padding: 1rem
- gap: 1rem between columns

Component Styles:
- .kanban-board-container { height: 100%; overflow: hidden; }
- .mobile-tabs { display: flex; border-bottom: 1px solid var(--border-color); }
- .error-state { text-align: center; padding: 2rem; }

Responsive:
- Desktop (>1024px): Three columns visible
- Tablet (768px-1024px): Two columns with horizontal scroll
- Mobile (<768px): Single column with tab navigation

API INTEGRATION

- loadData() → GET /api/youth-accounts/{youthAccountId}/dashboard → Load initial kanban data
- onTaskSubmit() → POST /api/youth-accounts/{youthAccountId}/fund-transfer → Create new task
- onTaskMove() → PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update task status

ERROR HANDLING

- API failure → show error state with retry button
- Drag-and-drop failure → revert task position and show snackbar
- Validation error → highlight invalid fields in modal

INTERACTION FLOW

User Action → UI → API
- Load page → ngOnInit → GET dashboard → populate columns
- Drag task → onTaskDrop → PUT spending-limit → update UI
- Click add task → openTaskModal → show modal → submit → POST fund-transfer
- Mobile tab switch → onMobileColumnSwitch → show/hide columns

### 4.2 KanbanColumnComponent

Component Type: Feature

Mapped From A1: Individual column container with header and task list

Purpose: Renders a single kanban column with header, task list, and drop zone functionality

Parent: KanbanBoardComponent

Children:
- TaskCardComponent (multiple instances)
- ButtonComponent (add task button)
- StatusIndicatorComponent (column status)
- BadgeComponent (task count)

Dependencies:
- DragDropModule
- CommonModule

Library Components Used:
- ButtonComponent → Add task button

TypeScript Specification

Inputs:
- column: KanbanColumn (column data with id, title, cards)
- tasks: Task[] (array of tasks in this column)
- isActive: boolean = false (for mobile visibility)

Outputs:
- taskDrop: EventEmitter<CdkDragDrop<Task[]>> (drag-drop event payload)
- addTask: EventEmitter<void> (add task button click)

State:
- isDragOver: boolean = false
- taskCount: number = 0

Methods:
- ngOnInit() → Initialize component
- onTaskDrop(event: CdkDragDrop<Task[]>) → Handle task drop
- onAddTaskClick() → Emit add task event
- onDragEnter() → Set drag over state
- onDragLeave() → Clear drag over state

HTML STRUCTURE (PSEUDO CODE ONLY)

<div class="kanban-column" 
     [class.active]="isActive"
     [class.drag-over]="isDragOver"
     cdkDropList
     [cdkDropListData]="tasks"
     (cdkDropListDropped)="onTaskDrop($event)"
     (cdkDropListEntered)="onDragEnter()"
     (cdkDropListExited)="onDragLeave()">
  
  <div class="column-header">
    <div class="column-title-section">
      <app-status-indicator [status]="column.id"></app-status-indicator>
      <h3 class="column-title">{{column.title}}</h3>
      <app-badge [count]="tasks.length" variant="secondary"></app-badge>
    </div>
    <app-button 
      iconName="plus"
      iconPosition="only"
      variant="secondary"
      size="sm"
      [attr.aria-label]="'Add task to ' + column.title"
      (userClick)="onAddTaskClick()">
    </app-button>
  </div>
  
  <div class="column-content" 
       [attr.aria-label]="column.title + ' tasks'">
    <app-task-card
      *ngFor="let task of tasks; trackBy: trackByTaskId"
      [task]="task"
      [draggable]="true"
      cdkDrag
      [cdkDragData]="task">
    </app-task-card>
    
    <div *ngIf="tasks.length === 0" class="empty-column">
      <p>No tasks in {{column.title}}</p>
      <app-button 
        label="Add First Task"
        variant="primary"
        size="sm"
        (userClick)="onAddTaskClick()">
      </app-button>
    </div>
  </div>
</div>

CSS SPECIFICATION

Layout:
- .kanban-column { display: flex; flex-direction: column; height: 100%; }
- .column-content { flex: 1; overflow-y: auto; }

Spacing:
- padding: 1rem
- gap: 0.75rem between tasks

Component Styles:
- .column-header { display: flex; justify-content: space-between; align-items: center; margin-bottom: 1rem; }
- .column-title-section { display: flex; align-items: center; gap: 0.5rem; }
- .drag-over { background-color: var(--primary-50); border: 2px dashed var(--primary-300); }
- .empty-column { text-align: center; padding: 2rem 1rem; color: var(--text-muted); }

Responsive:
- Mobile: .kanban-column:not(.active) { display: none; }
- Tablet/Desktop: All columns visible

API INTEGRATION

- None (receives data from parent KanbanBoardComponent)

ERROR HANDLING

- Drop validation → prevent invalid drops and show feedback
- Empty state → show helpful message and add task button

INTERACTION FLOW

User Action → UI → API
- Drop task → onTaskDrop → emit to parent → API call
- Click add task → onAddTaskClick → emit to parent → open modal
- Drag enter → onDragEnter → visual feedback

  <div class="task-metadata">

        [src]="task.assignee.avatar"

        size="sm">
      </app-avatar>
      <span class="assignee-name">{{task.assignee.name}}</span>
    </div>
    
    <div class="task-due-date" *ngIf="task.dueDate">
      <span class="due-date" 
            [class.overdue]="isOverdue(task.dueDate)"
            [class.due-soon]="isDueSoon(task.dueDate)">
        {{task.dueDate | date:'MMM d'}}

    </div>
  </div>
  
  <div class="task-tags" *ngIf="task.tags?.length">
    <app-badge 
      *ngFor="let tag of task.tags | slice:0:3"
      [label]="tag"
      variant="info"
      size="xs">
    </app-badge>

      +{{task.tags.length - 3}} more
    </span>
  </div>

Component Type: Feature

Mapped From A1: Modal for creating and editing tasks

Purpose: Provides form interface for task creation and editing with validation

Parent: KanbanBoardComponent

Children:
- ButtonComponent (submit, cancel buttons)
- FormInputComponent (task fields)

- CommonModule
- ModalModule

TypeScript Specification

Inputs:
- isOpen: boolean = false
- task: Task | null = null
- defaultColumn: string = 'todo'

Outputs:

- close: EventEmitter<void> (modal close event)
