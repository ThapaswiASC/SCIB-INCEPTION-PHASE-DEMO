## 1. Youth Account Dashboard API

### Endpoint: `/youth-accounts/{youthAccountId}/dashboard`
**Method**: `GET`
**Description**: Retrieves consolidated youth account dashboard information

#### Path Parameters
- `youthAccountId` (string, required): Unique identifier for the youth account

#### Request Headers
```json
{
  "Authorization": "Bearer {access_token}",
  "Content-Type": "application/json",
  "X-Correlation-ID": "uuid"
}
```

#### Request Schema
```json
// No request body required
```

#### Response Schema (200 OK)
```json
{
  "youthAccountId": "string",
  "accountBalance": "number",
  "spendingLimit": "number",
  "currency": "string",
  "accountStatus": "string",
  "lastUpdated": "string (ISO 8601)",
  "recentTransactions": [
    {
      "transactionId": "string",
      "date": "string (ISO 8601)",
      "description": "string",
      "amount": "number",
      "transactionType": "debit | credit",
      "merchant": "string",
      "category": "string"
    }
  ],
  "spendingLimitStatus": {
    "weeklyLimit": "number",
    "currentWeekSpending": "number",
    "remainingLimit": "number",
    "weekStartDate": "string (ISO 8601)"
  }
}
```

#### Error Handling
- **400 Bad Request**: Invalid youth account ID format
- **401 Unauthorized**: Invalid or expired authentication token
- **403 Forbidden**: User not authorized to access this youth account
- **404 Not Found**: Youth account not found
- **500 Internal Server Error**: Server processing error
- **503 Service Unavailable**: Service temporarily unavailable

#### Error Response Schema
```json
{
  "error": {
    "code": "string",
    "message": "string",
    "details": "string",
    "timestamp": "string (ISO 8601)",
    "correlationId": "string"
  }
}
```

## 2. Fund Transfer API

### Endpoint: `/youth-accounts/{youthAccountId}/fund-transfer`
**Method**: `POST`
**Description**: Transfers funds from parent account to youth account

#### Path Parameters
- `youthAccountId` (string, required): Target youth account identifier

#### Request Headers
```json
{
  "Authorization": "Bearer {access_token}",
  "Content-Type": "application/json",
  "X-Correlation-ID": "uuid",
  "Idempotency-Key": "uuid"
}
```

#### Request Schema
```json
{
  "sourceAccountId": "string",
  "transferAmount": "number",
  "currency": "string",
  "description": "string (optional)",
  "scheduledDate": "string (ISO 8601, optional)"
}
```

#### Request Validation
- `sourceAccountId`: Must be a valid parent account ID
- `transferAmount`: Must be positive, maximum 2 decimal places
- `currency`: Must match account currency (USD, EUR, GBP)
- `description`: Maximum 100 characters
- `scheduledDate`: Must be future date if provided

#### Response Schema (200 OK)
```json
{
  "transferId": "string",
  "status": "SUCCESS | PENDING | FAILED",
  "sourceAccountId": "string",
  "targetAccountId": "string",
  "transferAmount": "number",
  "currency": "string",
  "updatedBalance": "number",
  "transactionDate": "string (ISO 8601)",
  "description": "string",
  "fees": {
    "amount": "number",
    "currency": "string",
    "description": "string"
  }
}
```

#### Error Handling
- **400 Bad Request**: Invalid request parameters or insufficient funds
- **401 Unauthorized**: Invalid authentication
- **403 Forbidden**: Not authorized for this operation
- **409 Conflict**: Duplicate transaction (idempotency check)
- **422 Unprocessable Entity**: Business rule violation
- **500 Internal Server Error**: Transfer processing error

#### Specific Error Codes
```json
{
  "INSUFFICIENT_FUNDS": "Source account has insufficient balance",
  "ACCOUNT_FROZEN": "Account is frozen or suspended",
  "TRANSFER_LIMIT_EXCEEDED": "Transfer amount exceeds daily/monthly limits",
  "INVALID_CURRENCY": "Currency mismatch between accounts",
  "DUPLICATE_TRANSACTION": "Transaction already processed"
}
```

## 3. Spending Limit Configuration API

### Endpoint: `/youth-accounts/{youthAccountId}/spending-limit`
**Method**: `PUT`
**Description**: Configures weekly spending limit for youth account

#### Path Parameters
- `youthAccountId` (string, required): Youth account identifier

#### Request Headers
```json
{
  "Authorization": "Bearer {access_token}",
  "Content-Type": "application/json",
  "X-Correlation-ID": "uuid"
}
```

#### Request Schema
```json
{
  "weeklyLimit": "number",
  "currency": "string",
  "effectiveDate": "string (ISO 8601, optional)",
  "notificationEnabled": "boolean",
  "notificationThreshold": "number (optional)"
}
```

#### Request Validation
- `weeklyLimit`: Must be positive, minimum $1, maximum $1000
- `currency`: Must match account currency
- `effectiveDate`: Must be current or future date
- `notificationThreshold`: Percentage (0-100) of limit for notifications

#### Response Schema (200 OK)
```json
{
  "youthAccountId": "string",
  "weeklyLimit": "number",
  "currency": "string",
  "effectiveDate": "string (ISO 8601)",
  "previousLimit": "number",
  "status": "UPDATED",
  "notificationSettings": {
    "enabled": "boolean",
    "threshold": "number",
    "methods": ["email", "sms", "push"]
  }
}
```

#### Error Handling
- **400 Bad Request**: Invalid limit amount or parameters
- **401 Unauthorized**: Invalid authentication
- **403 Forbidden**: Not authorized to modify limits
- **422 Unprocessable Entity**: Limit violates business rules
- **500 Internal Server Error**: Configuration update error

## 4. Transaction History API

### Endpoint: `/youth-accounts/{youthAccountId}/transactions`
**Method**: `GET`
**Description**: Retrieves paginated transaction history for youth account

#### Path Parameters
- `youthAccountId` (string, required): Youth account identifier

#### Query Parameters
- `startDate` (string, optional): Start date for transaction range (ISO 8601)
- `endDate` (string, optional): End date for transaction range (ISO 8601)
- `page` (integer, optional): Page number (default: 1)
- `pageSize` (integer, optional): Records per page (default: 20, max: 100)
- `transactionType` (string, optional): Filter by type (debit, credit, all)
- `category` (string, optional): Filter by transaction category
- `minAmount` (number, optional): Minimum transaction amount
- `maxAmount` (number, optional): Maximum transaction amount

#### Request Headers
```json
{
  "Authorization": "Bearer {access_token}",
  "Content-Type": "application/json",
  "X-Correlation-ID": "uuid"
}
```

#### Response Schema (200 OK)
```json
{
  "youthAccountId": "string",
  "transactions": [
    {
      "transactionId": "string",
      "date": "string (ISO 8601)",
      "merchant": "string",
      "description": "string",
      "amount": "number",
      "transactionType": "debit | credit",
      "category": "string",
      "status": "completed | pending | failed",
      "balanceAfter": "number",
      "location": {
        "city": "string",
        "state": "string",
        "country": "string"
      },
      "paymentMethod": "card | transfer | cash"
    }
  ],
  "pagination": {
    "page": "number",
    "pageSize": "number",
    "totalRecords": "number",
    "totalPages": "number",
    "hasNext": "boolean",
    "hasPrevious": "boolean"
  },
  "summary": {
    "totalDebits": "number",
    "totalCredits": "number",
    "netAmount": "number",
    "transactionCount": "number"
  }
}
```

#### Error Handling
- **400 Bad Request**: Invalid query parameters or date range
- **401 Unauthorized**: Invalid authentication
- **403 Forbidden**: Not authorized to view transactions
- **404 Not Found**: Youth account not found
- **500 Internal Server Error**: Data retrieval error

## Rate Limiting

- **General APIs**: 100 requests per minute per user
- **Fund Transfer API**: 10 requests per minute per user
- **Dashboard API**: 60 requests per minute per user
- **Rate Limit Headers**: Included in all responses

## Security Considerations

### Authentication
- OAuth 2.0 with PKCE flow
- JWT tokens with 15-minute expiration
- Refresh token rotation
- Multi-factor authentication for sensitive operations

### Authorization
- Role-based access control (RBAC)
- Parent-youth account relationship validation
- Resource-level permissions
- Audit logging for all operations

### Data Protection
- TLS 1.3 for all communications
- Request/response encryption
- PII data masking in logs
- Input validation and sanitization

### Standard Error Format
```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
```

### HTTP Status Codes
- **200 OK**: Successful operation
- **201 Created**: Resource created successfully
- **400 Bad Request**: Invalid request parameters
- **401 Unauthorized**: Authentication required
- **403 Forbidden**: Access denied
- **404 Not Found**: Resource not found
- **409 Conflict**: Resource conflict
- **422 Unprocessable Entity**: Business rule violation
- **429 Too Many Requests**: Rate limit exceeded
- **500 Internal Server Error**: Server error
- **503 Service Unavailable**: Service temporarily unavailable
