# API Contract Outline
# Youth Account Management System

## Overview
This document outlines the API contracts for the Youth Account Management System, providing detailed specifications for all endpoints, request/response schemas, and error handling patterns.

## Base Configuration
- **Base URL**: `https://api.bank.com/v1`
- **Authentication**: OAuth 2.0 Bearer Token
- **Content-Type**: `application/json`
- **API Version**: v1
- **Rate Limiting**: 1000 requests per hour per user

## Common Headers
```
Authorization: Bearer {jwt_token}
Content-Type: application/json
X-Request-ID: {unique_request_id}
X-API-Version: v1
```

## Common Error Response Schema
```json
{
  "error": {
    "code": "string",
    "message": "string",
    "details": "string",
    "timestamp": "datetime",
    "requestId": "string"
  }
}
```

## API Endpoints

### 1. Youth Account Dashboard API
**Reference**: SCIB-26

#### Endpoint: GET /api/youth-accounts/{youthAccountId}/dashboard

**Description**: Retrieves consolidated youth account information including balance, spending limit, and recent transactions.

**Path Parameters**:
- `youthAccountId` (string, required): Unique identifier for the youth account

**Request Headers**:
```
Authorization: Bearer {jwt_token}
X-Request-ID: {unique_request_id}
```

**Request Schema**: N/A (GET request)

**Response Schema** (200 OK):
```json
{
  "youthAccountId": "string",
  "accountBalance": "decimal",
  "spendingLimit": "decimal",
  "currency": "string",
  "weeklySpentAmount": "decimal",
  "limitResetDate": "date",
  "recentTransactions": [
    {
      "transactionId": "string",
      "date": "datetime",
      "description": "string",
      "merchant": "string",
      "amount": "decimal",
      "transactionType": "debit|credit",
      "status": "completed|pending|failed"
    }
  ],
  "accountStatus": "active|suspended|closed",
  "lastUpdated": "datetime"
}
```

**Error Handling**:
- `400 Bad Request`: Invalid youth account ID format
- `401 Unauthorized`: Invalid or expired authentication token
- `403 Forbidden`: User not authorized to access this youth account
- `404 Not Found`: Youth account not found

- `503 Service Unavailable`: Core banking system unavailable

### 2. Fund Transfer API
**Reference**: SCIB-27

#### Endpoint: POST /api/youth-accounts/{youthAccountId}/fund-transfer

**Description**: Allows parents to transfer funds from their primary account to a youth account.

**Path Parameters**:
- `youthAccountId` (string, required): Unique identifier for the youth account

**Request Schema**:
```json
{
  "sourceAccountId": "string",
  "transferAmount": "decimal",
  "currency": "string",
  "description": "string",
  "transferType": "immediate|scheduled",
  "scheduledDate": "date" // Optional, required if transferType is 'scheduled'
}
```

**Request Validation**:
- `sourceAccountId`: Must be a valid parent account ID
- `transferAmount`: Must be positive, minimum $1.00, maximum $10,000.00
- `currency`: Must match account currency (USD, EUR, GBP)
- `description`: Optional, max 255 characters

**Response Schema** (200 OK):
```json
{
  "transferId": "string",
  "status": "SUCCESS|PENDING|FAILED",
  "updatedBalance": "decimal",
  "transferAmount": "decimal",
  "transactionReference": "string",
  "estimatedCompletionTime": "datetime",
  "fees": {
    "transferFee": "decimal",
    "currency": "string"
  }
}
```

**Error Handling**:
- `400 Bad Request`: Invalid request data or insufficient funds
  ```json
  {
    "error": {
      "code": "INSUFFICIENT_FUNDS",
      "message": "Source account does not have sufficient balance",
      "details": "Available balance: $50.00, Requested: $100.00"
    }
  }
  ```
- `401 Unauthorized`: Invalid authentication
- `403 Forbidden`: Not authorized to transfer from source account
- `404 Not Found`: Youth account or source account not found
- `409 Conflict`: Duplicate transfer request
- `422 Unprocessable Entity`: Transfer limits exceeded
- `500 Internal Server Error`: System error
- `503 Service Unavailable`: Payment service unavailable

**Example Request**:
```json
{
  "sourceAccountId": "PA-987654321",
  "transferAmount": 50.00,
  "currency": "USD",
  "description": "Weekly allowance",
  "transferType": "immediate"
}
```

**Example Response**:
```json
{
  "transferId": "TRF-123456789",
  "status": "SUCCESS",
  "updatedBalance": 200.75,
  "transferAmount": 50.00,
  "transactionReference": "REF-987654321",
  "estimatedCompletionTime": "2024-01-10T15:05:00Z",
  "fees": {
    "transferFee": 0.00,
    "currency": "USD"
  }
}
```

### 3. Spending Limit Configuration API
**Reference**: SCIB-28

#### Endpoint: PUT /api/youth-accounts/{youthAccountId}/spending-limit

**Description**: Allows parents to configure weekly spending limits for youth accounts.

**Path Parameters**:
- `youthAccountId` (string, required): Unique identifier for the youth account

**Request Schema**:
```json
{
  "weeklyLimit": "decimal",
  "currency": "string",
  "effectiveDate": "date",
  "notificationsEnabled": "boolean",
  "notificationThresholds": [
    {
      "percentage": "number",
      "enabled": "boolean"
    }
  ]
}
```

**Request Validation**:
- `weeklyLimit`: Must be positive, minimum $10.00, maximum $5,000.00
- `currency`: Must match account currency
- `effectiveDate`: Cannot be in the past
- `notificationThresholds`: Percentage values between 0-100

**Response Schema** (200 OK):
```json
{
  "youthAccountId": "string",
  "weeklyLimit": "decimal",
  "currency": "string",
  "effectiveDate": "date",
  "previousLimit": "decimal",
  "status": "UPDATED|PENDING",
  "nextResetDate": "date",
  "notificationsEnabled": "boolean",
  "lastModified": "datetime"
}
```

**Error Handling**:
- `400 Bad Request`: Invalid limit value or request data
- `401 Unauthorized`: Invalid authentication
- `403 Forbidden`: Not authorized to modify spending limits
- `404 Not Found`: Youth account not found
- `422 Unprocessable Entity`: Limit exceeds account balance or policy limits
- `500 Internal Server Error`: System error

**Example Request**:
```json
{
  "weeklyLimit": 75.00,
  "currency": "USD",
  "effectiveDate": "2024-01-15",
  "notificationsEnabled": true,
  "notificationThresholds": [
    {
      "percentage": 50,
      "enabled": true
    },
    {
      "percentage": 80,
      "enabled": true
    }
  ]
}
```

**Example Response**:
```json
{
  "youthAccountId": "YA-123456789",
  "weeklyLimit": 75.00,
  "currency": "USD",
  "effectiveDate": "2024-01-15",
  "previousLimit": 100.00,
  "status": "UPDATED",
  "nextResetDate": "2024-01-22",
  "notificationsEnabled": true,
  "lastModified": "2024-01-10T15:30:00Z"
}
```

### 4. Transaction History API
**Reference**: SCIB-29

#### Endpoint: GET /api/youth-accounts/{youthAccountId}/transactions

**Description**: Retrieves detailed transaction history for youth accounts with filtering and pagination support.

**Path Parameters**:
- `youthAccountId` (string, required): Unique identifier for the youth account

**Query Parameters**:
- `startDate` (date, optional): Start date for transaction filter (ISO 8601 format)
- `endDate` (date, optional): End date for transaction filter (ISO 8601 format)
- `page` (integer, optional): Page number for pagination (default: 1)
- `pageSize` (integer, optional): Number of records per page (default: 20, max: 100)
- `transactionType` (string, optional): Filter by transaction type (debit|credit|all)
- `status` (string, optional): Filter by status (completed|pending|failed|all)
- `minAmount` (decimal, optional): Minimum transaction amount filter
- `maxAmount` (decimal, optional): Maximum transaction amount filter
- `sortBy` (string, optional): Sort field (date|amount|merchant) (default: date)
- `sortOrder` (string, optional): Sort order (asc|desc) (default: desc)

**Request Schema**: N/A (GET request with query parameters)

**Response Schema** (200 OK):
```json
{
  "transactions": [
    {
      "transactionId": "string",
      "date": "datetime",
      "merchant": "string",
      "description": "string",
      "amount": "decimal",
      "transactionType": "debit|credit",
      "status": "completed|pending|failed",
      "category": "string",
      "location": {
        "city": "string",
        "state": "string",
        "country": "string"
      },
      "balanceAfterTransaction": "decimal",
      "authorizationCode": "string"
    }
  ],
  "pagination": {
    "page": "number",
    "pageSize": "number",
    "totalRecords": "number",
    "totalPages": "number",
    "hasNextPage": "boolean",
    "hasPreviousPage": "boolean"
  },
  "summary": {
    "totalDebits": "decimal",
    "totalCredits": "decimal",
    "netAmount": "decimal",
    "transactionCount": "number",
    "dateRange": {
      "startDate": "date",
      "endDate": "date"
    }
  }
}
```

**Error Handling**:
- `400 Bad Request`: Invalid query parameters or date range
- `401 Unauthorized`: Invalid authentication
- `403 Forbidden`: Not authorized to view transaction history
- `404 Not Found`: Youth account not found
- `422 Unprocessable Entity`: Invalid date range or filter parameters
- `500 Internal Server Error`: System error

**Example Request**:
```
GET /api/youth-accounts/YA-123456789/transactions?startDate=2024-01-01&endDate=2024-01-31&page=1&pageSize=10&transactionType=all&sortBy=date&sortOrder=desc
```

## Security Considerations

### Authentication
- OAuth 2.0 Bearer Token required for all endpoints
- Token expiration: 1 hour
- Refresh token mechanism available
- Multi-factor authentication support

### Authorization
- Role-based access control (RBAC)
- Parent role: Full access to linked youth accounts
- Youth role: Read-only access to own account
- Admin role: System administration access

### Data Protection
- All sensitive data encrypted in transit (TLS 1.3)
- PII data tokenization
- Request/response logging excludes sensitive fields
- Rate limiting to prevent abuse

### Input Validation
- All input parameters validated
- SQL injection prevention
- XSS protection
- CSRF token validation for state-changing operations

## Rate Limiting
- 1000 requests per hour per authenticated user
- 100 requests per minute per IP address
- Burst limit: 10 requests per second
- Rate limit headers included in responses:
  ```
  X-RateLimit-Limit: 1000
  X-RateLimit-Remaining: 999
  X-RateLimit-Reset: 1609459200
  ```
