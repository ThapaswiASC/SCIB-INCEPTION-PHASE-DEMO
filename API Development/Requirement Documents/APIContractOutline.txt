# API Contract Outline
# Youth Account Management System

## Overview
This document outlines the API contracts for the Youth Account Management System, providing detailed specifications for all endpoints, request/response schemas, and error handling patterns.

## Base Configuration
- **Base URL**: https://api.youthbanking.com/v1
- **Authentication**: OAuth 2.0 Bearer Token
- **Content Type**: application/json
- **API Version**: v1

## Authentication
All API endpoints require OAuth 2.0 authentication with appropriate scopes:
- `youth:read` - Read access to youth account data
- `youth:write` - Write access for fund transfers and configuration
- `youth:admin` - Administrative access for account management

## Common Response Headers
```
Content-Type: application/json
X-Request-ID: {unique-request-identifier}
X-Rate-Limit-Remaining: {remaining-requests}
X-Rate-Limit-Reset: {reset-timestamp}
```

## API Endpoints

### 1. Youth Account Dashboard

**Endpoint**: `GET /api/youth-accounts/{youthAccountId}/dashboard`

**Description**: Retrieves consolidated youth account information including balance, spending limit, and recent transactions.

**Path Parameters**:
- `youthAccountId` (string, required): Unique identifier for the youth account

**Request Headers**:
```
Authorization: Bearer {oauth-token}
Content-Type: application/json
```

**Request Schema**: N/A (GET request)

**Response Schema** (200 OK):
```json
{
  "youthAccountId": "string",
  "accountBalance": "decimal",
  "spendingLimit": "decimal",
  "currency": "string",
  "limitUtilization": {
    "weeklySpent": "decimal",
    "remainingLimit": "decimal",
    "utilizationPercentage": "number"
  },
  "recentTransactions": [
    {
      "transactionId": "string",
      "date": "datetime",
      "description": "string",
      "merchant": "string",
      "amount": "decimal",
      "transactionType": "debit|credit",
      "status": "completed|pending|failed"
    }
  ],
  "accountStatus": "active|suspended|closed",
  "lastUpdated": "datetime"
}
```

**Error Handling**:
- `400 Bad Request`: Invalid youth account ID format
- `401 Unauthorized`: Missing or invalid authentication token
- `403 Forbidden`: Insufficient permissions to access youth account
- `404 Not Found`: Youth account not found
- `500 Internal Server Error`: Server processing error

**Example Response**:
```json
{
  "youthAccountId": "YA-123456789",
  "accountBalance": 150.75,
  "spendingLimit": 100.00,
  "currency": "USD",
  "limitUtilization": {
    "weeklySpent": 35.50,
    "remainingLimit": 64.50,
    "utilizationPercentage": 35.5
  },
  "recentTransactions": [
    {
      "transactionId": "TXN-987654321",
      "date": "2024-01-15T14:30:00Z",
      "description": "Coffee Shop Purchase",
      "merchant": "Starbucks",
      "amount": 5.75,
      "transactionType": "debit",
      "status": "completed"
    }
  ],
  "accountStatus": "active",
  "lastUpdated": "2024-01-15T14:35:00Z"
}
```

### 2. Fund Transfer to Youth Account

**Endpoint**: `POST /api/youth-accounts/{youthAccountId}/fund-transfer`

**Description**: Transfers funds from parent account to youth account with validation and security checks.

**Path Parameters**:
- `youthAccountId` (string, required): Target youth account identifier

**Request Schema**:
```json
{
  "sourceAccountId": "string",
  "transferAmount": "decimal",
  "currency": "string",
  "transferNote": "string",
  "scheduledDate": "datetime" // Optional for future transfers
}
```

**Response Schema** (201 Created):
```json
{
  "transferId": "string",
  "status": "SUCCESS|PENDING|FAILED",
  "sourceAccountId": "string",
  "targetAccountId": "string",
  "transferAmount": "decimal",
  "currency": "string",
  "updatedBalance": "decimal",
  "transactionId": "string",
  "processedAt": "datetime",
  "estimatedCompletion": "datetime"
}
```

**Error Handling**:
- `400 Bad Request`: Invalid transfer amount or missing required fields
- `401 Unauthorized`: Authentication required
- `403 Forbidden`: Insufficient permissions or account relationship not verified
- `409 Conflict`: Insufficient funds in source account
- `422 Unprocessable Entity`: Transfer amount exceeds daily/monthly limits
- `500 Internal Server Error`: Transfer processing failure

**Example Request**:
```json
{
  "sourceAccountId": "PA-123456789",
  "transferAmount": 50.00,
  "currency": "USD",
  "transferNote": "Weekly allowance"
}
```

**Example Response**:
```json
{
  "transferId": "TRF-987654321",
  "status": "SUCCESS",
  "sourceAccountId": "PA-123456789",
  "targetAccountId": "YA-123456789",
  "transferAmount": 50.00,
  "currency": "USD",
  "updatedBalance": 200.75,
  "transactionId": "TXN-111222333",
  "processedAt": "2024-01-15T15:00:00Z",
  "estimatedCompletion": "2024-01-15T15:00:00Z"
}
```

### 3. Configure Spending Limit

**Endpoint**: `PUT /api/youth-accounts/{youthAccountId}/spending-limit`

**Description**: Configures or updates the weekly spending limit for a youth account.

**Path Parameters**:
- `youthAccountId` (string, required): Youth account identifier

**Request Schema**:
```json
{
  "weeklyLimit": "decimal",
  "limitEnabled": "boolean",
  "effectiveDate": "datetime",
  "notificationPreferences": {
    "alertAt50Percent": "boolean",
    "alertAt75Percent": "boolean",
    "alertAt90Percent": "boolean",
    "alertOnLimitReached": "boolean"
  }
}
```

**Response Schema** (200 OK):
```json
{
  "youthAccountId": "string",
  "weeklyLimit": "decimal",
  "limitEnabled": "boolean",
  "previousLimit": "decimal",
  "effectiveDate": "datetime",
  "status": "UPDATED|CREATED",
  "notificationPreferences": {
    "alertAt50Percent": "boolean",
    "alertAt75Percent": "boolean",
    "alertAt90Percent": "boolean",
    "alertOnLimitReached": "boolean"
  },
  "updatedAt": "datetime",
  "updatedBy": "string"
}
```

**Error Handling**:
- `400 Bad Request`: Invalid limit value (negative or exceeds maximum)
- `401 Unauthorized`: Authentication required
- `403 Forbidden`: Insufficient permissions to modify spending limits
- `404 Not Found`: Youth account not found
- `422 Unprocessable Entity`: Limit configuration conflicts with account policies
- `500 Internal Server Error`: Configuration update failure

**Example Request**:
```json
{
  "weeklyLimit": 75.00,
  "limitEnabled": true,
  "effectiveDate": "2024-01-16T00:00:00Z",
  "notificationPreferences": {
    "alertAt50Percent": true,
    "alertAt75Percent": true,
    "alertAt90Percent": true,
    "alertOnLimitReached": true
  }
}
```

### 4. Transaction History

**Endpoint**: `GET /api/youth-accounts/{youthAccountId}/transactions`

**Description**: Retrieves paginated transaction history for a youth account with filtering options.

**Path Parameters**:
- `youthAccountId` (string, required): Youth account identifier

**Query Parameters**:
- `startDate` (datetime, optional): Filter transactions from this date
- `endDate` (datetime, optional): Filter transactions until this date
- `transactionType` (string, optional): Filter by type (debit, credit, all)
- `merchant` (string, optional): Filter by merchant name
- `minAmount` (decimal, optional): Minimum transaction amount
- `maxAmount` (decimal, optional): Maximum transaction amount
- `page` (integer, optional, default: 1): Page number for pagination
- `pageSize` (integer, optional, default: 20, max: 100): Number of records per page
- `sortBy` (string, optional, default: date): Sort field (date, amount, merchant)
- `sortOrder` (string, optional, default: desc): Sort order (asc, desc)

**Response Schema** (200 OK):
```json
{
  "transactions": [
    {
      "transactionId": "string",
      "date": "datetime",
      "merchant": "string",
      "description": "string",
      "amount": "decimal",
      "transactionType": "debit|credit",
      "category": "string",
      "status": "completed|pending|failed|cancelled",
      "balanceAfter": "decimal",
      "location": {
        "city": "string",
        "state": "string",
        "country": "string"
      },
      "paymentMethod": "card|online|mobile"
    }
  ],
  "pagination": {
    "page": "number",
    "pageSize": "number",
    "totalRecords": "number",
    "totalPages": "number",
    "hasNextPage": "boolean",
    "hasPreviousPage": "boolean"
  },
  "summary": {
    "totalDebits": "decimal",
    "totalCredits": "decimal",
    "netAmount": "decimal",
    "transactionCount": "number"
  },
  "filters": {
    "appliedFilters": "object",
    "availableCategories": ["string"],
    "dateRange": {
      "earliest": "datetime",
      "latest": "datetime"
    }
  }
}
```

**Error Handling**:
- `400 Bad Request`: Invalid query parameters or date range
- `401 Unauthorized`: Authentication required
- `403 Forbidden`: Insufficient permissions to view transaction history
- `404 Not Found`: Youth account not found
- `500 Internal Server Error`: Transaction retrieval failure

### 5. Account Status Management

**Endpoint**: `PATCH /api/youth-accounts/{youthAccountId}/status`

**Description**: Updates the status of a youth account (active, suspended, closed).

**Request Schema**:
```json
{
  "status": "active|suspended|closed",
  "reason": "string",
  "effectiveDate": "datetime",
  "notifyYouth": "boolean"
}
```

**Response Schema** (200 OK):
```json
{
  "youthAccountId": "string",
  "previousStatus": "string",
  "newStatus": "string",
  "reason": "string",
  "effectiveDate": "datetime",
  "updatedBy": "string",
  "updatedAt": "datetime"
}
```

## Rate Limiting
- **Standard Rate Limit**: 1000 requests per hour per authenticated user
- **Burst Rate Limit**: 100 requests per minute
- **Fund Transfer Limit**: 10 transfers per hour per account
